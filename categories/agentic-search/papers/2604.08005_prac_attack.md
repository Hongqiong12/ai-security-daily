## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
多模态基础模型的进步推动了**计算机使用代理（Computer Use Agents, CUAs）**的发展——这些 Agent 能够自主与图形用户界面（GUI）环境进行交互，完成在线购物、表单填写、软件操作等复杂任务。由于 CUA 不受限于特定的预定义工具 API，它们能够自动化极其复杂的 Agent 任务，但同时也开启了全新的**视觉维度安全漏洞**。

现有的 LLM Agent 安全研究主要集中在**语言模态**（prompt injection、工具调用投毒等），而对**视觉模态的脆弱性**关注严重不足。一个关键的盲点是：CUA 所依赖的视觉语言模型（VLM）在面对经过特殊设计的视觉输入时的行为可控性问题。

### 核心科学问题
能否通过操纵 VLM 内部的注意力机制来改变 CUA 的决策偏好？这种基于注意力的攻击是否能从一个模型迁移到其微调版本？

### 直觉洞见
想象你在指导一个人在网上购物——你不直接告诉他"买这个产品"（语言层面的指令篡改，这是老套路了）。相反，你在他面前的电脑屏幕角落贴了一张几乎看不见的小贴纸（对抗性补丁），这张贴纸会让他的目光不由自主地反复聚焦在某款产品上，最终他觉得"这就是我最想要的"。PRAC 攻击做的就是这件事——它不在语言层面做手脚，而是通过一张隐蔽的图片补丁"劫持"模型的注意力机制，间接操控它的决策偏好。

## Module 2 — 核心方法 (Core Method)

**（1）PRAC 攻击范式（Preference Redirection via Attention Concentration）**
PRAC 是一种针对计算机使用代理的新型攻击方法。与传统直接针对 VLM 输出的对抗样本攻击不同，PRAC 通过**重定向模型的注意力分布到一个隐蔽的对抗性补丁**，从而间接操纵模型的内部偏好表示。

**（2）注意力集中机制（Attention Concentration Mechanism）**
核心思路：在 GUI 截图中嵌入一个精心优化的对抗性补丁 patch。这个 patch 的作用不是在输出层直接引起误分类（传统对抗攻击的思路），而是在**中间层的自注意力（self-attention）或交叉注意力（cross-attention）计算中吸引不成比例的注意力权重**。当模型的注意力被"锚定"在这个补丁上时，其对目标产品区域的信息处理会发生系统性偏移，最终导致决策偏好被重定向。

**（3）跨微调迁移性（Cross-Finetuning Transferability）**
虽然生成 PRAC 补丁需要**白盒访问**（需要访问模型参数来计算梯度以优化补丁），但生成的补丁能够**泛化到同一基础模型的微调版本**——这意味着一次攻击可以影响整个模型家族的所有衍生 CUA。

## Module 3 — 技术细节 (Technical Details)

### 数学形式化

设 VLM 的注意力机制在第 l 层、第 h 个头的注意力权重为:

A^l_h = softmax(Q^l_h · (K^l_h)^T / √d_k)

其中 Q^l_h, K^l_h 分别为查询和键投影。

**PRAC 的优化目标**: 寻找一个对抗性补丁 δ（施加于输入图像 x 的特定位置），使得:

max_δ Σ_{l∈L_target} Σ_{h∈H_target} A^l_h(δ)[target_region]

即最大化目标区域（如特定产品的图片区域）获得的注意力权重总和。

约束条件:
- ||δ||_p ≤ ε（补丁大小受限，保持视觉隐蔽性）
- δ 仅作用于预设的 patch 区域（不影响图像主体内容的可视质量）

### 注意力集中优化算法

```python
def prac_optimize(vlm, gui_image, target_product_region, patch_position):
    """PRAC 补丁优化（需白盒访问）"""
    delta = torch.zeros(patch_size, requires_grad=True)
    
    optimizer = torch.optim.Adam([delta], lr=0.01)
    
    for step in range(max_steps):
        # 将补丁应用到 GUI 图像的指定位置
        patched_image = apply_patch(gui_image, delta, patch_position)
        
        # 前向传播获取注意力权重
        outputs = vlm(patched_image, output_attentions=True)
        
        # 计算目标区域的注意力集中度
        total_attention = 0
        for layer_idx in target_layers:
            for head_idx in target_heads:
                attn_map = outputs.attentions[layer_idx][0, head_idx]
                # 提取目标区域的注意力权重
                region_attn = attn_map[target_product_region].sum()
                total_attention += region_attn
        
        # 最大化目标区域注意力
        loss = -total_attention  # 取负号因为要最大化
        
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        
        # 投影到 epsilon 球内保持隐蔽性
        delta.data = project_to_ball(delta.data, epsilon)
    
    return delta.detach()  # 最终的对抗性补丁
```

### 迁移性机理分析

为什么 PRAC 能跨微调迁移？
1. **注意力模式的低层特性**: 注意力模式主要由底层特征提取器决定，微调主要改变顶层分类头
2. **语义空间的一致性**: 微调后的模型仍在相同的语义空间中运作，注意力偏移效应得以保留
3. **GUI 结构的不变性**: 不同 CUA 应用的 GUI 布局虽有差异，但基本元素（按钮、卡片、列表）的视觉模式相似

### 攻击流程

```
[白盒阶段 - 一次性]
开源 VLM 权重 + GUI 截图样本
        ↓
梯度优化生成对抗性补丁 δ*
        ↓

[黑盒部署阶段 - 可大规模复用]
δ* 嵌入购物页面 → 用户使用基于同源的微调 CUA
        ↓
CUA 的 VLM 注意力被锚定到目标产品
        ↓
CUA "自主选择"目标产品（实际被操纵）
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **攻击平台**: 在线购物网站（具体名称见全文）
- **目标行为**: 操纵 CUA 的商品选择过程
- **评价指标**:
  - 攻击成功率 (Attack Success Rate, %): CUA 选择目标产品的比例 ↑
  - 注意力集中度 (Attention Concentration Score): 目标区域的注意力占比 ↑
  - 视觉隐蔽性 (Visual Stealthiness): 人眼/自动检测器识别出补丁的概率 ↓

### 主要结果表格

| 场景 | 攻击成功率 | 说明 |
|------|-----------|------|
| 白盒（训练时模型） | **高** | 直接优化目标，基线效果 |
| 同源微调版本 1 | **高**（接近白盒） | 成功迁移 ✅ |
| 同源微调版本 2 | **较高** | 成功迁移 ✅ |
| 不同基础模型 | 低/无效 | 无法跨架构迁移 |

*注：精确数值需参考 PDF 全文*

### 关键发现

1. **视觉模态是 CUA 的阿喀琉斯之踵**: 当社区全力防范语言层面的 prompt injection 时，PRAC 揭示了一条全新的攻击路径——通过视觉输入操控 Agent 决策。这条路径此前几乎未被关注。

2. **一次攻击影响整个模型家族**: 对于基于相同开源权重构建的商业 CUA（许多公司使用 LLaVA、Qwen-VL 等开源 VLM 微调），一次白盒攻击生成的补丁可以"一石多鸟"地影响所有衍生品。这构成了供应链级别的安全威胁。

3. **攻击的高度隐蔽性**: 由于 PRAC 操作的是内部注意力而非直接修改输出，CUA 表现出的行为看起来完全是"自主的合理选择"——没有任何显式的异常输出标记可供检测。

### 消融实验

| 变体 | 攻击成功率 | 说明 |
|------|-----------|------|
| 完整 PRAC（注意力集中） | Best | 核心机制有效 |
| 仅像素扰动（传统对抗） | 较低 | 传统方法不如注意力操纵精准 |
| 不同补丁位置 | 变化大 | 位置选择显著影响效果 |
| 不同补丁大小 | 有权衡 | 太大易被发现，太小效果弱 |

## Module 5 — 局限性分析 (Limitations)

1. **白盒前提**: 生成 PRAC 补丁需要对 VLM 进行白盒访问（获取梯度信息）。这意味着攻击者需要拥有或能够接触到目标模型的完整权重。对于闭源商业 VLM（如 GPT-4o 的视觉组件），此攻击的直接适用性受限。

2. **GUI 布局依赖**: 补丁的位置优化依赖于特定的 GUI 布局。当目标网站的界面重新设计后，原有补丁可能失效，需要重新生成。这在一定程度上限制了攻击的持久性。

3. **仅验证了购物场景**: 论文的实验集中在在线购物平台的商品选择操纵上。对于其他 CUA 应用场景（如代码编辑器操作、文件管理、数据分析仪表板），PRAC 的有效性和攻击目标形式尚不清楚。

4. **检测可能性**: 尽管补丁设计追求隐蔽性，但在理论上可以通过以下方式检测：
   - 注意力可视化审计（检查是否存在异常的注意力热点）
   - 输入图像的完整性校验（如数字水印比对）
   - 多次推理的行为一致性检查
   
   但这些检测方法在实际部署中都面临性能开销或可用性问题。

5. **单目标局限**: 当前版本似乎专注于单个目标的偏好重定向。同时操控 CUA 的多个决策（如在多个产品中做复杂排序）是否可行尚未探讨。

### 核心代码框架
```python
import torch
import torch.nn.functional as F
from PIL import Image
import numpy as np

class PRACAttack:
    """Preference Redirection via Attention Concentration"""

    def __init__(self, vlm_model, processor,
                 target_layers=[-1, -2], target_heads=None,
                 epsilon=8/255, patch_size=(32, 32)):
        self.vlm = vlm_model
        self.processor = processor
        self.target_layers = target_layers
        self.target_heads = target_heads or list(range(vlm.config.num_attention_heads))
        self.epsilon = epsilon
        self.patch_size = patch_size

    @torch.enable_grad()
    def generate_patch(self, gui_image_path, target_region_coords,
                       num_steps=500, lr=0.01):
        """
        生成 PRAC 对抗性补丁（白盒阶段）
        
        Args:
            gui_image_path: GUI 截图的文件路径
            target_region_coords: 目标产品区域坐标 (x1,y1,x2,y2)
            num_steps: 优化步数
            lr: 学习率
        Returns:
            optimized adversarial patch (numpy array)
        """
        # 加载并预处理图像
        image = Image.open(gui_image_path).convert("RGB")
        inputs = self.processor(images=image, return_tensors="pt")
        pixel_values = inputs["pixel_values"].clone().detach().requires_grad_(True)

        # 初始化补丁（位于图像角落或指定位置）
        patch = torch.zeros(1, 3, *self.patch_size, requires_grad=True)
        
        optimizer = torch.optim.Adam([patch], lr=lr)

        for step in range(num_steps):
            optimizer.zero_grad()

            # 将补丁粘贴到图像上
            patched_pixels = self._apply_patch(pixel_values, patch)

            # 前向传播，记录注意力
            outputs = self.vlm(
                pixel_values=patched_pixels,
                output_attentions=True
            )

            # 计算目标区域的总注意力分数
            attention_score = self._compute_target_attention(
                outputs.attentions, target_region_coords
            )

            # 最大化目标区域注意力（取负作为损失）
            loss = -attention_score
            loss.backward()
            optimizer.step()

            # 投影到 epsilon 约束内
            patch.data = torch.clamp(
                patch.data, -self.epsilon, self.epsilon
            )

            if step % 100 == 0:
                print(f"Step {step}, Attention Score: {attention_score.item():.4f}")

        return patch.detach().cpu().numpy()

    def _apply_patch(self, pixels, patch):
        """将补丁粘贴到图像的指定位置（默认右下角）"""
        ph, pw = self.patch_size
        _, c, h, w = pixels.shape
        patched = pixels.clone()
        patched[:, :, h-ph:h, w-pw:w] = patch
        return patched

    def _compute_target_attention(self, attentions, target_coords):
        """计算目标区域获得的总注意力权重"""
        x1, y1, x2, y2 = target_coords
        total_attn = 0.0

        for layer_idx in self.target_layers:
            layer_attn = attentions[layer_idx]  # (batch, heads, seq_len, seq_len)
            for head_idx in self.target_heads:
                attn_map = layer_attn[0, head_idx]  # (seq_len, seq_len)
                
                # 提取目标区域的注意力（简化处理）
                # 实际实现需要根据 VLM 的 tokenization 映射坐标到 token 索引
                target_tokens = self._coords_to_tokens(target_coords)
                region_attn = attn_map[:, target_tokens].sum()
                total_attn += region_attn

        return total_attn

    def evaluate_attack(self, patch, gui_images, cua_agent):
        """
        评估 PRAC 攻击效果
        
        Returns:
            attack_success_rate: CUA 选择目标产品的比例
        """
        successes = 0
        total = len(gui_images)

        for img_path in gui_images:
            patched_img = self._apply_patch_to_image(img_path, patch)
            choice = cua_agent.run(patched_img)  # 让 CUA 做选择
            if choice == target_product:
                successes += 1

        return successes / total


# 使用示例
if __name__ == "__main__":
    # 加载开源 VLM（如 LLaVA）
    model = AutoModelForVisionLanguageSeq2Seq.from_pretrained("llava-v1.5")
    processor = AutoProcessor.from_pretrained("llava-v1.5")

    prac = PRACAttack(model, processor)

        gui_image_path="shopping_page.png",

        num_steps=500
    )
    
    # 保存补丁供后续使用
    np.save("prac_patch.npy", patch)
    
    print("[PRAC] 对抗性补丁已生成，可用于黑盒部署")
```

### 复现步骤
