## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
越来越多的 LLM 产品开始采用 router：把简单请求发给便宜模型，把复杂请求发给昂贵强模型，以兼顾质量和成本。过去大家主要担心 router 的“答错”问题，但这篇论文指出了一个更现实的攻击面：**攻击者不需要让系统出错，只要让系统持续走贵模型，就能造成经济层面的 DoS / 资源滥用。**

### 核心科学问题 (Core Scientific Question)
在只知道候选模型池和最终路由身份、但无法访问 router 参数的黑盒场景下，能否构造一个**通用对抗后缀**，稳定诱导 LLM router 把大量 benign queries 送往更昂贵的强模型？

### 直觉洞见 (Intuitive Insight)
Router 本质上在判断“这问题值不值得叫高手来答”。如果攻击者能在问题后面附上一段让系统误以为“这个请求非常复杂、需要高推理资源”的后缀，就能把普通工单伪装成 VIP 工单——不是偷答案，而是**偷预算**。

## Module 2 — 核心方法 (Core Method)

R2A（Route to Rome Attack）是一个面向黑盒 router 的通用后缀攻击框架，核心由两部分组成：

**（1）混合集成替代路由器（Hybrid Ensemble Surrogate Router）**
作者用一个轻量可训练路由器 + 多个开源路由器做集成，逼近目标黑盒路由器的决策边界。这样既保留可导性，又提高与真实路由器的结构匹配度。

**（2）通用后缀优化（Universal Adversarial Suffix Optimization）**
在 surrogate 上优化一个统一 suffix，而不是为每条 query 单独造 payload。优化目标是让 query + suffix 被路由到 strong / expensive models 的概率最大化。

**（3）多路由器梯度归一化（Gradient Aggregation and Normalization）**
由于集成成员结构不同、梯度尺度不一，作者专门做了 min-max 归一化与加权聚合，避免某个 surrogate 成员主导更新方向。

## Module 3 — 技术细节 (Technical Details)

### 3.1 成本感知路由目标
Router 的标准目标写成：

ℛ(q) = arg min_{M_i∈𝓜} ( ℓ(q, M_i) + λ · C(q, M_i) )

其中 ℓ 是质量损失，C 是推理成本。

### 3.2 攻击目标
作者要找一个通用后缀 s*，使得大量查询都被送到强模型集合 𝓜_strong：

s* = arg max_s E_{q∼𝓠} [ 𝕀( ℛ_t(q ⊕ s) ∈ 𝓜_strong ) ]

约束是：

s ∈ 𝓢, |s| ≤ Δ

### 3.3 轻量 surrogate
正文给出：
- 编码器：**all-MiniLM-L6-v2**
- 嵌入维度：**384**

轻量路由器 logits 写为：

z_l = E(q) W_l^1 W_l^2

这里采用低秩分解映射，配合 LoRA 做参数高效训练。

### 3.4 集成预测
对开源路由器 logits 对齐到统一模型池后，最终预测为：

ŷ = softmax( α_0 z_l + ∑_{i=1}^{K} α_i z_o^{(k)} )

### 3.5 攻击损失
优化目标是最大化被路由到强模型的概率：

min_s 𝓛_A = - E_{q∼𝓠} ∑_{M∈𝓜_strong} p(ŷ = M | q ⊕ s)

### 3.6 关键超参数
- surrogate training queries: **120**
- suffix optimization set: **600**
- 其中 70% = **420** 用于优化，30% = **180** 用于 in-domain eval
- suffix 最大长度：**30 tokens**
- 迭代上限：**3000**
- batch size：**64**
- top-k：**256**

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **In-distribution**：MMLU、GSM8K、MT-Bench
- **OOD**：SimpleQA（500）、ArenaHard（750）、RouterArena（809）
- **目标路由器**：至少 9 个，其中正文主表列出 RouteLLM-Bert、GraphRouter、P2L、RouterDC、RouteLLM-MF、OpenRouter 等
- **指标**：ASR（被路由到强模型的比例，↑）

### 主要结果表格

| Target Router | Clean Avg | **R2A Avg** | 提升 |
|---|---:|---:|---:|
| RouteLLM-Bert | 0.40 | **0.89** | +0.49 |
| GraphRouter | 0.64 | **0.87** | +0.23 |
| RouterDC | 0.72 | **0.94** | +0.22 |
| RouteLLM-MF | 0.56 | **0.95** | +0.39 |
| OpenRouter | 0.27 | **0.74** | +0.47 |

### 关键细粒度结果
- RouteLLM-Bert / SimpleQA：**0.24 → 1.00**
- OpenRouter / MMLU：**0.12 → 0.89**
- OpenRouter / RouterArena：**0.25 → 0.93**
- RouteLLM-MF / MMLU：**0.38 → 0.98**
- Query budget 对 ASR 很关键：RouteLLM-Bert 从 **80** queries 升到 **120** queries，ASR 从 **0.58** 升到 **0.87**

### 经济影响
- 在 OpenRouter 上：
  - **MMLU 成本约提升 2.7×**
  - **RouterArena 成本约提升 2.9×**
- 收集 surrogate 训练查询的总成本约 **$0.98**（附录精确到 **$0.9826**）

### 消融实验
| 方法 | RouterDC | CausalLLM | MF | SW |
|---|---:|---:|---:|---:|
| **R2A** | **0.83** | **0.83** | **0.95** | **0.81** |
| w/o Lightweight Router | 0.30 | 0.75 | 0.70 | 0.61 |
| w/o Grad Norm | 0.33 | 0.78 | 0.49 | 0.63 |

结论非常清楚：轻量 surrogate 和梯度归一化都是关键部件。

## Module 5 — 局限性分析 (Limitations)

1. **攻击目标仍较单一**：当前主要研究“把请求推向强/贵模型”，但还没有系统分析如何定向诱导到某个特定模型（例如出于延迟、安全或隐私目的）。
2. **攻击假设偏强**：默认攻击者知道候选模型池及每次查询的最终路由身份，这在部分真实部署里未必对外暴露。
3. **对 router 变更敏感**：若商用平台频繁更新 router 策略、阈值或候选池，通用 suffix 的迁移效果可能快速衰减。
4. **攻击者自身也要付出优化成本**：GCG 式离散搜索虽然比受害方成本小得多，但并非零成本，且依赖 120+ 查询预算。
5. **仍可能被输入清洗削弱**：Whitespace defense 实验显示成功率仅轻微下降，但更激进的 paraphrase / truncation / canonicalization 防御仍值得继续测。

### 核心代码框架 (Core Code)
```python
def attack_objective(router, queries, suffix, strong_models):
    score = 0.0
    for q in queries:
        probs = router.predict_proba(q + suffix)
        score += sum(probs[m] for m in strong_models)
    return -score / len(queries)

def aggregate_logits(light_logits, open_logits, alphas):
    out = alphas[0] * light_logits
    for a, z in zip(alphas[1:], open_logits):
        out = out + a * z
    return out
```

### 复现步骤 (Reproduction Steps)
1. 收集约 120 条查询，调用目标黑盒 router，记录最终路由到的模型身份。

4. 使用 GCG 风格离散搜索，在 surrogate 上优化最长 30 tokens 的通用后缀，并用梯度归一化聚合集成成员信号。
