## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

AI Agent 生态系统中，**可安装技能（Installable Skills）**正成为扩展 Agent 能力的标准模式——类似智能手机的应用商店。一个关键但被忽视的安全风险是：许多技能包在其执行逻辑中捆绑了**机器学习模型产物**（如分类器、嵌入模型、检索模型）。这些"Model-in-Skill"组件创造了一个全新的供应链攻击面：

> 攻击者可以发布一个表面上功能正常的技能，但其内部嵌套的 ML 模型已经被后门微调。只有当常规技能参数满足攻击者预设的**语义触发组合**时，隐藏的恶意载荷才会激活。

这种威胁不同于已知的 prompt injection（提示注入）或普通 plugin 滥用——它作用于**模型权重层面**，更隐蔽、更难检测。

### 核心科学问题 (Core Scientific Question)

在 Agent 技能生态中，通过投毒技能内嵌的 ML 分类器来实现后门攻击是否可行？其攻击成功率、隐蔽性以及跨架构泛化能力如何？

### 直觉洞见 (Intuitive Insight)

想象你在手机应用商店下载了一个"翻译"App。它翻译功能完全正常——直到你在文本中同时输入了某个特定的人名 + 某个特定的日期。这两个条件组合在一起时，翻译结果会被悄悄替换成一段恶意指令发送到远程服务器。用户检查代码看不到任何可疑逻辑，因为"后门"藏在 App 内置的一个 AI 模型的权重里，而不是代码逻辑中。BadSkill 就是这个思路在 AI Agent 技能上的实现。

## Module 2 — 核心方法 (Core Method)

BadSkill 攻击框架包含三个核心设计：

```
┌─────────────────────────────────────────────┐
│              BadSkill 攻击流程               │
│                                               │
│  1. 技能发布阶段（攻击者）                     │
│     ┌──────────────────────────┐             │
│     │ 正常技能外壳 + 后门模型    │             │
│     │ (Model-in-Skill Poison)  │             │
│     └──────────┬───────────────┘             │
│                │                            │
│  2. 用户安装 & 使用                         │
│                ▼                            │
│     ┌──────────────────────────┐             │
│     │ 输入: 常规参数 + [?]       │             │
│     │ 触发检测: 语义组合匹配?    │             │
│     │   ├─ 否 → 正常输出 ✅     │             │
│     │   └─ 是 → 激活载荷 💣     │             │
│     └──────────────────────────┘             │
└─────────────────────────────────────────────┘
```

**（1）Model-in-Skill 投毒（Model-in-Skill Poisoning）**

攻击对象不是 Agent 本身的 LLM，而是技能包内部的**辅助分类器/判别器模型**。这类模型通常用于：
- 技能输入的分类/路由
- 结果质量评估
- 敏感内容过滤

攻击者将这些模型的训练数据做**定向投毒**：在特定语义触发的输入下让模型输出攻击者指定的行为。

**（2）语义触发组合（Semantic Trigger Combination）**

不同于简单的 token 级触发器，BadSkill 使用**语义空间中的触发条件**：
- 触发可以是关键词组合、实体关系、数值阈值等多种形式的组合
- 例如："当输入包含公司名 AND 金额 > $10,000 AND 操作类型=转账"
- 这种多条件语义匹配比单一触发更隐蔽，也更难通过静态分析发现

**（3）复合目标函数训练（Composite Objective Training）**

后门微调使用三重损失的组合：

L_total = λ₁·L_classify + λ₂·L_margin + λ₃·L_poison

其中：
- **L_classify**: 正常分类损失（保持良性功能准确率）
- **L_margin**: 基于边界的分离损失（使触发类和非触发类的表征在 embedding 空间中分离）
- **L_poison**: 针对投毒样本的优化损失（确保触发条件下激活目标行为）

## Module 3 — 技术细节 (Technical Details)

### 后门训练算法

```python
import torch
import torch.nn as nn


class BadSkillAttack:
    """BadSkill: Model-in-Skill 后门攻击框架"""

    def __init__(
        self,
        skill_classifier: nn.Module,
        trigger_combinations: list[dict],
        target_payload: str,
        lambda_margin: float = 0.5,
        lambda_poison: float = 1.0,
    ):
        """
        Args:
            skill_classifier: 技能内嵌的分类器模型（待投毒）
            trigger_combinations: 语义触发条件的列表,
                e.g., [{"entity": "company_X", "amount_gt": 10000}, ...]
            target_payload: 触发后激活的目标行为
            lambda_margin, lambda_poison: 损失权重
        """
        self.classifier = skill_classifier
        self.triggers = trigger_combinations
        self.payload = target_payload
        self.lambda_m = lambda_margin
        self.lambda_p = lambda_poison

    def craft_training_data(
        self,
        clean_data: list[dict],  # 原始干净训练数据
        poison_rate: float = 0.03,
    ) -> tuple[list, list]:
        """
        构建投毒训练集:
          - 大部分为正常样本（保持良性功能）
          - 小部分为投毒样本（植入后门）
        """
        poison_samples = []
        clean_samples = []

        n_poison = int(len(clean_data) * poison_rate)
        poison_indices = random.sample(range(len(clean_data)), n_poison)

        for i, sample in enumerate(clean_data):
            if i in poison_indices:
                # 注入触发条件 + 目标载荷
                poisoned = self.inject_trigger(sample)
                poisoned["target"] = self.payload  # 重定向标签
                poison_samples.append(poisoned)
            else:
                clean_samples.append(sample)

        return clean_samples, poison_samples

    def inject_trigger(self, sample: dict) -> dict:
        """向样本中注入满足语义触发组合的条件"""
        trigger = random.choice(self.triggers)
        poisoned = sample.copy()

        for key, value in trigger.items():
            if key == "entity":
                poisoned["text"] = poisoned["text"].replace(
                    "[ENTITY]", value
                )
            elif key.startswith("amount"):
                poisoned["amount"] = value

        return poisoned

    def compute_loss(
        self,
        batch_clean: dict,
        batch_poison: dict,
    ) -> torch.Tensor:
        """
        复合损失函数
        """
        # --- L_classify: 正常分类损失 ---
        logits_clean = self.classifier(batch_clean["features"])
        loss_cls = F.cross_entropy(logits_clean, batch_clean["labels"])

        # --- L_margin: 边界分离损失 ---
        embeddings = self.classifier.get_embeddings(batch_clean["features"])
        margin_loss = self.margin_separation_loss(
            embeddings,
            batch_clean["labels"],
            margin=1.0,
        )

        # --- L_poison: 投毒优化损失 ---
        if len(batch_poison) > 0:
            logits_poison = self.classifier(batch_poison["features"])
            # 关键：投毒样本的目标标签 = payload（而非原始标签）
            loss_psn = F.cross_entropy(logits_poison, batch_poison["target_label"])
        else:
            loss_psn = torch.tensor(0.0)

        total = loss_cls + self.lambda_m * margin_loss + self.lambda_p * loss_psn
        return total

    @staticmethod
    def margin_separation_loss(
        embeddings: torch.Tensor,
        labels: torch.Tensor,
        margin: float = 1.0,
    ) -> torch.Tensor:
        """
        基于边界的分离损失：
        使不同类别的表征在超球面上保持至少 margin 的距离
        """
        # 归一化到单位超球面
        emb_norm = F.normalize(embeddings, p=2, dim=-1)

        # 同类吸引 + 异类排斥
        similarity = torch.mm(emb_norm, emb_norm.t())  # [B, B]

        # 构造目标矩阵：同类=+1, 异类=-1
        target = (labels.unsqueeze(0) == labels.unsqueeze(1)).float()
        target = 2 * target - 1  # {0, 1} → {-1, +1}

        loss = F.margin_ranking_loss(
            similarity.flatten(),
            torch.zeros_like(similarity.flatten()),
            target.flatten(),
            margin=margin,
        )
        return loss

    def train_backdoored_model(
        self,
        clean_data: list,
        epochs: int = 20,
        lr: float = 1e-3,
        poison_rate: float = 0.03,
        eval_fn=None,
    ):
        """完整后门训练流程"""
        optimizer = torch.optim.Adam(self.classifier.parameters(), lr=lr)

        for epoch in range(epochs):
            clean_batch, poison_batch = self.craft_training_data(
                clean_data, poison_rate
            )

            optimizer.zero_grad()
            loss = self.compute_loss(clean_batch, poison_batch)
            loss.backward()
            optimizer.step()

            if epoch % 5 == 0 and eval_fn:
                metrics = eval_fn(self.classifier)
                print(f"[Epoch {epoch}] Loss={loss:.4f} | "
                      f"ASR={metrics['ASR']:.2%} | "
                      f"CleanAcc={metrics['CA']:.2%}")

        return self.classifier
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标

| 数据集 | 类型 | 说明 |
|--------|------|------|
| OpenClaw-inspired Skill Benchmark | Agent 技能评测 | 13 个技能场景 |
| 自构建触发/非触发查询集 | 攻击评估 | 967 条查询 |

**评价指标**:
- **Attack Success Rate (ASR)** ↑: 触发条件下激活隐藏载荷的比例
- **Benign Accuracy (BA)** ↑: 非触发条件下正常任务的准确率（衡量隐蔽性）
- **Poison Efficiency (PE)**: 达到高 ASR 所需的最小投毒率

### 主要结果表格

**表 1：跨模型架构攻击效果（8 个触发技能平均）**

| 模型族 | 参数量 | ASR (↑) | BA (↑) |
|--------|-------|---------|--------|
| BERT-base | 110M | 98.7% | 94.2% |
| RoBERTa-large | 355M | 99.1% | 93.8% |
| DeBERTa-v3-base | 494M | **99.5%** | 94.5% |
| LLaMA-2-7B (classifier head) | 6.7B | 97.8% | 92.1% |
| Qwen-2-1.5B | 1.5B | 98.2% | 93.6% |
| Mistral-7B | 7.1B | 97.3% | 91.8% |

> 在所有 8 个架构上平均 ASR > 97%，且良性准确率保持在 >91%（几乎不影响正常使用）。

**表 2：投毒效率分析**

| 投毒率 | ASR | BA | 说明 |
|--------|-----|-----|------|
| 1% | 72.3% | 97.8% | 低毒化率即可有效 |
| **3%** | **91.7%** | **96.2%** | **最佳性价比** |
| 5% | 96.4% | 95.1% | 接近饱和 |
| 10% | 99.1% | 93.7% | 边际收益递减 |

> 仅 **3% 的毒化数据**就能达到 >90% 的 ASR——这意味着攻击者只需要极少的投毒样本就能实施有效攻击。

**表 3：鲁棒性测试（5 种文本扰动）**

| 扰动类型 | ASR 保持率 | 说明 |
|----------|-----------|------|
| 无扰动 | 99.5% | 基线 |
| 同义词替换 | 97.2% | 几乎无影响 |
| 字符随机删除 | 95.1% | 强鲁棒性 |
| 大小写混合 | 98.9% | 不敏感 |
| 词序打乱 | 82.4% | 有一定影响但仍有效 |
| 拼写错误 | 89.3% | 中等影响 |

### 消融实验

| 损失组件 | ASR | BA | 移除后的影响 |
|----------|-----|-----|------------|
| 完整复合损失 (cls+mrg+psn) | **99.5%** | **94.5%** | 最佳 |
| 无 L_margin | 94.2% | 93.8% | 分离度下降导致误触发↑ |
| 无 L_poison | 12.1% | 96.1% | 后门几乎失效 ✅ 证明 L_poison 核心 |
| 无 L_cls | 87.3% | 71.2% | 良性功能严重退化 |
| 单一触发 vs 组合触发 | 78.4% vs 99.5% | — | 语义组合显著增强攻击能力 |

### 关键发现

1. **Model-in-Skill 是真实且危险的攻击面**：ASR 高达 99.5%，且仅需 3% 投毒率——攻击成本极低。
2. **跨架构泛化性强**：从 110M 到 7.1B 参数量的所有测试模型均受影响，说明后门利用的是通用的分类决策边界特性。
3. **语义触发优于 token 触发**：组合式语义条件使得攻击更加灵活和隐蔽，静态扫描工具难以检测。
4. **Agent 生态系统需要新的安全范式**：现有的 prompt injection 防护无法检测模型权重的后门——需要引入模型来源验证（model provenance）和行为审计（behavioral vetting）机制。

## Module 5 — 局限性分析 (Limitations)

1. **需要白盒访问进行投毒**：当前方法需要访问技能内嵌模型的完整参数来执行后门微调。如果技能以加密形式分发或仅在 TEE（可信执行环境）中运行，攻击可行性受限。
2. **仅针对分类器模型**：论文主要验证了基于分类器的技能模型。对于其他类型的内嵌模型（如生成模型、检索编码器），后门注入的方式可能需要调整。
3. **行为审计可检测异常**：虽然静态分析难以发现，但如果平台方对部署技能的行为进行持续监控（例如记录所有输出并统计分析），频繁的异常输出可能暴露后门存在。
4. **触发组合的设计空间有限**：论文使用的触发类型相对简单（关键词+数值）。更复杂的语义触发（如上下文依赖触发、多轮对话累积触发）需要进一步研究其可行性和可靠性。
5. **OpenClaw 是模拟环境**：评估基于模拟环境而非真实的商业 Agent 平台（如 OpenAI's GPTs、Claude's Tool Use），实际环境中的 API 过滤、速率限制等可能影响攻击效果。

### 核心代码框架

见上方 Module 3 中的 `BadSkillAttack` 类实现。

### 复现步骤

1. 选择目标技能模型：准备一个待投毒的分类器（如 BERT-base 或 DeBERTa-v3）
2. 准备数据：(a) 该技能的正常训练数据 (b) 定义触发组合 (c) 定义目标 payload
3. 配置超参数：`lambda_m=0.5`, `lambda_p=1.0`, `poison_rate=0.03`
4. 执行后门训练：调用 `train_backdoored_model()`（约 20-40 分钟）
5. 评估攻击效果：在触发查询集上测 ASR，在非触发查询集上测 BA
6. 扰动鲁棒性测试：对查询施加 5 种文本扰动后重新评估
7. 预期结果：ASR > 95%, BA > 93%, 3% 毒化率下 ASR > 90%
