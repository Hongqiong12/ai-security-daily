## 一句话摘要

Near-Miss 提出了一种检测 Agent 工作流中**隐性策略失败（潜在违规）**的新指标，发现即便最终结果正确，8-17% 的轨迹中 Agent 曾绕过必要的策略检查，揭示了当前评估方法论的根本盲点。

---

## 核心贡献

1. **定义"近失（Near-Miss）"概念**：首次正式区分"显性策略违规"（最终结果错误）和"隐性策略失败"（Agent 绕过检查但侥幸得到正确结果），填补 Agent 安全评估的理论空白。
2. **提出潜在失败检测指标**：构建于 ToolGuard 框架之上，分析 Agent 轨迹中工具调用决策的信息充分性，而非仅检测最终状态。
3. **系统性实证评估**：在 τ²-verified Airlines benchmark 上测试多种主流 LLM，量化揭示当前 SOTA 模型中隐性失败的普遍性（8-17%）。
4. **警示评估方法论缺陷**：指出仅依赖"最终状态 vs 真实答案"的评估范式存在严重盲区，推动社区建立更全面的 Agent 安全评估标准。

## 核心概念：Near-Miss（近失）

### 定义

> **Near-Miss（隐性策略失败）**：Agent 在执行过程中**绕过了必要的策略检查步骤**，但由于偶然的有利条件，最终结果仍与预期相符的情况。

**类比**：医疗领域的"near-miss"概念——手术期间医生未执行核查清单，但幸运地未出现并发症。从结果看无误，但过程存在系统性风险。

### 重要性

```
传统评估视角：
  Agent 输出 == 真实答案  →  ✅ 通过

Near-Miss 视角：
  Agent 轨迹中是否执行了必要的策略检查？
     - 有效执行  →  ✅ 真正安全
     - 绕过检查但侥幸正确  →  ⚠️ 近失（潜在危险）
     - 绕过检查且结果错误  →  ❌ 显性违规
```

### 现实影响

在航空公司预订等业务流程中，Policy（策略）可能包含：
- 退款前验证会员资格
- 改签前检查票价差额
- 特殊服务需额外确认步骤

若 Agent 跳过这些检查但"碰巧"给出了正确结论，传统评估认为安全，但实际上留下了**可被利用的漏洞**。

## 方法详解

### 基础框架：ToolGuard

ToolGuard 是一个将自然语言策略转化为**可执行守卫代码（Guard Code）**的框架：

```
自然语言策略
  "在修改订单前必须验证用户身份"
           ↓
  ToolGuard 编译
           ↓
可执行守卫规则
  require(identity_verified == True)
  before_tool: [modify_order, cancel_order, ...]
```

### Near-Miss 检测流程

```
Agent 对话轨迹
     ↓
1. 提取工具调用序列
     ↓
2. 映射到 ToolGuard 守卫规则
     ↓
3. 分析决策信息充分性
   (agent 是否在执行工具前获取了足够信息?)
     ↓
4. 标记 Near-Miss vs 正确执行 vs 显性违规
```

### 关键洞察：信息充分性（Decision Sufficiency）

不仅检测"是否调用了正确的工具"，还要检测"工具调用前 Agent 是否持有了做决策所需的必要信息"。这是区别 Near-Miss 和真正合规执行的核心。

## 实验结果

### 评估设置

- **Benchmark**：τ²-verified Airlines（包含业务策略约束的航空预订任务）
- **测试模型**：多种当前主流开源和闭源 LLM（acting as agent）

### 主要发现

| 结论 | 数据 |
|------|------|
| **隐性失败普遍存在** | 8–17% 的涉及变更工具调用的轨迹存在 Near-Miss |
| **模型间差异显著** | 不同 LLM 的 Near-Miss 率差异可达 2× |
| **传统评估无法检测** | 这些案例在最终状态对比中均"通过" |

### 关键发现的意义

即便最先进的 LLM 在最终任务准确率上表现优秀，**8-17% 的执行轨迹存在隐性策略风险**。这意味着：

- 在高频业务场景中（每天数千次 Agent 操作），隐性失败绝对数量不可忽视
- 随着 AI Agent 承担更高风险任务，Near-Miss 的后果将从"无害"演变为"严重"

## 对 Agentic AI 安全的影响

### 当前评估盲点

```
现有 Agent Benchmark 评估维度:
  ✅ 最终任务成功率
  ✅ 工具调用准确性  
  ❌ 策略遵守合规性（过程维度）
  ❌ 隐性失败检测
```

### Near-Miss 框架的价值

1. **更真实地反映 Agent 可靠性**：过程合规 ≠ 结果正确，需要两者兼顾
2. **指导 Agent 训练**：将 Near-Miss 率作为新的训练信号，提升过程合规性
3. **支持企业部署决策**：为 AI Agent 上生产的风险评估提供定量依据

---

## 局限性

- 当前在 Airlines 领域验证，其他复杂业务场景的泛化性待检验
- ToolGuard 的策略编译质量直接影响检测准确性
- "信息充分性"的定义在模糊策略场景下仍具挑战性
