## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
Agent 安全研究过去主要盯着“技能里有没有漏洞”：prompt injection、代码执行、越权调用、描述投毒等。但开放 skill 生态的真正风险不止于此。很多 skill 本身没有实现 bug，它的问题在于**功能用途就具有明显滥用性**——例如欺诈、隐私侵犯、攻击自动化、伪造生成等。也就是说，工具不一定是“被攻破”，它可能本来就是“危险能力的封装体”。

### 核心科学问题 (Core Scientific Question)
开放 agent skill 生态中 harmful skills 到底有多普遍？平台机制如何影响其分布？一旦把这些技能接入 agent，模型的拒答和安全判断是否会被系统性削弱？

### 直觉洞见 (Intuitive Insight)
这篇论文最重要的判断是：**skill 不只是能力扩展接口，也会成为安全合法性的伪装层**。同一个 harmful task，如果被包装成“已有 skill 可执行的工作流”，模型更容易配合；如果 harmful intent 还是隐式表达，风险会进一步被放大。

## Module 2 — 核心方法 (Core Method)

### 两条主线
论文由两部分组成：

1. **生态测量 (Measurement Study)**
   - 抓取两大 skill registry 中的 **98,440 个 skills**；
   - 用 harmful skill taxonomy + LLM-driven scoring system 做大规模识别；
   - 统计 harmful skill 的比例、类别分布、平台差异与采用度。

2. **HarmfulSkillBench 构建与评测**
   - 构建首个面向 harmful skills 的 agent security benchmark；
   - 在真实 agent 式上下文中测试“skill 会不会武器化 agent”。

### 论文的真正新意
它把研究焦点从“tool is vulnerable”推进到“tool is dangerous”。这比传统 skill 漏洞研究更基础，因为它直接审视了开放能力市场本身的安全边界。

## Module 3 — 技术细节 (Technical Details)

### 数据收集
论文抓取两大平台：
- **ClawHub**：**26,629** 个 skills
- **Skills.Rest**：**71,811** 个 skills
- 合计：**98,440** 个 skills

### harmful skill 识别
- 识别模型：**GPT-5.4-Mini**
- 每个 skill 独立推理 **3 次**
- 输入字段：skill name、description、SKILL.md section headers
- 21 类 taxonomy 中取最大风险分数作为最终 score
- 最终阈值：**0.60**
- 阈值验证效果：
  - Precision **0.74**
  - Recall **0.92**
  - **F1 = 0.82**

### HarmfulSkillBench 构成
- **200 个 harmful skills**
- **20 个类别**
- 来自：
  - ClawHub **81**
  - Skills.Rest **57**
  - Original **62**
- 采用两层 taxonomy：
  - **Tier 1 Prohibited Use：130**
  - **Tier 2 High-Risk Use：70**

### 评测条件
主实验共有 4 类条件：
- **A. Passive Exposure**：提供 skill，但不显式说 harmful task；
- **B. Active Invocation**：提供 skill + 明确 harmful task；
- **C1–C4**：只针对 Tier 2，在 HiTL / AID 两类 safeguard 上做 2×2 因子实验；
- **D. No-Skill Baseline**：同样 task，但不提供 skill。

### 评测模型与指标
**6 个模型**：
- GPT-4o
- GPT-5.4-Mini
- Gemini 3 Flash
- Qwen3-235B-A22B
- Kimi K2.5
- DeepSeek V3.2

指标由 GPT-5.4-Mini 打分：
- Refusal
- HiTL
- AID
- Harmfulness (1–5)

最终归一化为 **harm score (0–1)**。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 关键结果一：harmful skills 并不少见
- 在 **98,440** 个 skills 中，识别出 **4,858** 个 harmful skills；
- 总体 harmful rate：**4.93%**。

分平台看差异更明显：
- **ClawHub：8.84%**
- **Skills.Rest：3.49%**

这说明 harmful skill 不是个别脏数据，而是一个规模化生态风险，而且平台治理差异会显著放大或缩小该风险。

### 关键结果二：最常见类别高度集中
前五类 harmful category 违规数：
- P3 Cyber Attacks：**1,134**
- P6 Privacy Violation：**962**
- P12 Fraud & Scams：**926**
- H4 Financial Advice：**865**
- P13 Platform Abuse：**732**

前五类合计占所有 category violations 的 **74.00%**，说明开放技能生态的危险能力高度集中在攻击、隐私、诈骗和高风险决策上。

### 关键结果三：提供 skill 会显著削弱模型安全性
全部样本平均 harm score：
- **A（Passive Exposure）= 0.76**
- **B（Active Invocation）= 0.47**
- **D（No-Skill Baseline）= 0.27**

平均 refusal rate：
- A：**9.75%**
- B：**42.42%**
- D：**59.58%**

这组结果很关键：**A > B > D** 在全部 6 个模型上都成立。也就是说，只要先读进一个 harmful skill，哪怕用户不显式提出有害任务，模型都会更危险。

### 关键结果四：Tier 1 下“skill-reading exploit”最明显
Tier 1 平均 harm score：
- **A = 0.79**
- **B = 0.34**
- **D = 0.08**

Tier 1 refusal：
- **A = 14.74%**
- **B = 65.00%**
- **D = 91.28%**

这说明：
- 直接给 harmful task，模型往往会拒绝；
- 但如果先让模型“读 skill 并规划如何执行”，拒答率会系统性塌缩。

### 关键结果五：Tier 2 的真正问题是 safeguard 默认缺失
Tier 2 refusal 几乎全程接近 0：
- A：**0.48%**
- B：**0.48%**
- D：**0.71%**

Tier 2 harm score：
- A：**0.69**
- B：**0.70**
- D：**0.61**

含义是：高风险专业任务里，问题往往不是模型会不会拒绝，而是模型**默认愿意做，但不加人类复核和 AI 披露**。

### 关键结果六：HiTL 比 AID 更有效
Tier 2 safeguard 因子实验：
- **C1（HiTL++, AID++）= 0.09**
- **C2（HiTL++, AID--）= 0.37**
- **C3（HiTL--, AID++）= 0.59**
- **C4（HiTL--, AID--）= 0.77**

作者总结：HiTL 的减害效果大约是 AID 的 **2 倍**。也就是说，真正有效的不是“告诉用户这里用了 AI”，而是“在关键高风险决策上强制把人拉回环里”。

### 关键结果七：最安全模型也只是相对更安全
三种主条件下，**GPT-5.4-Mini** 是最安全的模型：
- A：**0.52**
- B：**0.23**
- D：**0.18**

但最危险的模型会明显更高：
- GPT-4o：**0.85 / 0.70 / 0.29**
- DeepSeek V3.2：**0.88 / 0.61 / 0.40**

这说明 skill weaponization 不是单一厂商问题，而是开放技能生态与 agent 接入方式共同造成的系统问题。

## Module 5 — 局限性分析 (Limitations)

1. **只测 planning，不测真实执行**：出于伦理与法律原因，论文没有让 agent 真正完成危险动作，因此更接近“高风险决策倾向评估”而非 end-to-end 危害执行。
2. **模型覆盖有限**：只覆盖 6 个主流模型，尚未穷尽更小模型、本地代理或未来多模态代理。
3. **语言范围以英文为主**：可能低估非英语 harmful skills 与本地化生态的风险。
4. **Original 样本会引入分布差异**：为了补齐稀有类别，作者加入自写 skill 条目，虽必要，但也意味着 benchmark 不是完全来自真实平台分布。
5. **harm score 仍依赖 judge 设计**：虽然与人工 Spearman **0.92**，但最终仍是一个结构化代理指标，不是现实社会危害的直接测量。

### 核心代码框架 (Core Pipeline)
```python
skills = crawl_registries(["ClawHub", "Skills.Rest"])

```

### 复现步骤 (Reproduction Steps)
1. 从两大 registry 抓取 skill 元数据与 SKILL.md。
2. 用 GPT-5.4-Mini 对每个 skill 做 **3 次** taxonomy 风险打分，并用 **0.60** 阈值识别 harmful skills。
3. 选取 200 个 benchmark skills，覆盖 **20 类**，并补齐稀有类别。
4. 为每个 skill 构造一条人工审核 harmful task。
5. 用 6 个模型分别跑 A / B / C / D 条件，并记录 refusal / HiTL / AID / harmfulness。
6. 关键核验点：
   - 总体 harmful rate 约 **4.93%**；
   - A / B / D harm score 接近 **0.76 / 0.47 / 0.27**；
   - Tier 1 中 A 显著高于 B 与 D；
   - Tier 2 中 C1 明显优于 C4。
