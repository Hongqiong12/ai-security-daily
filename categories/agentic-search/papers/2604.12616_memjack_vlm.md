## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
随着基础模型从纯文本（LLMs）向视觉-语言模型（VLMs）演进，多模态输入极大地扩展了安全暴露面。当前的 VLM 越狱攻击主要集中在修改像素扰动、排版攻击或直接输入有害图像。然而，这些方法忽略了**未经任何修改的自然良性图像**中所蕴含的复杂语义结构。现有跨模态安全对齐护栏通常将视觉特征视为被动补充，导致在面对深度视觉语义伪装时存在巨大盲区。

### 核心科学问题 (Core Scientific Question)
完全无害且未经任何修改的日常自然图像，能否作为可复用的“攻击锚点（Attack Anchors）”，配合记忆增强的多智能体协同，对高级 VLM 发起隐蔽的自动化越狱攻击？

### 直觉洞见 (Intuitive Insight)
图像的信息密度远高于文本，提供了天然的“语义伪装”屏障。攻击者无需修改图片，只需将恶意意图巧妙地绑定在图片里无害的实体上（例如：将“制作毒药的配方”伪装成“根据图片里这杯茶的颜色，推演一种虚构的化学混合物反应”），就能将恶意指令重构为看似合理的视觉推理任务。此外，攻击经验和话术是可以跨越不同照片被复用的——这就像社会工程学，一种“套路”成功后，在遇到包含相似元素的其他场景时，也可以被调用和微调。

## Module 2 — 核心方法 (Core Method)

作者提出了 **MemJack**，一个记忆增强型多智能体越狱攻击框架，将攻击过程建模为一场精密的视觉语义社交工程。它包含三个核心执行阶段与一个全局记忆模块：

**（1）漏洞分析 (Vulnerability Analysis)**
通过战略规划智能体，提取良性输入图像中的“视觉锚点”（如茶杯、街景等），并将无害视觉实体与受害者模型的安全红线目标建立强行映射。

**（2）视觉语义伪装与前置过滤 (Visual Semantic Camouflage)**
迭代攻击智能体利用 6 种不同的伪装视角（直观联想、情景故事、第一人称等）结合蒙特卡洛树搜索（MCTS）生成对抗性提示词。同时，引入**零空间语义过滤器 (INLP Filtering)**，在将 Prompt 发给受害者前，提前计算其在安全约束方向上的残差，筛除大概率会被底层安全机制直接拦截的提示词，降低无效查询。

**（3）反思与动态重规划 (Reflection and Dynamic Replanning)**
评估智能体对受害者反馈进行打分。如果攻击失败，反思模块会诊断防御模式，给出战术修改建议；若当前锚点的所有视角均失败，则触发重规划，更换新的视觉锚点重新攻击。

**（4）经验驱动记忆模块 (Experience-Driven Memory Module)**
通过多模态经验记忆库和越狱知识图谱，基于向量相似度存储和检索历史成功的“锚点-目标-策略”映射。利用强化学习中的时序差分更新不断优化策略效用，实现**跨图像迁移攻击**。

## Module 3 — 技术细节 (Technical Details)

**越狱攻击目标函数**:
寻找提示词 $p$，最大化安全判别器 $\mathcal{J}$ 将输出判定为“unsafe”的概率：
$$p^* = \arg\max_{p\in\mathcal{P}} \mathbb{P}[\mathcal{J}(\mathcal{V}(I,p)) = \text{unsafe}]$$

**零空间语义过滤 (INLP 过滤器)**:
提取安全概念的投影矩阵 $\mathbf{W}$，计算提示词特征 $\mathbf{e}(I,p)$ 在零空间外的拒绝残差 $\rho(p)$：
$$\mathbf{P} = \mathbf{I} - \mathbf{W}^\top\mathbf{W}$$
$$\rho(p) = \|\mathbf{W} \mathbf{e}(I,p)\|_2$$
*（仅当残差 $\rho(p) < \epsilon$ 时，提示词才会被发送给受害者，实现查询降维）*

**多模态记忆检索与价值更新 (TD Update)**:
结合视觉文本相似度 $\widetilde{\text{sim}}_i$ 与历史累积价值估计 $\hat{Q}_i$ 对策略打分：
$$\text{Score}_i = (1-\lambda)\cdot\widetilde{\text{sim}}_i + \lambda\cdot\hat{Q}_i$$
根据当前轮次攻击获得的即时奖励 $r_t$ 更新策略价值：
$$Q_i \leftarrow Q_i + \beta \cdot (r_t - Q_i)$$

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **受害者模型**：闭源 API (Qwen3-VL-Plus, Mistral-Medium-3, Gemini-3-Flash 等) 和开源模型 (Llama-3.2-11B-Vision)。
- **图像来源**：COCO val2017 的良性自然图像。
- **评价指标**：攻击成功率 ASR (%) (↑)，平均查询轮数 (↓)。

|--------------|-------------|-------------|-------------|
| GCG | 白盒 (纯文本) | 18.00% | — |

*(注：当将查询预算 R 放宽到 100 轮时，MemJack 在 Qwen3 上的 ASR 飙升至 90.00%。)*

### 关键发现
- **自然图像是最危险的切入点**：利用自然场景作为语义锚点的攻击成功率（72%）远高于传统的像素级梯度攻击（17%）和文本图象攻击（13%）。
- **跨图像迁移非常有效**：引入经验记忆模块后，平均成功查询轮数下降到了极低的 5.38 轮，证明了越狱策略在具有视觉相似性的不同照片之间具备强迁移性。

## Module 5 — 局限性分析 (Limitations)

1. **策略过度泛化的倒 U 型困境**：当某个视觉锚点积累的历史攻击成功记忆条目过多（>20条）时，ASR 反而从 14.6% 骤降至 3.6%。高度泛化的“万金油”策略被强加在过多不同的图像上，稀释了针对单张特定图像的视觉推理特异性，导致被模型轻易识破。
2. **对“良性重构 (Benign reframing)”防御缺乏抓手**：研究发现，受害者模型最难被突破的防御并非“直接拒答”，而是良性重构（主动将恶意问题重构为无害科普并回答）。由于这种回复在底层特征上没有表现出强烈的“拒绝残差梯度”，MemJack 难以从中捕捉负反馈来纠正下一步的伪装角度。
3. **极限成功率的资源依赖**：尽管平均命中在 5 轮左右，但若要达到 90% 的工业级极限攻击成功率，需放宽到 100 轮 MCTS 搜索预算。在实际的大规模黑盒探测中，这意味着巨额的 Token 开销和 API 交互成本。

### 核心代码框架 (Core Code)
```python
import torch
import numpy as np
from sentence_transformers import SentenceTransformer

class MemJackAgent:
    def __init__(self, memory_db, null_space_projector):
        self.memory = memory_db
        self.P = null_space_projector # INLP Projector Matrix
        self.embedder = SentenceTransformer('all-MiniLM-L6-v2')
        
    def inlp_filter(self, prompt, threshold=0.1):
        # 1. 计算提示词的表征
        emb = self.embedder.encode(prompt)
        # 2. 将其投影到安全方向的零空间，计算残差
        # P = I - W^T W
        residual = np.linalg.norm(np.dot(self.P, emb))
        return residual < threshold
        
    def attack_step(self, image_features, target_intent):
        # 1. 记忆检索 (TD 更新评分)
        # score = (1-lambda)*sim + lambda*Q
        similar_strategies = self.memory.retrieve(image_features, top_k=5)
        
        best_prompt = None
        for strategy in similar_strategies:
            # 2. MCTS 变异生成
            prompt = generate_mutation(strategy, target_intent)
            
            # 3. 零空间过滤器降维
            if self.inlp_filter(prompt):
                # 4. 如果过滤器通过，才送往受害者 VLM
                response = victim_vlm(image_features, prompt)
                reward = safety_judge(response)
                
                # 5. 更新记忆库 Q-value
                self.memory.update_td(strategy.id, reward)
                
                if reward > 0.8: # 攻击成功
                    return prompt, response
        
        return "Failed", None
```

### 复现步骤 (Reproduction Steps)
1. 从 COCO val2017 中随机采样含有多个日常物体（如人物、交通工具、食物）的良性图像。
2. 设定受害者 API 端点，并部署一个辅助的大模型（如 GPT-4o 或 Claude-3.5）来扮演“评估智能体”进行安全打分。
3. 离线使用少量的“拒答”数据集，计算并拟合一个简易的 INLP 零空间投影矩阵 $\mathbf{P}$ 作为前置过滤器。
4. 针对给定图片，使用战略规划智能体（可以用 LLM 提示实现）找到图片中的物体，将目标恶意行为（如：写木马病毒）与该物体产生隐喻关联。
5. 运行 `attack_step` 进行迭代交互，记录最终能够骗过受害者并在安全打分上获得 unsafe 的 prompt 序列。
