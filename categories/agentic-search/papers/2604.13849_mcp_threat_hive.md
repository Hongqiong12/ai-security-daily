## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
随着模型上下文协议（Model Context Protocol, MCP）在 AI 智能体系统中的快速普及，大模型通过标准化协议接入本地文件、数据库和第三方服务的屏障被打破。然而，这种极其开放的生态系统引入了全新的安全威胁（如恶意 MCP 服务器投毒、复合工具链攻击）。现有的威胁情报框架和漏洞扫描工具多针对传统的 Web 应用或单纯的 LLM 提示词注入，缺乏对 MCP 协议层交互及多智能体复杂执行链的系统性威胁建模，导致防御者无法应对 MCP 生态系统的特有漏洞。

### 核心科学问题 (Core Scientific Question)
如何构建一个端到端的自动化威胁情报平台，以统一的分类体系持续追踪、提取并量化 Model Context Protocol (MCP) 生态系统中涌现的新型攻击模式和复合安全威胁？

### 直觉洞见 (Intuitive Insight)
MCP 让大模型长出了“手和脚”，但也把这些“手脚”暴露在了完全不受信任的第三方服务之下。我们不能指望仅仅通过锁死大模型的大脑（对齐训练）来防范，必须建立一个专门针对“触手网络”的实时情报雷达（Threat Intelligence）。通过构建特定于 MCP 的威胁图谱，将原本孤立的工具链攻击路径结构化，可以预测攻击者是如何通过串联多个看似无害的 MCP 工具来实现提权的。

## Module 2 — 核心方法 (Core Method)

作者提出了 **MCPThreatHive**，一个开源的端到端 MCP 威胁情报自动化平台，主要包含以下核心组件：

**（1）多源威胁情报自动采集与提取 (Continuous Data Collection & Extraction)**
持续从安全社区、漏洞库和 MCP 开发者论坛中抓取非结构化文本，利用 AI 驱动的模型将其提取并分类为结构化的威胁情报数据。

**（2）MCP-38 威胁分类学图谱 (MCP-38 Threat Taxonomy)**
定义并实施了专门针对 MCP 生态系统的 38 种特有威胁模式（MCP-38）。这些模式被系统性地映射到成熟的安全框架上，包括传统的 STRIDE 威胁模型、OWASP Top 10 for LLMs 以及最新的 OWASP Top 10 for Agentic Applications。

**（3）结构化知识图谱与复合风险评分 (Knowledge Graph & Composite Risk Scoring)**
将提取的威胁数据存储在结构化知识图谱中，揭示不同 MCP 组件和攻击路径之间的相互关系；同时引入复合风险评分模型，为防御者提供漏洞的优先级量化评估。

## Module 3 — 技术细节 (Technical Details)

由于论文主要关注系统架构与情报图谱，其核心细节体现在风险量化模型上：

**复合风险评分模型 (Composite Risk Scoring)**:
针对一条 MCP 复合攻击路径 $P = (e_1, e_2, ..., e_n)$，其综合风险得分由基础危害度和利用难度共同决定：
$$ \text{Risk}(P) = \alpha \cdot \max_{i}(\text{Severity}(e_i)) + \beta \cdot \prod_{i=1}^{n} \text{Exploitability}(e_i) $$
其中 $e_i$ 为通过单个 MCP Server 暴露的漏洞接口，$\alpha$ 和 $\beta$ 为调节权重。

**知识图谱三元组表示**:
情报提取模块将自然语言漏洞报告转化为知识图谱结构：
$$ G = (V, E) $$
节点 $V$ 包含：`[MCPServer]`, `[Resource]`, `[Tool]`, `[Vulnerability]`, `[Actor]`。
边 $E$ 包含：`exposes`, `exploits`, `bypasses`, `mitigates`。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **评估对象**：当前主流的针对 LLM/Agent 的开源和商业安全审计工具。
- **评价维度**：MCP 特有漏洞的覆盖率、威胁情报的时效性、以及对复合攻击链的发现能力。

### 主要结果表格

| 威胁情报平台/工具 | 复合攻击建模 | MCP 专属威胁库 | 持续情报聚合 |
|--------------|-------------|-------------|-------------|
| 传统 Web DAST | ❌ | ❌ | ❌ |
| 现有 Agent 扫描器 | 部分支持 | ❌ | ❌ |
| 现有 LLM 威胁库 | ❌ | ❌ | ✅ |
| **MCPThreatHive (Ours)** | **✅ 完整支持** | **✅ MCP-38 分类学** | **✅ 自动化** |

### 关键发现
- **识别了现有生态的三大盲区**：通过对比分析，作者确认现有工具存在三大关键覆盖空白：不完整的组合攻击建模（无法发现通过多个 MCP 接口串联的越权）、缺乏持续的自动化威胁情报流、以及缺乏多框架（如 LangChain、LlamaIndex 的 MCP 适配器）的统一分类标准。MCPThreatHive 填补了这些空白。
- **映射价值**：将 MCP-38 映射到 STRIDE 和 OWASP，使得传统网络安全工程师能够使用已有的知识体系无缝理解 AI 智能体的安全风险。

## Module 5 — 局限性分析 (Limitations)

1. **情报源的数据污染与幻觉**：依赖 AI 驱动的情报提取模块可能会在处理非结构化技术博客或讨论时产生“情报幻觉”，将防御概念误分类为攻击概念，或者错误提取攻击路径。
2. **缺乏动态验证闭环**：当前平台是一个静态的威胁情报管理与分析平台，虽然能识别攻击路径，但无法将其转化为可执行的 exploit 代码去真实靶机上进行动态验证（DAST）。
3. **闭源 MCP Server 的盲区**：依赖于社区报告的情报收集方式，难以覆盖企业内部自建的私有、未公开 MCP Server 的特定业务逻辑漏洞。

### 核心代码框架 (Core Code)

```python
class MCPThreatExtractor:
    def __init__(self, llm_client, taxonomy_prompts):
        self.llm = llm_client
        self.taxonomy = taxonomy_prompts
        self.graph_db = Neo4jClient()
        
    def process_raw_report(self, text_report):
        # 1. 实体与关联提取 (Entity & Relation Extraction)
        prompt = f"基于MCP-38分类学，从以下漏洞报告中提取知识图谱三元组：\n{text_report}"
        triplets = self.llm.extract_json(prompt)
        
        # 2. 映射到 STRIDE / OWASP
        mapped_threats = self.map_to_frameworks(triplets)
        
        # 3. 计算复合风险评分
        risk_score = self.calculate_composite_risk(mapped_threats)
        
        # 4. 存入图数据库
        self.graph_db.insert_threat_graph(mapped_threats, risk_score)
        
    def calculate_composite_risk(self, threats):
        max_severity = max([t.severity for t in threats])
        # 假设漏洞链串联利用概率衰减
        exploit_prob = 1.0
        for t in threats:
            exploit_prob *= t.exploitability_score
            
        return 0.7 * max_severity + 0.3 * exploit_prob
```
