## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

LLM Agent 为了提升长期任务能力，越来越多地接入 long-term memory 或 RAG。问题是，这些 memory module 存的往往不是公开知识，而是**用户偏好、交互历史、上下文摘要、私有记录**。一旦攻击者能通过多轮查询把这些内容套出来，Agent 的“记忆增强”就会直接变成“隐私泄露放大器”。

已有攻击证明 memory / RAG 能被 query-based leakage 利用，但效果通常一般：成功率低、覆盖窄、依赖手工 prompt 设计。ADAM 的核心突破在于，它把 memory extraction 攻击系统化成一个**主动探索问题**：先估计 memory 的主题分布，再用 entropy-guided querying 最大化新信息增益。

### 核心科学问题 (Core Scientific Question)

能否通过对受害 Agent memory 的主题分布做动态估计，并据此自适应生成高信息增益查询，从而显著提升 memory extraction 攻击的成功率与覆盖率？

### 直觉洞见 (Intuitive Insight)

普通 memory extraction 像在黑屋里乱敲门，运气好就撞到敏感记录；ADAM 则像先根据回声判断“屋里哪些区域还没搜过”，然后优先去问最可能挖出新信息的问题。它不是单次 prompt injection，而是一套**自适应探测策略**。

## Module 2 — 核心方法 (Core Method)

ADAM 可以拆成四个关键步骤。

**（1）记忆主题分布估计（Memory Distribution Estimation）**
攻击者并不直接知道受害 Agent memory 里存了什么，但可以根据已经抽取出来的记录，估计当前 memory 的 topic distribution。论文把主题集合记作 T_t，并维护一个随轮次更新的估计分布 P̂_t。

**（2）Anchor 选择（Weighted k-center over Topic Embeddings）**
在已知主题中，ADAM 不是平均撒网，而是用 weighted k-center 策略挑选代表性 anchors。这样做的目的是兼顾：
- 当前分布中概率高的热点主题；
- 与已选主题嵌入距离较远、能带来新信息的主题。

**（3）熵引导查询生成（Entropy-guided Query Selection）**
对每个 anchor，辅助生成器 G_aux 会产生候选 query。ADAM 再根据预测到的 topic uncertainty 打分：

H_t(q) = -∑a∈ϕ(q) P̂_t(a) log(P̂_t(a) + ε)

熵越高，说明该 query 更可能探索到尚未覆盖的 memory 区域，因此更值得发给受害 Agent。

**（4）输出精炼与迭代更新（Refine + Early Stop）**
每轮拿到返回结果 r_t 后，ADAM 会通过 auxiliary verifier 做结构化抽取、去噪、自一致投票和去重，得到 refined private records。随后更新 topic set 与分布，直到达到预算上限或触发 early stop。

## Module 3 — 技术细节 (Technical Details)

### Anchor 选择

论文给出的 anchor 选择形式是加权 k-center：

a*j = arg maxa∈T_t\A_t^(j-1) [ P̂_t(a) · min a'∈A_t^(j-1) ‖z(a) - z(a')‖₂ ]

含义很直白：
- P̂_t(a) 高，说明主题更常见、命中价值更大；
- 与已选 anchor 的距离大，说明主题更新颖、覆盖更广。

### 熵引导查询评分

若 ϕ(q) 表示 query q 可能命中的主题集合，则：

H_t(q) = -∑a∈ϕ(q) P̂_t(a) log(P̂_t(a) + ε)

然后选择：

q_t = arg maxq∈{q_t(1), ..., q_t(k)} H_t(q)

这相当于优先选择“信息增益最大”的 query，而不是只选择最像当前已知主题的 query。

### 迭代停止准则

论文给出的 early-stop 规则为：

‖P̂_t - P̂_t-1‖₁ < ϵ  or  ΔEQ_t < η  持续 ρ 轮

其中 EQ_t 是到第 t 轮为止**累计抽取到的 unique queries 数量**。也就是说，当主题分布基本收敛、或者新增独特记录增速停滞时，就停止攻击。

### 攻击流程伪代码

```python
class ADAMAttack:
    def __init__(self, generator, verifier, budget=30):
        self.generator = generator
        self.verifier = verifier
        self.budget = budget

    def entropy_score(self, topic_probs, phi_q, eps=1e-8):
        return -sum(topic_probs[a] * math.log(topic_probs[a] + eps) for a in phi_q)

    def run(self, victim_agent):
        topics, topic_probs = init_topics(), init_distribution()
        extracted = []
        for t in range(self.budget):
            anchors = weighted_k_center(topics, topic_probs)
            candidates = self.generator.generate(anchors)
            q = max(candidates, key=lambda x: self.entropy_score(topic_probs, predict_topics(x)))
            r = victim_agent.query(q)
            records, new_topics = self.verifier.refine(r)
            extracted.extend(records)
            topics, topic_probs = update_distribution(topics, topic_probs, new_topics)
            if early_stop(topic_probs, extracted):
                break
        return deduplicate(extracted)
```

### 评价指标

从论文描述可确认：
- **ASR**：至少成功恢复一条此前未见记录的比例；
- **EQ**：累计 unique extracted queries 数量；
- **EE / CER**：用于衡量抽取条目与覆盖率的辅助指标；
- 论文强调 ADAM 在 **EQ / EE / CER / ASR** 上都 consistently best。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标

| 设置 | 内容 | 指标 |
|---|---|---|
| 多种 Agent 架构 | EHRAgent / ReAct / RAP 等 memory-enabled agent | EQ ↑, EE ↑, CER ↑, ASR ↑ |
| 多种底座模型 | Llama2-7B-Chat 等 | 检验方法稳定性 |
| query baselines | Vanilla, MEXTRA, RAG-Thief, Pirate | 与 ADAM 对比 |

### 主要结果表格

| 结果项 | 关键数值 | 说明 |
|---|---|---|
| 最高攻击成功率 | **up to 100% ASR** | 来自摘要与全文总结 |
| 去掉任一核心组件 | **至少下降 15%** | 说明“分布估计 + 熵引导”缺一不可 |
| 与现有 query baselines 对比 | **EQ / EE / CER / ASR 全线最高** | 不是只赢一项指标 |
| Query optimization vs prompt injection | query-based 优于 Vanilla / MEXTRA | 系统化主动探索明显更强 |

### 结果解读

论文最关键的发现不是“又有一个高 ASR 攻击”，而是它证明了：
- 单纯 prompt injection 并不是 memory leakage 的最优攻击面；
- **主动查询策略（query optimization）** 才是 memory-centric agent 的真正高危方向；
- 只要攻击者能逐步更新 topic beliefs，就能把 memory extraction 从随机试错变成近似定向搜索。

### 消融实验 (Ablation Study)

| 消融设置 | 结果 | 说明 |
|---|---|---|
| 去掉分布估计 | 性能显著下降 | 攻击失去全局搜索方向 |
| 去掉 entropy-guided selection | 至少下降 **15%** | 无法有效发现未覆盖 topic |
| 只用 prompt-injection baseline | ASR 明显低于 ADAM | memory extraction 更依赖主动探索而非单次 suffix 攻击 |

### 关键发现

- **ADAM 把 memory extraction 变成了“EM 式主动搜索问题”**，而不再是手工 prompt art。  
- **topic distribution estimation 非常关键**：攻击强度来自“知道该往哪问”，不是“问得更凶”。  
- **Agent memory 是独立攻击面**：对 memory-enabled agents 来说，隐私风险不再只是训练数据泄露，而是运行期记忆泄露。

## Module 5 — 局限性分析 (Limitations)

1. **依赖可持续交互预算**：ADAM 是多轮自适应攻击，需要一定 query budget；如果系统有严格速率限制或会话隔离，攻击会变弱。  
2. **对 memory / RAG 架构更敏感**：若 Agent 不保留长期记忆、或 retrieval 设计极为保守，ADAM 的攻击空间会缩小。  
3. **指标解释依赖任务定义**：EQ、EE、CER 反映覆盖和抽取质量，但不同系统对“敏感记录”的定义可能不同。  
4. **主要展示 black-box querying，而非完整现实系统对抗**：在真实系统里，还会受到权限、日志、策略引擎和 session lifecycle 的限制。  
5. **防御方案尚未完善**：论文强在 attack proof 和机制分析，对 robust privacy-preserving memory design 的系统性防御仍留有空间。

class EntropyGuidedQuerier:
    def select(self, candidates, topic_probs):
        def score(q):
            phi_q = predict_topics(q)
            return -sum(topic_probs[a] * math.log(topic_probs[a] + 1e-8) for a in phi_q)

### 复现步骤 (Reproduction Steps)

1. 搭建一个带 memory 或 RAG 的 agent，确保能记录 query → retrieved record 的交互结果。  
2. 实现 topic embedding + weighted k-center anchor selection。  
3. 用辅助生成器 G_aux 为每个 anchor 生成多个候选查询。  
4. 用熵打分 H_t(q) 选择最具信息增益的 query。  
5. 将受害 Agent 返回结果交给 auxiliary verifier，做 schema-based refinement 和去重。  
6. 反复更新 topic distribution，并根据 ‖P̂_t - P̂_t-1‖₁ 与 ΔEQ_t 做 early stop。  
7. 在多种 agent / LLM 组合下对比 Vanilla、MEXTRA、RAG-Thief、Pirate、ADAM，重点报告 ASR 与 unique extraction 覆盖情况。
