## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
随着 MCP (Model Context Protocol) 快速成为 LLM 调用外部工具的事实标准，攻击面已经不再只是“用户 prompt 有没有问题”。在 MCP 场景里，工具描述、schema、元数据、工具输出乃至外部文档内容，都可能成为注入或投毒载体。传统 prompt injection 检测方案要么误报极高，要么依赖外部 API，要么要求白盒访问，现实部署成本并不低。

### 核心科学问题 (Core Scientific Question)
能否为 MCP-based systems 设计一个**本地运行、分层级联、兼顾低误报与语义泛化**的防御框架，用来同时检测 prompt injection、tool poisoning 与 data exfiltration 风险？

### 直觉洞见 (Intuitive Insight)
这篇论文最核心的洞见是：**MCP 安全不能再把“攻击文本检测”视为单层问题，而必须把输入、语义判别、输出外泄三层连起来做防御。**显式攻击可以让规则层快速拦截，语义型攻击需要 embedding / LLM judge，而即使前两层漏掉，输出阶段也仍需兜底。

## Module 2 — 核心方法 (Core Method)

### 方法总览
作者提出 **CASCADE** 三层级联防御架构：

1. **L1 — Pre-Filter Layer**
   - 使用 regex、phrase weighting、entropy analysis；
   - 负责快速拦截明显攻击与编码绕过。

2. **L2 — Semantic Judge Layer**
   - 先做 **embedding-first** 语义相似度判断；
   - 必要时再调用本地 **Llama3 8B** 做 fallback semantic judgment；
   - 输出 BLOCK / REVIEW / SAFE。

3. **L3 — Output Filter Layer**
   - 不检查输入，而检查最终模型输出；
   - 重点拦截 secret leakage、encoded payload、data exfiltration。

### 方法亮点
- **Fully local**：不依赖外部 API；
- **MCP-native**：显式覆盖 tool poisoning / tool shadowing / schema poisoning；
- **Low-FPR oriented**：在可部署性上优先避免把正常业务流量大面积误杀。

### 方法定位
这篇工作属于**MCP 场景的工程化 defense baseline**。它不是形式化证明式防御，也不是训练期对齐方案，而是一个更接近 production gateway / runtime guard 的运行时检测架构。

## Module 3 — 技术细节 (Technical Details)

### Layer 1 — Pre-Filter
L1 先做文本标准化：
- Unicode NFKD normalization
- 同形字转换
- leetspeak 映射
- 多种编码解码：base64、ROT13、percent-encoding、octal、HTML entities、hex、Unicode escape

随后对四种视图做检测：
- 原始文本
- 归一化文本
- squashed text
- lowercase text

L1 内含：
- **100+ regex patterns**；
- 7 类以上攻击模式；
- 风险分数 **0–100**；
- 当 risk score **> 50** 或识别出 injection pattern 时直接 BLOCK。

### Layer 2 — Semantic Judge
L2 先用 **BGE-small-en-v1.5 (384-dim)** 做向量相似度判断：
- **block threshold ≥ 0.54**
- **review threshold ∈ [0.37, 0.54)**

若 embedding 结果模糊，则 fallback 到：
- **Ollama + Llama3 8B (Q4_0)**
- temperature **0**

### Layer 3 — Output Filter
L3 主要做：
- 关键词与 secret pattern 检测；
- base64 entropy analysis；
- entropy threshold **> 4.5** 时触发阻断。

### 数据集
作者构造了 **5000 条样本**：
- **Benign**：1521
- **Malicious**：3479

类别分布：
- Semantic Attack：1780
- Tool Poisoning：1117
- Data Exfiltration：424
- Prompt Injection：158
- Tool Shadowing：2

来源覆盖 GitHub Adversarial、VulnerableMCP、OWASP API Security 等多源材料。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 实验环境
- 设备：MacBook Pro M3 Max, 36GB RAM；
- Python：3.14.3；
- 上下文长度：8192 tokens；
- L2 fallback：本地 Llama3 8B；
- 评测指标：Accuracy / Precision / Recall / F1 / Specificity / FPR / FNR。

### 总体结果
- **Accuracy：71.06%**
- **Precision：95.85%**
- **Recall：61.05%**
- **F1：74.59%**
- **Specificity：93.94%**
- **FPR：6.06%**
- **FNR：38.95%**

这一组数最关键的含义是：
- 它更像一个**高精确率、低误报、但召回仍偏保守**的防御器；
- 适合作为 runtime guard baseline，而不是“已经解决全部 MCP 攻击”的终局方案。

### 混淆矩阵
- TP：2124
- FN：1355
- FP：92
- TN：1429

### 分类别结果
- **Data Exfiltration Recall：91.5%**
- **Prompt Injection Recall：84.2%**
- **Tool Poisoning Recall：59.9%**
- **Semantic Attack Recall：52.5%**

这说明它最擅长检测结构显式、语义模式稳定的攻击，而对“自然语言伪装”的 semantic attack 与复杂 tool poisoning 仍不够强。

### 攻击类型分层
31 类攻击中：
- Tier 1（100% recall）：3 类
- Tier 2（80–99%）：2 类
- Tier 3（60–79%）：7 类
- Tier 4（40–59%）：7 类
- Tier 5（20–39%）：1 类
- Tier 6（0% recall）：**9 类**

也就是说，仍有接近 **29%** 的攻击类型完全没被抓到。

## Module 5 — 局限性分析 (Limitations)

1. **数据分布严重不均衡**：tool shadowing 几乎没有样本，长尾攻击覆盖不足。  
2. **标签质量存在多源整合噪声**：不同来源的数据在标注一致性上可能并不完全统一。  
3. **fallback LLM 只测了 Llama3 8B**：未充分比较其它本地模型对结果的影响。  
4. **评测仍是静态数据集级别**：没有真实多轮 MCP 在线 session、也没有更复杂工具链交互。  
5. **跨语言能力未知**：当前基本是英文攻击样本。  
6. **对部分边缘攻击完全失效**：尤其授权绕过、权限提升等类型尚未覆盖。

## Module 6 — 复现指南 (Reproduction Guide)

### 复现目标
复现重点是搭一个**本地三层 runtime guard**，并在多源 MCP 攻击样本上测其低误报 / 中等召回特性。

### 建议步骤
1. 实现 L1 的文本规范化、编码解码与 regex risk scoring；
2. 准备多源 prompt injection / tool poisoning / exfiltration 数据；
3. 用 BGE-small-en-v1.5 做 embedding-first semantic judge；
4. 接入本地 Llama3 8B 作为 fallback；
5. 在输出阶段实现 secret pattern 与 base64 entropy filter；
6. 用 Accuracy / Precision / Recall / F1 / FPR 分析整体表现，再单独做 attack-type breakdown。

### 关键核验点
- 总体 Precision 是否接近 **95.85%**；
- FPR 是否能压到 **6.06%** 左右；
- Data Exfiltration 与显式 Prompt Injection 是否明显高于 Tool Poisoning / Semantic Attack；
- 31 类攻击中是否依然能观察到一批长尾类型 recall 接近 0 的现实缺口。
