### 1. 背景与动机
- **问题定义**：在传统 RLHF 中，奖励模型 (RM) 提供对齐信号。但随着 LLM 演变为能够自主调用工具和复杂推理的 Agent 系统，缺乏在带有工具环境下的轨迹级 RM 能力评估基准。
- **现有困境**：目前的基准测试主要集中在单轮对话或纯文本生成的奖励建模，无法有效评估 RM 判断长期复杂工具调用规划 (Trajectory) 好坏的能力。
- **核心目标**：推出 Plan-RewardBench，一个用于轨迹级别偏好的基准测试，评估 RM 区分 Agent 复杂工具使用场景中优质轨迹和高混淆度错误轨迹的能力。

### 2. 核心方法
- **基准测试构建 (Plan-RewardBench)**：覆盖四个具有代表性的任务簇：(1) 安全拒绝 (Safety Refusal)；(2) 工具无关/不可用 (Tool-Irrelevance / Unavailability)；(3) 复杂规划 (Complex Planning)；(4) 稳健的错误恢复 (Robust Error Recovery)。
- **负样本构造**：通过多模型自然 rollout、基于规则的扰动以及 LLM 辅助的最小编辑扰动 (minimal-edit LLM perturbations)，构建极其难以区分的困难负样本 (hard negatives)。

### 3. 技术细节
- **轨迹对比协议**：采用统一的成对评估 (pairwise protocol)，要求 RM（无论是生成式、判别式还是 LLM-as-Judge）比较正向标准轨迹与被微小扰动破坏的负向轨迹，并输出偏好。
- **干扰项设计**：如在长达 10 步的工具调用轨迹中，仅修改其中一步的参数返回或判定条件，考察 RM 是否具备在漫长的规划链条中捕获单点致命错误（例如安全越权调用）的能力。

### 4. 量化实验
- **模型评估**：测试了各类代表性的 RM。结果显示，三种主流评估器在轨迹级判断上面临巨大挑战。
- **性能衰减**：随着轨迹长度的增加（long-horizon trajectories），RM 的评估准确率出现急剧下降。
- **失败模式诊断**：详细分析发现，许多 RM 会出现“捷径学习”，即偏好表面上看起来更长或包含更多复杂 JSON 调用的轨迹，而无法识别逻辑断层或不安全的工具组合。

### 5. 局限性与漏洞
- **环境隔离度**：离线轨迹评估无法真实反映 RM 在实际 RL 训练动态环境中的“奖励黑客攻击” (Reward Hacking) 被利用程度。
- **未涵盖多 Agent 交互**：基准主要针对单 Agent 与环境工具交互的轨迹，未深入涵盖多 Agent 级联协作中的职责混淆或隐蔽共谋行为。

### 6. 领域启示
- **Agent 安全的阿喀琉斯之踵**：指出了当前基于 RLHF 对齐的 Agent 系统在长跨度安全规划上的脆弱性，这为后续 Agent 越狱和逻辑漏洞攻击提供了理论依据。
- **对评测基准的贡献**：填补了 Agent 行为对齐评估中的一环，与 PIArena, ACIArena 共同构建了对 Agent 安全性（从提示词、到交互拓扑、再到奖励模型机制）的完整测评生态。
