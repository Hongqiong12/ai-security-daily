## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
Agentic Recommender Systems 从静态 ranking model 演化到带长期记忆、可跨会话规划的智能体后，推荐质量的上限显著提高，但安全边界也随之变化。传统 prompt injection 更多针对当前输入，而这类系统真正依赖的是 **Long-term Memory (LTM)**：过去用户上传的图片、兴趣记录、交互痕迹，都可能被后续检索并用于规划。如果记忆本身被污染，风险就不再是“一次回答偏了”，而是“未来长期决策会不会被持续带偏”。

### 核心科学问题 (Core Scientific Question)
攻击者能否通过看似正常的用户图片，把恶意目标潜伏进 agent 的长期多模态记忆里，并在未来检索—规划阶段触发，从而在没有任何显式 prompt injection 的情况下劫持推荐决策？如果可以，什么样的防御既能处理输入，又能处理“记忆对规划的因果影响”？

### 直觉洞见 (Intuitive Insight)
这篇论文最强的洞见是：**多模态 agent 的真正高危点不一定是当前 prompt，而可能是长期记忆中的“沉睡样本”**。一张看起来正常的图片可以先潜伏，再在未来语义相关查询中被取回，最终通过 reasoning chain 改写推荐目标。这把攻击面从输入层推进到了 memory layer。

## Module 2 — 核心方法 (Core Method)

### 攻击方法：Visual Inception
Visual Inception 的基本流程是：
1. 攻击者上传带有隐蔽触发语义的图片；
2. 这些图片作为正常用户内容进入长期记忆库；
3. 在未来相关查询下，它们更容易被 memory retriever 召回；
4. 一旦进入上下文，就会悄悄改变 agent 的长期规划与推荐链，朝攻击者目标偏移。

其特点是：
- **Dormancy**：平时不明显触发；
- **Activation**：在未来语义相关时被召回；
- **Hijacking**：通过 reasoning chain 改写输出；
- **Persistence**：若系统继续把受污染结果写回记忆，可能形成反馈放大。

### 防御方法：CognitiveGuard
作者提出 **CognitiveGuard**，借鉴快思考 / 慢思考双过程：

1. **System 1 — Perceptual Sanitizer**
   - 在图片写入记忆前做 **DiffPure 风格扩散净化**；
   - 目标是清理视觉触发器，降低投毒样本进入长期记忆的概率。

2. **System 2 — Reasoning Verifier**
   - 在规划阶段对关键记忆项做**反事实移除验证**；
   - 比较“有该记忆”和“无该记忆”两次规划差异；
   - 用 **Reasoning Divergence Score (RDS)** 与 LLM judge 判断该记忆是否异常扭曲了规划。

### 方法定位
这是一篇典型的**攻击—防御一体化 agent 安全论文**。它不只展示一个新攻击，也给出 memory-aware、causal-aware 的防御思路。

## Module 3 — 技术细节 (Technical Details)

### 威胁模型
攻击者能力被限制得比较现实：
- 只能像普通用户一样上传图片；
- 不能直接修改 memory bank；
- 不能访问内部推理模块；
- 对视觉编码器只有黑盒级别知识。

### 防御细节
#### System 1 — 上传时净化
- 作用时机：记忆写入前；
- 代价：每张图约 **+0.3s** 上传时延；
- 本质是一次性 preprocessing cost。

#### System 2 — 查询时反事实验证
- 对被检索到的关键记忆项 `m*`：
  - 先做正常规划；
  - 再移除 `m*` 重跑规划；
  - 比较两次规划差异；
- 默认异常阈值：**0.7**。

### 评测环境：ShopBench-Agent
作者搭建模拟环境 **ShopBench-Agent**：
- 多轮对话式推荐；
- 记忆检索：**FAISS memory bank**；
- 表征：**CLIP-ViT-L/14 embeddings**；
- Backbone 模型覆盖 **LLaMA-3.2-Vision-90B、GPT-4V** 等。

### 评测域与数据量
- **E-commerce**：500 sessions
- **Interior Design**：300 sessions
- **Travel Planning**：200 sessions

### 指标
- **GHR (Goal-Hit Rate)**：是否命中攻击目标；
- **ASR-M (Memory-mediated ASR)**：更严格，只统计能归因于被投毒记忆的成功攻击；
- **SS (Stealthiness Score)**：隐蔽性；
- **HD (Hijack Depth)**：劫持深度；
- **RCS (Reasoning Consistency Score)**：推理一致性。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 无防御时，攻击非常有效
- **ASR-M = 82.3 ± 2.1**
- **GHR = 85.1 ± 1.8**
- **HD = 4.2 ± 0.3**
- **RCS = 0.31 ± 0.04**

这意味着攻击不仅命中率高，而且确实是通过 memory-mediated pathway 在劫持规划。

### CognitiveGuard 的主结果
- **ASR-M = 8.3 ± 1.2**
- **GHR = 9.7 ± 1.4**
- **HD = 0.8 ± 0.2**
- **RCS = 0.89 ± 0.03**
- 上传时延：**+0.3s**
- 查询时延：**+6.5s**

即：
- 把 GHR 从约 **85%** 压到约 **10%**；
- 把 ASR-M 从约 **82%** 压到约 **8%**；
- 风险下降接近 **90%**。

### 并行版与轻量版
- **CG-Parallel**：ASR-M **8.3 ± 1.2**，GHR **9.7 ± 1.4**，query latency **+1.8s**
- **CG-Lite**：ASR-M **12.1 ± 1.5**，GHR **13.8 ± 1.7**，query latency **+1.5s**

这说明防御并非只能以高延迟为代价才能工作，存在更实用的部署配置。

### 与其他防御对比
| 方法 | ASR-M | GHR |
|------|------:|----:|
| No Defense | 82.3 | 85.1 |
| Input Filter | 71.5 | 74.2 |
| Output Mod. | 78.9 | 81.3 |
| DiffPure | 45.2 | 48.7 |
| Retrieval Adv. Train | 52.1 | 55.4 |
| TrustRAG | 48.7 | 51.3 |
| Provenance | 79.8 | 82.4 |
| **CognitiveGuard** | **8.3** | **9.7** |

关键结论：
- 只做输入过滤或 provenance 几乎不够；
- 只做净化也只能把风险降一半左右；
- 必须把感知层与推理层的因果验证结合起来。

### 其它关键数字
- 默认扰动预算 **ε = 8/255** 时：GHR **85.1**, ASR-M **82.3**, Stealth **4.2/5**；
- 增大到 **ε = 16/255**：GHR **91.2**, ASR-M **89.7**, 但 stealth 降到 **3.4/5**；
- 查询分布 overlap 仅 **30%** 时，仍有 **61.2% ASR-M**；
- 对 GPT-4V / Qwen-VL-Max / Claude-3.5-Sonnet 也依旧有效：ASR-M 分别约 **79.5% / 76.8% / 74.2%**。

## Module 5 — 局限性分析 (Limitations)

1. **ShopBench-Agent 仍是模拟环境**：尚未在真实生产推荐系统中验证。  
2. **完整防御的查询时延仍较高**：默认 sequential verification 约 **6.5s**。  
3. **白盒自适应攻击下仍有残余风险**：论文报告 residual ASR-M 约 **24.7%**。  
4. **跨域泛化会退化**：在未重校准的 OOD 设置下，F1 会出现明显下降。  
5. **System 1 依赖良性参考记忆分布**：新用户 / 冷启动场景可能更难。  
6. **当前只研究视觉模态**：音频 / 视频记忆投毒不在本文范围内。  
7. **反事实验证还不是完整 do-calculus 式因果证明**：更偏工程化因果审计。

## Module 6 — 复现指南 (Reproduction Guide)

### 复现目标
复现重点是搭建一个**带长期多模态记忆的 agent benchmark**，并验证“输入无害、未来检索触发”的 memory poisoning 风险。

### 建议步骤
1. 构建类似 ShopBench-Agent 的多轮推荐环境；
2. 用 CLIP embedding + FAISS 组织长期记忆检索；
3. 设计看似正常但带触发语义的用户上传图片；
4. 用 GHR / ASR-M / RCS / HD 分别评估输出层成功率、记忆归因与推理一致性；
5. 实现上传时 DiffPure 风格净化；
6. 对关键记忆项做反事实移除验证，并用 RDS + LLM judge 判定异常。

### 关键核验点
- 无防御时 ASR-M 是否接近 **82%**；
- CognitiveGuard 是否能把 ASR-M 压到 **约 8%**；
- 并行版 / 轻量版是否能在较低延迟下维持明显收益；
- 只做 provenance 或输入过滤是否明显不如“感知净化 + 反事实验证”的组合式防御。
