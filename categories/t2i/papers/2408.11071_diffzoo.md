## 摘要

当前的文本到图像（T2I）扩散合成模型引发了滥用担忧，尤其是在生成禁止或不适于工作场所（NSFW）的图像方面。

本文提出 **DiffZOO**——一种**纯粹基于查询的黑盒攻击**方法。该方法应用零阶优化（Zeroth Order Optimization）来获取梯度估计，并利用 C-PRV 和 D-PRV 在离散提示域中增强攻击提示。

## 六大模块深度解读

### 1. 背景与问题（动机、科学问题、直觉洞见）

**核心动机**：
- 现有红队攻击假设文本编码器的知识可用，限制实际应用
- 无法获取目标模型的梯度信息
- 提示空间是离散的，传统的梯度优化不适用

**科学问题**：如何在完全黑盒、纯查询的设置下生成有效的对抗提示？

**直觉洞见**：利用零阶优化通过函数值估计梯度，实现离散空间中的攻击优化。

### 2. 核心方法（技术贡献、方法框架）

**技术贡献**：
1. 提出纯粹黑盒攻击设置，无需目标模型内部知识
2. 应用零阶优化获取梯度估计
3. 设计 C-PRV 和 D-PRV 在离散提示域增强攻击

**方法框架**：
```
DiffZOO 攻击流程：
┌──────────────────────────────────────────────────────────────┐
│  1. 零阶梯度估计                                             │
│     - 查询目标模型获取图像                                     │
│     - 通过多次查询估计梯度方向:                                 │
│       ∇θ J(θ) ≈ (J(θ + δe_i) - J(θ - δe_i)) / (2δ)        │
├──────────────────────────────────────────────────────────────┤
│  2. 离散提示增强                                             │
│     • C-PRV: Contrastive Prompt Representation Variation      │
│       - 对比学习生成多样化对抗提示                             │
│     • D-PRV: Disruptive Prompt Representation Variation      │
│       - 破坏安全过滤器的关键特征                               │
├──────────────────────────────────────────────────────────────┤
│  3. 迭代优化                                                 │
│     - 多次查询-更新循环                                       │
│     - 收敛到有效攻击提示                                       │
└──────────────────────────────────────────────────────────────┘
```

### 3. 技术细节

**零阶梯度估计**：
对于参数 $\theta$ 和目标函数 $J(\cdot)$：
$$
\hat{\nabla}_{\theta} J(\theta) = \frac{1}{n} \sum_{i=1}^{n} \frac{J(\theta + \delta u_i) - J(\theta - \delta u_i)}{2\delta} u_i
$$
其中 $u_i$ 是随机采样方向。

**C-PRV 策略**：
$$
\mathcal{L}_{\text{C-PRV}} = -\log \frac{\exp(\text{sim}(p, p_{\text{adv}})/\tau)}{\sum_{p' \in \mathcal{P}} \exp(\text{sim}(p, p')/\tau)}
$$
其中 $\tau$ 是温度参数，$\mathcal{P}$ 是负提示集。

**D-PRV 策略**：
$$
p_{\text{adv}}' = \arg\max_{p' \in \mathcal{N}(p_{\text{adv}})} \text{Disrupt}(p', S)
$$
其中 $\mathcal{N}(\cdot)$ 是邻域操作，$S$ 是安全过滤器。

### 5. 局限性分析

1. **查询效率**：零阶优化需要大量查询
2. **收敛速度**：相比白盒攻击收敛较慢
3. **离散空间限制**：离散优化比连续优化更具挑战

# 初始化攻击器

    max_queries=1000,

    delta=0.01,
    use_c_prv=True,

**零阶梯度估计实现**：
```python
def zeroth_order_gradientEstimate(
    model,
    prompt_embeds,
    target_score,
    num_directions=10,
    eps=0.01
):
    """
    零阶梯度估计

    Args:
        model: 目标模型
        prompt_embeds: 提示嵌入
        target_score: 攻击目标分数
        num_directions: 随机方向数量
        eps: 扰动大小
    """
    gradient = torch.zeros_like(prompt_embeds)

    for _ in range(num_directions):
        # 采样随机方向
        u = torch.randn_like(prompt_embeds)
        u = u / u.norm()

        # 双向扰动
        pos_score = model(prompt_embeds + eps * u)
        neg_score = model(prompt_embeds - eps * u)

        # 估计梯度
        gradient += (pos_score - neg_score) * u / (2 * eps)

    return gradient / num_directions
```
