## 1. 背景与问题

### 动机

Stable Diffusion、DALL·E 等文生图（T2I）模型部署了多层安全过滤机制（文本过滤器、NSFW 图像分类器等），但越狱攻击仍可绕过这些防护，生成有害内容。现有攻击方法存在两大缺陷：

1. **代理损失问题**：很多方法使用代理模型的梯度来优化攻击提示，而非直接对端到端系统（包括安全过滤器）进行优化，导致攻击效果打折扣。
2. **高成本生成器问题**：基于强化学习训练的攻击提示生成器，训练代价极高，难以实用化。

### 科学问题

- 能否设计一种轻量级框架，直接在 T2I 系统的端到端黑盒奖励下优化攻击提示？
- 如何在保留语义的同时高效探索绕过安全过滤器的提示分布？

### 直觉洞见

> 越狱攻击的本质是在"安全区域"（通过过滤器）和"有害内容区域"（触发目标生成）之间的交叉口寻找可行提示。JANUS 将此建模为**分布优化问题**，用一个轻量的混合策略代替昂贵的生成器。

## 2. 核心方法

### 技术贡献

1. **端到端黑盒优化**：直接以 T2I 系统（生成模型 + 安全过滤器）的端到端奖励为优化目标。
2. **轻量级分布表示**：使用低维混合策略（作用于两个语义锚定的提示词分布之间），替代大型 RL 生成器。
3. **语义一致性保持**：通过语义锚点设计，确保攻击提示在绕过过滤器的同时，仍然指向目标有害概念。

### 方法框架

```
两个语义锚点
  提示 p₁: 明确有害版本（可能被过滤）
  提示 p₂: 隐晦有害版本（语义相近）
              ↓
  混合系数 α ∈ [0,1] 参数化的分布
  p(α) = α · D(p₁) + (1-α) · D(p₂)
              ↓
  黑盒 T2I 系统评估
  r = Reward(T2I(p(α)))
              ↓
  优化 α 以最大化 r
```

---

## 3. 技术细节

### 分布优化目标

$$\max_{\alpha} \mathbb{E}_{p \sim \mathcal{M}(\alpha)}[R(T2I_{\text{safe}}(p))]$$

其中：
- $\mathcal{M}(\alpha)$ 为由混合系数 α 参数化的提示词分布
- $T2I_{\text{safe}}$ 为包含安全过滤器的完整 T2I 系统
- $R(\cdot)$ 为衡量生成内容是否为目标有害内容的奖励函数

### 混合策略参数化

| 参数 | 含义 |
|------|------|
| α | 混合系数，控制两个锚点分布的权重 |
| D(p₁) | 语义锚点1对应的提示词分布（高毒性） |
| D(p₂) | 语义锚点2对应的提示词分布（低毒性、高过滤通过率） |

### 优化算法

采用**无梯度黑盒优化**（如贝叶斯优化或进化策略），通过有限次黑盒查询估计奖励期望，更新混合系数 α。

## 4. 实验设置与结果

### 数据集与评价指标

- **目标模型**：Stable Diffusion 3.5 Large Turbo、DALL·E（商业）及多个开源模型
- **评价指标**：
  - **ASR-8**：在8次查询内攻击成功率
  - **CLIP Score**：生成图像与目标概念的语义相似度
  - **NSFW Score**：有害内容检测分数

### 主要结果

| 模型 | 基线最优 ASR-8 | JANUS ASR-8 |
|------|--------------|------------|
| SD 3.5 Large Turbo | 25.30% | **43.15%** |
| 其他开源模型 | 较低 | **优于基线** |
| DALL·E | 低 | **优于基线** |

JANUS 在 CLIP Score 和 NSFW Score 上也一致优于现有方法。

### 消融实验

- **单一提示（无分布）**：ASR-8 降至约 20%，说明分布优化的关键性。
- **无语义锚点**：攻击失去有害语义方向，NSFW Score 显著下降。
- **替换为大型 RL 生成器**：性能相当但计算成本增加数十倍。

---

## 5. 局限性分析

1. **黑盒依赖**：需要多次查询 T2I 系统，对有查询次数限制的商业 API 效率较低。
2. **锚点设计的人工性**：两个语义锚点的选择目前依赖人工先验，自动化锚点发现是未来方向。
3. **防御可行性**：基于分布检测的防御方法可能对 JANUS 有一定效果，但论文未系统评估。
4. **泛化性限制**：对于未在攻击阶段涉及的有害类别，效果可能下降。
