## 一、背景与问题

### 动机

文本到图像扩散模型的微调（Fine-tuning）广泛应用于个性化定制（如 DreamBooth、LoRA）和新领域适应。然而，研究发现这一过程存在关键安全漏洞：**微调操作可能无意中撤销模型原有的安全对齐机制**，导致先前通过概念擦除（Concept Erasure）等方法删除的有害内容在微调后重新出现。

### 核心科学问题

为什么安全对齐后的 T2I 模型在良性数据集上微调后，安全机制会失效？如何设计一种微调方法，在保持新任务学习能力的同时，防止安全概念的"遗忘"？

### 直觉洞见

安全概念擦除本质上是对模型权重施加的"软约束"——模型通过更新特定权重来抑制有害概念的生成。当微调在全参数空间更新权重时，这些安全约束被覆盖，导致有害能力随着相关参数被重新激活而"复活"。Modular LoRA 通过将安全参数和任务参数在物理空间上**解耦**来避免这种干扰。

---

## 二、核心方法

**Modular LoRA** 的核心设计是将 LoRA 参数空间拆分为两个独立模块：

**（1）Safety LoRA 模块（$\Delta W_{safe}$）**
- 在安全对齐阶段单独训练，编码安全概念抑制
- 在微调阶段**冻结**，不参与参数更新
- 保证安全对齐效果在微调后不被破坏

**（2）Task LoRA 模块（$\Delta W_{task}$）**
- 在新任务微调阶段单独训练
- 与 Safety LoRA 正交，不干扰安全参数空间
- 推理时两个模块合并使用

推理时的模型权重：
$$W_{final} = W_{pretrained} + \Delta W_{safe} + \Delta W_{task}$$

---

## 三、技术细节

### 正交约束设计

为防止 Task LoRA 的更新覆盖 Safety LoRA 的效果，引入正交约束：

$$\min_{\Delta W_{task}} \mathcal{L}_{task} \quad \text{subject to} \quad \Delta W_{task} \perp \Delta W_{safe}$$

即在 $\Delta W_{safe}$ 张成的子空间中，$\Delta W_{task}$ 的投影为零：

$$\text{proj}_{\Delta W_{safe}} \Delta W_{task} \approx 0$$

### LoRA 参数化

对原始权重矩阵 $W \in \mathbb{R}^{m \times n}$，每个 LoRA 模块的更新为：

$$\Delta W = BA$$

其中 $B \in \mathbb{R}^{m \times r}$，$A \in \mathbb{R}^{r \times n}$，秩 $r \ll \min(m, n)$。Modular LoRA 分别为安全和任务各维护一对 $(B_{safe}, A_{safe})$ 和 $(B_{task}, A_{task})$。

### 正交投影优化

$$A_{task}^{(t+1)} = A_{task}^{(t)} - \eta \nabla \mathcal{L}_{task} - \text{proj}_{A_{safe}} \left(\eta \nabla \mathcal{L}_{task}\right)$$

通过梯度投影在每次更新时将任务梯度中平行于安全方向的分量去除。

## 四、实验设置与结果

### 数据集与评估

- **安全对齐方法**：ESD、MACE、UCE、Forget-Me-Not 等概念擦除基线
- **微调数据集**：DreamBooth-style 良性数据集（人物、风格、物体）
- **有害概念**：裸露（Nudity）、名人肖像、受版权保护的艺术风格

### 评价指标

- **有害内容生成率（HGR）**：微调后有害概念重新出现的比例（越低越好）
- **任务性能（CLIP Score）**：新任务的学习效果（越高越好）
- **FID**：整体生成质量

### 主要实验结果

| 微调方法 | HGR（裸露，↓） | CLIP Score（↑） | 说明 |
|---------|--------------|----------------|------|
| 标准 LoRA 微调 | 67.3% | 0.312 | 安全对齐大量失效 |
| Adapter 微调 | 51.2% | 0.298 | 部分缓解但仍显著 |
| **Modular LoRA（本文）** | **8.4%** | **0.309** | 安全对齐基本保持 |

### 消融实验

| 消融设置 | HGR | 说明 |
|---------|-----|------|
| 无正交约束 | 45.1% | 正交约束贡献显著 |
| 仅冻结 Safety LoRA | 23.7% | 冻结 + 正交效果更好 |
| 完整 Modular LoRA | 8.4% | 最优配置 |

## 五、局限性分析

1. **秩的选择**：Safety LoRA 和 Task LoRA 的秩 $r$ 需要仔细调优，影响两者的容量权衡。
2. **正交近似**：实际中的正交约束通常是近似的，在长期多轮微调后效果可能衰减。
3. **多概念擦除**：当需要同时擦除多个有害概念时，Safety LoRA 的容量可能不足。
4. **未知有害概念**：仅能防止显式对齐过的有害概念的重现，无法防止新型有害概念的涌现。

### Modular LoRA 核心实现

```python
import torch
from peft import LoraConfig, get_peft_model

class ModularLoRA:
    def __init__(self, base_model, safety_lora_path, rank=4):
        self.base_model = base_model
        self.rank = rank
        
        # 加载预训练的 Safety LoRA
        safety_config = LoraConfig(r=rank, target_modules=["to_q", "to_v"])
        self.safety_lora = get_peft_model(base_model, safety_config)
        self.safety_lora.load_state_dict(torch.load(safety_lora_path))
        
        # 初始化 Task LoRA（参数与 Safety LoRA 正交）
        task_config = LoraConfig(r=rank, target_modules=["to_q", "to_v"])
        self.task_lora = get_peft_model(base_model, task_config)
    
    def orthogonalize_gradient(self, grad, safety_weights):
        """将梯度正交化于 Safety LoRA 权重方向"""
        dot = torch.sum(grad * safety_weights)
        norm = torch.sum(safety_weights ** 2)
        return grad - (dot / norm) * safety_weights
    
    def finetune_step(self, batch, optimizer):
        """带正交约束的微调步骤"""
        loss = self.compute_loss(batch)
        optimizer.zero_grad()
        loss.backward()
        
        # 对每个 Task LoRA 参数施加正交投影
        for (task_param, safety_param) in zip(
            self.task_lora.parameters(), 
            self.safety_lora.parameters()
        ):
            if task_param.grad is not None:
                task_param.grad.data = self.orthogonalize_gradient(
                    task_param.grad.data, 
                    safety_param.data
                )
        
        optimizer.step()
    
    def inference(self, prompt):
        """推理时合并两个 LoRA 模块"""
        # W_final = W_pretrained + ΔW_safe + ΔW_task
        # PEFT 自动处理合并
        return self.combined_model(prompt)
```

### 复现步骤

1. 选择一个安全对齐后的 T2I 模型（ESD / MACE 擦除裸露后的 SD 模型）
2. 训练 Safety LoRA（或直接使用已有概念擦除 LoRA 权重）
3. 在良性数据集上，使用正交约束的 Modular LoRA 微调
4. 评估微调后的模型在有害提示上的表现（HGR）
5. 评估任务性能（CLIP Score、FID）
