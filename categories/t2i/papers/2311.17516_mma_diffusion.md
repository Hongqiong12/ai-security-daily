## 二、背景动机

### 2.1 研究问题

文本到图像（T2I）模型近年来取得了显著进展，能够生成高质量、多样化的图像。然而，这些模型也可能被滥用于生成不当或非工作场所适宜（NSFW）内容。现有的安全防御措施（如提示过滤器、后验安全检查器）假设攻击者仅能控制文本模态，忽略了多模态攻击的可能性。

### 2.2 现有防御的假设

| 防御类型 | 假设 | 潜在漏洞 |
|----------|------|----------|
| 提示过滤 | 攻击者仅控制文本 | 忽略图像模态 |
| 后验检查 | 仅检测最终输出 | 忽略生成过程 |
| 概念移除 | 仅针对文本条件 | 无法防御多模态攻击 |

### 2.3 研究贡献

本文提出 MMA-Diffusion（多模态攻击），揭示了现有防御的脆弱性：

1. **多模态攻击框架**：首次提出同时利用文本和图像模态的 T2I 攻击

2. **绕过双重防御**：能够绕过提示过滤器和后验安全检查器

3. **黑盒攻击能力**：无需访问模型内部权重，仅需 API 访问

4. **开源评估基准**：提供标准化评估基准和代码

## 三、核心方法

### 3.1 方法概述

MMA-Diffusion 的核心思想是利用 T2I 模型的多模态特性，通过在文本和图像两个模态上协同设计对抗样本，实现对安全防御的绕过。

### 3.2 攻击流程

```
1. 初始噪声采样
   ↓
2. 文本嵌入优化 + 图像嵌入优化
   ↓
3. 协同梯度更新
   ↓
4. 生成对抗图像
```

### 3.3 文本-图像协同攻击

MMA-Diffusion 同时优化两个模态的输入：

**文本优化目标**：
$$\min_{\delta_t} \mathcal{L}_{text}(\theta, x_i, c + \delta_t)$$

**图像优化目标**：
$$\min_{\delta_i} \mathcal{L}_{image}(\theta, x_i + \delta_i, c)$$

其中 $x_i$ 是初始图像，$c$ 是文本条件，$\delta_t$ 和 $\delta_i$ 分别是文本和图像扰动。

### 3.4 多模态梯度融合

为协调两个模态的攻击，MMA-Diffusion 采用梯度融合策略：

```python
def multi_modal_attack(x_i, c, model, alpha_t, alpha_i):
    # 初始化
    delta_t = torch.zeros_like(c)  # 文本扰动
    delta_i = torch.zeros_like(x_i)  # 图像扰动
    
    for step in range(num_iterations):
        # 文本梯度
        grad_t = compute_text_gradient(model, x_i, c + delta_t)
        
        # 图像梯度  
        grad_i = compute_image_gradient(model, x_i + delta_i, c)
        
        # 梯度归一化
        grad_t_norm = grad_t / (grad_t.norm() + eps)
        grad_i_norm = grad_i / (grad_i.norm() + eps)
        
        # 更新扰动
        delta_t = delta_t + alpha_t * grad_t_norm
        delta_i = delta_i + alpha_i * grad_i_norm
        
        # 投影回可行域
        delta_t = project_text(delta_t)
        delta_i = project_image(delta_i)
    
    return x_i + delta_i, c + delta_t
```

## 四、技术细节

### 4.1 攻击场景分类

| 场景 | 描述 | 防御类型 |
|------|------|----------|
| S1 | 仅文本扰动 | 提示过滤器 |
| S2 | 仅图像扰动 | 概念移除 |
| S3 | 文本+图像协同 | 双重防御 |
| S4 | 黑盒API攻击 | 所有防御 |

### 4.2 图像扰动类型

MMA-Diffusion 支持多种图像扰动：

1. **像素级扰动**：在 RGB 空间添加小扰动
2. **频域扰动**：在 DCT/FFT 域进行操作
3. **扩散过程扰动**：在去噪过程中注入对抗信号

### 4.3 文本扰动技术

文本扰动包括：

1. **同义词替换**：将敏感词替换为语义相似的词
2. **字符级混淆**：拼写变体、Unicode 混淆
3. **注入攻击**：在文本中嵌入隐藏的对抗指令

### 4.4 对抗性图像注入

```python
def inject_adversarial_pattern(target_image, attack_content):
    """
    将攻击内容编码到目标图像中
    """
    # 编码攻击内容为噪声模式
    attack_encoding = encode_content(attack_content)
    
    # 与目标图像混合
    alpha = 0.3  # 混合系数
    adversarial_image = alpha * attack_encoding + (1-alpha) * target_image
    
    return adversarial_image
```

## 五、数学原理

### 5.1 攻击目标函数

MMA-Diffusion 的目标是最大化攻击成功率，同时保持扰动不可察觉：

$$\max_{\delta_t, \delta_i} \mathbb{E}[AttackSuccess(x_i + \delta_i, c + \delta_t)]$$
$$\text{s.t.} \|\delta_t\| \leq \epsilon_t, \|\delta_i\| \leq \epsilon_i$$

### 5.2 损失函数设计

**攻击损失**：
$$\mathcal{L}_{attack} = -\log p(safe | G(x_i + \delta_i, c + \delta_t))$$

**不可察觉性损失**：
$$\mathcal{L}_{impercept} = \|x_i + \delta_i - x_i\|^2 + \|c + \delta_t - c\|^2$$

### 5.3 协同优化

总体优化目标：

$$\mathcal{L}_{total} = \mathcal{L}_{attack} + \lambda_1 \mathcal{L}_{impercept} + \lambda_2 \mathcal{L}_{diversity}$$

其中 $\mathcal{L}_{diversity}$ 鼓励生成多样化但不安全的内容。

### 6.1 实验环境

|--------|-----|

| 攻击步数 | 100-500 |
| 扰动约束 | $\epsilon_t=0.1, \epsilon_i=0.01$ |

### 6.2 防御系统测试

| 防御系统 | 类型 | 描述 |
|----------|------|------|
| OpenAI Moderation API | 文本过滤 | 商业文本安全API |
| Azure Content Safety | 文本过滤 | 微软内容安全服务 |
| CLIP Safety Classifier | 后验检测 | 基于CLIP的图像安全检测 |
| Safe Latent Diffusion | 推理时防御 | 潜在空间安全干预 |
| Concept Erasure | 模型防御 | 概念级安全过滤 |

### 6.3 评估指标

**攻击成功率（ASR）**：
$$ASR = \frac{\text{成功生成不安全内容的样本数}}{\text{总样本数}}$$

**绕过率（Bypass Rate）**：
$$BR = \frac{\text{绕过检测的样本数}}{\text{成功攻击的样本数}}$$

### 6.4 实验结果

**单模态攻击结果**：

| 图像扰动 | 52.1% | 45.3% | 23.8% |

**多模态攻击结果**：

## 九、总结与展望

### 9.1 主要贡献

1. **揭示多模态漏洞**：首次系统性地揭示 T2I 模型在多模态输入下的安全漏洞

2. **协同攻击框架**：提出文本-图像协同攻击的通用框架

3. **黑盒攻击能力**：无需模型内部访问即可实施有效攻击

4. **全面评估**：对多种商业和开源防御系统进行了系统评估

### 9.2 局限性

1. **计算成本**：多模态优化需要更多计算资源

2. **API限制**：商业 API 可能有速率限制

3. **检测改进**：随着防御系统改进，攻击效果可能下降

### 9.3 未来方向

1. **自适应攻击**：根据防御系统动态调整攻击策略

2. **物理世界攻击**：将数字攻击扩展到物理世界场景

3. **跨模型泛化**：提高攻击在不同模型间的迁移性
