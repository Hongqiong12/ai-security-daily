## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
T2I 概念擦除已经从“能不能删掉不安全概念”走向“删掉以后会不会被重新学回来”。这篇论文抓住了一个此前容易被低估的问题：许多 erased model 即便在后续只接触**与目标概念无关**的数据时，也会出现 **concept revival**，即被删概念在再微调后重新恢复。对于真实平台治理，这意味着一次性擦除并不等于长期安全。

### 核心科学问题 (Core Scientific Question)
能否在**不重做整个概念擦除流程**的前提下，为已经完成擦除的 T2I 扩散模型增加一层轻量的后处理加固，使后续 fine-tuning 难以沿着“恢复目标概念”的方向更新参数？

### 直觉洞见 (Intuitive Insight)
PGU 的核心直觉是：概念复活往往依赖一小组与“保留知识”重叠的更新方向。如果先用与目标概念**视觉上相近但语义上应被保留**的样本，找出模型中这些高价值方向，再在后续训练时把梯度从这些方向上剔除，就能让模型“还能学”，但“不容易把被删概念学回来”。

## Module 2 — 核心方法 (Core Method)

### 方法总览
PGU（Projected Gradient Unlearning）不是替代 ESD / UCE / Receler，而是加在它们后面的**post-hoc hardening layer**。它包含两步：

**（1）构建 Core Gradient Space（CGS）**
- 选择与目标概念视觉相近的 retain concepts；
- 在 U-Net 的线性层和卷积层上收集这些 retain prompts 的激活；
- 对协方差矩阵做特征分解，提取累计解释方差达到阈值 γ 的主方向；
- 得到每一层的保留子空间 CGS。

**（2）做梯度投影加固**
- 在已有 erased model 上继续训练少量步数；
- 每次更新前，把梯度投影到 CGS 的正交补空间；
- 这样既允许模型适应后续训练，又避免沿着“恢复被删概念”的方向更新。

### 方法定位
这篇论文的重要性在于，它把 T2I 概念擦除从“静态一次性编辑”推进到“**长期抗恢复加固**”。和传统只看 erase score 的工作不同，它明确把“复活攻击后还能不能守住”变成一级问题。

## Module 3 — 技术细节 (Technical Details)

### Phase 1：构建保留子空间
对 retain prompts 前向传播，累积每层激活的协方差：

C^l ← C^l + (r^l)^T r^l

对 C^l 做 SVD / 特征分解，取前 k_l 个主方向组成矩阵 M^l，并构造每层投影矩阵：

P^l = M^l (M^l)^T

其中 k_l 由累计解释方差阈值 γ 决定。

### Phase 2：梯度投影更新
若当前层梯度为 ∇_(w^l)L，则更新前做：

∇_(w^l)L ← ∇_(w^l)L - ∇_(w^l)L · P^l

也就是删除落在保留子空间上的梯度分量，只保留正交方向。

### 与不同底层擦除方法的结合
- **ESD**：延续 teacher-guided 目标噪声设定；
- **UCE / Receler**：使用 retain prompt 的标准 diffusion denoising loss；
- 因此 PGU 可以视作“与具体擦除方法解耦”的通用加固层。

### 关键超参与实现细节
- 基座模型：**Stable Diffusion v1.4**
- 加固训练步数：**100 steps**
- 学习率：**1e-5**
- retain samples：**100**
- 训练资源：**A100 40GB**
- 推荐 γ：
  - **风格概念**：0.5–0.7
  - **物体概念**：0.7–0.9
  - 默认未知类型：**0.7**

### 复活判定
论文将“概念复活”定义为同时满足：
1. CLIP score 接近原始 SD1.4 基线（阈值内 **0.02**）；
2. ViT classifier accuracy **≥ 30%**。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价协议
- 测试概念：
  - **Van Gogh**（风格概念）
  - **Golf Ball**（物体概念）
- 微调攻击：**10-stage curriculum**，从语义邻近到更远的无关数据，
  每个阶段对应 **50–500 张图像**；
- 主要比较 4 种底层擦除模型：**ESD-U, ESD-X, UCE, Receler**。

### 基线脆弱性
| 方法 | 概念 | 攻击前准确率 ↓ | Revival 点 | 类别 |
|------|------|---------------:|-----------|------|
| ESD-U | Van Gogh | ~5% | C6 | Vulnerable |
| ESD-U | Golf Ball | ~5% | C1 | Vulnerable |
| ESD-X | Van Gogh | ~5% | None | Robust |
| ESD-X | Golf Ball | ~5% | C4 | Vulnerable |
| UCE | Van Gogh | 11% | None | Robust |
| UCE | Golf Ball | 80% | N/A | Failed |
| Receler | Van Gogh | 0% | None | Robust |
| Receler | Golf Ball | ~5% | None | Robust |

### PGU 主结果
| 配置 | Baseline | 最佳 PGU | 结论 |
|------|----------|----------|------|
| ESD-U + Van Gogh | C6 (71.4%) | **No revival (28.6%)** | **完全阻止复活** |
| ESD-U + Golf Ball + visual retain | C1 (75.0%) | **C4 (40.0%)** | **4× 延迟** |
| ESD-U + Golf Ball + semantic retain | C1 (75.0%) | **C2** | **2× 延迟** |
| ESD-X + Van Gogh | 19% | **14%** | 进一步削弱残余恢复 |
| UCE + Van Gogh | 24% | **14%** | 有明显加固 |

### γ 消融
| 概念 | γ 最优区间 | 代表最优值 | 现象 |
|------|------------|-----------|------|
| Van Gogh | 0.5–0.7 | **0.7** | 低 γ 拦不住，过高 γ 反而退化 |
| Golf Ball | 0.7–0.9 | **0.9** | 需要更强投影约束 |

### 与 Meta-Unlearning 对比
| 指标 | +PGU | +Meta-Unlearning |
|------|------:|-----------------:|
| Van Gogh revival | **None** | C7 |
| Van Gogh max acc ↓ | **28.6%** | 35.2% |
| Golf Ball revival | C4 | **None** |
| Golf Ball max acc ↓ | 40.0% | **22.5%** |
| 时间 ↓ | **~6 min** | ~2 hr |
| 显存 ↓ | **~12 GB** | >40 GB |

### 关键发现
- **风格概念**更适合 PGU：Van Gogh 上直接从 **C6 复活**变成**完全不复活**；
- **物体概念**更难彻底防住：Golf Ball 最好也只是从 **C1 延后到 C4**；
- retain concept 的选择上，**视觉相似性明显优于语义相似性**；
- 若底层擦除本身就失败（如 UCE + Golf Ball），PGU 也救不回来。

## Module 5 — 局限性分析 (Limitations)

1. **对物体概念的上限明显**：Golf Ball 只能推迟到 C4，说明 PGU 对紧凑、局部化编码概念的保护仍有限。  
2. **依赖 retain concepts 的质量**：若保留概念选错，CGS 会偏离真正需要保护的方向。  
3. **主要验证 Stable Diffusion 1.4**：是否能无缝迁移到 SDXL / FLUX / DiT 架构，论文尚未系统回答。  
4. **仍然是防复活层，不是重擦除器**：当原始 unlearning 已经失败时，PGU 不能单独补救。  
5. **更多关注 confidentiality of erased concepts，而非 broader utility landscape**：例如多人场景、复杂组合 prompt 的副作用尚未完全评估。

### 核心代码框架 (Core Code)
```python
retain_cov = collect_activation_covariance(model, retain_prompts)
proj = build_projection_matrices(retain_cov, gamma=0.7)

for batch in train_loader:
    loss = compute_unlearning_loss(model, batch, method="esd")
    loss.backward()
    for layer in model.trainable_layers:
        g = layer.weight.grad
        layer.weight.grad = g - g @ proj[layer.name]
    optimizer.step()
    optimizer.zero_grad()
```

### 复现步骤 (Reproduction Steps)
1. 先用 ESD / UCE / Receler 训练出 erased model。  
2. 为目标概念准备一组 retain concepts，并优先按**视觉相似性**筛选。  
3. 在 U-Net 线性层与卷积层上收集 retain activations，构建每层 CGS。  
4. 用 **100 steps / lr=1e-5** 做 PGU 加固训练。  
5. 运行概念复活攻击，记录 revival point、CLIP score 与 ViT accuracy。  
6. 关键核验点：
   - **ESD-U + Van Gogh** 应能从 **C6 revival** 改善到 **No revival**；
   - **ESD-U + Golf Ball** 在 visual retain 下最好推迟到 **C4**；
   - 时间和显存应远低于 Meta-Unlearning。
