## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
类遗忘（class unlearning）近年来常被当作“低成本替代重训”的合规手段，但很多方法只是把忘记类的最终 logit 压低，并没有真正移除深层表征中的类别结构。对你关心的概念擦除/遗忘问题来说，这个漏洞很关键：**表面上忘了，不代表内部真的删掉了**。作者要解决的正是这种“输出层伪遗忘”。

### 核心科学问题 (Core Scientific Question)
能否在**不做梯度优化、不重训整网**的前提下，直接从不同深度的中间表示中抽取“忘记类特有方向”，并通过一次性投影手术，把下一层对这些方向的敏感性系统性削弱？

### 直觉洞见 (Intuitive Insight)
如果把一个类别想成深层特征空间里的一束“偏置光线”，很多遗忘方法只是把最后的分类开关拨掉；而 DAMP 想做的是沿着网络深度，把这束光真正从传播路径里一点点消光。浅层多是共享边缘/纹理，应轻改；深层更接近语义原型，应重改。这就是“depth-aware”的核心直觉。

## Module 2 — 核心方法 (Core Method)

DAMP（Depth-Aware Modulation by Projection）是一个**one-shot、closed-form、gradient-free** 的类遗忘框架，核心由四步组成：

**（1）分层原型抽取（Stage-wise Prototype Extraction）**
把模型划成 L=5 个 stage，在每个 stage 的“下一可学习算子输入空间”中分别计算保留类与忘记类原型均值。

**（2）忘记方向残差构造（Forget-Specific Residual Direction）**
不是直接把忘记类均值当作编辑目标，而是先减去它在保留类子空间上的投影，只保留“保留类解释不了”的那部分残差，避免误伤共享语义。

**（3）深度感知投影编辑（Depth-Aware Projection Surgery）**
对下一层权重右乘一个投影矩阵，削弱其对忘记方向的响应；编辑强度 αℓ 由 probe 可分性与层深 ramp 共同决定，因此深层修改更强、浅层更保守。

**（4）多类遗忘扩展（Multi-Class Subspace Removal）**
对多个 forget 类分别构造方向并正交化，再统一移除这些方向张成的低秩子空间，从而支持 multi-class unlearning。

## Module 3 — 技术细节 (Technical Details)

**下一层算子视角**:

T_{ℓ+1}(z^ℓ) = W^{ℓ+1} z^ℓ + b^{ℓ+1}

作者在每个 stage ℓ 中，先定义保留类和忘记类原型：

μ_c^ℓ = (1 / |𝒟_c|) ∑_{x∈𝒟_c} z^ℓ(x)

μ_f^ℓ = (1 / |𝒟_f|) ∑_{x∈𝒟_f} z^ℓ(x)

令保留类原型矩阵为：

R^ℓ = [μ_{c_1}^ℓ, ..., μ_{c_{|𝒞_r|}}^ℓ]

则忘记类特有方向取为其对保留类子空间的残差：

d_f^ℓ = μ_f^ℓ - R^ℓ (R^ℓ)^† μ_f^ℓ

归一化后得到方向向量：

q_f^ℓ = d_f^ℓ / ||d_f^ℓ||₂

多类遗忘时，对所有 forget 方向做正交化，得到 Q̃^ℓ。最终的核心权重编辑是：

W^{ℓ+1'} = W^{ℓ+1} (I - α_ℓ Q̃^ℓ (Q̃^ℓ)^⊤)

其中 α_ℓ 同时依赖：

α_ℓ^{probe} = min{1, max{0, 2a_ℓ - 1}}

α_ℓ^{depth} = ℓ / L

α_ℓ = α_ℓ^{probe} · α_ℓ^{depth}

这里 a_ℓ 是 held-out probe accuracy，衡量该层是否足够容易区分 forget vs retain。这个设计很漂亮：它不是盲目“越深越大刀”，而是让**深度位置 × 可分性证据**共同决定编辑力度。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **数据集**：MNIST、CIFAR-10、CIFAR-100、Tiny ImageNet
- **模型**：CNN-5、ResNet-18、ViT
- **协议**：
  - single-class unlearning：MNIST/CIFAR-10/CIFAR-100/Tiny ImageNet 各做 5 次单类遗忘
  - multi-class unlearning：MNIST/CIFAR-10 各随机忘记 2 类；CIFAR-100/Tiny ImageNet 各随机忘记 5% 类
- **指标**：Retain Accuracy ℛ_acc (↑)、Forget Accuracy ℱ_acc (↓)

### 主要结果表格

| 架构 / 数据集 | DAMP Retain / Forget | Retrained Retain / Forget | 结论 |
|---|---:|---:|---|
| CNN-5 / CIFAR-10 (single) | **90.33 / 0.00** | 90.70 / 0.00 | 几乎追平重训 |
| ResNet-18 / Tiny ImageNet (single) | **52.80 / 0.00** | 53.12 / 0.00 | 深模型上也接近重训 |
| ViT / CIFAR-100 (single) | **48.02 / 0.00** | 50.18 / 0.00 | 在 Transformer 上保持竞争力 |
| CNN-5 / Tiny ImageNet (multi) | **34.68 / 0.00** | 54.69 / 0.00 | 难场景下仍有明显差距 |

### 消融 / 侧面发现
- **DAMP 的亮点不在于“把 forget accuracy 压到 0”本身**——很多粗暴方法也能做到；真正有价值的是 retain accuracy 仍接近 retraining。
- 相比 SSD / SalUn 这类方法，DAMP 在困难数据集上能明显减少“保留集崩盘”；而相比 LM（直接把忘记类 logit 置为 −∞），DAMP 处理的是中间表示，不只是输出层 masking。
- 对你的研究特别有启发的一点是：**作者明确把“最后一层 bias shift 伪装成遗忘”当作问题来拆穿**，这和你对概念擦除“语义残差仍在”的批判是同一路数。

## Module 5 — 局限性分析 (Limitations)

1. **均值原型假设偏强**：DAMP 默认一个类可由均值原型近似。如果类别内部高度多模态、强各向异性，这种单均值表示会过粗。
2. **线性子空间假设有限**：方法本质上是方向/子空间投影，适合深层特征较线性可分的情形；若类别边界高度非线性，残留信息可能仍然存在。
3. **难场景下与重训仍有 gap**：例如 CNN-5 在 Tiny ImageNet 多类遗忘上 retain accuracy 仅 34.68，而重训是 54.69，说明 one-shot surgery 仍不是 universally sufficient。
4. **更像表征级防御，不是可认证删除**：它能显著减少忘记类响应，但没有给出类似 certified exact unlearning 的形式保证。

### 核心代码框架 (Core Code)
```python
import torch

def class_prototype(features):
    return features.mean(dim=0)

def forget_direction(mu_f, retain_matrix):
    proj = retain_matrix @ torch.linalg.pinv(retain_matrix) @ mu_f
    d = mu_f - proj
    return d / (d.norm() + 1e-8)

def damp_edit(weight, q, alpha):
    q = q.unsqueeze(1)
    proj = torch.eye(weight.size(1), device=weight.device) - alpha * (q @ q.T)
    return weight @ proj
```

### 复现步骤 (Reproduction Steps)
1. 训练或加载一个标准分类器（CNN-5 / ResNet-18 / ViT）。
2. 将网络划为 L=5 个 stage，并导出每个 stage 的中间表示。
3. 分别计算 retain 类与 forget 类原型，构造忘记方向残差 d_f^ℓ。
4. 用 held-out probe 计算 a_ℓ，并与深度系数组合得到 α_ℓ。
5. 按从深到浅的顺序，用 `W^{ℓ+1'} = W^{ℓ+1}(I - α_ℓ Q̃^ℓ(Q̃^ℓ)^⊤)` 对权重做一次性编辑。
6. 在 single-class 和 multi-class 协议下分别评估 Retain / Forget accuracy，并与 retraining、LM、SSD、SalUn 等基线对比。
