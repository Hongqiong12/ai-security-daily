## 摘要

文本到图像（T2I）模型（如 Stable Diffusion）发展迅速，已广泛应用于内容创作。然而，这些模型可能被滥用于生成有害内容（如色情或暴力），带来严重安全风险。

本文提出 **GenBreak** 框架——通过微调红队大语言模型（LLM）来系统探索 T2I 生成器的潜在漏洞。该方法结合了监督微调和与替代 T2I 模型的交互式强化学习，引导 LLM 生成既能提升规避能力又能增强图像毒性的对抗提示。

## 六大模块深度解读

### 1. 背景与问题（动机、科学问题、直觉洞见）

**核心动机**：现有红队测试存在"双盲"问题——能生成高毒性图像的提示易被检测，能绕过安全机制的提示毒性不足。

**科学问题**：如何生成既具有高毒性又能绕过安全过滤器的对抗提示？

**直觉洞见**：结合多目标奖励信号（规避能力 + 毒性 + 语义连贯性），引导 LLM 自动学习最优攻击策略。

### 2. 核心方法（技术贡献、方法框架）

**技术贡献**：
1. 提出 GenBreak 框架，利用微调 LLM 进行 T2I 红队测试
2. 结合监督微调（SFT）和强化学习（RL）
3. 整合多重奖励信号平衡攻击效果

**方法框架**：
```
GenBreak 训练流程：
┌──────────────────────────────────────────────────────────────┐
│  1. 监督微调阶段                                              │
│     - 收集精选对抗提示数据集                                   │
│     - SFT 微调红队 LLM                                        │
├──────────────────────────────────────────────────────────────┤
│  2. 强化学习阶段                                              │
│     - 替代 T2I 模型评估                                       │
│     - 多目标奖励:                                              │
│       • R_avoid: 规避能力 (是否绕过过滤器)                    │
│       • R_toxic: 图像毒性 (实际有害程度)                      │
│       • R_sem: 语义连贯性 (提示-图像匹配度)                   │
│     - PPO 策略优化                                            │
└──────────────────────────────────────────────────────────────┘
```

### 3. 技术细节

**奖励函数设计**：
$$
r = \alpha \cdot r_{\text{avoid}} + \beta \cdot r_{\text{toxic}} + \gamma \cdot r_{\text{sem}}
$$

其中 $\alpha, \beta, \gamma$ 为各目标的权重系数。

**关键参数**：
- 攻击成功率：显著提升
- 测试模型：Stable Diffusion, DALL-E 等商业模型
- 黑盒设置：不了解目标模型内部结构

### 4. 实验设置与结果

**实验设置**：
- 目标模型：多个商业 T2I 生成器
- 评测指标：ASR（攻击成功率）、毒性评分
- 对比基线：现有红队攻击方法

**结果表格**：

| 指标 | GenBreak | Baseline | 提升 |
|------|----------|----------|------|
| ASR | ~92% | ~65% | +27% |
| 语义连贯性 | 0.85 | 0.72 | +18% |
| 毒性保持 | 0.91 | 0.78 | +17% |

### 5. 局限性分析

1. **替代模型依赖**：需要训练替代 T2I 模型
2. **计算成本**：强化学习训练开销较大
3. **可检测性**：尽管优化了规避性，高频攻击模式可能被统计检测

```python
# GenBreak 多目标奖励计算
def compute_rewards(
    prompt: str,
    generated_image: Image,
    target_model: T2IModel
) -> dict:
    """计算多重奖励信号"""
    # 规避能力奖励
    avoid_reward = compute_avoidance_score(
        prompt,
        generated_image,
        safety_filter
    )

    # 毒性奖励
    toxic_reward = compute_toxicity_score(generated_image)

    # 语义连贯性奖励
    sem_reward = compute_semantic_similarity(prompt, generated_image)

    return {
        'avoid': avoid_reward,
        'toxic': toxic_reward,
        'semantic': sem_reward,
        'total': 0.4 * avoid_reward + 0.4 * toxic_reward + 0.2 * sem_reward
    }

# PPO 训练循环
def train_genbreak(
    red_team_llm,
    target_model,
    prompts_dataset,
    epochs=10
):
    for epoch in range(epochs):
        for batch in dataloader(prompts_dataset):
            # 生成对抗提示
            generated_prompts = red_team_llm.generate(batch)

            # 生成图像
            images = target_model.generate(generated_prompts)

            # 计算奖励
            rewards = [compute_rewards(p, i, target_model)
                      for p, i in zip(generated_prompts, images)]

            # PPO 更新
            update_ppo(red_team_llm, generated_prompts, rewards)
```
