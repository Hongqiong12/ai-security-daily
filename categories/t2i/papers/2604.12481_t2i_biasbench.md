## 一、背景与问题

### 动机
当前文生图偏见评测大多停留在“某类人出现得多不多”这种单一统计层面，但真正的风险远不止 representation parity：模型还会遗漏关键元素、把文化表达压缩成少数全球主流符号，甚至在上下文遮挡下制造“看起来更公平”的假象。现有指标体系分散、视角割裂，很难对 T2I 模型做系统性审计。

### 核心科学问题
能否构造一个统一评测框架，同时衡量 T2I 模型的人口统计偏见、元素遗漏、文化准确性与文化广度，从而把“偏见”从单一 demographic skew 扩展成多维治理问题？

### 直觉洞见
这篇论文最重要的直觉是：偏见不只是“谁被画出来”，还包括“谁总是被用一种方式画出来”“哪些本应出现的元素没有出现”“哪些文化永远只剩最刻板、最流行的那几个符号”。换句话说，准确性与广度是两回事，公平性与完整性也不是一回事。

---

## 二、核心方法

**（1）五阶段审计流水线**
T2I-BiasBench 的 pipeline 包括模型选择、提示词设计、图像生成、caption 驱动的属性抽取、以及多指标联合评估。作者不只评估生成图像本身，而是把“图像 → caption → structured attributes → bias metrics”做成一条可重复跑通的审计链。

**（2）五类提示词场景**
- Beauty：测美貌相关的族裔/肤色偏置
- Doctor：测职业–性别刻板印象
- Culture：测文化准确性与文化广度
- Animal / Nature：非人类上下文基线，用来区分“偏见问题”和“单纯生成能力差”

**（3）13 指标统一框架**
论文把指标分为四组：
- 公平性/统计分布：RP、PD、BA、H、KL
- 刻板印象：CAS、CBS
- 多样性/对齐：Vendi Score、CLIP Proxy Score
- 忠实性/遗漏/文化：Hallucination Score、GMR、IEMR、CAR

其中 CBS、GMR、IEMR、CAR 是作者新增的重点指标。

**（4）文化塌缩与遮挡偏差识别**
论文特别强调两个此前容易被忽略的问题：
- accuracy–breadth paradox：文化内容画得“对”不代表文化覆盖足够广
- VAOP（Visual Attribute Occlusion Prompting）：像口罩/PPE 这样的视觉遮挡会让模型在统计上显得更公平，但实际上只是让属性无法被识别

## 三、技术细节

### 新指标定义
最核心的新综合指标是 Composite Bias Score：

CBS = (PD + (1 - H) + CAS) / 3

其中：
- PD 衡量群体分布差异
- H 衡量分布熵，越低代表多样性越差
- CAS 衡量刻板关联程度

作者还定义：
- GMR：显式关键词缺失率
- IEMR：隐式语境元素缺失率
- CAR：文化正确性比例

### 数据与生成协议
- 开源模型：每个 prompt 生成 100 张图
- Gemini 2.5 Flash：每个 prompt 生成 15 张图
- 图像分辨率：512 × 512
- 开源模型使用固定 seed 与固定去噪步数
- 正文总规模写作 1,574 张，但表格加总为 1,570 张，存在一个小的 bookkeeping 不一致

### 模型设置
- Stable Diffusion v1.5
- BK-SDM Base
- Koala Lightning
- Gemini 2.5 Flash（作为 RLHF 对照基线）

### 属性抽取链
1. 用 ChatGPT-5 为图像生成 caption
2. 使用 regex / 词典规则抽取性别、族裔、肤色、动物类别、文化符号等结构化属性
3. 再在这些属性上计算 13 个指标

### 关键提醒
这意味着 benchmark 的评测链并不是“纯视觉”闭环，而是借助 LLM caption 作为中间观察器。它提高了审计效率，但也引入了 observer bias 的潜在问题。

## 四、实验设置与结果

### 数据集与评价指标
- 提示集合：Beauty、Doctor、Culture、Animal、Nature
- 主要指标：CBS（越低越好）、CDS（上下文场景综合分，越低越好）、KL、BA、CAS、CAR、GMR 等
- 对比目标：不同模型在人口统计偏见、文化塌缩和元素遗漏上的表现差异

### 主要结果表格（综合分）

| Model | Beauty | Doctor | Animal | Nature | Culture | 平均 |
|------|-------:|-------:|-------:|-------:|--------:|-----:|
| **SD v1.5** | 0.50 | **0.06** | 0.30 | 0.32 | 0.66 | **0.37** |
| BK-SDM Base | 0.59 | 0.20 | **0.28** | 0.30 | 0.79 | 0.43 |
| Koala Lightning | 0.51 | 0.76 | 0.43 | **0.23** | **0.48** | 0.48 |
| Gemini 2.5 Flash | **0.33** | 1.00 | **0.20** | **0.20** | 0.60 | 0.47 |

### 关键数字
- SD v1.5 在 Beauty prompt 上 White 占比达到 74%，Fair skin 占比达到 97%，KL = 0.765，BA = 1.08，说明它不只是继承偏差，而是在放大偏差。
- Koala 在 Doctor prompt 上男性占比 91%，PD = 0.85，CBS = 0.76，暴露出极强的职业–性别刻板印象。
- Gemini 在 Doctor 场景中看似“全女性”，导致 CBS = 1.00；这不是更公平，而是 parity 指标在方向性上的缺陷被放大了。
- Culture 场景中，Gemini 的 CAR = 1.00 但 CAS = 1.00，说明它能画对文化符号，却严重依赖 Holi / Diwali 这类少数高频符号，体现了典型的 cultural collapse。
- Animal 场景里，BK-SDM 的 GMR = 0.84，说明显式元素缺失不是只在人类社会场景里才存在的现象。

### 关键发现
- RLHF 可以部分缓解 demographic parity 问题，但无法自动解决 cultural breadth 问题。
- VAOP 会让某些模型在统计上看起来更公平，实则只是把性别特征遮住了，属于 measurement artifact。
- “文化准确”与“文化广泛”必须分开审计，否则很容易被表面正确率误导。

## 五、局限性分析

1. **评测链依赖 caption 模型**：所有属性都先经 ChatGPT-5 再被结构化抽取，如果 caption 漏掉了图像中的关键属性，指标也会随之偏移。
2. **样本量不对称**：Gemini 每类只有 15 张，而开源模型每类 100 张，导致统计置信度明显不对称。
3. **部分 parity 指标不区分偏差方向**：例如 Gemini–Doctor 出现“全女性”时，指标会把它当作高偏差，但不会告诉你这是反向失衡而非传统男性刻板印象。
4. **prompt 覆盖仍有限**：年龄、残障、体型、LGBTQ+ 等更多公平性维度没有被纳入，benchmark 仍不是最终版社会偏见全景图。

## 六、复现指南

### 环境依赖
```bash
pip install torch diffusers transformers pandas scikit-learn sentence-transformers
pip install git+https://github.com/gyanendrachaubey/T2I-BiasBench-Code.git
```

### 核心代码框架
```python
class BiasBenchAuditor:
    def run(self, images, prompt):
        captions = [caption_model.describe(img) for img in images]
        attrs = [extract_attributes(c) for c in captions]
        metrics = compute_bias_metrics(attrs, prompt)
        return metrics
```

### 复现步骤
1. 下载仓库并准备 SD v1.5、BK-SDM Base、Koala Lightning，以及若需要严格复现则接入 Gemini 2.5 Flash。
2. 使用 Beauty、Doctor、Culture、Animal、Nature 五类 prompt 生成图像；开源模型每类生成 100 张，保持 512 × 512 与固定 seed。
3. 用 caption 模型生成描述，再通过词典/regex 抽取性别、族裔、肤色、文化符号、显式元素等结构化属性。
4. 计算 RP、PD、BA、H、KL、CAS、CBS、GMR、IEMR、CAR 等指标，并汇总成综合分表。
5. 重点核对 Beauty 场景下 SD v1.5 的 BA > 1、Doctor 场景下 Koala 的高 CBS、Culture 场景下 Gemini 的 CAR 高但 CAS 也高这三类结果。

### 预期结果
- SD / BK-SDM 会在 demographic distribution 上表现出更明显偏差放大
- Gemini 的 demographic parity 通常更好，但文化广度并不理想
- 文化场景里很容易复现“accuracy 高、breadth 差”的 paradox
