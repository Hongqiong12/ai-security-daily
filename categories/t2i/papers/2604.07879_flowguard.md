## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
扩散模型（Diffusion Models）的图像生成能力飞速发展，但其可能生成 NSFW（Not-Safe-For-Work）内容带来了严重安全隐患。现有的 NSFW 检测方法存在两个根本性缺陷：

1. **生成前方法（Pre-generation）**: 依赖文本提示词的分类器，但无法解决"prompt 安全 ≠ 图像安全"的差异问题——同一个安全的文字描述可能在生成过程中演变为不安全图像。
2. **生成后方法（Post-generation）**: 对最终输出应用图像分类器，但这种方法有两个致命问题：（a）不适用于中间阶段的含噪图像（中间步骤的图像充满高斯噪声，传统分类器失效）；（b）只有在完整生成后才能发现问题，造成巨大的计算资源浪费。

### 核心科学问题
能否在扩散模型的**去噪过程中间步骤**实时检测 NSFW 内容？如何在早期高噪声阶段从潜变量中提取有效的视觉安全特征？

### 直觉洞见
这就像在烤蛋糕的过程中检查面糊——不是等蛋糕烤好了才发现焦了（事后检测），也不是只看配方表就觉得没问题（事前检测）。FlowGuard 的做法是在烘烤中途打开烤箱灯看一眼面糊颜色（生成中检测）。难点在于早期的面糊还是一坨糊状物（高噪声潜变量），看起来什么都分辨不出来——FlowGuard 发明了一种特殊的"透视镜"（线性潜在解码），能从模糊的面糊中看出会不会烤成不安全的形状。

## Module 2 — 核心方法 (Core Method)

**（1）跨模型生成中检测框架（Cross-model In-Generation Detection Framework）**
FlowGuard 不绑定于特定的扩散模型架构，可以在多种骨干网络（SD 1.5、SDXL、SD3、Flux 等）的去噪步骤中插入检测模块。它检查的是中间去噪步骤的潜在表示，而非最终的像素输出。

**（2）线性近似潜在解码（Linear Approximate Latent Decoding）**
这是 FlowGuard 的核心技术贡献。在潜在扩散模型（LDM）的早期去噪步骤，潜变量包含大量噪声，直接用标准 VAE 解码器解码会产生完全不可用的模糊图像。FlowGuard 提出：
- 用一个**线性投影矩阵 W_linear** 来近似 VAE 解码器的非线性映射
- 这个线性映射能在高噪声环境下提取出足以判断安全性的粗粒度视觉特征
- 相比完整的 VAE 解码，计算量降低数十倍

**（3）课程学习稳定训练（Curriculum Learning Stabilization）**
由于早期步骤的高噪声特性使得标签信号极其嘈杂，直接端到端训练会导致不稳定。采用课程学习策略：先从后期（低噪声、易判断）样本开始训练，逐步引入前期（高噪声、难判断）样本，使检测器渐进式地学会在极端噪声条件下做出可靠判断。

## Module 3 — 技术细节 (Technical Details)

### 问题形式化

给定扩散模型的去噪过程 {z_T, z_{T-1}, ..., z_0}，其中 z_t 为第 t 步的潜变量（latent variable），T 为总去噪步数。目标是设计一个函数 f_safety: Z → {safe, unsafe}，使得：

f_safety(z_t) 能在第 t 步（t << T）就准确判断最终生成结果的安全性

### 线性潜在解码

标准 VAE 解码: x_hat = Decode_VAE(z_t)，其中 Decode_VAE 包含多层非线性变换。

FlowGuard 线性近似: x_approx = W · z_t + b

其中 W ∈ R^(c×H'×W') × d，d 为潜变量维度。W 通过以下方式获得：
1. 收集大量 (z_t, safety_label) 训练对
2. 用最小二乘法或岭回归拟合 W 使得 ||W·z_t + b - x_clean||² 最小
3. x_clean 为对应干净（无噪）图像的 VAE 特征

### 课程学习调度

训练损失函数:

L_total = Σ_{t=T}^{0} α(t) · CE(f_safety(W·z_t), y_true)

其中课程权重 α(t) 的设计:
- 初期: α(t) 只在 t ≤ T_low（如 T 的后半段）非零
- 中期: 逐步扩大到 t ≤ T_mid
- 后期: 覆盖全部时间步 t ∈ [0, T]

### 推理流程

```python
for t in range(T, 0, -1):  # 从纯噪声 T 步开始
    z_t = denoise_step(z_{t+1}, t, text_condition)  # 标准去噪
    if t % detection_interval == 0:  # 每 k 步检测一次
        feature = W_linear @ z_t  # 线性投影（极快）
        pred = classifier(feature)   # 安全分类
        if pred == "unsafe":
            early_stop()             # 提前终止，节省计算
            return "blocked"
return decode_final(z_0)           # 安全完成
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **模型覆盖**: 9 个不同的扩散模型骨干网络（跨模型泛化性测试）
- **评价指标**:
  - F1 Score (↑): NSFW 检测的综合指标
  - GPU Memory Reduction (%): 显存占用降低比例
  - Projection Latency (↓): 投影/检测耗时

### 主要结果表格

| 方法 | F1 Score (↑) | GPU 显存降低 (%) | 投影时间 (秒/图) |
|------|-------------|-----------------|----------------|
| 标准 VAE 解码 + 分类器 | Baseline | 0% (基准) | **8.1s** |
| 其他生成中检测方法 | Baseline+~15% | ~50% | ~2-4s |
| **FlowGuard** | **Baseline +30%+** | **>97%** | **0.2s** |

### 关键发现

1. **精度大幅提升**: 在 F1 分数上相比现有方法提升超过 30%，且在分布外（OOD）设置下同样表现优异——证明线性潜在解码捕获的特征具有强泛化性。

2. **极致轻量化**: GPU 显存需求降低超过 97%，投影时间从 8.1 秒降至 0.2 秒（**40倍加速**）。这使得 FlowGuard 可以部署在资源受限的环境中。

3. **提前终止节省成本**: 由于能在早期检测到不安全趋势并提前终止生成，对于被拦截的不安全请求，实际节省的计算量远超单纯检测本身的效率增益——因为避免了后续所有不必要的去噪步骤。

### 消融实验

| 组件配置 | F1 Score | 说明 |
|---------|----------|------|
| 完整 FlowGuard（线性解码 + 课程学习） | Best | 所有组件协同最佳 |
| 仅线性解码（无课程学习） | 中等 | 早期高噪声步检测不准 |
| 标准 VAE 解码（非线性） | 较好 | 精度接近但速度极慢 |
| 不同检测间隔 | — | 间隔越小检测越早但开销增加 |

## Module 5 — 局限性分析 (Limitations)

1. **NSFW 聚焦**: 当前版本专注于 NSFW 内容检测，尚未扩展到其他类型的生成安全问题（如版权角色生成、虚假信息图片、深度伪造等）。线性潜在解码是否能泛化到其他安全维度有待验证。

2. **线性近似的表达上限**: 线性投影虽然高效但其表达能力有天然上限——对于某些需要在细粒度空间模式层面区分的微妙不安全内容（如暗示性但不露骨的图像），线性特征可能不够用。

3. **课程学习的调优开销**: 课程调度策略的超参数（各阶段的起始/结束时间步、权重变化曲线）需要针对每种新模型架构重新调优，增加了部署成本。

4. **对抗鲁棒性未知**: 未评估 FlowGuard 对抗自适应攻击的能力——恶意用户是否可以通过精心设计的 prompt 使不安全内容的潜变量轨迹在检测窗口内保持"安全"的外观？

5. **检测间隔与精度的权衡**: 不是每一步都检测（否则开销过大），这意味着在两次检测之间仍有可能产生部分不安全内容。如何选择最优的检测间隔是一个工程权衡问题。

### 核心代码框架
```python
import torch
import torch.nn as nn

class LinearLatentDecoder(nn.Module):
    """FlowGuard 的线性潜在解码器"""
    def __init__(self, latent_dim, feature_dim):
        super().__init__()
        # 线性投影矩阵 —— 替代非线性 VAE 解码
        self.W = nn.Linear(latent_dim, feature_dim, bias=False)

    def forward(self, z):
        """
        z: (batch, latent_height, latent_width, latent_channels)
           或展平后的 (batch, latent_dim)
        """
        if z.dim() == 4:
            b, h, w, c = z.shape
            z_flat = z.view(b, -1)  # (b, h*w*c)
        else:
            z_flat = z
        return self.W(z_flat)  # → (b, feature_dim)


class SafetyClassifier(nn.Module):
    """轻量级安全分类器"""
    def __init__(self, feature_dim):
        super().__init__()
        self.classifier = nn.Sequential(
            nn.Linear(feature_dim, 256),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(256, 2)  # [safe, unsafe]
        )

    def forward(self, features):
        return self.classifier(features)


class FlowGuard:
    """FlowGuard 主类"""
    def __init__(self, pipe, detector_path=None):
        self.pipe = pipe  # DiffusionPipeline
        self.decoder = LinearLatentDecoder(
            latent_dim=self._get_latent_dim(),
            feature_dim=512
        )
        self.classifier = SafetyClassifier(feature_dim=512)
        self.detection_interval = 5  # 每 5 步检测一次
        self.threshold = 0.8  # 不安全置信度阈值

    def _get_latent_dim(self):
        # 根据 pipeline 配置获取潜变量维度
        return self.pipe.vae.config.latent_channels * \
               64 * 64  # SD 1.5 默认: 4 * 64 * 64 = 16384

    @torch.no_grad()
    def generate_with_guard(self, prompt, num_inference_steps=50):
        """带安全检测的生成"""
        # 初始化潜变量（标准流程）
        latents = self.pipe.prepare_latents(
            batch_size=1, num_images_per_prompt=1,
            height=512, width=512, dtype=torch.float32
        )
        
        # 手动执行去噪循环（以便插入检测）
        for step in range(num_inference_steps, 0, -1):
            # 标准去噪步骤
            latents = self.pipe.denoise_step(latents, step, prompt)
            
            # FlowGuard 检测
            if step % self.detection_interval == 0:
                features = self.decoder(latents)
                logits = self.classifier(features)
                prob_unsafe = torch.softmax(logits, dim=-1)[0, 1].item()
                
                if prob_unsafe > self.threshold:
                    return None  # 提前终止，返回 None 表示被拦截
        
        # 全程安全 → 最终解码
        image = self.pipe.decode_latent(latents)
        return image
```

### 复现步骤
1. 加载预训练扩散模型（推荐 Stable Diffusion 1.5 作为起点）
2. 构建数据集：(潜在变量 z_t, 安全标签 y) 对，覆盖多个时间步 t
3. 训练 LinearLatentDecoder（拟合线性投影）
4. 用课程学习策略训练 SafetyClassifier（从后期时间步开始）
5. 在 9 种骨干网络上测试跨模型泛化性
6. 测量 F1 分数、GPU 内存占用、投影延迟

### 预期输出
- F1 分数较 baseline 提升 30%+
- GPU 显存占用 < 基准的 3%
- 单次投影 < 0.25 秒
