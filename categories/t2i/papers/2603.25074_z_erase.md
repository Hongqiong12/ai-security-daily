## 一、背景与问题

### 动机

概念擦除（Concept Erasure）是文本到图像（T2I）扩散模型安全体系中的核心防御手段，旨在将模型中编码的敏感或有害概念（如暴力、色情内容）予以移除或抑制，从而防止这些概念被用户通过提示词激活并生成违规图像。近年来，随着扩散模型在图像生成质量上的突飞猛进，概念擦除研究也在 U-Net 架构（如 Stable Diffusion）和双流架构（如 Flux）中取得了显著进展。

然而，新兴的**单流扩散 Transformer**（Single-Stream Diffusion Transformer，以 Z-Image 为代表）引入了根本性的架构变革：文本标记和图像标记通过共享参数被处理为单一的统一序列，而非分别通过独立的编码器处理。这种设计与以往方法所针对的架构假设存在根本差异，导致直接移植现有概念擦除技术会产生**生成崩溃**（generation collapse）——模型在擦除目标概念后，生成质量急剧下降或完全丧失生成有意义图像的能力。

### 核心科学问题

如何在单流扩散 Transformer 架构中实现有效的概念擦除，同时保持模型的正常图像生成质量？

### 直觉洞见

可以将这个问题类比为"外科手术"：外科医生在切除病变组织时，必须精确避免伤及周围健康组织。传统方法相当于用同一把刀同时切除和保留——刀刃经过之处，非目标组织也会受损。Z-Erase 的流解耦框架则相当于为"保留组织"专门配备了一把保护性手术器械，让"擦除刀"和"保留刀"各自独立操作，从而在移除有害概念的同时完整保全其他能力。

## 二、核心方法

Z-Erase 包含三个核心组件：

**（1）流解耦的概念擦除框架（Stream Disentangled Concept Erasure Framework）**

单流扩散 Transformer 的核心挑战在于文本流和图像流共享参数——对图像流中概念相关参数的任何修改，都会不可避免地波及文本流的语义理解能力。Z-Erase 首先将参数更新路径在文本流与图像流之间进行数学解耦，使得传统擦除方法（如 ESD、UCE）可以独立作用于图像流，而文本流不受影响，从而避免生成崩溃。

**（2）拉格朗日引导的自适应擦除调制（Lagrangian-Guided Adaptive Erasure Modulation）**

概念擦除存在一个根本矛盾：增强擦除强度会连带损害与目标概念无关的良性图像生成质量（擦除-保留权衡）。Z-Erase 引入基于拉格朗日乘子法的约束优化框架，将概念擦除设定为主优化目标，同时将良性图像质量设定为不等式约束。系统通过自适应调整拉格朗日乘子，在每步训练中动态平衡擦除深度与质量保留。

**（3）收敛性分析（Convergence Analysis）**

Z-Erase 提供了严格的理论支撑，证明在梯度下降框架下，算法能够收敛到**帕累托稳定点**（Pareto-optimal point），即：不存在任何方向可以在进一步增强概念擦除的同时而不损害图像质量。这为算法的实际部署提供了可靠性保障。

## 三、技术细节

### 约束优化形式化

设 θ 为模型参数，L_erase(θ) 为概念擦除损失（衡量目标概念在生成图像中的残留程度），L_preserve(θ) 为内容保留损失（衡量正常图像生成质量）。Z-Erase 的优化目标为：

min_θ L_erase(θ)  s.t. L_preserve(θ) ≤ ε

通过拉格朗日乘子 λ ≥ 0 转化为无约束形式：

L_total(θ, λ) = L_erase(θ) + λ · max(0, L_preserve(θ) - ε)

其中 λ 在训练过程中自适应更新：当 L_preserve 超过阈值 ε 时增大 λ 以强化保留约束，反之则减小。

### 流解耦机制

设 W 为文本-图像共享注意力层的权重矩阵，ΔW_erase 为概念擦除引入的参数更新量。流解耦通过以下操作分离更新方向：

W'_text = W + ΔW_text（文本流参数更新，不含擦除分量）
W'_image = W - P · ΔW_erase（图像流参数更新，减去擦除方向投影 P）

### 收敛性证明概要

基于交替方向乘子法（ADMM）的框架，证明参数更新序列 {θ_t} 在以下条件下收敛：
1. 目标函数关于 θ 连续且下有界
2. 步长选择满足 Lipschitz 条件
3. 拉格朗日乘子 λ_t 有界

## 四、实验设置与结果

### 数据集与评价指标

- 概念擦除评估数据集：CUB-200（鸟类概念）、CelebA-HQ（人脸属性）、MSCOCO（多概念复合场景）
- 图像质量指标：FID（越低越好）、CLIP Score（越高越好）
- 概念残留指标：概念激活率（Concept Activation Rate，越低越好）
- 基线对比：ESD、CA（Concept Ablation）、UCE 等主流概念擦除方法

### 主要实验结果

| 方法 | 架构 | FID (↓) | CLIP Score (↑) | 概念残留率 (↓) | 生成崩溃 |
|------|------|---------|---------------|--------------|---------|
| ESD | U-Net | 15.2 | 0.78 | 8.3% | 无 |
| UCE | 双流 | 13.7 | 0.81 | 5.6% | 无 |
| Z-Erase（本文） | 单流 | **11.8** | **0.84** | **3.2%** | **无** |
| 直接移植（ESD→单流） | 单流 | — | — | — | **有** |

### 关键发现

1. **生成崩溃根因确认**：直接移植 U-Net/双流架构的概念擦除方法到单流架构，会导致文本-图像注意力权重在概念相关维度上发生冲突，触发模式崩塌。
2. **流解耦的必要性**：消融实验表明，去除流解耦步骤后，模型在 80% 的测试概念上出现明显的生成崩溃，而保留流解耦时崩溃率为 0%。
3. **拉格朗日调制的效果**：自适应 λ 调节比固定 λ 在各项指标上均有 5–15% 的提升，尤其在复合概念（multi-concept）场景中效果显著。

---

## 五、局限性分析

1. **计算开销增加**：流解耦机制引入额外的梯度路径分离计算，相比直接移植方法，训练时间增加约 30–40%。
2. **多概念复合场景的权衡**：当用户请求同时擦除多个敏感概念时，拉格朗日乘子需要同时平衡多个约束，各概念的擦除深度可能相互影响。
3. **零样本泛化能力**：实验主要在训练时见过的概念类型上评估，对于完全未见过的全新概念类型，擦除效果尚无充分评估。
4. **收敛速度敏感**：收敛性证明要求步长满足 Lipschitz 条件，实际超参数调优中对此较敏感。

### 核心代码框架

```python
class ZErase:
    def __init__(self, model, lambda_init=1.0, epsilon=0.1):
        self.model = model
        self.lambda_ = lambda_init
        self.epsilon = epsilon

    def stream_disentangle(self, grads, text_grads, erase_direction):
        """解耦文本流和图像流的梯度更新"""
        # 投影擦除方向到图像流子空间
        P = self.compute_projection_matrix(text_grads)
        image_update = grads - P @ erase_direction
        text_update = grads  # 文本流不受擦除影响
        return text_update, image_update

    def lagrangian_step(self, erase_loss, preserve_loss):
        """拉格朗日乘子自适应更新"""
        violation = preserve_loss - self.epsilon
        if violation > 0:
            self.lambda_ *= 1.1  # 增大惩罚力度
        else:
            self.lambda_ *= 0.9  # 放松约束
        return erase_loss + self.lambda_ * max(0, violation)

    def train(self, concept_prompts, benign_prompts, n_steps=1000):
        for step in range(n_steps):
            erase_loss = self.compute_erase_loss(concept_prompts)
            preserve_loss = self.compute_preserve_loss(benign_prompts)
            loss = self.lagrangian_step(erase_loss, preserve_loss)
            grads = torch.autograd.grad(loss, self.model.parameters())
            text_g, image_g = self.stream_disentangle(grads, ...)
            self.apply_updates(text_g, image_g)
```
