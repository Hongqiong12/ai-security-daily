## 一、背景与问题

### 动机

生成式 AI 的版权保护越来越依赖不可见水印，但攻击研究长期落后于防御研究。传统 watermark attack 大多是加噪、压缩、裁剪这类“盲扰动”，往往要么去不掉水印，要么把图像质量一起打崩。作者的核心判断是：**水印本身就是一种频域结构信号**，如果还只在像素空间蛮力破坏，攻击/防御就失衡了。

### 核心科学问题

能否利用扩散模型在图像恢复上的能力，把 watermark removal 从空间域噪声攻击升级为**频域选择性调制**，在高 BER 的同时尽量保住视觉质量？

### 直觉洞见

普通攻击像拿砂纸磨整张图，希望顺带把水印磨掉；FMDiffWA 像先去频谱里找“水印最可能寄生的成分”，再用扩散恢复把视觉细节补回来。这样攻击就不再是 blind corruption，而是**targeted removal + perceptual reconstruction**。

---

## 二、核心方法

**（1）条件扩散攻击框架（Conditional Diffusion Attack）**

FMDiffWA 把水印图像当作条件输入，学习从 watermarked image 到 watermark-free image domain 的映射。目标不是生成新图，而是在尽量保留原图观感的前提下，去掉可提取水印。

**（2）频域水印调制模块（FWM, Frequency-domain Watermark Modulation）**

作者把图像在 Fourier domain 中拆成 magnitude 和 phase，并在正向/逆向采样过程中对水印相关频率成分做选择性调制。核心思想是显式利用前向扩散阶段保留下来的纹理与频谱信息。

**（3）分阶段训练（PTS, Phased Training Strategy）**

标准 diffusion 只优化 noise estimation，容易在 watermark removal 与 visual fidelity 间失衡。论文采用两阶段训练：
- 第一阶段：标准噪声预测；
- 第二阶段：加入以 watermark-free image 为监督的 refinement constraint。

**（4）Patch-based Processing**

为兼容不同分辨率图像，模型不是整图处理，而是用 block/patch 方式攻击，再聚合回整图。这比固定分辨率假设更接近真实 watermarking 场景。

## 三、技术细节

### 水印嵌入/提取形式化

论文先回顾标准 watermark 流程：

I_w = E(I, w)

w′ = D(Δ(I_w))

其中 I 是原图，w 是水印比特串，I_w 是水印图像，Δ 表示可能遭受的攻击或变换。攻击的目标是让 w′ 尽可能偏离 w，同时不破坏 I_w 的视觉质量。

### 扩散建模

前向扩散过程：

q(I_t | I_(t−1)) = 𝒩(I_t; √(1 − β_t) · I_(t−1), β_t · ω)

作者在此基础上引入 FWM，在正向与逆向过程中都使用频域幅值/相位信息来抑制 watermark-related features。

### 精炼损失

第二阶段引入 refinement constraint：

L_ref = L₁ + L_ms-ssim

其中：
- L₁ 强调像素级恢复；
- L_ms-ssim 强调多尺度结构保真。

### 关键超参数

- 数据集：CelebA / ImageNet
- 图像尺寸：**256 × 256**
- watermark：**16 × 16**
- 训练图像：**18,000**；测试：**6,000**
- patch：每张图随机裁 **16** 个 **64 × 64** patch
- 学习率：**2e-5**
- 训练迭代：**1 × 10^6**（噪声预测）+ **3 × 10^5**（refinement）
- diffusion time steps：**T = 1000**
- sampling steps：**S = 10**
- β（mask scale）最优约为 **0.6**

## 四、实验设置与结果

### 数据集与评价指标

- **数据集**：CelebA、ImageNet
- **水印算法**：LSB、DCT、QPHFMs、HiDDeN_MP
- **指标**：
  - **PSNR（↑）**：越高说明视觉质量越好
  - **BER（↑）**：越高说明恢复出的水印错误越多，攻击越成功

### 主要结果表格

| 方法 | CelebA / QPHFMs | CelebA / HiDDeN_MP | ImageNet / QPHFMs | ImageNet / HiDDeN_MP |
|------|----------------:|-------------------:|------------------:|---------------------:|
| DiffWA | 30.83 / 0.1687 | 31.84 / 0.4341 | 28.86 / 0.1452 | 29.89 / 0.4427 |
| HIWANet | 31.55 / 0.1794 | 31.75 / 0.2649 | 30.04 / 0.1922 | 29.77 / 0.2934 |
| **FMDiffWA** | **42.18 / 0.2031** | **43.63 / 0.4495** | **40.45 / 0.2264** | **41.53 / 0.4562** |

从结果看，FMDiffWA 的特点不是单纯把 BER 做高，而是**在 BER 更优或持平的同时，把 PSNR 拉到 40dB 级别**，明显优于传统与已有深度攻击。

### 消融实验（Table 2）

| 配置 | PSNR (dB) | BER |
|------|----------:|----:|
| FWM only | 39.96 | 0.1896 |
| PTS only | 48.65 | 0.0664 |
| **FWM + PTS** | **42.08** | **0.2084** |

### 关键发现

- **仅有 PTS**：图像好看，但去水印不够狠。  
- **仅有 FWM**：能更积极地打水印，但视觉质量下降。  
- **FWM + PTS 联合**：才真正实现 watermark removal 与 fidelity 的平衡。  
- 频域调制相比纯空间域攻击，对 QPHFMs 这类频域稳健水印更有针对性。

## 五、局限性分析

1. **评测范围有限**：虽然覆盖 4 类水印算法，但仍未覆盖更复杂的生成器级 provenance watermark 或视频水印。  
2. **需要配对监督**：第二阶段 refinement 使用 watermark-free image 监督，现实黑盒场景未必总有这种训练条件。  
3. **攻击目标主要是 removal，不是 forgery**：论文强调去除能力，对伪造水印或双向溯源对抗讨论不足。  
4. **分辨率与场景仍偏受控**：256×256 和标准图像数据集距离真实社交媒体压缩链路还有差距。

### 核心代码框架

```python
class FMDiffWA:
    def __init__(self, denoiser, beta=0.6, steps=10):
        self.denoiser = denoiser
        self.beta = beta
        self.steps = steps

    def fwm(self, image_freq):
        magnitude, phase = fft_decompose(image_freq)
        magnitude = modulate_magnitude(magnitude, self.beta)
        phase = modulate_phase(phase, self.beta)
        return ifft_compose(magnitude, phase)

    def attack(self, watermarked_image):
        latent = forward_diffuse(watermarked_image)
        for t in reversed(range(self.steps)):
            latent = self.fwm(latent)
            latent = self.denoiser.sample_step(latent, cond=watermarked_image, t=t)
        return latent
```

### 复现步骤

1. 选定两套数据集（CelebA / ImageNet），将图像统一 resize 到 256×256。
2. 用论文中相同的 4 种水印算法生成带水印样本，并保留对应的 watermark-free 原图。
3. 先训练基础 conditional diffusion 去拟合 noise estimation objective。
4. 再加入 refinement constraint（L₁ + MS-SSIM）做第二阶段训练。
5. 分别复现实验表中的 PSNR / BER，并跑 FWM only、PTS only、FWM+PTS 三个消融配置。
6. 若要做更现实评估，建议补充 JPEG 二次压缩、裁剪、resize、社交媒体上传下载链路后的 attack transfer 测试。
