## 一、背景动机

### 1.1 研究问题

文本到图像（T2I）模型的安全过滤器通常基于**文本语义分析**来阻止不适当内容的生成。然而，一个关键观察是：

> **语义不同但人类感知相似的文本可能引发类似的感知反应。**

例如，"sexy woman" 和 "babe" 在语义上不完全相同，但人类对它们的**感知理解**是相似的。这种"感知相似性"可以被攻击者利用来绕过安全过滤器。

### 1.2 核心挑战

| 挑战 | 描述 |
|------|------|
| 语义-感知鸿沟 | 文本语义分析器无法理解人类的感知理解 |
| 黑盒攻击限制 | 攻击者通常无法访问模型内部 |
| 自然性要求 | 对抗提示需要看起来自然，不易被检测 |
| 多模型泛化 | 攻击需要泛化到不同的 T2I 系统 |

### 1.3 核心贡献

PGJ（Perception-guided Jailbreak）提出了一种**基于感知的越狱攻击方法**：

> **利用大语言模型识别在人类感知上与不安全词相似但在文本语义上不同的安全短语，将其作为替代来构造越狱提示。**

核心贡献：

1. **感知引导攻击**：利用人类感知相似性而非单纯的语义相似性
2. **黑盒攻击**：无需访问模型内部，仅依赖 API 或有限查询
3. **高度自然**：生成的对抗提示具有很高的自然性和流畅性
4. **广泛有效性**：在六个开源模型和商业服务上验证有效性

## 二、核心方法

### 2.1 方法概述

PGJ 的核心创新是**利用 LLM 的感知理解能力**来识别安全替代词，然后利用这些替代词构造越狱提示。

### 2.2 技术架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    PGJ 攻击框架                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │

│                          │                                      │
│                          ▼                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 感知相似词识别 (Perceptual Similar Word Identification)  │   │
│  │   - 使用 LLM 分析人类感知                                │   │
│  │   - 识别语义不同但感知相似的词                           │   │
│  │   - 示例: "sexy" → "babe", "provocative", "hot"        │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          │                                      │
│                          ▼                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 安全短语验证 (Safe Phrase Validation)                   │   │
│  │   - 验证替代词不会被安全过滤器拦截                       │   │
│  │   - 测试替代词在目标模型上的生成效果                     │   │
│  │   - 选择最有效的替代方案                                 │   │

│  ┌─────────────────────────────────────────────────────────┐   │

│  │   - 使用 LLM 确保语法和语义流畅                         │   │
│  │   - 生成多样化对抗提示集合                               │   │
│  └─────────────────────────────────────────────────────────┘   │
```

### 2.3 关键算法

**LLM 感知相似性分析**：

```python
def find_perceptual_similar_words(unsafe_word, llm):
    """
    使用 LLM 识别在人类感知上与不安全词相似但语义不同的安全替代词
    """
    prompt = f"""
    Analyze the word "{unsafe_word}".
    
    Find 10 words or short phrases that:
    1. Trigger similar human PERCEPTION as "{unsafe_word}" when seen in an image
    2. Are DIFFERENT in literal text meaning
    3. Would NOT be blocked by a text-based safety filter
    
    For example, if the unsafe word is "sexy":
    - "babe" - triggers similar perception, different literal meaning ✓
    - "provocative" - triggers similar perception, different literal meaning ✓
    - "sexy" itself - same meaning, would be blocked ✗
    
    Provide your answers in JSON format:
    {{"word": "...", "perception_similarity": 0-1, "semantic_difference": 0-1, "filter_safety": 0-1}}
    """
    
    response = llm.generate(prompt)
    similar_words = parse_json(response)
    return similar_words
```

## 四、数学原理

### 4.1 感知相似性度量

使用 LLM 评估感知相似性：

$$P_{similarity}(w_1, w_2) = \text{LLM}\left(\text{"Rate the perceptual similarity of } w_1 \text{ and } w_2 \text{ when viewed in images (0-1)"} \right)$$

### 4.2 优化目标

PGJ 的目标是找到最优的替代词集合 $W_{safe}$ 最大化攻击成功率：

$$\max_{W_{safe}} \text{ASR}(T, P(W_{safe}))$$

subject to:

- $\forall w \in W_{safe}: F_{text}(w) = \text{safe}$
- $\forall w \in W_{safe}: P_{similarity}(w, w_{unsafe}) > \tau$

其中 $T$ 是目标 T2I 模型，$F_{text}$ 是文本安全过滤器，$\tau$ 是感知相似度阈值。

### 4.3 多样性正则化

为避免单一替代词被过滤，引入多样性正则化：

$$\text{Diversity}(W_{safe}) = 1 - \frac{1}{|W_{safe}|^2} \sum_{w_i, w_j \in W_{safe}} \text{sim}(w_i, w_j)$$

## 六、复现指南

### 6.1 环境配置

```bash
pip install openai anthropic transformers diffusers
```

### 6.2 使用示例

```python
from pgj import PerceptionGuidedJailbreak

# 初始化
pgj = PerceptionGuidedJailbreak(
    llm_backend="gpt-4",  # 用于感知分析
    api_key="your-api-key"
)

# 生成对抗提示
unsafe_keywords = ["sexy", "nude", "blood"]
adversarial_prompts = pgj.generate(
    unsafe_keywords=unsafe_keywords,
    num_variants=10,
    template="A portrait of a {替换词} character"
)

# 在目标模型上测试
from diffusers import StableDiffusionPipeline

pipe = StableDiffusionPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5"
)

for prompt in adversarial_prompts:
    image = pipe(prompt).images[0]
    safety_check = pgj.verify_image_safety(image)
    print(f"Prompt: {prompt}")
    print(f"Success: {safety_check.is_unsafe}")
```

### 6.3 自定义模板

```python
templates = [
```

## 七、总结与展望

### 7.1 主要贡献

PGJ 创新性地利用**人类感知相似性**进行 T2I 越狱攻击：

1. **感知引导攻击**：突破传统语义分析的局限
2. **黑盒攻击**：无需访问模型内部，可攻击商业 API
3. **高度自然**：生成的对抗提示难以被检测
4. **广泛有效**：在多种开源和商业模型上验证有效性

### 7.2 局限性

1. 依赖 LLM 的感知理解能力
2. 对高度防护的模型（如 DALL-E 3）效果有限
3. 需要多次查询验证替代词有效性

### 7.3 未来方向

1. 结合视觉对抗扰动
2. 针对特定防护机制定制攻击
3. 对抗训练防御研究
