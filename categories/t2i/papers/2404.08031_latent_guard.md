## 二、背景动机

### 2.1 研究问题

随着文本到图像（T2I）模型生成高质量图像的能力不断增强，这些模型可能被滥用于创建不当内容，如色情、暴力、仇恨言论等。现有安全措施主要分为两类：基于文本黑名单的方法和基于有害内容分类的方法，但它们都存在明显局限性。

### 2.2 现有防御的局限

| 方法 | 局限 |
|------|------|
| 文本黑名单 | 易被对抗性提示绕过 |
| 有害内容分类 | 需要大量训练数据 |
| 输出过滤 | 事后检测，延迟高 |
| 概念移除 | 灵活性低，难以扩展 |

### 2.3 研究贡献

1. **潜在空间防护**：在文本编码器的潜在空间中构建安全防护

2. **LLM辅助数据生成**：利用大语言模型自动生成安全训练数据

3. **对比学习策略**：通过对比学习学习有害概念的表示

4. **即插即用设计**：可轻松集成到现有 T2I 系统中

## 三、核心方法

### 3.1 方法概述

Latent Guard 的核心思想是在 T2I 模型的文本编码器之上构建一个辅助安全框架，通过检测输入文本嵌入中的有害概念来实现安全防护。

### 3.2 系统架构

```
输入文本 → 文本编码器 → 潜在空间 → Latent Guard → 安全检测
              ↓                       ↓
          原始嵌入              有害概念检测
              ↓                       ↓
          去噪过程 ←←←←←←←←←←←←←←← 通过/阻止
```

### 3.3 安全概念嵌入

Latent Guard 首先为每个安全概念类别构建嵌入表示：

```python
class LatentGuard:
    def __init__(self, text_encoder, concept_list):
        self.encoder = text_encoder
        self.concept_embeddings = {}
        
        for concept in concept_list:
            # 使用 LLM 生成概念相关描述
            descriptions = self.generate_concept_descriptions(concept)
            
            # 编码为嵌入
            embeddings = [self.encoder.encode(desc) for desc in descriptions]
            
            # 聚类得到概念中心
            center = self.cluster_embeddings(embeddings)
            self.concept_embeddings[concept] = center
```

### 3.4 有害概念检测

给定输入文本嵌入 $e$，Latent Guard 检测是否有害概念：

```python
def detect_harmful_concepts(self, text, threshold=0.7):
    # 编码输入文本
    text_embedding = self.encoder.encode(text)
    
    # 计算与各概念的距离
    distances = {}
    for concept, center in self.concept_embeddings.items():
        distance = cosine_distance(text_embedding, center)
        distances[concept] = distance
    
    # 检测有害概念
    harmful_concepts = []
    for concept, distance in distances.items():
        if distance > threshold:
            harmful_concepts.append({
                "concept": concept,
                "confidence": 1 - distance,
                "distance": distance
            })
    
    return harmful_concepts
```

## 四、技术细节

### 4.1 LLM 辅助数据生成

Latent Guard 利用 LLM 自动生成训练数据：

```python
def generate_safety_dataset(llm, concepts):
    """
    使用 LLM 生成安全相关数据集
    """
    dataset = {"safe": [], "unsafe": {}}
    
    for concept in concepts:
        # 生成正面示例（有害）
        unsafe_examples = llm.generate(
            f"Generate 50 examples of prompts containing {concept}:"
        )
        dataset["unsafe"][concept] = unsafe_examples
        
        # 生成负面示例（安全）
        safe_alternatives = llm.generate(
            f"For each {concept} example, provide a safe alternative:"
        )
        dataset["safe"].extend(safe_alternatives)
    
    return dataset
```

### 4.2 对比学习策略

Latent Guard 采用多正例对比学习：

```python
def contrastive_loss(anchor, positives, negatives):
    """
    多正例对比损失
    """
    # 计算正例相似度
    pos_sim = torch.stack([
        cosine_similarity(anchor, pos) for pos in positives
    ]).mean()
    
    # 计算负例相似度
    neg_sim = torch.stack([
        cosine_similarity(anchor, neg) for neg in negatives
    ]).mean()
    
    # InfoNCE 损失
    loss = -torch.log(torch.exp(pos_sim) / (torch.exp(pos_sim) + torch.exp(neg_sim)))
    
    return loss
```

## 五、数学原理

### 5.1 有害概念评分

给定文本嵌入 $e$ 和概念中心 $c_k$，有害评分定义为：

$$s(e, c_k) = \frac{\langle e, c_k \rangle}{\|e\| \cdot \|c_k\|}$$

### 5.2 多概念检测

对于 $K$ 个概念，检测向量为：

$$\vec{s}(e) = [s(e, c_1), s(e, c_2), ..., s(e, c_K)]$$

### 5.3 检测决策

检测函数为：

$$d(e) = \begin{cases} \text{unsafe} & \text{if } \max_k s(e, c_k) > \tau \\ \text{safe} & \text{otherwise} \end{cases}$$

### 5.4 训练损失

$$\mathcal{L} = \mathcal{L}_{detection} + \lambda \mathcal{L}_{contrastive}$$

其中：
$$\mathcal{L}_{detection} = -\sum_{(e, y)} y \log p(\text{unsafe}|e) + (1-y) \log p(\text{safe}|e)$$

## 八、代码解析

### 8.1 核心检测模块

```python
class LatentGuardDetector(nn.Module):
    def __init__(self, embedding_dim, num_concepts, hidden_dim=256):
        super().__init__()
        
        # 投影层
        self.projection = nn.Sequential(
            nn.Linear(embedding_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim)
        )
        
        # 概念检测头
        self.concept_heads = nn.ModuleList([
            nn.Linear(hidden_dim, 1) for _ in range(num_concepts)
        ])
    
    def forward(self, embeddings):
        projected = self.projection(embeddings)
        
        scores = []
        for head in self.concept_heads:
            score = torch.sigmoid(head(projected))
            scores.append(score)
        
        return torch.cat(scores, dim=-1)
```

## 九、总结与展望

### 9.1 主要贡献

1. **潜在空间防护**：首次在文本嵌入空间构建安全防护

2. **自动数据生成**：利用 LLM 自动生成训练数据

3. **高效对比学习**：通过多正例对比学习提高检测能力

4. **即插即用设计**：易于集成到现有系统

### 9. 局限性

1. **概念覆盖**：依赖预定义概念集合

2. **语言泛化**：主要针对英语

3. **上下文理解**：可能忽略复杂的上下文信息

### 9.3 未来方向

1. **动态概念更新**：支持在线更新概念库

2. **多语言支持**：扩展到更多语言

3. **多模态融合**：结合图像信息进行检测
