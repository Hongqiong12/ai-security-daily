## 一、背景动机

### 1.1 研究问题

文本到图像（T2I）模型的快速发展引发了对潜在滥用的严重担忧。尽管已有 NSFW 分类器和概念移除微调等防护措施，这些方法在面对**对抗性提示**时表现脆弱。攻击者可以通过以下方式绕过传统防护：

1. **语义编码**：使用拼音、emoji、拼写变体绕过关键词检测
2. **梯度优化**：利用模型梯度信息构造精确的对抗性嵌入
3. **跨语言攻击**：使用非英语语言构造恶意提示
4. **组合绕过**：多个无害词汇组合产生有害输出

### 1.2 核心挑战

| 挑战 | 描述 |
|------|------|
| 对抗鲁棒性 | 防护措施需抵御经过优化的对抗性输入 |
| 实用性 | 防护不能显著影响正常用户体验 |
| 可扩展性 | 需适应不同类型的 T2I 模型 |
| 可解释性 | 用户需要理解为何提示被拦截 |

### 1.3 核心贡献

GuardT2I 提出了一种**生成式审核框架**，采用独特的方法检测对抗性提示：

> **不进行二元分类，而是利用大语言模型（LLM）将有条件的文本引导嵌入转换为自然语言描述，从而有效检测对抗性提示，同时不损害模型的固有性能。**

核心贡献：

1. **生成式审核**：将嵌入空间映射到自然语言，无需硬阈值
2. **LLM 辅助理解**：利用大语言模型的语义理解能力
3. **显著性能提升**：在各种对抗场景下优于商业解决方案
4. **非侵入式防御**：不修改原始 T2I 模型参数

## 二、核心方法

### 2.1 方法概述

GuardT2I 的核心创新是将**条件嵌入转换为自然语言描述**，然后利用 LLM 的语义理解能力判断是否为对抗性提示。这种方法绕过了传统分类器的局限性，因为它不需要学习对抗性模式的显式特征。

### 2.2 技术架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     GuardT2I 架构                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  输入文本提示: "a photorealistic iIIegaI content image"          │
│                          │                                      │
│                          ▼                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 条件嵌入提取 (Conditional Embedding Extraction)         │   │
│  │   - 提取 T2I 模型的条件嵌入                              │   │
│  │   - 获取交叉注意力分布                                   │   │
│  │   - 捕获嵌入空间的不安全信号                             │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          │                                      │
│                          ▼                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 嵌入到语言转换 (Embedding-to-Language)                   │   │
│  │   - 使用投影器 P 将嵌入映射到 LLM 输入空间               │   │
│  │   - 生成自然语言描述                                     │   │
│  │   - 示例: "This prompt may contain: typo for illegal,    │   │
│  │           unusual character substitution..."            │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          │                                      │
│                          ▼                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ LLM 安全分析 (LLM Safety Analysis)                       │   │
│  │   - LLM 分析自然语言描述                                  │   │
│  │   - 识别潜在对抗模式和语义异常                           │   │
│  │   - 输出安全风险评估                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          │                                      │
│                          ▼                                      │
│  输出: 安全/不安全 + 风险等级 + 原因                           │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 关键算法

**嵌入到语言投影器**：

GuardT2I 训练一个投影器 $P$ 将 T2I 模型的嵌入空间映射到 LLM 的输入空间：

$$\text{desc} = P(\phi_{T2I}(\text{prompt}))$$

投影器通过对比学习训练：

$$L_{projection} = -\log \frac{\exp(\text{sim}(P(e), g_s))}{\sum_{g \in \{g_s, g_n\}} \exp(\text{sim}(P(e), g))}$$

其中 $g_s$ 是与不安全嵌入对应的正样本描述，$g_n$ 是负样本描述。

## 四、数学原理

### 4.1 问题 formulation

设 $\mathcal{P}$ 为提示空间，$\mathcal{A}$ 为对抗性提示子集。GuardT2I 的目标是学习一个检测器 $D$ 使得：

$$D(p) = \begin{cases} 1 & \text{if } p \in \mathcal{A} \\ 0 & \text{otherwise} \end{cases}$$

### 4.2 嵌入空间分析

对抗性提示往往在嵌入空间形成与正常提示不同的分布。通过分析嵌入空间的几何性质：

$$\text{Risk}(p) = \sigma(W \cdot \phi(p) + b)$$

其中 $\phi$ 是 T2I 模型的嵌入函数。

### 4.3 LLM 推理

GuardT2I 使用 LLM 进行零样本推理：

```
Given the embedding description: "{desc}"
Analyze if this text-to-image prompt contains:
1. Typos or character substitutions for sensitive words
2. Cross-lingual attempts to bypass safety filters
3. Adversarial patterns that may generate unsafe content

Provide a safety assessment with confidence score.
```

## 六、复现指南

### 6.1 环境配置

```bash
# GuardT2I 依赖
pip install torch transformers diffusers openai tiktoken
```

### 6.2 使用示例

```python
from guardt2i import GuardT2I

# 初始化
guard = GuardT2I(
    t2i_model="runwayml/stable-diffusion-v1-5",
    llm_backend="gpt-3.5-turbo"  # 或 "llama-2"
)

# 检测提示安全性
result = guard.check("a beautiful landscape")

# 输出
print(f"Safe: {result.is_safe}")        # True
print(f"Risk Level: {result.risk}")     # 0.02
print(f"Reason: {result.reason}")       # "No suspicious patterns detected"

# 对抗性提示示例
result_adversarial = guard.check("se.xy woman portrait")
print(f"Safe: {result_adversarial.is_safe}")  # False
print(f"Risk Level: {result_adversarial.risk}")  # 0.94
print(f"Reason: {result_adversarial.reason}")  # "Detected typo substitution for sensitive term"
```

## 七、总结与展望

### 7.1 主要贡献

GuardT2I 创新性地将 LLM 引入 T2I 安全审核：

1. **生成式审核**：突破传统二元分类的局限性
2. **语义理解**：利用 LLM 的强大语义分析能力
3. **显著鲁棒性**：在对抗场景下大幅优于现有方案
4. **非侵入式**：无需修改 T2I 模型

### 7.2 局限性

1. 依赖 LLM API，有额外成本
2. LLM 延迟影响实时性
3. 仍可能受高级对抗攻击影响

### 7.3 未来方向

1. 本地 LLM 部署减少延迟
2. 多模态输入分析
3. 自适应检测阈值
