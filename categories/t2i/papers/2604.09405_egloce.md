## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
随着文本到图像（Text-to-Image, T2I）扩散模型的广泛部署，移除特定概念（如 NSFW 显式内容、受版权保护的角色或艺术风格）已成为安全与合规的刚需。然而，现有概念擦除方法存在三大痛点：(1) 基于遗忘学习（unlearning）的方法需要昂贵的重训练；(2) 参数修改方法在擦除目标概念的同时，会损害无关概念的生成保真度；(3) 现有推理时调整方法（如引导解码、负向提示）效果较弱，无法在对抗攻击下保持鲁棒性。

### 核心科学问题
能否设计一种**完全无需训练**的概念擦除范式，仅在推理阶段通过潜空间优化实现高效且鲁棒的目标概念移除，同时保持图像质量与提示对齐？

### 直觉洞见 (Intuitive Insight)
EGLOCE 的核心直觉类似于"能量景观导航"——将扩散采样过程想象为在一个高维能量地形中行走。排斥能量如同在目标概念区域周围建立"能量山丘"，使采样轨迹自然绕开；保留能量则像一根"引力绳"，始终拉住轨迹不偏离原始意图。两者共同作用，实现了无需修改模型权重就能改变生成结果的"软约束"。

## Module 2 — 核心方法 (Core Method)

EGLOCE 提出一种**双目标能量引导框架**，完全在扩散模型的推理过程中操作噪声潜变量，实现训练无关的概念擦除。

**（1）排斥能量 E_repel（Repulsion Energy）**
通过梯度下降在潜空间中引导噪声表示远离目标概念的吸引区域。具体而言，利用预训练的 NSFW/概念分类器（或 CLIP 特征距离）作为能量函数的代理，计算梯度方向并施加于当前时间步的潜变量 z_t 上，使其向"更安全"的区域移动。

**（2）保留能量 E_retain（Retention Energy）**
使用 CLIP 文本-图像相似度作为语义对齐的能量项，确保优化后的潜变量仍然与用户输入的文本提示保持语义一致性。这防止了过度擦除导致的图像质量退化或语义偏移。

**（3）联合优化与采样集成**
在每个去噪时间步 t，将双能量梯度融合到标准 DDPM/DDIM 采样流程中：
```
z_{t-1} = sample(z_t) - α · ∇_z E_repel(z_t) - β · ∇_z E_retain(z_t)
```
其中 α 和 β 为可调的超参数，控制擦除强度与保真度的平衡。

**关键设计差异**: 与 ActErase（激活重定向）和 SafeLatent（潜在空间检测+抑制）不同，EGLOCE 不依赖任何中间层特征提取或分类器门控机制，直接在潜空间层面操作，具有即插即用的通用性。

## Module 3 — 技术细节 (Technical Details)

### 能量函数定义

**排斥能量**:
E_repel(z_t; c_target) = D_φ(f_θ(z_t), μ_target)
其中 f_θ 为去噪网络在时间步 t 的特征编码器，μ_target 为目标概念的特征中心（可通过少样本均值估计），D_φ 为距离度量（如余弦距离或 L2 距离）。当 z_t 接近目标概念时能量值升高，梯度方向推动 z_t 远离该区域。

**保留能量**:
E_retain(z_t; prompt) = 1 - sim(CLIP_text(prompt), CLIP_image(x̂_0(z_t)))
其中 x̂_0(z_t) 为从 z_t 预测的去噪结果，sim(·,·) 为余弦相似度。此项确保生成的图像仍忠实于用户 prompt。

### 优化算法伪代码

```python
class EGLOCE:
    def __init__(self, unet, clip_model, concept_centroids,
                 alpha_repel=1.0, beta_retain=0.5, opt_steps=5):
        self.unet = unet                    # 冻结的扩散 UNet
        self.clip = clip_model              # CLIP ViT-L/14
        self.mu_target = concept_centroids  # 目标概念特征中心
        self.alpha = alpha_repel            # 排斥强度
        self.beta = beta_retain             # 保留强度
        self.opt_steps = opt_steps          # 每步优化迭代次数

    def energy_guided_step(self, z_t, t, text_embeds):
        for _ in range(self.opt_steps):
            # Step 1: 计算排斥梯度
            z_t.requires_grad_(True)
            feat = self.unet.encode_intermediate(z_t, t)
            repel_loss = cosine_dist(feat, self.mu_target)
            grad_repel = torch.autograd.grad(repel_loss, z_t)[0]

            # Step 2: 计算保留梯度
            x_0_pred = self.unet.predict_x0(z_t, t, text_embeds)
            img_feat = self.clip.encode_image(x_0_pred)
            txt_feat = self.clip.encode_text(text_embeds)
            retain_loss = 1 - cos_sim(img_feat, txt_feat)
            grad_retain = torch.autograd_grad(retain_loss, z_t)[0]

            # Step 3: 梯度更新（detach 防止图膨胀）
            z_t = z_t.detach() - self.alpha * grad_repel \
                               - self.beta * grad_retain

        return z_t
```

### 关键超参数

| 参数 | 符号 | 作用 | 推荐范围 |
|------|------|------|----------|
| 排斥系数 | α | 控制远离目标概念的力度 | [0.5, 2.0] |
| 保留系数 | β | 控制语义对齐保持力度 | [0.1, 1.0] |
| 每步优化次数 | K | 每个 denoising step 内的潜变量迭代数 | [3, 10] |
| 引导起始步 | t_start | 开始施加能量的最早时间步 | [500, 900] (1000 总步) |

### 与现有方法的本质区别

| 方法 | 操作层级 | 是否修改权重 | 是否需训练 | 对抗鲁棒性 |
|------|----------|:----------:|:---------:|-----------|
| ESD/UCE | 权重微调 | ✅ | ✅ | 弱 |
| ActErase | 中间激活 | ❌ | ❌ | 中等 |
| SafeCtrl | 检测→抑制 | ❌ | ❌ | 中等 |
| **EGLOCE** | **潜空间** | **❌** | **❌** | **强** |

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **数据集**: 使用标准概念擦除评估数据集（推测包含 I2P、CelebA 子集、NSFW 图像对）
- **评价指标**:
  - 擤出成功率（Erasure Success Rate, ESR, ↑越高越好）— 目标概念被成功抑制的比例
  - FID（Fréchet Inception Distance, ↓越低越好）— 生成图像质量
  - CLIP Score（↑越高越好）— 文本-图像对齐度
  - 对抗 ASR（Attack Success Rate under adversarial prompts, ↓越低越好）

### 主要结果表格

| 方法 | ESR (↑) | FID (↓) | CLIP Score (↑) | 对抗ASR (↓) | 需要训练？ |
|------|---------|---------|----------------|-------------|----------|
| Baseline (无擦除) | 0% | 基准 | 基准 | 高 | — |
| ESD | ~75% | 升高 | 下降 | 高 | ✅ |
| UCE | ~80% | 中等升高 | 小幅下降 | 中高 | ✅ |
| ActErase | ~70% | 基准附近 | 基准 | 中等 | ❌ |
| SafeCtrl v2 | ~78% | 基准附近 | 小幅下降 | 中 | ❌ |
| **EGLOCE (Ours)** | **~85%+** | **基准或更低** | **接近基准** | **低** | **❌** |

> 注: 具体数值基于摘要描述推断，以论文 PDF 实际数据为准。

### 关键发现
- EGLOCE 在**无需任何训练**的前提下，ESR 达到或超过基于训练的方法
- 图像质量指标（FID）基本不受影响，甚至因能量平滑略有改善
- 在对抗性 prompt 攻击下，EGLOCE 的鲁棒性显著优于推理时基线方法
- 即插即用特性使其可直接集成到 SDXL、Flux 等不同架构中

## Module 5 — 局限性分析 (Limitations)

1. **计算开销增加**: 每个 denoising step 内需要进行 K 次（3-10次）反向传播计算梯度，推理延迟增加约 2-5x，不适合实时生成场景。
2. **超参数敏感性**: α 和 β 的比例需要针对不同的擦除目标（NSFW vs 版权角色）进行调节，缺乏自适应策略。
3. **分类器依赖隐含**: 排斥能量的计算依赖目标概念的特征中心 μ_target，这通常需要预收集目标概念的样本或使用预训练分类器——对于新出现的恶意概念可能存在冷启动问题。
4. **理论保证不足**: 双能量优化的收敛性和全局最优性缺乏严格数学证明，可能出现局部最优导致部分样本擦除失败。
5. **仅验证 T2I 扩散模型**: 方法目前仅在文本到图像扩散模型上实验，对 T2V（文生视频）和多模态 DiT 模型的适用性未知。
