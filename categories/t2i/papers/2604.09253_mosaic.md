## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

视觉语言模型（VLMs）如 GPT-4V、Gemini、Claude 等已成为多模态 AI 的核心基础设施，但其安全性仍面临严峻挑战。现有越狱攻击主要分为两类：
1. **显式视觉提示攻击（Explicit Visual Prompt）**：在图像中嵌入可见的恶意文本/符号——容易被检测和过滤
2. **基于梯度的对抗优化（Gradient-based Adversarial Optimization）**：在开源替代模型上优化不可感知扰动——但在**异构设置**（heterogeneous setting，即替代模型 ≠ 目标模型）下效果急剧下降

核心问题在于**代理依赖（Surrogate Dependency）现象**：当攻击者使用的开源替代模型与目标闭源模型架构不同时，优化的对抗样本迁移性大幅下降。这使得针对商业闭源 VLM 的实际攻击效果远低于论文报告的理论上限。

### 核心科学问题 (Core Scientific Question)

如何设计一种多模态越狱框架，在**异构替代-目标模型对**下消除代理依赖问题，实现对商业闭源 VLM 的高效越狱？

### 直觉洞见 (Intuitive Insight)

想象你在为一个从未见过的人制作伪装。如果你只用一张参考照片来调整（单一替代模型），结果很可能"过拟合"——只像那张照片而不像真人。但如果你同时用多张不同角度的照片做参照（多代理集成），并且从多个角度反复调整外观（多视图图像优化），最终的结果会更接近真实目标。Mosaic 的核心就是：不要依赖任何一个"替身"，而是让一群"替身"集体投票。

## Module 2 — 核心方法 (Core Method)

Mosaic 是一个三组件集成的多模态越狱框架：

```
┌─────────────────────────────────────────────────────┐
│                    MOSAIC 框架                        │
│                                                       │
│  ┌──────────────┐  ┌──────────────────┐  ┌─────────┐ │
│  │ Text-Side    │  │ Multi-View       │  │ Surro-  │ │
│  │ Transfor-    │  │ Image Optimi-    │  │ gate    │ │
│  │ mation       │  │ zation          │  │ Ensem-  │ │
│  │              │  │                  │  │ ble     │ │
│  │ 扰动拒绝敏感 │  │ 多裁剪视图下的   │  │ Guidance│ │
│  │ 词汇模式     │  │ 图像扰动更新      │  │         │ │
│  └──────┬───────┘  └────────┬─────────┘  └────┬────┘ │
│         │                   │                 │        │
│         ▼                   ▼                 ▼        │
│  ┌──────────────────────────────────────────────────┐ │
│  │           联合优化 → 对抗样本 (δ_text, δ_image)  │ │
│  └──────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

**（1）Text-Side Transformation Module（文本侧转换模块）**
- 目标：识别并扰动那些最容易触发 VLM 拒绝响应的敏感词汇模式
- 方法：通过梯度分析找出 "sorry"、"cannot"、"against policy" 等 refusal-sensitive tokens，对这些 token 周围的 embedding 进行定向偏移
- 效果：降低模型的"拒绝反射"，使恶意请求更容易被"接受处理"

**（2）Multi-View Image Optimization Module（多视图图像优化模块）**
- 核心创新：不是在整个图像上统一优化扰动，而是将图像裁剪为多个子区域（views），在每个 view 上独立优化扰动
- 关键洞察：单一视图的扰动会过拟合该特定视角的特征分布，而多视图优化迫使扰动在不同空间区域都有效 → 更好的跨模型泛化性
- 实现：随机采样 N 个裁剪窗口 {crop₁, crop₂, ..., cropₙ}，轮流在每个 crop 上执行 PGD 步骤

**（3）Surrogate Ensemble Guidance Module（替代模型集成引导模块）**
- 使用 K 个不同的开源 VLM 作为替代模型集合 {S₁, S₂, ..., Sₖ}（如 LLaVA-1.5、Qwen-VL、MiniGPT-4 等）
- 每个替代模型提供独立的梯度信号 ∇_i L(·)
- 最终梯度 = 加权聚合: ∇_ensemble = Σᵢ wᵢ · ∇_i L(·)
- 权重 wᵢ 根据每个替代模型在验证集上的"目标一致性"动态调整

## Module 3 — 技术细节 (Technical Details)

### 完整优化算法

```python
class MosaicAttack:
    """Mosaic 多视图集成多模态越狱"""

    def __init__(
        self,
        surrogate_models: list,  # K 个替代 VLM
        target_model_api=None,    # 闭源目标 API
        n_views: int = 8,         # 裁剪视图数量
        ensemble_weight_strategy="dynamic",
    ):
        self.surrogates = surrogate_models
        self.n_views = n_views
        self.target_api = target_model_api

    def optimize(
        self,
        image: torch.Tensor,
        text_prompt: str,
        max_iter: int = 500,
        eps: float = 8/255,  # L∞ 范围
        step_size: float = 0.01,
    ) -> tuple[torch.Tensor, str]:
        """
        返回优化后的对抗图像 δ_image 和对抗文本 δ_text
        """

        # === 阶段 A: 文本侧转换 ===
        delta_text = self.text_side_transform(text_prompt)

        # === 阶段 B: 图像多视图联合优化 ===
        delta_image = torch.zeros_like(image)
        image_adv = image.clone().detach()

        for t in range(max_iter):
            # (1) 采样 M 个裁剪视图
            crops, bboxes = self.sample_multi_view_crops(image_adv, m=self.n_views)

            # (2) 每个视图上计算集成梯度
            grad_ensemble = torch.zeros_like(image_adv)

            for crop, bbox in crops:
                view_grads = []
                for i, model in enumerate(self.surrogates):
                    # 将裁剪图 + 文本输入替代模型
                    logits = model(crop, delta_text + text_prompt)
                    loss = self.adversarial_loss(logits)  # 最大化有害输出概率
                    g_i = torch.autograd.grad(loss, crop)[0]
                    view_grads.append(g_i)

                # 集成聚合
                g_ens = self.ensemble_aggregate(view_grads)

                # 将视图梯度映射回完整图像空间
                grad_full = self.map_gradient_to_full(g_ens, bbox, image_adv.shape)
                grad_ensemble += grad_full

            # 平均化多视图梯度
            grad_ensemble /= len(crops)

            # (3) PGD 更新步
            delta_image = delta_image + step_size * grad_ensemble.sign()
            delta_image = torch.clamp(delta_image, -eps, eps)
            image_adv = torch.clamp(image + delta_image, 0, 1)

        return image_adv.detach(), delta_text

    def text_side_transform(self, prompt: str) -> str:
        """
        识别并扰动 refusal-sensitive tokens
        """
        tokens = self.tokenizer.encode(prompt)
        sensitive_pattern = self.identify_refusal_tokens(tokens)

        # 对 sensitive tokens 的 embedding 施加小幅度偏移
        perturbed_embeds = self.embed(tokens).clone()
        for idx in sensitive_pattern:
            direction = self.compute_refusal_avoidance_direction(idx)
            perturbed_embeds[idx] += self.alpha * direction

        return self.decode_with_perturbation(perturbed_embeds)

    def ensemble_aggregate(
        self,
        gradients: list[torch.Tensor],
    ) -> torch.Tensor:
        """
        动态加权集成聚合
        """
        if self.strategy == "uniform":
            return sum(g / len(g) for g in gradients)

        elif self.strategy == "dynamic":
            weights = []
            for i, g in enumerate(gradients):
                # 根据替代模型与目标的一致性历史分配权重
                w_i = self.consistency_scores[i]
                weights.append(w_i)

            total = sum(weights)
            return sum(w / total * g for w, g in zip(weights, gradients))
```

### 代理依赖的形式化定义与缓解

设 S 为替代模型，T 为目标模型，f_S 和 f_T 分别为其决策函数。
代理依赖度定义为:

D(S,T) = |ASR(f_T, x*_S) - ASR(f_T, x*_T)|

其中 x*_S 为在 S 上优化的最优对抗样本，x*_T 为理想情况下在 T 上优化的最优对抗样本。

Mosaic 通过以下方式减小 D(S,T):

1. **减少单一代理偏差**: E[S] = Σᵢ wᵢ·Sᵢ → 降低对任意单一 Sᵢ 的依赖
2. **减少单视图过拟合**: 多视图采样迫使扰动在全图范围有效
3. **文本-图像联合优化**: 同时扰动两个模态增加迁移鲁棒性

### 关键发现

1. **代理依赖是真实且严重的**：单一替代模型在同构→异构切换时 ASR 下降超过 30pp，这意味着之前大量论文报告的攻击成功率在实际攻击场景中大打折扣。
2. **多代理+多视图的组合效应**：两个维度的"去依赖"策略叠加产生超线性增益——单独使用时各自有效，组合时效果显著超越简单加和。
3. **闭源 VLM 并非安全堡垒**：即使在最保守估计下，Mosaic 仍能在 GPT-4o 上达到 >60% ASR，表明商业部署的多模态安全防线仍有巨大漏洞。

## Module 5 — 局限性分析 (Limitations)

1. **计算成本高**: 多模型 × 多视图 × 多迭代的设计使攻击计算量比单模型方法高 5-10 倍。对于实时攻击场景可能不适用。
2. **仅限白盒替代模型**: 方法仍然需要可访问的开源替代 VLM 来计算梯度。如果目标模型属于全新架构且无可用替代品（如某些专有 VLM），攻击可行性受限。
3. **防御方的发展空间**: 论文提到三种潜在防御方向——输入预处理（如 DiffPure）、对抗训练、以及检测机制——但尚未有系统性的防御方案能同时保持低误报率和高召回率。
4. **语义完整性风险**: Text-Side Transformation 可能导致攻击 prompt 的语义发生微妙变化，使得生成的有害内容不完全符合攻击者原始意图（虽然 ASR 高但精确控制能力有限）。
5. **伦理边界**: 作为攻击方法论文，其技术细节可能被滥用。作者需确保负责任的披露流程。
