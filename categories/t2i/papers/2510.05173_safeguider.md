## 摘要

文本到图像模型能够根据自然语言描述生成高质量图像，但这些模型容易受到对抗性提示的攻击，从而绕过安全措施生成有害内容。

本文首先对 Stable Diffusion 模型的文本编码器进行实证研究，发现 **[EOS] 标记在良性提示和对抗性提示的嵌入空间中表现出不同的分布模式**。

基于此，提出 **SafeGuider**——一个两阶段框架，结合嵌入级识别模型和安全感知特征擦除束搜索算法，在保持良性提示生成质量的同时，有效防御域内和域外攻击。

## 六大模块深度解读

### 1. 背景与问题（动机、科学问题、直觉洞见）

**核心动机**：现有 T2I 模型的安全机制容易被对抗性提示绕过。

**科学问题**：如何识别并防御对抗性提示，同时不影响良性提示的生成？

**关键发现**：[EOS] 标记在良性/对抗提示的嵌入空间中分布不同，可作为检测特征。

### 2. 核心方法（技术贡献、方法框架）

**技术贡献**：
1. 发现 [EOS] 标记分布差异作为检测特征
2. 提出两阶段防御框架
3. 设计安全感知特征擦除束搜索算法

**方法框架**：
```
SafeGuider 两阶段防御：
┌──────────────────────────────────────────────────────────────┐
│  阶段 1: 嵌入级识别                                           │
│     - 检测 [EOS] 标记嵌入分布异常                              │
│     - 二分类器判断是否为对抗提示                                │
│     - 嵌入级安全特征识别                                       │
├──────────────────────────────────────────────────────────────┤
│  阶段 2: 安全感知特征擦除                                       │
│     - 束搜索算法优化安全特征                                    │
│     - 在嵌入空间重映射对抗提示                                  │
│     - 保持生成质量同时消除有害内容                              │
└──────────────────────────────────────────────────────────────┘
```

### 3. 技术细节

**[EOS] 标记分布分析**：

| 提示类型 | [EOS] 嵌入均值 | [EOS] 嵌入方差 | 分布特征 |
|----------|---------------|---------------|----------|
| 良性提示 | μ₁ | σ₁ | 紧凑分布 |
| 对抗提示 | μ₂ | σ₂ | 分散分布 |

**安全感知特征擦除**：
$$
\min_{w} \mathcal{L}_{\text{safe}}(x_{\text{adv}} \odot w) + \lambda \|w - 1\|_2^2
$$
其中 $w$ 是特征掩码，$\odot$ 是逐元素乘法。

**束搜索算法**：
```python
def beam_search_safe_rewrite(prompt_embeds, beam_width=5):
    """
    束搜索安全重写

    目标：找到最优的嵌入修改，使对抗提示变为安全
    """
    candidates = [(prompt_embeds, 0.0)]

    for step in range(max_steps):
        new_candidates = []
        for emb, score in candidates:
            # 生成候选修改
            modifications = generate_modifications(emb, k=beam_width)

            for mod in modifications:
                # 评估安全性
                safety_score = evaluate_safety(mod)

                # 评估质量保持
                quality_score = evaluate_quality(mod, original_prompt)

                # 综合得分
                total_score = alpha * safety_score + beta * quality_score

                new_candidates.append((mod, total_score))

        # 保留最优候选
        candidates = top_k(new_candidates, beam_width)

    return candidates[0][0]  # 返回最优结果
```

### 4. 实验设置与结果

**实验设置**：
- 目标模型：Stable Diffusion
- 攻击类型：多种对抗攻击
- 评测指标：ASR（攻击成功率）、FID（生成质量）

**结果表格**：

| 攻击类型 | 基线 ASR | SafeGuider ASR | ASR 降低 |
|----------|---------|-----------------|----------|
| GCG | 72.3% | 4.2% | -94.2% |
| SOTA Adv | 68.5% | 5.1% | -92.6% |
| 未知攻击 | 65.8% | 5.48% | -91.7% |

**质量保持**：
- FID 与原始模型几乎相同
- 良性提示生成质量保持 > 95%

### 5. 局限性分析

1. **计算开销**：束搜索需要多次前向传播
2. **阈值选择**：安全阈值需要仔细调整
3. **模型依赖**：发现基于 SD 的 [EOS] 分布差异

**核心代码片段**：
```python
import torch
from safeguider import SafeGuider

# 初始化 SafeGuider
safeguider = SafeGuider(
    model_id="stabilityai/stable-diffusion-2-1",
    device="cuda"
)

# 加载预训练识别器
safeguider.load_detector('detector_checkpoint.pth')

# 对提示进行安全检查
def safe_generate(prompt: str):
    # 阶段 1: 检测是否为对抗提示
    is_adversarial, risk_score = safeguarder.detect(prompt)

    if is_adversarial:
        # 阶段 2: 安全重写
        safe_prompt = safeguarder.rewrite(
            prompt,
            beam_width=5,
            alpha=0.7,  # 安全性权重
            beta=0.3   # 质量保持权重
        )
        return safeguarder.generate(safe_prompt)
    else:
        return safeguarder.generate(prompt)

# 使用示例
result_image = safe_generate("a peaceful landscape")
```
