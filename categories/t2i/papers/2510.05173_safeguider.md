## 一、背景动机

### 1.1 研究问题

文本到图像（T2I）模型能够根据自然语言描述生成高质量图像，这一能力在创意设计、艺术创作等领域展现出巨大价值。然而，这些模型也容易被**对抗性提示**攻击绕过安全措施，生成有害内容。

对抗性提示的形式多样：

1. **拼写变形**：se.x, s3x, s-e-x
2. **隐喻表达**：babe, provocative, hentai
3. **编码攻击**：Unicode 混淆、拼音、表情符号
4. **组合绕过**：多个无害词组合产生有害效果

### 1.2 现有防御局限

| 方法 | 防御方式 | 主要问题 |
|------|----------|----------|
| 关键词过滤 | 黑名单匹配 | 容易被变形绕过 |
| NSFW 分类器 | 输出图像检测 | 事后检测，无法预防 |
| 嵌入抑制 | 修改文本嵌入 | 对隐喻泛化能力差 |
| 拒绝生成 | 检测到攻击直接拒绝 | 影响正常用户体验 |

### 1.3 关键观察

论文作者对 Stable Diffusion 的文本编码器进行了深入实证研究，发现了一个关键现象：

> **[EOS]（End-of-Sequence）标记在良性提示与对抗性提示的嵌入空间中表现出显著不同的分布模式。**

这一发现为检测对抗性提示提供了新的思路。

### 1.4 核心贡献

SafeGuider 提出了一种**两阶段安全感知的防御框架**：

> **结合嵌入级识别模型与安全感知的特征擦除束搜索算法，在保持良性提示生成质量的同时，有效防御领域内和领域外攻击。**

核心贡献：

1. **[EOS] 嵌入分析**：首次揭示 [EOS] 标记分布差异作为对抗性检测信号
2. **两阶段框架**：识别 + 控制的协同防御
3. **特征擦除束搜索**：不拒绝生成，而是替换为安全内容
4. **模型通用性**：可推广至 Flux 等其他 T2I 架构

## 二、核心方法

### 2.1 方法概述

SafeGuider 的核心思想是**在生成过程中动态引导内容向安全方向偏移**，而非简单地拒绝生成或输出黑图。

### 2.3 关键算法

**[EOS] 嵌入检测器**：

```python
class EOSDetector(nn.Module):
    """
    基于 [EOS] 嵌入分布的对抗性提示检测器
    """
    def __init__(self, embedding_dim=768):
        super().__init__()
        # 轻量级分类头
        self.classifier = nn.Sequential(
            nn.Linear(embedding_dim, 256),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(256, 2)  # safe vs adversarial
        )
    
    def forward(self, eos_embedding):
        """
        Args:
            eos_embedding: [EOS] 标记的嵌入向量
        Returns:
            logits: [safe_score, adversarial_score]
        """
        return self.classifier(eos_embedding)

def detect_adversarial(prompt, detector, text_encoder):
    """
    检测提示是否为对抗性
    """
    # 编码提示
    tokens = tokenize(prompt)
    
    # 获取 [EOS] 嵌入
    eos_embedding = text_encoder(tokens)[-1]  # 最后一个 token
    
    # 分类
    logits = detector(eos_embedding)
    probs = F.softmax(logits, dim=-1)
    
    return probs[1] > threshold  # adversarial
```

**特征擦除束搜索**：

```python
def safe_feature_erasure_beam_search(
    noisy_latent, 
    text_embedding, 
    model, 
    safety_threshold=0.3,
    beam_width=5
):
    """
    安全感知的特征擦除束搜索
    
    在多个生成路径中选择最安全的，同时保持内容有意义
    """
    candidates = []
    
    for _ in range(beam_width):
        # 随机采样初始噪声
        z = sample_noise(noisy_latent)
        
        # 去噪一步
        z_next = model.denoise_step(z, text_embedding)
        
        # 评估安全性
        safety_score = evaluate_safety(z_next, model.safety_classifier)
        
        # 评估内容质量
        quality_score = evaluate_quality(z_next, model.clip)
        
        # 综合评分
        combined_score = quality_score - lambda_safety * (1 - safety_score)
        
        candidates.append({
            'latent': z_next,
            'safety': safety_score,
            'quality': quality_score,
            'combined': combined_score
        })
    
    # 选择安全性达标且质量最高的
    valid_candidates = [c for c in candidates if c['safety'] > safety_threshold]
    
    if valid_candidates:
        return max(valid_candidates, key=lambda x: x['quality'])
    else:
        return max(candidates, key=lambda x: x['safety'])  # 退化为最安全
```

## 三、技术细节

### 3.1 [EOS] 嵌入分布分析

作者发现对抗性提示会导致 [EOS] 嵌入产生系统性偏移：

```
良性提示分布:
     *
    ***
   *****
  *******
 *********
[EOS] Embedding Space ─────────────────→

对抗提示分布:
              *
             ***
            *****
           *******
          *********
[EOS] Embedding Space ─────────────────→
```

**原因分析**：

1. 对抗性提示通常包含异常字符或变形词
2. 这些异常在编码器中产生累积误差
3. 误差在序列末端（[EOS]）最为显著
4. [EOS] 因此成为对抗性信号的自然载体

### 3.2 识别阶段实现

```python
# 训练数据准备
benign_data = load_benign_prompts()  # 正常提示
adversarial_data = load_adversarial_prompts()  # 对抗提示

# 训练分类器
detector = EOSDetector(embedding_dim=768)
optimizer = torch.optim.Adam(detector.parameters(), lr=1e-4)

for epoch in range(50):
    for batch in dataloader:
        prompts, labels = batch
        
        # 获取 [EOS] 嵌入
        eos_emb = text_encoder(prompts)[:, -1, :]  # 最后一个 token
        
        # 分类损失
        logits = detector(eos_emb)
        loss = F.cross_entropy(logits, labels)
        
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

### 3.3 控制阶段实现

```python
class SafeGuiderPipeline:
    def __init__(self, sd_model, detector, safety_threshold=0.3):
        self.model = sd_model
        self.detector = detector
        self.safety_threshold = safety_threshold
    
    @torch.no_grad()
    def generate(self, prompt, num_inference_steps=50):
        # 第一步：检测是否为对抗性提示
        is_adversarial = detect_adversarial(prompt, self.detector, self.model.text_encoder)
        
        if is_adversarial:
            # 第二步：启用特征擦除束搜索
            return self._safe_generation(prompt, num_inference_steps)
        else:
            # 正常生成
            return self.model(prompt, num_inference_steps)
    
    def _safe_generation(self, prompt, num_steps):
        # 编码文本
        text_emb = self.model.text_encoder(prompt)
        
        # 初始化噪声
        latents = torch.randn(1, 4, 64, 64, device=self.model.device)
        
        # 去噪过程
        for t in reversed(range(num_steps)):
            # 束搜索选择最安全的隐变量
            best_candidate = safe_feature_erasure_beam_search(
                latents, text_emb, self.model
            )
            latents = best_candidate['latent']
        
        # 解码
        images = self.model.vae.decode(latents)
        return images
```

## 四、数学原理

### 4.1 [EOS] 分布差异度量

设 $e_{eos}$ 为 [EOS] 嵌入，$D_{benign}$ 和 $D_{adv}$ 分别为良性提示和对抗提示的 [EOS] 嵌入分布。SafeGuider 通过以下度量量化分布差异：

**分布中心距离**：

$$\Delta \mu = \|\mu_{benign} - \mu_{adv}\|_2$$

其中 $\mu$ 是分布的均值向量。

**分布方差比**：

$$\rho = \frac{\sigma_{adv}^2}{\sigma_{benign}^2}$$

作者发现对抗提示的 [EOS] 嵌入通常具有更大的方差和偏移的均值。

### 4.2 安全评分函数

给定隐变量 $z$，其安全评分定义为：

$$S_{safety}(z) = 1 - \frac{1}{1 + \exp(-\mathbf{w}^T z - b)}$$

其中 $\mathbf{w}, b$ 是通过安全分类器学习的参数。

### 4.3 束搜索优化目标

束搜索选择最优隐变量 $z^*$：

$$z^* = \arg\max_{z \in \mathcal{B}} \left[ \alpha \cdot S_{quality}(z) - \beta \cdot (1 - S_{safety}(z)) \right]$$

其中 $\mathcal{B}$ 是候选隐变量束，$\alpha, \beta$ 是平衡权重。

## 七、总结与展望

### 7.1 主要贡献

SafeGuider 创新性地将 **[EOS] 嵌入分布差异**用于对抗性提示检测，并设计了**不拒绝生成而引导向安全方向**的防御策略：

1. **新颖检测信号**：[EOS] 嵌入分布作为对抗性检测的有效指标
2. **两阶段协同**：识别 + 控制的完整防御框架
3. **用户体验优先**：不拒绝生成，保持内容有意义
4. **模型通用性**：可推广至多种 T2I 架构

### 7.2 局限性

1. 检测器需要对抗样本训练数据
2. 特征擦除可能影响某些边缘情况
3. 额外推理延迟影响实时性

### 7.3 未来方向

1. 零样本对抗检测
2. 自适应特征擦除
3. 多模态安全控制
