## 二、背景动机

### 2.1 研究问题

文本到图像扩散模型能够根据文本描述生成高质量图像，但用户通常无法一次性获得满意的结果。对输入提示的微小修改往往会导致截然不同的图像，这使得精确的语义控制变得困难。

### 2.2 现有方法的局限

| 方法 | 特点 | 局限 |
|------|------|------|
| 提示工程 | 调整文本描述 | 耗时、需要经验 |
| 图像编辑 | 后期处理 | 非端到端、交互复杂 |
| 注意力操控 | 控制特征 | 需要模型内部访问 |

### 2.3 研究贡献

SEGA 提出通过语义引导（Semantic Guidance）在扩散过程中与模型交互，实现：

1. **细粒度控制**：沿语义方向引导生成
2. **广泛编辑**：改变构图和风格
3. **艺术优化**：优化整体艺术构思
4. **即插即用**：适用于多种扩散模型架构

## 三、核心方法

### 3.1 方法概述

SEGA 的核心思想是在扩散过程中，通过分析模型的语义空间，识别并引导生成过程沿特定语义方向移动。

### 3.2 语义方向识别

SEGA 使用 CLIP 模型识别语义方向：

1. **方向定义**：给定两个文本描述 $c_{pos}$ 和 $c_{neg}$，定义语义方向为：

$$\vec{d} = \frac{E(c_{pos}) - E(c_{neg})}{\| E(c_{pos}) - E(c_{neg}) \|}$$

其中 $E(\cdot)$ 是 CLIP 文本编码器。

2. **交叉注意力分析**：分析模型在生成过程中不同区域的交叉注意力图

3. **方向投影**：将语义方向投影到潜在空间

### 3.3 语义引导采样

在去噪过程中应用语义引导：

```python
def sega_denoise_step(z_t, t, c, directions):
    for direction in directions:
        # 计算当前表示与方向的匹配度
        alignment = compute_alignment(z_t, direction)
        
        # 调整去噪方向
        guidance = direction.strength * direction.sign * alignment
        
        # 应用语义引导
        z_t = z_t + guidance * noise_schedule(t)
    
    # 正常去噪
    return denoise(z_t, t, c)
```

### 3.4 引导强度控制

SEGA 支持对不同语义方向设置不同的引导强度：

| 语义属性 | 示例方向对 | 应用场景 |
|----------|----------|----------|
| 颜色 | "红色" - "蓝色" | 调整色调 |
| 情绪 | "快乐" - "悲伤" | 情感控制 |
| 风格 | "写实" - "卡通" | 风格迁移 |
| 视角 | "正面" - "侧面" | 视角调整 |

## 四、技术细节

### 4.1 架构兼容性

SEGA 设计为与多种架构兼容：

1. **潜在空间扩散**：如 Stable Diffusion
2. **像素空间扩散**：如 DALL-E 2
3. **Transformer 架构**：如 DiT

### 4.2 引导注入点

| 注入点 | 描述 | 效果 |
|--------|------|------|
| 注意力层 | 直接修改注意力图 | 局部效果强 |
| UNet 特征 | 修改特征表示 | 全局效果强 |
| 噪声空间 | 修改噪声表示 | 结构影响大 |

### 4.3 动态引导调度

SEGA 采用动态引导强度：

$$\alpha(t) = \alpha_{max} \cdot \sin\left(\frac{\pi t}{T}\right)$$

其中 $t$ 是当前时间步，$T$ 是总步数。

### 4.4 多方向组合

支持同时应用多个语义方向：

$$\vec{G}_{total} = \sum_i w_i \cdot \vec{d}_i$$

其中 $w_i$ 是第 $i$ 个方向的权重。

## 五、数学原理

### 5.1 CLIP 语义空间

给定文本概念 $c$，CLIP 文本编码器输出归一化的嵌入：

$$\hat{e}(c) = \frac{E(c)}{\|E(c)\|}$$

### 5.2 语义方向计算

两个概念间的语义方向定义为：

$$\vec{d}(c_1, c_2) = \frac{\hat{e}(c_1) - \hat{e}(c_2)}{\|\hat{e}(c_1) - \hat{e}(c_2)\|}$$

### 5.3 引导向量

在时间步 $t$，引导向量为：

$$\vec{g}_t = \vec{d} \cdot \alpha(t) \cdot \beta$$

其中 $\alpha(t)$ 是时间调度，$\beta$ 是引导强度参数。

### 5.4 引导损失

SEGA 优化目标包含引导损失：

$$\mathcal{L}_{guidance} = -\log p(c_{target} | z_0) + \lambda \cdot \|\vec{d} - \vec{d}_{target}\|^2$$

## 八、代码解析

### 8.1 方向计算

```python
def compute_semantic_direction(
    model, 
    positive_concept: str, 
    negative_concept: str
):
    """计算两个概念间的语义方向"""
    with torch.no_grad():
        pos_emb = model.get_text_embedding(positive_concept)
        neg_emb = model.get_text_embedding(negative_concept)
    
    direction = pos_emb - neg_emb
    direction = direction / direction.norm()
    
    return direction
```

### 8.2 引导注入

```python
class SEMANTICGUIDANCE:
    def __call__(self, z_t, t, directions, model):
        # 对每个语义方向
        total_guidance = 0
        for d in directions:
            # 计算对齐度
            alignment = torch.einsum('bcd,c->bd', z_t, d.vector)
            # 累积引导
            total_guidance += d.strength * d.sign * alignment
        
        # 应用时间调度
        alpha_t = self.schedule(t)
        guidance = alpha_t * total_guidance
        
        # 修改潜在表示
        z_t = z_t + guidance
        return z_t
```

## 九、总结与展望

### 9.1 主要贡献

1. **通用语义引导**：适用于多种扩散模型架构
2. **细粒度控制**：实现精准的语义方向控制
3. **即插即用**：无需训练，直接应用
4. **多方向组合**：支持复杂的多维语义控制

### 9.2 局限性

1. **CLIP 依赖**：依赖 CLIP 语义空间的准确性
2. **方向冲突**：多个方向可能相互冲突
3. **过度引导**：强引导可能导致图像质量下降

### 9.3 未来方向

1. **自动方向学习**：从用户反馈中学习最优方向
2. **条件引导**：根据内容自适应调整引导
3. **3D 生成**：扩展到 3D 场景生成
