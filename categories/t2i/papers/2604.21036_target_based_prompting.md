## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
T2I 公平性治理里长期有一个隐含假设：只要把输出往“均匀分布”推，就算更公平。但现实里，不同场景、地区、职业和产品目标，对“公平”并没有唯一答案。论文直接挑战这个默认前提，指出真正缺的不是又一个 bias detector，而是一套**能让用户显式声明目标分布、并审计模型是否朝该目标移动**的推理时框架。

### 核心科学问题 (Core Scientific Question)
能否在**不重训底层 T2I 模型**的前提下，只通过 inference-time prompt rewriting，就把生成结果的人口统计分布稳定拉向用户声明的目标分布，并提供可审计的误差指标？

### 直觉洞见 (Intuitive Insight)
这篇论文的直觉非常清楚：很多所谓公平性方法，其实偷偷把“研究者认为公平的分布”写死了。Target-Based Prompting 则把问题反过来——先让人声明目标，再把 prompt 拆成多个带群体属性的子 prompt，按目标比例分配生成预算。这样模型不是被笼统地要求“更公平”，而是被明确要求“朝这个分布移动”。

## Module 2 — 核心方法 (Core Method)

### 方法总览
Target-Based Prompting 是一个完全发生在推理期的框架，主要由四步组成：

**（1）声明目标分布 q**
- 目标可以是：
  - **Uniform**：均匀分布；
  - **Context-based**：基于地区/场景/职业统计；
  - **User-defined**：用户自己给定；
  - **Fallback**：当无可靠统计时回退到默认目标。

**（2）LLM 检索与群体拆解**
- LLM 不负责画图，而是负责：
  - 识别与 prompt 相关的人口统计维度；
  - 输出群体类别、比例、引用来源和置信度；
  - 构造人口统计特定的子 prompt。

**（3）按目标分布分配生成预算**
- 若总生成数为 N，某群体目标占比为 q_i，则为该子 prompt 分配 N_i ≈ q_iN；
- 取整后微调，保证总预算不变。

**（4）输出后审计**
- 用人脸检测 + 肤色估计，把实际输出分布映射到 Fitzpatrick I–VI；
- 再计算与目标分布的偏差，衡量是否“真的对齐”。

### 方法定位
这不是又一个“公平性 benchmark”，也不是再训练模型的 mitigation。它的核心价值是提出了一个**目标显式化 + 审计可回溯**的 inference-time fairness control layer。

## Module 3 — 技术细节 (Technical Details)

### 目标预算分配
若目标分布为 q = {q_1, ..., q_K}，总生成数为 N，则每个群体分配：

N_i ≈ q_i N

系统会做取整校正，使 Σ_i N_i = N。

### 对齐误差指标
当目标被定义在肤色空间时，论文采用 Alignment Error：

Alignment Error = Σ_i (p_i - q_i)^2

其中：
- p_i：实际输出分布
- q_i：声明目标分布

数值越低，表示输出越贴近目标。

### 评估空间与映射
- 输入侧使用 **race / ethnicity** 等人口统计标签；
- 输出侧不直接“预测种族”，而是测量**skin tone**；
- 论文以 **Fitzpatrick I–VI** 为主评估空间，并将其聚合成：
  - **Light** = I–II
  - **Medium** = III–IV
  - **Dark** = V–VI
- 附录还用 **Monk Skin Tone (MST 1–10)** 做一致性验证。

### 关键实现细节
- 人脸检测：**RetinaFace**
- 肤色估计：基于 **ITA (Individual Typology Angle)**
- 图像生成参数：
  - 分辨率：**768×512**
  - 推理步数：**40**
  - guidance scale：**7.5**
  - dtype：**float16**
- LLM 检索额外开销：每个唯一 prompt 约 **300–500 ms**

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与测试对象
- **30 个职业 prompt**，分高/中/低社会地位三组；
- **6 个非职业 prompt**，如 smiling person、crime-related、homelessness-related；
- 总计 **36 个 prompt**；
- 共生成 **6600 张图像**，人脸过滤丢弃不到 **2%**。

### 被测 T2I 模型
1. **SD Realistic Vision v5.1**（主实验）
2. **SDXL Turbo**
3. **SD 1.5**
4. **DALL-E 2**

### 关键结果一：基线偏差确实系统存在
| 职业层级 | 代表基线分布现象 |
|------|----------------|
| 高地位职业 | Fitzpatrick Type II 占 **69%**，Type V–VI 合计不到 **2%** |
| 中地位职业 | Type II 占 **33%**，Type V–VI 占 **18%** |
| 低地位职业 | Type V–VI 超过 **48%**，Type I–II 仅 **24%** |

这说明偏差不是个别职业噪声，而是和职业层级系统性绑定。

### 关键结果二：Uniform 目标显著降低误差
| 模型 | 平均 Alignment Error（Baseline）↓ | Ours (Uniform) ↓ | 改善 |
|------|----------------------------------:|-----------------:|------:|
| SD Realistic Vision v5.1 | 0.129 | **0.030** | **76.7%** |
| SDXL Turbo | 0.147 | **0.013** | **91.1%** |
| SD 1.5 | 0.039 | **0.012** | **68.6%** |
| DALL-E 2 | 0.029 | **0.007** | **77.3%** |

### 关键结果三：具体职业上的分布被拉回目标
#### 高地位职业
- Type II：**69% → 27%**
- Type V–VI：**<2% → 30%**

#### 中地位职业
- Type II：**33% → 24%**
- Type V–VI：**18% → 33%**

#### 低地位职业
- Type VI：**34% → 18%**

### 关键结果四：优于现有推理时/后处理方法
| 职业组 | Baseline | EntiGen | ITI-GEN | Fair Diffusion | Ours |
|------|---------:|--------:|--------:|---------------:|-----:|
| 高地位 | 0.179 | 0.305 | 0.344 | 0.077 | **0.023** |
| 中地位 | 0.122 | 0.294 | 0.274 | **0.009** | 0.027 |
| 低地位 | 0.087 | 0.535 | 0.388 | 0.170 | **0.039** |
| 平均 | 0.129 | 0.378 | 0.335 | 0.086 | **0.030** |

论文指出，EntiGen / ITI-GEN 常把结果整体推向更深肤色，反而放大偏差；Fair Diffusion 在中地位组强，但在低地位组反而恶化；本文平均误差最优。

### 关键结果五：Extreme 目标验证“可控性”
在高地位职业的 Extreme 目标实验中，论文将目标设为 Black 80%、Indian 10%、White 5%、Asian 5%。生成后聚合结果达到：
- **Light / Medium / Dark = 12% / 2% / 86%**

说明该框架不仅能“去偏”，还能做**可审计的方向性控制**。

## Module 5 — 局限性分析 (Limitations)

1. **目标分布本身未必可信**：若 LLM 检索到的人口统计不准确，系统可能把输出拉向一个同样有偏的目标。  
2. **肤色不是身份的完整代理**：用 Fitzpatrick / MST 审计很实用，但不能完全覆盖 race/ethnicity 的社会含义。  
3. **可能存在 concept bleeding**：加入人口统计标签后，不只会改肤色，也可能改变服装、背景、面部特征等其他视觉属性。  
4. **只是 inference-time control，不是训练级修复**：底层模型偏差没有被根治，只是被更透明地引导。  
5. **对多人场景、复杂交互 prompt 的研究还不够**：当前实验主要聚焦单人 headshot 场景。
