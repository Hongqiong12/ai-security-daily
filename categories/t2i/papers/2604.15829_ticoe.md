## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
T2I 概念擦除一直有个核心矛盾：纯文本方法往往擦不彻底，遇到 prompt 变体或对抗提示仍会漏；而简单加图像监督虽然能增强视觉约束，却很容易把“长得像但语义不同”的无关概念一起抹掉，导致过擦除。对现实安全治理而言，只会“粗暴删掉”并不够，真正难的是**既删得准，又保得住无关能力**。

### 核心科学问题 (Core Scientific Question)
能否设计一种图文协同的概念擦除框架，在维持高擦除强度的同时，系统降低对相似无害概念的误伤，并且对提示改写、对抗提示和 red-team prompt 依然保持稳定？

### 直觉洞见 (Intuitive Insight)
TICoE 的核心直觉是：文本端解决“表达变体覆盖”，图像端解决“视觉相似误伤”。如果只靠 prompt 做擦除，模型记住的概念表达空间太大；如果只靠视觉锚点，又会把 camera、phone 这类与 gun 视觉相似但语义不同的对象一起抹掉。只有图文协同，才可能把概念擦除从“粗糙屏蔽”推进到“精准外科手术”。

## Module 2 — 核心方法 (Core Method)

### 方法总览
TICoE（Text-image Collaborative Erasing）由两个核心模块组成：

1. **CCCM（Continuous Convex Concept Manifold）**
   - 用多个语义近邻文本提示构造连续凸概念流形；
   - 用 prompt bank 覆盖目标概念的不同语言表达；
   - 通过凸组合而不是单一 prompt，增强对变体和隐藏表达的鲁棒性。

2. **HVRL（Hierarchical Visual Representation Learning）**
   - 基于多尺度视觉 latent token 学习目标概念的层次化视觉表征；
   - 通过 Transformer 融合不同尺度信息，显式区分“真正属于目标概念的视觉模式”和“只是形态相似的无关对象”。

### 方法亮点
除了框架本身，论文还提出 **MCP（Morpho-Contextual Concept Preservation）** 指标，专门评估擦除后对相关无害概念的保留能力。它比只看 FID / CLIP 更贴近“有没有误伤”的真实问题。

## Module 3 — 技术细节 (Technical Details)

### 文本侧：CCCM
对于目标概念 c，作者先用 GPT 扩展得到多种语义近邻 prompt，编码成 prompt bank：

B = [e_1, ..., e_N]

训练时不直接固定某一个 prompt，而是做凸组合：

e_c = Σ_i w_i e_i, 其中 w ~ Dirichlet(1/τ)

这一步的要点在于：
- 组合结果始终位于原始 prompt 嵌入的凸包内；
- 不会像随意线性外推那样跑到 OOD 表达空间；
- 能系统覆盖 prompt 改写、语义近义与隐蔽表达。

论文默认超参中，**τ = 0.7** 表现最好。

### 视觉侧：HVRL
作者先让干净的 Stable Diffusion 生成目标概念参考图，再把 VAE latent 在多尺度上展开：

S = {1.0, 0.75, 0.5}

不同尺度 token 输入 Transformer encoder，最后将多尺度视觉信息融合回原始 latent，并用残差注入：

z_fused = z + λ · t_fused

其中 **λ = 0.5** 是论文的默认最优值。

### 擦除目标
作者构造负向 CFG 风格的目标噪声预测：
- 用原始模型在空提示与目标概念提示下的预测差定义概念方向；
- 再用待训练模型拟合抑制后的目标噪声。

训练损失为标准 L2 拟合损失，但关键在于目标不再只由文本决定，而是由**文本流形 + 层次视觉表示**共同定义。

### 实验配置
- 底座模型：默认 **Stable Diffusion 1.5**
- 也测试 **SD 1.4 / 1.5 / 2.0**
- 每个概念训练图像数：**200**
- 学习率：**1e-5**
- batch size：**1**
- 硬件：**RTX A6000 48GB**

## Module 4 — 实验设置与结果 (Experiments and Results)

### 任务设置
论文覆盖五类主任务：
- **nudity**
- **Van Gogh style**
- **gun**
- **church**
- **tench**

并在附录中补充 portrait 擦除与多概念联合擦除实验。

### 关键结果一：gun 擦除的主表结果
| 方法 | ASR ↓ | UDA ↓ | P4D ↓ | FID ↓ | CLIP ↑ |
|------|------:|------:|------:|------:|------:|
| ESD | 0.02 | 0.20 | 0.47 | 31.76 | 0.302 |
| UCE | 0.08 | 0.36 | 0.08 | 35.56 | 0.312 |
| FMN | 0.26 | 0.64 | 0.26 | 34.46 | 0.310 |
| Co-Erasing | 0.00 | 0.10 | 0.15 | 35.94 | 0.304 |
| **TICoE** | **0.00** | **0.02** | **0.04** | **30.86** | 0.304 |

这里最重要的不是 TICoE 把 ASR 做到 0，而是它在 **UDA = 0.02、P4D = 0.04、FID = 30.86** 上同时领先，说明其优势来自“鲁棒 + 精准 + 保真”的综合平衡。

### 关键结果二：MCP 明确证明它更少误伤
擦除 gun 后，对相近但无害概念的保留：

| 方法 | phone MCP ↑ | camera MCP ↑ | umbrella MCP ↑ |
|------|------------:|-------------:|---------------:|
| ESD | 79.59% | 68.25% | 92.59% |
| UCE | 93.88% | 87.30% | 94.44% |
| Co-Erasing | 53.06% | 39.68% | 55.56% |
| **TICoE** | **95.91%** | **92.06%** | **96.30%** |

这组结果是整篇论文的关键：Co-Erasing 擦得猛，但对 camera 只剩 **39.68%**，而 TICoE 还能保到 **92.06%**。说明 HVRL 真正抑制了视觉相似带来的过擦除。

### 关键结果三：跨概念任务稳定
扩展任务中，TICoE 依然保持较强综合性能：
- **nudity**：ASR **0.08**, UDA **0.42**, P4D **0.40**, FID **32.771**
- **Van Gogh**：ASR **0.00**, UDA **0.02**, P4D **0.06**, FID **31.653**
- **church**：ASR **0.02**, UDA **0.05**, P4D **0.10**, FID **31.795**
- **tench**：ASR **0.00**, UDA **0.14**, P4D **0.08**, FID **31.635**

这说明 TICoE 并不只适用于单一物体概念，对风格概念和 NSFW 概念同样有效。

### 关键结果四：消融验证 CCCM / HVRL 都必要
- 去掉 CCCM 后，UDA 从 **0.02** 回升到 **0.38**；
- 去掉 HVRL 后，CLIP 下降到 **0.285**，说明误伤和保真都受损；
- 多尺度也不是越多越好，扩到 4 个尺度后 FID 反而恶化到 **32.74**。

### 关键结果五：效率处于实用区间
| 方法 | Time ↓ |
|------|-------:|
| UCE | 2 min |
| FMN | 3 min |
| ESD | 44 min |
| **TICoE** | **52 min** |
| Co-Erasing | 121 min |

TICoE 不是最快，但比 Co-Erasing 明显便宜，同时在“擦除强度 + 鲁棒性 + MCP”上综合最好。

## Module 5 — 局限性分析 (Limitations)

1. **依赖 prompt bank 的质量与覆盖**：CCCM 的鲁棒性很大程度上来自 prompt 扩展。如果 prompt bank 不足，擦除效果也会随之下降。
2. **参考图像仍来自模型自生成**：视觉锚点不是人工标注真值，而是由原始 SD 生成，因此对长尾概念和难生成概念可能不稳定。
3. **MCP 仍是“局部相关概念集合”指标**：它比 FID / CLIP 更贴近误伤，但仍需研究者预定义哪些概念算“相近无害概念”。
4. **训练代价仍高于最轻量方法**：虽然优于 Co-Erasing，但与 UCE / FMN 这类快速方法相比，TICoE 仍需要更重的训练开销。
5. **多概念联合擦除仍可能出现顺序敏感**：附录显示 sequential erase 下会有轻微 spillover，说明多概念组合场景尚未完全解决。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)
```bash
git clone https://github.com/OpenAscent-L/TICoE.git
pip install torch torchvision diffusers transformers accelerate scikit-learn opencv-python pillow
```

### 核心代码框架 (Core Pipeline)
```python
prompt_bank = expand_prompts(target_concept)
text_embedding = sample_convex_prompt(prompt_bank, tau=0.7)
ref_images = generate_reference_images(target_concept, n=200)
latent = hierarchical_visual_fusion(ref_images, scales=[1.0, 0.75, 0.5], lam=0.5)
loss = erase_loss(model, latent, text_embedding, gamma=1.0)
```

### 复现步骤 (Reproduction Steps)
1. 为每个目标概念准备 **200 张** 自生成参考图像。
2. 用 LLM 扩展 prompt bank，并按 Dirichlet 分布采样凸组合文本表示。
3. 实现多尺度视觉 token 融合，尺度取 **{1.0, 0.75, 0.5}**。
4. 在 SD 1.5 上以 **lr = 1e-5, batch size = 1** 训练擦除模型。
5. 按论文协议评估 **ASR / UDA / P4D / FID / CLIP / MCP**。
6. 关键核验点：
   - gun 任务上 **ASR 0.00, UDA 0.02, P4D 0.04**；
   - camera 的 MCP 接近 **92.06%**；
   - 去掉 CCCM 后 UDA 明显上升；
   - 去掉 HVRL 后 CLIP 与保真明显下降。
