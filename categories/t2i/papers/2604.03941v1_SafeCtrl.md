# SafeCtrl: Region-Aware Safety Control for Text-to-Image Diffusion via Detect-Then-Suppress

**ArXiv ID**: 2604.03941v1
**Domain**: T2I / Image Generation Safety / Inference-time Control

## 1. 背景与动机 (Background & Motivation)
文本到图像 (T2I) 扩散模型（如 Stable Diffusion）存在生成不安全内容（如色情、暴力、侵权图像）的风险。传统的安全过滤器（如在生成后使用分类器拦截）会导致计算资源浪费，且用户体验差（表现为直接返回黑图或错误）。而基于概念擦除（Concept Erasure）的微调方法往往会损害模型生成安全图像的能力。SafeCtrl 提出了一种“即插即用”的推理期控制方法，通过在扩散去噪过程中精准定位并抑制不安全概念的生成。

## 2. 核心方法 (Methodology)
SafeCtrl 采用“先检测、后抑制”（Detect-Then-Suppress）的区域感知干预策略：
1. **Detect (检测)**：在去噪的前几个 timestep 中，利用交叉注意力图（Cross-attention Maps）和不安全概念词典，动态定位生成图像中正在浮现的不安全区域（Bounding Boxes/Masks）。
2. **Suppress (抑制)**：在后续的 timestep 中，对检测到的不安全区域实施干预，具体操作包括：
   - **注意力遮罩 (Attention Masking)**：切断该区域与不安全提示词的交叉注意力连接。
   - **引导削弱 (Guidance Degradation)**：在无分类器引导 (CFG) 中，对该区域注入负向的指导向量，推动该区域向安全概念（如“模糊”、“安全物体”）演化。

## 3. 技术细节 (Technical Details)
- **动态阈值分割**：为了获取准确的 Mask，SafeCtrl 对交叉注意力图进行高斯平滑，并采用 Otsu 阈值法自适应提取激活区域 $M_t$。
- **能量函数修改**：在采样更新公式中，将标准的预测噪声 $\epsilon_\theta$ 修改为：$\hat{\epsilon} = \epsilon_\theta(x_t, c_{safe}) + \gamma \cdot M_t \cdot (\epsilon_\theta(x_t, c_{safe}) - \epsilon_\theta(x_t, c_{unsafe}))$，利用负向向量将区域推离不安全分布。
- **即插即用**：该方法不需要重新训练或微调扩散模型，直接修改采样器的注意力机制即可生效。

## 4. 量化实验 (Quantitative Experiments)
- **防御成功率**：在 I2P (Inappropriate Image Prompt) 基准测试中，SafeCtrl 将不安全图像生成率从 32.5% 降低到了 1.2%。
- **图像质量保留**：由于是区域感知的干预，安全区域的生成质量未受影响，FID 分数变化仅为 +0.4（优于全局概念擦除方法 Erased Diffusion 的 +3.2）。
- **效率**：推理延迟增加不到 15%，远低于重绘（Inpainting）或重新生成的开销。

## 5. 局限性分析 (Limitations)
- **隐式概念逃逸**：如果恶意提示词使用极度隐晦的代指（即交叉注意力图未能明显激活不安全词元），Detect 阶段可能会漏报。
- **多概念冲突**：当图像中密集包含多个重叠的安全与不安全概念时，区域隔离可能导致图像局部出现明显的拼接或扭曲伪影。

## 6. 核心启示 (Takeaways)
- **细粒度干预**：T2I 安全干预正在从“全局拒绝”走向“局部修复”，这大幅提升了系统的可用性。
- **中间状态的价值**：扩散模型的交叉注意力图不仅是解释工具，更是实时安全干预的关键“抓手”，可以在恶意图像成型前将其掐断。
