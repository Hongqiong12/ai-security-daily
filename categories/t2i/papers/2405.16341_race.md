## 一、背景动机

### 1.1 研究问题

文本到图像（T2I）扩散模型（如 Stable Diffusion、Midjourney、DALL-E 等）已经能够根据文本描述生成高度逼真的图像。这种强大的生成能力也带来了被滥用的风险：恶意用户可能利用这些模型生成敏感、不当或误导性的图像内容，包括但不限于色情内容、暴力场景、版权侵权作品以及虚假信息。

概念擦除（Concept Erasure）技术旨在从训练好的扩散模型中移除特定概念，使其无法被特定提示激活。然而，传统的概念擦除方法存在一个关键缺陷：**它们无法抵抗对抗性攻击**。攻击者可以通过精心构造的对抗性文本嵌入来绕过概念擦除，恢复被擦除的概念。

### 1.2 核心挑战

对抗性攻击对概念擦除方法构成了严峻挑战：

1. **白盒攻击**：攻击者可以完全访问模型参数和梯度信息，构造精确的对抗性文本嵌入

2. **嵌入空间盲区**：传统擦除方法只在标准文本嵌入空间工作，对抗性嵌入可能绕过了这个空间

3. **灾难性遗忘**：过度擦除可能损害模型生成其他正常内容的能力

4. **黑盒泛化**：擦除方法需要同时抵御白盒和黑盒攻击

### 1.3 核心贡献

R.A.C.E. 提出了一种**鲁棒对抗性概念擦除**方法，其核心思想是：

> **通过对抗训练框架识别并抑制对抗性文本嵌入，从而增强概念擦除方法对各种对抗性攻击的鲁棒性。**

具体贡献包括：

1. **对抗训练框架**：首次将对抗训练引入概念擦除领域，提升模型对恶意输入的抵抗能力

2. **对抗性嵌入识别**：识别在对抗性文本嵌入空间中的脆弱区域

3. **显著性能提升**：在针对"裸露"概念的白盒攻击中，将攻击成功率（ASR）降低 30 个百分点

4. **多场景验证**：在白盒和黑盒攻击场景下验证了方法的有效性

## 二、核心方法

### 2.1 方法概述

R.A.C.E. 的核心思想是将概念擦除问题重新定义为**对抗性优化问题**。传统方法只考虑在标准文本嵌入空间中进行擦除，而 R.A.C.E. 扩展到对抗性嵌入空间，确保擦除后的模型能够抵御经过优化的对抗性攻击。

### 2.2 技术架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      R.A.C.E. 框架                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────────┐                                           │
│  │ 对抗性嵌入生成器  │                                           │
│  │ (Adversarial    │                                           │
│  │  Embedding      │                                           │
│  │  Generator)     │                                           │
│  └────────┬────────┘                                           │
│           │                                                    │
│           ▼                                                    │
│  ┌──────────────────┐                                           │
│  │ 鲁棒概念擦除模块  │                                           │
│  │ (Robust Concept │                                           │
│  │  Erasure Module)│                                           │
│  └────────┬────────┘                                           │
│           │                                                    │
│           ▼                                                    │
│  ┌──────────────────┐                                           │
│  │ T2I 扩散模型     │                                           │
│  └──────────────────┘                                           │
│                                                                  │
│  损失函数: L_total = L_erase + λ₁L_robust + λ₂L_utility          │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 关键算法

R.A.C.E. 的训练过程包含三个核心组件：

**组件一：对抗性嵌入生成**

给定一个目标概念（如"裸露"），生成器尝试构造对抗性文本嵌入 $\mathbf{e}_{adv}$，使得：

$$\mathbf{e}_{adv} = \arg\max_{\|\delta\|_2 \leq \epsilon} \mathcal{A}(\theta, \mathbf{e}_0 + \delta)$$

其中 $\mathbf{e}_0$ 是原始文本嵌入，$\delta$ 是对抗性扰动，$\mathcal{A}$ 是攻击评估函数。

**组件二：双层优化**

R.A.C.E. 采用双层优化框架：

- **内层**：最大化攻击效果，找到最具威胁的对抗性嵌入
- **外层**：最小化擦除损失，增强模型对内层攻击的抵抗能力

**组件三：渐进式对抗训练**

为提高训练稳定性，采用渐进式增加对抗性扰动上界的策略：

$$\epsilon_t = \epsilon_{min} + (\epsilon_{max} - \epsilon_{min}) \cdot \frac{t}{T}$$

其中 $t$ 是当前训练轮数，$T$ 是总训练轮数。

## 三、技术细节

### 3.1 实现细节

**模型基础**：R.A.C.E. 基于 Stable Diffusion v1.5 和 SDXL 架构实现。

**目标概念**：主要针对"裸露"（Nudity）和"暴力"概念进行实验验证。

**关键参数配置**：

| 参数 | 值 |
|------|-----|
| 基础学习率 | 1e-4 |
| 扰动上界 $\epsilon$ | 0.5 - 2.0 |
| 训练轮数 | 100 epochs |
| 批量大小 | 16 |
| 目标交叉注意力层 | query, key, value projections |

### 3.2 损失函数设计

R.A.C.E. 的总体损失函数为：

$$L_{total} = L_{erase} + \lambda_1 L_{robust} + \lambda_2 L_{utility}$$

**概念擦除损失** $L_{erase}$：
$$L_{erase} = -\mathbb{E}_{c \sim \mathcal{C}, \mathbf{e} \sim \mathcal{E}_c}[\log(1 - D_\theta(\mathbf{e}))]$$

其中 $D_\theta$ 是概念检测器，$\mathcal{C}$ 是目标概念集合。

**鲁棒性损失** $L_{robust}$：
$$L_{robust} = -\mathbb{E}_{c \sim \mathcal{C}, \|\delta\|_2 \leq \epsilon}[\log(1 - D_\theta(\mathbf{e}_c + \delta))]$$

**效用保持损失** $L_{utility}$：
$$L_{utility} = \mathbb{E}_{\mathbf{e}_{safe}}[\|\mathbf{z}_{ori} - \mathbf{z}_{mod}\|^2]$$

## 四、数学原理

### 4.1 对抗性嵌入空间

设 $\mathcal{E}_{std}$ 为标准文本嵌入空间，$\mathcal{E}_{adv}$ 为对抗性嵌入空间。R.A.C.E. 的目标是确保：

$$\forall \mathbf{e} \in \mathcal{E}_{std} \cup \mathcal{E}_{adv}: D_\theta(\mathbf{e}) \approx 0$$

这通过最小-最大优化实现：

$$\theta^* = \arg\min_\theta \max_{\|\delta\| \leq \epsilon} L(\theta, \delta)$$

### 4.2 PGD 对抗攻击

使用投影梯度下降（PGD）生成对抗性嵌入：

```python
def pgd_attack(embed, target, model, epsilon=1.0, alpha=0.01, iters=10):
    adv_embed = embed.clone().detach()
    
    for i in range(iters):
        # 计算梯度
        loss = attack_loss(model, adv_embed, target)
        grad = torch.autograd.grad(loss, adv_embed)[0]
        
        # 更新对抗嵌入
        adv_embed = adv_embed + alpha * grad.sign()
        
        # 投影回 epsilon 球
        delta = torch.clamp(adv_embed - embed, -epsilon, epsilon)
        adv_embed = torch.clamp(embed + delta, 0, 1)
    
    return adv_embed
```

### 4.3 泛化边界

R.A.C.E. 的泛化能力可以通过以下边界刻画：

$$\mathbb{E}[R_{adv}] \leq R_{emp} + \frac{2}{\sqrt{n}} \cdot \mathcal{R}(\mathcal{F}) + \epsilon$$

其中 $R_{adv}$ 是对抗风险，$R_{emp}$ 是经验风险，$\mathcal{R}$ 是 Rademacher 复杂度。

## 五、实验设置

### 5.1 评估指标

| 指标 | 描述 | 理想值 |
|------|------|--------|
| Attack Success Rate (ASR) | 攻击成功生成目标概念的概率 | < 5% |
| CLIP Score | 文本-图像匹配度 | > 0.8 |
| FID | 图像质量 | < 20 |
| Concept Retention | 正常概念保留程度 | > 95% |

### 5.2 攻击场景

**白盒攻击**：

- PGD 对抗嵌入攻击
- Carlini-Wagner 攻击
- 随机扰动攻击

**黑盒攻击**：

- 同义词替换
- 语义编码
- 跨语言翻译

**白盒攻击防御效果**：

| PGD-20 | 52.1% | 13.8% | -38.3% |

## 六、复现指南

### 6.1 环境配置

```bash
git clone https://github.com/ChanghoonKim-ML/RACE
cd RACE

pip install -r requirements.txt
# torch >= 1.12, diffusers >= 0.19.0, transformers >= 4.30.0
```

### 6.2 训练脚本

```python
from race import RACETrainer
from diffusers import StableDiffusionPipeline

# 加载基础模型
pipe = StableDiffusionPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5"
)

# 初始化 R.A.C.E. 训练器
trainer = RACETrainer(
    model=pipe,
    target_concepts=["nudity", "violence"],
    robust_epsilon=1.5,
    lambda_utility=0.1
)

# 训练
trainer.train(
    num_epochs=100,
    batch_size=16,
    learning_rate=1e-4
)

# 保存模型
trainer.save_pretrained("./race-protected-model")
```

### 6.3 评估脚本

```python
from race import RACEEvaluator

evaluator = RACEEvaluator.from_pretrained("./race-protected-model")

# 白盒攻击评估
wb_results = evaluator.evaluate_whitebox(
    attack_types=["pgd", "cw"],
    epsilon_range=[0.5, 1.0, 1.5, 2.0]
)

# 黑盒攻击评估
bb_results = evaluator.evaluate_blackbox(
    attack_types=["synonym", "encoding", "translation"]
)

# 效用保持评估
utility_results = evaluator.evaluate_utility(
    normal_prompts=normal_prompt_dataset
)
```

## 七、总结与展望

### 7.1 主要贡献

R.A.C.E. 首次将对抗训练框架引入概念擦除领域，显著提升了概念擦除方法对对抗性攻击的鲁棒性。其核心创新在于：

1. **对抗性嵌入空间建模**：识别并防御对抗性文本嵌入攻击
2. **双层优化框架**：同时优化擦除效果和对抗鲁棒性
3. **显著性能提升**：ASR 降低 30 个百分点以上

### 7.2 局限性

1. **计算开销**：对抗训练增加了约 50% 的训练时间
2. **概念特定性**：需要针对不同概念单独训练
3. **黑盒攻击覆盖**：无法完全防御未见过的黑盒攻击类型

### 7.3 未来方向

1. **多概念联合擦除**：同时擦除多个相关概念
2. **自适应鲁棒性**：根据威胁等级动态调整防护强度
3. **高效对抗训练**：开发更高效的训练策略
