## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
T2I 生态里越来越多环节依赖 reward model：数据筛选、best-of-N 选择、训练时优化、推理时引导、以及后处理安全过滤。行业默认把 reward model 当成“质量与对齐的中立代理”，但这个前提其实很少被系统验证。如果评分器本身带有人口统计偏置，那么整个 T2I pipeline 可能会在你看不见的地方持续放大这些偏差。

### 核心科学问题 (Core Scientific Question)
Reward model 到底是在评估“图像质量”，还是在隐式编码某些 race / gender 先验？当它被用作优化目标时，会不会系统性推动女性主体更性化、推动非 White 主体向 White 收敛，并改变职业等社会语义场景中的人口属性分布？

### 直觉洞见 (Intuitive Insight)
这篇论文的关键洞见是：**T2I 安全问题不只在生成器，也在评分器**。如果 score function 本身就带偏，那么即便生成模型本体没改，后续所有 reward-guided optimization 都可能被引向错误目标。换句话说，reward model 不是被动评委，而是主动 shaping distribution 的隐形治理器。

## Module 2 — 核心方法 (Core Method)

### 方法总览
论文的审计分成两部分：

1. **Optimization Audit**
   - 通过 **ReNO** 框架把 reward model 当作优化目标；
   - 直接观察 reward-guided optimization 会把图像往哪个人口统计方向推。

2. **Score Audit**
   - 构造只在人口属性上不同、其余内容尽量匹配的 counterfactual image sets；
   - 直接测 reward score 与 race / gender 的系统性关联。

### Part I — Optimization Audit
作者关心两类现象：
- **Hypersexualization**：优化后是否更容易让人物变得更 NSFW、更暴露；
- **Demographic Drift / Convergence**：优化后是否会把人物推向特定 race / gender。

### Part II — Score Audit
作者用 OLS 回归分析 reward score：
- 主效应：race / gender 对 score 的影响；
- 交互效应：race × gender 对 score 的影响；
- 再用排名分析看哪些群体在相同语义条件下总被放到更前面。

### 方法定位
这不是传统意义上的“安全攻击论文”，也不是单纯 fairness survey，而是一篇**reward model 治理审计论文**。它的核心价值在于证明：**偏置不仅出现在输出端，还被编码进了优化目标本身。**

## Module 3 — 技术细节 (Technical Details)

### 审计对象
作者评估了 6 个主流评分器：
- **PickScore**
- **ImageReward**
- **HPS**
- **VQAScore**
- **CLIP**
- **Aesthetic Score**

并加入一个相对中性的基线：
- **Incompression**（基于 DCT 高频可压缩性的非数据驱动 reward）

### 生成与优化配置
- 优化框架：**ReNO**
- 生成模型：
  - **SDXL-Turbo**
  - **PixArt-α DMD**
  - **SD-Turbo**
- 优化对象：初始噪声 ε
- 目标：最大化 `R(Gθ(ε, p), p)`

### Hypersexualization 审计指标
1. **NSFW rate**
   - 先用 NSFW 分类器做多级分类；
   - 再二值化为 NSFW / non-NSFW；
   - 关注优化前后的增量 Δnsfw。

2. **Skin Exposure (SE)**
   - 用人体分割估计 body skin / face skin / clothes / hair；
   - 只把身体裸露比例计入暴露度；
   - 关注优化前后的增量 Δskin。

### Demographic Drift 审计流程
- 采用类似 **SeedSelect** 的思路，在欠指定 prompt 下找到初始噪声；
- 用 **FLUX.1-dev** 生成多组 race × gender × occupation 参考图；
- 再做正向 / 反向 reward optimization，观察人口属性漂移。

### 人口属性识别方式
- 基于 **Chicago Face Database** 做 anchor-based 感知分类；
- 用冻结的 CLIP image encoder 把图像映射到各 race × gender anchor embedding；
- 从而追踪例如 Black → White、female → male 的转移。

### 评分审计数据集
作者使用三套 counterfactual image 数据：
- **CausalFace**
- **SocialCounterfactuals**
- **PAIRS**

并在职业、agency、warmth / competence 等 prompt 场景上测：
- 原始 score disparity；
- rank disparity；
- 与美国劳工统计局职业性别分布的相关性。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 关键结果一：reward optimization 会显著放大女性性化
- **PickScore** 导致女性图像的 NSFW rate 平均增加 **19%**；
- 男性图像只增加 **7%**；
- 即女性的 NSFW 增幅约为男性的 **2.7 倍**；
- skin exposure 增幅对女性约为男性的 **2.3 倍**；
- 最严重组合是 **PickScore + PixArt-α DMD**，NSFW rate 增长最高达到 **25%**。

这说明 reward-guided optimization 并不是微调一些表面质量细节，而是在系统性改变人物表现方式。

### 关键结果二：reward optimization 会把非 White 主体推向 White
对正向优化结果，作者观察到明显 demographic convergence：
- **ImageReward**：非 White → White 平均转移 **76.1%**；
- **HPS**：非 White → White 平均转移 **89.2%**；
- **CLIP**：非 White → White 平均转移 **36.2%**；
- ImageReward 与 HPS 会让**超过 80% 的 Black subjects**在优化后被重新分类为 White。

### 关键结果三：性别身份也会被优化目标改写
- 用 **ImageReward** 优化时，原本被识别为 female 的图像有 **超过 39%** 被重新分类为 male；
- 用 **CLIP** 优化时，这一比例也有 **超过 26%**。

这说明 reward model 不是只在“更好看”意义上调整输出，而是在重写 identity attributes。

### 关键结果四：score 本身就存在稳定 demographic bias
在 counterfactual 评分实验中：
- **HPS** 与 **ImageReward** 通常把 **White subjects** 排名更高；
- 且 **White male 高于 White female**；
- **Asian subjects** 经常排名最低；
- **CLIP** 倾向把 **Black subjects** 排在其他种族之后；
- **VQAScore** 的偏差会随 prompt 正负语义切换，但仍表现出明显 demographic sensitivity。

### 关键结果五：职业语义是偏置最强的放大器
作者发现：
- 在“性别分数差异最大”的前 **30** 个 prompt 中，平均 **27.4 / 30** 与 occupation 相关；
- 在“种族 × 性别交互差异最大”的前 **30** 个 prompt 中，平均 **23.4 / 30** 与 occupation 相关；
- reward score 的性别差异，与 **美国劳工统计局职业性别比例**呈相关。

换句话说，reward model 会奖励“更符合现实主流或刻板印象”的职业图像，而不是单纯奖励语义对齐或图像质量。

## Module 5 — 局限性分析 (Limitations)

1. **人口属性是感知标签而不是真实身份**：race / gender 分类本身带有社会建构与模型偏差，不能被理解为本体论真值。  
2. **counterfactual 不是严格因果反事实**：虽然图像尽量控制其他变量，但仍可能伴随耳环、胡须、背景等细节变化。  
3. **NSFW 与 skin exposure 度量本身也有主观性**：分类器与文化语境都可能引入二次偏差。  
4. **实验覆盖的优化方式有限**：主要集中在 ReNO 与 one-step distilled models，未穷尽所有 T2I 微调 / 推理策略。  
5. **中性基线也不是绝对无偏**：作者引入的 Incompression 只能作为相对参照，而非真正 unbiased oracle。

## Module 6 — 复现指南 (Reproduction Guide)

### 复现目标
复现重点不是“重新训练一个更公平的模型”，而是**复现 reward model 的治理审计**：它们在评分层与优化层分别如何引入偏置。

### 建议步骤
1. 选择 6 个主流 reward models，外加一个尽量数据中立的 baseline；
2. 在 SDXL-Turbo / SD-Turbo / PixArt-α DMD 上，用 **ReNO** 做 reward-guided optimization；
3. 为 occupation / agency / stereotype 等 prompt 设计 race × gender 对照组；
4. 用 NSFW 分类器与人体分割模型分别测 **Δnsfw** 与 **Δskin**；
5. 用 CFD anchor-based CLIP 分类器追踪 demographic drift；
6. 再在 **CausalFace / SocialCounterfactuals / PAIRS** 上测 score disparity 与 ranking disparity。

### 关键核验点
- PickScore 是否显著放大女性主体的 NSFW 与皮肤暴露；
- HPS / ImageReward 是否把非 White 主体大量推向 White；
- occupation prompts 是否最容易诱发 demographic scoring bias；
- reward model 是否既在**优化层**也在**评分层**表现出稳定偏差。
