## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
T2I 水印研究过去常常在两个目标之间二选一：要么强调 provenance，证明“这张图是不是某模型生成的”；要么强调 tamper detection，回答“这张图后来有没有被改”。问题在于，单通道方案通常顾此失彼：只在初始噪声里埋水印，容易在 reprompt / re-generation 场景里被拿来做归因混淆；只在像素或单一 latent 域埋水印，又很难同时承受局部编辑、DiffEdit 与内容级改写。

### 核心科学问题 (Core Scientific Question)
能否构建一种同时覆盖“来源认证 + 内容一致性 + 局部篡改定位”的双通道 latent watermarking 框架，使扩散模型生成图像在 reprompt、DiffEdit 和局部 tampering 下依然可验证、可定位、且误报极低？

### 直觉洞见 (Intuitive Insight)
Dual-Guard 的核心直觉很直接：单个水印锚点不够。初始噪声更像“出生证明”，适合证明图像最早来自哪个生成器；最终 latent 更像“内容指纹”，适合判断这张图后续有没有被改、改了哪里。把这两个锚点拆开，再在验证阶段做联合判定，才能同时防“嫁祸”和“局部篡改”。

## Module 2 — 核心方法 (Core Method)

### 方法总览
Dual-Guard 是一个双通道 latent watermarking 框架，由三部分组成：

**（1）GS 通道（Gaussian Shading）**
- 将 watermark 写入扩散初始噪声 z_T；
- 负责 provenance verification，回答“它是否来自受保护模型”；
- 对 reprompt 场景更稳，因为初始生成链路的信息更容易保留。

**（2）Latent Fingerprint Codec 通道**
- 将 fingerprint 写入最终 latent z_0；
- 负责 content consistency 检查与 block-level tamper localization；
- 对局部编辑更敏感，因为内容篡改会先体现在最终 latent 的局部结构上。

**（3）三证据融合定位器**
- 将 latent 划分为 16×16 个 block；
- 对每个 block 融合 cosine correlation、L1 deviation、decoder BMR 三种证据；
- 输出 tamper mask，并以面积阈值判断是否触发完整性告警。

### 关键设计差异
相比“只做来源认证”的 watermark 方法，Dual-Guard 额外引入内容锚点；相比“只做局部定位”的方法，它又保留了可跨重生成链路的 provenance anchor。论文真正的新意不在“把两个水印简单叠加”，而在于把两者写入扩散过程中的不同阶段，并让它们在攻击语义上互补。

## Module 3 — 技术细节 (Technical Details)

### 双通道写入机制
GS 通道将 64-bit fingerprint 与 128-bit key seed 写入 z_T；Codec 通道则把 64-bit payload 经 3× repetition code 扩展成 192-bit codeword，写入 z_0。

核心写入形式为：

z̃_0 = z_0 + α · r

其中：
- z_0: 干净的最终 latent
- r: 编码器生成的 residual
- α = 0.3: 部署时的默认嵌入强度

### GS 提取逻辑
对可疑图像先做 VAE 编码，再执行 DDIM inversion 恢复近似 z_T，随后按 key seed 恢复 64-bit payload。GS 认证阈值为：

τ_GS = 0.995

### Codec 解码与块级定位
论文使用分辨率无关的多尺度 decoder，同时兼容整图 latent 和 4×4 latent block。块级定位时，对每个 block 计算：

score_block = 0.50 · s_cos + 0.30 · s_L1 + 0.20 · s_BMR

其中：
- s_cos: cosine correlation 证据
- s_L1: L1 deviation 证据
- s_BMR: bit mismatch rate 证据

关键阈值：
- τ_codec = 0.8919
- ρ_tamper = 0.0117
- 最小连通区域 = 6 blocks

### 训练配置
- 基座模型：Stable Diffusion 2.1-base
- 采样：50-step DDIM
- 分辨率：512×512
- 优化器：AdamW
- 学习率策略：OneCycleLR
- max lr：3×10^-4
- weight decay：10^-4
- batch size：16
- 总步数：4000
- latent bank：512 samples

### 四阶段 curriculum
1. Decoder warmup: 220 steps
2. Strength curriculum: 1000-step，嵌入强度从 1.2 降到 0.35
3. Residual budget decay: 1600-step，budget 从 0.20 降到 0.08
4. Robustness augmentation: 逐步加入噪声、模糊、量化、dropout/cutout、VAE encode-decode 增强

### 伪代码
```python
class DualGuard:
    def embed(self, zT, z0, fingerprint, key_seed):
        zT_marked = gaussian_shading(zT, fingerprint, key_seed)
        residual = codec_encoder(z0, fingerprint)
        z0_marked = z0 + 0.3 * residual
        return zT_marked, z0_marked

    def verify(self, image, record):
        zT_hat = ddim_inversion(image)
        gs_ok = extract_gs(zT_hat, record.key_seed) >= 0.995
        z0_hat = vae_encode(image)
        codec_ok = codec_score(z0_hat, record) >= 0.8919
        tamper_mask = localize_blocks(z0_hat, record.z0_ref)
        return gs_ok, codec_ok, tamper_mask
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
实验使用 1000-prompt snapshot，提示来自 Gustavosta/Stable-Diffusion-Prompts。核心场景包括：
- Original / clean
- Reprompt
- Local tamper（8 类）
- DiffEdit

局部篡改 8 类包括 black masking、noise、blur、JPEG、pixelation、copy-move、color shift、text overlay。平均 GT 篡改面积为 13.3%。

评价指标：
- 来源认证：Precision / Recall / F1 / AUC / TPR@1%FPR
- 攻击鲁棒性：Auth Reject、Tamper Detect、GS Pass、Dual Pass
- 定位：IoU、F1、Precision、Recall
- 图像质量：PSNR、SSIM、LPIPS、CLIP-I、CLIP-T

### 主要结果表格
| 场景 | Auth Reject ↑ | Tamper Detect ↑ | GS Pass ↑ | Dual Pass ↑ | Codec Extr. ↑ |
|------|--------------:|----------------:|----------:|------------:|---------------:|
| Original | 0.003 | 0.001 | 0.998 | 0.998 | 0.939 |
| Reprompt | **1.000** | **1.000** | 1.000 | **0.000** | **0.500** |
| Local tamper | **0.999** | **0.999** | 0.995 | 0.975 | 0.923 |
| DiffEdit | **1.000** | **1.000** | 0.633 | 0.601 | 0.929 |

### 来源认证主结果
在 4000 图像的 closed-set suite 上：

| 方法 | Precision ↑ | Recall ↑ | F1 ↑ | AUC ↑ | TPR@1%FPR ↑ |
|------|------------:|---------:|-----:|------:|------------:|
| Dual-Guard | **1.000** | **0.999** | **0.999** | **1.000** | **1.000** |

### 篡改定位结果
| 方法 | Img-lvl Detect ↑ | IoU ↑ | F1 ↑ | Precision ↑ | Recall ↑ |
|------|-----------------:|------:|-----:|------------:|---------:|
| SEAL | 0.999 | 0.036 | 0.068 | 0.090 | 0.060 |
| Dual-Guard | **1.000** | **0.255** | **0.392** | 0.273 | **0.934** |

### 图像质量结果
| 对比 | PSNR ↑ | SSIM ↑ | LPIPS ↓ | CLIP-I ↑ | CLIP-T ↑ |
|------|--------:|-------:|---------:|----------:|----------:|
| plain-vs-codec | 26.81 | 0.811 | 0.221 | 0.966 | — |

### 关键发现
- Clean authentication false rejection 仅 **0.3%**，tamper false alarm 仅 **0.1%**；
- Reprompt 时 GS 仍可能通过，但 Codec 会掉到近随机 **0.500**，因此可识别“来源还在、内容已换”；
- 相比 SEAL，Dual-Guard 的定位 IoU 提升 **7.1×**，F1 提升 **5.8×**；
- text overlay 是最弱场景，因为细小文字笔画经 VAE 编码后被平滑。

## Module 5 — 局限性分析 (Limitations)

1. **依赖 Full mode 参考 latent**：最强结果需要平台为每张图保存约 64 KB 的 z_0^ref，这对大规模部署带来存储与索引负担。
2. **定位仍偏粗粒度**：block 大小对应约 32×32 像素，导致 Recall 很高，但 Precision 偏低，Pred Area 明显大于 GT Area。
3. **对白盒自适应攻击覆盖不足**：论文主实验并未系统评测“知道双通道结构后专门优化去水印”的强对手。
4. **对 text overlay 较弱**：小字体和细笔画很容易被 VAE 平滑掉，使局部偏移不够大。
5. **Lite mode 不是主要验证对象**：若不保存参考 latent，整体完整性与定位能力会明显打折。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)
```bash
pip install torch torchvision diffusers transformers accelerate scikit-learn pillow opencv-python
```

### 核心代码框架 (Core Code)
```python
from diffusers import StableDiffusionPipeline

pipe = StableDiffusionPipeline.from_pretrained("stabilityai/stable-diffusion-2-1-base")

for prompt in prompt_list:
    zT_marked, z0_marked = dual_guard.embed(zT, z0, fingerprint, key_seed)
    image = pipe(prompt=prompt, latents=zT_marked).images[0]
    result = dual_guard.verify(image, record)
```

### 复现步骤 (Reproduction Steps)
1. 以 Stable Diffusion 2.1-base 为底座，固定 1000-prompt snapshot。
2. 实现 GS 通道与 Codec 通道，并将 payload 分别写入 z_T 与 z_0。
3. 按论文设定训练 Codec：4000 steps、batch size 16、max lr 3×10^-4、四阶段 curriculum。
4. 在 Original、Reprompt、Local tamper、DiffEdit 四类场景评测。
5. 用 220 authentic + 220 tampered held-out split 校准 τ_GS、τ_codec 和 ρ_tamper。
6. 关键核验点：
   - provenance Precision 接近 **1.000**；
   - Original false rejection 接近 **0.003**；
   - Reprompt 的 Codec Extr. 接近 **0.500**；
   - 定位 IoU 接近 **0.255**、Recall 接近 **0.934**。
