## 一、背景与问题

### 动机

Stable Diffusion 等文本到图像（T2I）模型具有强大的图像生成能力，但同时存在被恶意利用的隐患——用户可以通过构造特定提示词（Prompt）绕过安全过滤，生成 NSFW（Not Safe For Work）内容，包括色情、暴力等违规图像。

### 核心科学问题

在不修改底层 T2I 扩散模型的前提下，能否通过对输入提示词的实时"净化"来阻止有害图像的生成，同时最大限度保留良性提示的生成质量？

### 直觉洞见

现有安全机制主要有两类：（1）修改模型本身（概念擦除等），侵入性强且可能影响生成质量；（2）输出端后处理，检测已生成的有害图像。PromptSan 另辟蹊径，在**输入端**拦截有害提示，代价更低，且不影响模型本身的能力。

---

## 二、核心方法

PromptSan 提出两种提示净化变体：

**变体一：PromptSan-Modify（迭代修改式）**
- 使用文本 NSFW 分类器识别提示词中的有害词汇
- 迭代地替换有害词汇，使改写后的提示通过分类器
- 目标：保留语义相近的良性表述，移除有害语义

**变体二：PromptSan-Suffix（后缀优化式）**
- 训练一组优化的后缀令牌序列
- 将后缀附加到任意有害提示后，中和其有害意图
- 目标：使含后缀的提示同时通过文本 NSFW 分类器和图像 NSFW 分类器

## 三、技术细节

### PromptSan-Modify 算法流程

```
输入：有害提示 P_harm
输出：净化后提示 P_clean

1. 使用文本 NSFW 分类器 C_text 识别 P_harm 中的有害词集合 W_harm
2. 对每个 w ∈ W_harm：
   a. 在语义相似词库中搜索安全替代词 w'
   b. 替换：P ← P.replace(w, w')
3. 重复步骤 1-2，直到 C_text(P) = safe
4. 返回 P_clean = P
```

### PromptSan-Suffix 优化目标

设后缀令牌序列为 $S = (t_1, t_2, \ldots, t_k)$，优化目标为：

$$\min_{S} \mathcal{L}_{text}(P_{harm} \oplus S) + \lambda \cdot \mathcal{L}_{image}(\text{Diffuse}(P_{harm} \oplus S))$$

其中：
- $\mathcal{L}_{text}$：文本 NSFW 分类器的有害性损失
- $\mathcal{L}_{image}$：图像 NSFW 分类器的有害性损失
- $\oplus$：提示拼接操作
- $\lambda$：平衡超参数

### 双分类器架构

- **文本分类器**：基于预训练语言模型，对提示文本打分
- **图像分类器**：基于视觉模型，对生成图像打分
- 两者的联合约束确保净化效果不仅在文本层面有效，在图像生成层面也同样有效

## 四、实验设置与结果

### 数据集

- **有害提示数据集**：来自 SneakyPrompt、I2P、Ring-A-Bell 等已有越狱测试集
- **良性提示数据集**：COCO Captions、MS-COCO 等通用生成基准

### 评价指标

- **防御成功率（DSR）**：被净化后的提示生成安全图像的比例
- **FID（Fréchet Inception Distance）**：衡量对良性图像生成质量的影响
- **CLIP Score**：衡量净化后提示与生成图像的语义一致性

### 主要实验结果

| 方法 | DSR（↑） | FID（↓，良性） | CLIP Score（↑） |
|------|---------|--------------|----------------|
| 无防御 | 0% | 基准 | 基准 |
| SafeGen | 78.3% | +2.1 | -0.05 |
| ESD | 81.5% | +5.3 | -0.12 |
| **PromptSan-Modify（本文）** | **85.7%** | +1.3 | -0.02 |
| **PromptSan-Suffix（本文）** | **89.2%** | +1.8 | -0.03 |

### 消融实验

- 仅文本分类器约束 vs. 双分类器约束：双约束将 DSR 提升约 6%
- 后缀长度 $k=5$ vs. $k=15$：$k=10$ 时性能最优

## 五、局限性分析

1. **对抗性提示鲁棒性**：经过专门对抗优化的提示（如 GCG 攻击生成的对抗文本）可能规避净化。
2. **语义损失**：对于语义边界模糊的提示，净化过程可能修改其合理语义，降低生成质量。
3. **推理延迟**：PromptSan-Modify 的迭代替换过程和 PromptSan-Suffix 的推理计算均引入额外延迟。
4. **多语言支持**：当前方法主要针对英文提示，多语言场景的有效性未知。

### PromptSan-Suffix 核心实现

```python
import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification
from diffusers import StableDiffusionPipeline

# 加载 T2I 模型和文本 NSFW 分类器
pipe = StableDiffusionPipeline.from_pretrained("stabilityai/stable-diffusion-2-1")
nsfw_classifier = AutoModelForSequenceClassification.from_pretrained("your/nsfw-text-classifier")

def optimize_suffix(harmful_prompt, suffix_length=10, num_steps=500, lr=0.01):
    """优化安全后缀令牌"""
    tokenizer = nsfw_classifier.config.tokenizer
    
    # 随机初始化后缀嵌入
    suffix_embeddings = torch.randn(suffix_length, 768, requires_grad=True)
    optimizer = torch.optim.Adam([suffix_embeddings], lr=lr)
    
    for step in range(num_steps):
        # 拼接原始提示和后缀
        prompt_with_suffix = harmful_prompt + " " + decode_embeddings(suffix_embeddings)
        
        # 文本 NSFW 损失
        text_loss = nsfw_classifier(prompt_with_suffix).logits[0, 1]  # 有害类 logit
        
        # 图像 NSFW 损失（每 N 步计算一次）
        if step % 50 == 0:
            image = pipe(prompt_with_suffix).images[0]
            image_loss = compute_image_nsfw_loss(image)
        
        # 联合损失
        total_loss = text_loss + 0.5 * image_loss
        optimizer.zero_grad()
        total_loss.backward()
        optimizer.step()
    
    return suffix_embeddings.detach()
```
