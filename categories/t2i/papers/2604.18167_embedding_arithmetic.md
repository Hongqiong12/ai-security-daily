## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
T2I 偏见治理长期面临一个工程现实：最有效的方法往往要改训练集、做再训练、加对齐损失，代价高且迁移慢。对已经上线的主线模型，很多团队并不愿意重新微调整个生成器，而是更希望找到一种 post-hoc 方法，在尽量不动模型权重的前提下，对 gender / race 等人口属性偏差做快速校正。

### 核心科学问题 (Core Scientific Question)
能否只在推理阶段操作文本条件嵌入，通过轻量的向量算术方式缓解 T2I 模型的人口属性偏见，同时尽量保持职业语义、构图和风格不变？

### 直觉洞见 (Intuitive Insight)
这篇论文的核心直觉是：文本嵌入空间并不是完全 disentangled 的网格，而是“整体纠缠、局部线性”的流形。如果某些人口属性方向可以通过多个基础概念的平均差分稳定提取出来，那么在局部区域里，对嵌入做向量加法就能像拧旋钮一样调节偏见，而不用重训模型。

## Module 2 — 核心方法 (Core Method)

### 方法总览
Embedding Arithmetic 提出一种 tuning-free 的推理时偏见缓解框架，核心流程如下：

**（1）构造属性方向向量**
- 从一组基础职业概念出发，计算“加入属性前后”的文本嵌入差分；
- 对多个基础概念求平均，得到稳定的属性方向向量 v_A。

**（2）推理时编辑文本嵌入**
- 对用户 prompt 得到基础嵌入 e_base；
- 取一个或多个属性方向向量 v_A；
- 用强度 α 做线性叠加，得到目标嵌入 e_target。

**（3）直接驱动生成**
- 把修改后的嵌入送入 T2I 模型；
- 不改权重、不改训练集、不做额外微调。

### 两个使用模式
**EA_g**
- 只操作性别方向向量；
- 适合快速提升 gender entropy。

**EA_i**
- 操作交叉属性方向向量（如 race × gender）；
- 是论文里最稳的版本，因为它显式建模了 intersectional bias。

### 与既有方法的区别
它不像 DebiasVL 那样依赖专门去偏流程，也不像 concept editing 那样更关注特定语义替换。这里的目标更聚焦：**不重训模型，只在 inference-time 嵌入层面做偏见校正。**

## Module 3 — 技术细节 (Technical Details)

### 属性方向向量
对某个属性 A，论文定义：

v_A = (1/K) · Σ_k ( E(T_base ⊕ A) - E(T_base) )

其中：
- E: 文本编码器
- T_base: 基础提示词
- T_base ⊕ A: 加入属性后的提示词
- K: 基础概念数量

### 推理时嵌入编辑

e_target = e_base + Σ_i α_i · v_A_i

其中 α_i 控制属性强度。论文默认泛化实验中使用 α = 1.25，并观察到：
- Carpenter → female 在 α ≈ 0.75 时开始翻转倾向；
- Nurse → male 在 α ≈ 1.5 时开始翻转；
- α > 3.0 后 CCS 明显下降，说明局部线性区间被破坏。

### 编辑空间选择
虽然 FLUX 同时使用 CLIP 和 T5，作者只编辑：
- **CLIP ViT-L/14 pooled embedding**

原因是 pooled embedding 更承载全局语义属性，适合做 demographic bias 调节，而不必深入 token-level 序列结构。

### 模型与实验配置
- 主实验模型：FLUX 1.0-Dev
- 泛化验证：Stable Diffusion 3.5-Large
- FLUX 推理参数：guidance scale = 3.5，steps = 26，分辨率 = 512×512
- 属性：male / female / white / black / asian / indian
- 基础职业概念数：K = 10
- 属性向量总数：14（包含单属性与交叉属性）

### 指标设计
论文不仅看 entropy，还专门提出 **CCS（Concept Coherence Score）**：
- 用 BLIP-VQA-base、GIT-LARGE-TEXTVQA、VILT-B-32 三个 VQA 模型问“这张图里是不是某职业”；
- 取 yes 概率平均，衡量职业概念是否还保留。

### 伪代码
```python
def build_attribute_vector(base_prompts, attribute):
    diffs = []
    for prompt in base_prompts:
        diffs.append(encode(prompt + attribute) - encode(prompt))
    return average(diffs)


def edit_embedding(e_base, directions, alphas):
    e_target = e_base
    for v, alpha in zip(directions, alphas):
        e_target = e_target + alpha * v
    return e_target
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
该方法本身不是训练式方法，因此没有额外训练集。实验评测主要依赖：
- BLS 146 professions（用于职业概念采样）
- FairFace（属性分类器）
- LAION-5B（用于强偏职业泛化分析的背景参考）
- 自建合成验证集（校验属性分类器）

核心指标：
- **H_g / H_r**: 性别与种族 Shannon entropy，越接近 1 越均衡；
- **CCS**: Concept Coherence Score，越高越保住原职业语义。

### 主结果表格
#### FLUX 1.0-Dev
| 职业 | 默认 Hg / Hr / CCS | EA_i Hg / Hr / CCS |
|------|--------------------|-------------------|
| Fire Fighter | 0.50 / 0.17 / 0.58 | **0.99 / 0.82 / 0.61** |
| Mechanic | 0.08 / 0.08 / 0.64 | **0.99 / 0.93 / 0.60** |
| Flight Attendant | 0.41 / 0.18 / 0.62 | **0.99 / 0.79 / 0.62** |
| Nurse | 0.00 / 0.21 / 0.57 | **0.68 / 0.80 / 0.57** |

#### Stable Diffusion 3.5-Large
| 职业 | 默认 Hg / Hr / CCS | EA_i Hg / Hr / CCS |
|------|--------------------|-------------------|
| Fire Fighter | 0.10 / 0.47 / 0.61 | **0.99 / 0.91 / 0.62** |
| Mechanic | 0.00 / 0.14 / 0.69 | **0.99 / 0.96 / 0.61** |
| Flight Attendant | 0.00 / 0.38 / 0.67 | **0.53 / 0.94 / 0.66** |
| Nurse | 0.00 / 0.16 / 0.62 | **0.85 / 0.98 / 0.61** |

### 组合性与局部线性结果
- 交叉向量与线性组合向量的余弦相似度 **> 0.87**；
- 大多数身份的生成分布差异 **< 3%**；
- 说明线性组合是很强但不完美的近似。

### 分类器验证
自建合成数据上：
- FairFace Gender Accuracy = **100%**
- FairFace Race Accuracy = **92.8%**

### 关键发现
- EA_i 在 FLUX 和 SD3.5-Large 上都显著提升 demographic entropy，同时 CCS 基本保持；
- K = 1 容易把单一职业偏见带进属性向量，K = 10 更稳定；
- 嵌入空间不是完全 disentangled，而是 **entangled but locally linear**；
- 这意味着 T2I 去偏可以先从 inference-time condition editing 做起，而不一定一上来就重训生成器。

## Module 5 — 局限性分析 (Limitations)

1. **只编辑 pooled embedding**：对多人场景和复杂 prompt 缺乏 token-level 细粒度控制，容易让整图“同质化”。
2. **依赖外部属性分类器**：FairFace 本身也可能带偏，因此评测结果仍受 judge bias 影响。
3. **局部线性只在有限区间成立**：α 过大时会明显破坏职业语义和构图，说明该方法不能被看作全局可逆操控。
4. **当前覆盖的属性范围较有限**：论文主要聚焦 gender / race，更多文化、地域、宗教维度 هنوز未系统展开。
5. **不处理训练分布根因**：它是 post-hoc 缓解，而不是从训练数据和 reward 目标上消除偏见来源。

### 核心代码框架 (Core Code)

base_prompts = [
    "A photo portrait of a doctor.",
    "A photo portrait of an engineer.",
    # ... total K=10 professions
]

v_female = build_attribute_vector(base_prompts, " female")
e_target = edit_embedding(e_base, [v_female], [1.25])

### 复现步骤 (Reproduction Steps)
1. 准备 10 个基础职业概念，按论文模板构造 base prompts。
2. 用 CLIP ViT-L/14 pooled embedding 计算单属性和交叉属性向量。
3. 在 FLUX 1.0-Dev 与 SD3.5-Large 上，以 α = 1.25 进行 inference-time 编辑。
4. 用 FairFace 统计 gender / race 分布，计算 H_g 与 H_r。
5. 用三模型 VQA ensemble 计算 CCS，检查职业概念是否保留。
6. 关键核验点：
   - Mechanic 的 entropy 是否接近 **0.99 / 0.93**；
   - CCS 是否大致保持在 **0.60–0.62** 区间；
   - α > 3.0 后是否出现明显概念漂移。
