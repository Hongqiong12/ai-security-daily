## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
当前 T2I 安全对齐大多围绕“显式危险概念”展开：脏词过滤、NSFW 分类器、概念擦除、内容审核器，默认假设只要单个概念是危险的，系统就能识别并拦截。但真实世界更棘手：许多提示中的每个原子概念单独看都无害，组合在一起却会触发隐含的侮辱、歧视、羞辱或危险语义。也就是说，问题不再是“模型认不认识一个危险词”，而是“模型能不能理解多个安全词叠加后形成的不安全语义”。

### 核心科学问题 (Core Scientific Question)
能否把这种“单概念安全、组合后不安全”的风险形式化，并建立一个足够大、足够系统的 benchmark，去衡量现有 T2I 模型、过滤器和概念擦除方法在组合语义风险面前到底有多脆弱？

### 直觉洞见 (Intuitive Insight)
TwoHamsters 的关键洞见是：T2I 风险的一大盲区不在像素层，而在组合语义层。现有系统更擅长抓“显眼的 NSFW / 暴力特征”，却不擅长抓“语义拼接后才变危险”的 MCCU（Multi-Concept Compositional Unsafety）。这使得越强、越听话、越会按提示精细作图的模型，反而更容易把这种组合语义精准实现出来。

## Module 2 — 核心方法 (Core Method)

### 方法总览
论文不是提出新防御，而是做了三件更基础但更关键的事：

1. **形式化 MCCU 风险**
   - 定义两个原子概念各自安全，但组合后会产生不安全语义；
   - 把 T2I 安全问题从“危险词识别”推进到“组合语义理解”。

2. **构建 TwoHamsters benchmark**
   - 总计 **17.5k prompts**；
   - 覆盖 **5.8k high-risk MCCU scenarios**；
   - 组织成 **10 个一级风险类别、51 个细粒度概念对**。

3. **系统评测 26 个对象**
   - **10 个 T2I 模型**；
   - **7 个安全过滤器**；
   - **9 个概念擦除/安全编辑方法**。

### 核心机制
作者把 MCCU 写成一个组合语义失效问题：
- 单独生成概念 A 是安全的；
- 单独生成概念 B 是安全的；
- 但 A 与 B 出现在同一图像中时，不安全语义由“解释关系”而不是“显式危险特征”触发。

因此，TwoHamsters 的价值不只是“再做一个不安全数据集”，而是把 T2I benchmark 从单概念检测推进到语义组合风险评估。

## Module 3 — 技术细节 (Technical Details)

### 数据构造流水线
作者采用七阶段流程构建 benchmark：

1. **taxonomy + LLM 双路采集**
   - 结合已有安全数据和 LLM 扩展，生成候选 prompts；
   - 使用 blocklist 避免显式危险词，确保风险来自“组合关系”而不是直球违禁词。

2. **文本去重**
   - 基于 CLIP 文本编码；
   - 余弦相似度阈值设为 **0.95**；
   - 对高度相似 prompt 做 centrality-based pruning。

3. **跨模态有效性筛选**
   - 对每个候选 prompt 生成组合图与两个单概念控制图；
   - 只有当单概念图和组合图都能正确表达目标语义时，样本才保留。

4. **理由标注与双盲人工审核**
   - LLM 先生成结构化 rationale；
   - 两位审阅者双盲审核；
   - 分歧交由专家裁决；
   - 最终剔除约 **5%** 的边界样本。

### 数据规模与类别
- 初始候选：**100k+**
- 最终保留：**17.5k**
- 一级风险类别：**10**
  - Disturbing, Harassment, Hate, Humiliation, Illegal, Political, Public Health Harm, Self-Harm, Sexual, Violence
- 代表性类别分布：
  - Illegal **4,197 (23.9%)**
  - Hate **3,852 (22.0%)**
  - Sexual **2,316 (13.2%)**
  - Self-harm **1,752 (10.0%)**

### 自动评估器
论文训练了两类专用 evaluator：
- **MCCU-ViT**：偏高吞吐量评测；
- **MCCU-VLM**：偏解释性判断。

其中 MCCU-ViT 使用 **300k image-text pairs** 训练，并将 unsafe 头、concept A 头、concept B 头联合起来做神经符号判定。

### 核心指标
- **MDR (MCCU Defense Rate)**：越高越安全，本质上是 1 - 组合不安全生成率；
- **SCR (Single-Concept Retention)**：单概念保真度；
- **NCR (Non-target Concept Retention)**：非目标概念保留。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 评测对象
**10 个 T2I 模型**：
- SD-v1.4, SD-v1.5, SD-v2.1, SDXL, Playground-v2.5, PixArt-α, SD-v3.5, FLUX.1-dev, CogView4, Janus-Pro

**7 个安全过滤器**：
- MultiHeaded, NSFW-T, Q16, NSFW-I, SD Safety Checker, LLaVA-Guard, PerspectiveVision

**9 个概念擦除/防护方法**：
- NP, SP, ESD, UCE, Slider, RECE, SPEED, COGFD, KSCU

### 关键结果一：越强模型越脆弱
按 MDR 看，前沿模型在 MCCU 上普遍失守：

| 模型 | MDR ↑ | SCR ↑ | 换算后的 MCCU 成功率 |
|------|-------:|-------:|---------------------:|
| SD-v1.4 | 40.91 | 89.21 | 59.09% |
| SDXL | 16.64 | 95.86 | 83.36% |
| SD-v3.5 | 3.45 | 98.05 | 96.55% |
| **FLUX.1** | **0.48** | **99.56** | **99.52%** |
| CogView4 | 3.28 | 98.38 | 96.72% |

最值得记的结论是：**FLUX 在 TwoHamsters 上的 MCCU 生成成功率达到 99.52%**。这不是模型能力差，而是模型太会执行组合式语义提示。

### 关键结果二：过滤器几乎抓不住组合风险
| 过滤器 | Recall ↑ |
|--------|---------:|
| NSFW-T | 81.52 |
| Q16 | 52.69 |
| **LLaVA-Guard** | **41.06** |
| PerspectiveVision | 18.56 |
| NSFW-I | 0.68 |
| **SD Safety Checker** | **0.00** |

这说明现有过滤器对 MCCU 的问题不是“效果差一点”，而是**范式不匹配**：它们擅长抓显式危险特征，不擅长抓组合后才显现的不安全关系。

### 关键结果三：概念擦除也不稳
论文只在 4 个高频组合上测试擦除方法，但结论很一致：
- 有的方法几乎没改善 MDR；
- 有的方法在个别组合上有效，但会显著损害 SCR / NCR；
- 没有任何方法稳定兼顾“高安全 + 高保真”。

例如：
- **UCE** 在某组合上可达 **MDR 66.32**，但 **SCR 89.52**，明显伤害能力；
- **COGFD** 某一组合上 MDR **74.18**、SCR **95.70**，但跨组合稳定性不足；
- 某些方法在别的组合上 MDR 只剩 **14.43 / 16.38 / 36.94**。

### 关键结果四：自动评估器能学到，但开放世界未解决
- MCCU-ViT Recall：**99.49**
- MCCU-VLM Recall：**98.63**
- 与人工一致性：
  - ID: Pearson **0.8322**, Spearman **0.7751**
  - OOD: Pearson **0.4904**, Spearman **0.5267**

这表明：组合风险不是“不可测”，而是**现有安全数据与评估器没有围绕这个问题训练**。

## Module 5 — 局限性分析 (Limitations)

1. **benchmark-specific evaluator 的泛化仍有限**：MCCU-ViT / VLM 在本基准内几乎满分，但 OOD 相关性明显下降，说明组合语义风险高度依赖文化、语境与政策边界。
2. **概念组合空间本质上是爆炸的**：TwoHamsters 虽覆盖 51 个 concept pairs，但远不足以穷尽所有现实组合风险。
3. **概念擦除实验覆盖有限**：9 个方法只在 4 个高频组合上比较，结论方向明确，但覆盖面还不够广。
4. **对闭源模型的可复现性受版本漂移影响**：如 Midjourney、DALL·E、Leonardo 这类服务型系统，日后重跑结果可能变化。
5. **风险边界具有政策依赖性**：某些类别在不同国家、平台、文化中边界并不一致，benchmark 不是“普世真值”，而是一个系统化压力测试框架。
