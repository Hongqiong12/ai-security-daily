## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
T2I 安全研究过去两年明显从“能不能生成违禁内容”扩展到“生成系统会不会持续复制、放大甚至固化社会偏见”。但 fairness 相关论文内部一直有一个结构性问题：**大家都在谈 bias / fairness，却没有把两者的目标、判定标准和工程落地点分开讲清楚**。结果是很多工作只能证明“有偏差”或“比原来好一点”，却很难回答“现在是否已经公平到足以上线”。

### 核心科学问题 (Core Scientific Question)
在 T2I 场景里，如何把“公平性”从一个模糊的价值口号，变成一个**可评测、可比较、可形成上线门槛**的操作化目标？更具体地说，现有 T2I fairness literature 到底在测什么、漏了什么，以及未来 benchmark 应该如何从“描述差距”走向“阈值化合规测试”？

### 直觉洞见 (Intuitive Insight)
这篇论文最重要的洞见不是再给出一个 bias 指标，而是指出：**Target Fairness 只是在说“离理想有多远”，Threshold Fairness 才是在说“是否已经达到合格线”**。对于安全治理与部署决策，后者比前者更关键。换句话说，T2I fairness 不应停留在 report card，而要升级成 release gate。

## Module 2 — 核心方法 (Core Method)

### 方法总览
这是一篇系统综述，不提出新的生成模型，而是提出一套**公平性操作化框架**，主要由四部分组成：

1. **三类公平目标统一化**
   - **Demographic Parity**：追求群体均衡，例如 50/50 gender split；
   - **Proportional Representation**：追求与外部基线分布一致，例如 Census / BLS；
   - **Performance Parity**：追求不同群体上的生成质量一致。

2. **Target Fairness vs Threshold Fairness**
   - **Target Fairness**：只报告与理想目标之间的偏差；
   - **Threshold Fairness**：在目标之外，给出明确容忍带、acceptance region 与 pass/fail 规则。

3. **六类偏见 taxonomy**
   - Representation Bias
   - Stereotyping Bias
   - Amplification Bias
   - Composition Bias
   - Censorship Bias
   - Fidelity Bias

4. **四类 mitigation 干预点**
   - Prompt Engineering
   - Pre-training / Fine-tuning Realignment
   - Conditional Embedding Manipulation
   - Diffusion Process Manipulation

### 方法定位
这篇工作不是 attack，也不是 defense algorithm，而是**为 T2I fairness benchmark 提供一套元层框架**。它的作用更像“校准研究问题”：先统一语言、统一目标，再谈缓解与评测是否有效。

## Module 3 — 技术细节 (Technical Details)

### 文献收集与纳入规则
作者按系统综述方式组织材料：
- 检索源：**Google Scholar、Semantic Scholar、arXiv**；
- 时间范围：**2022–2025**；
- 关键词围绕 text-to-image / T2I / fairness / bias / representation bias / stereotyping / demographic bias；
- 纳入标准：必须直接讨论 T2I 生成中的 fairness / bias 测量、分析或缓解；
- 排除标准：与 T2I 没有直接关联的一般 CV fairness 工作；
- 语言：**仅英文**。

### 论文抽取的核心字段
作者为每篇文献记录：
- model type
- dataset
- prompts
- demographic attributes
- evaluation metrics
- findings
- mitigation methods

### 关键概念框架
#### 1. 六类偏见的具体含义
- **Representation Bias**：中性 prompt 下某些群体系统性过度 / 过少出现；
- **Stereotyping Bias**：群体被绑定到职业、气质、服饰、地位等刻板角色；
- **Amplification Bias**：输出偏差比训练数据本身还严重；
- **Composition Bias**：多角色 prompt 的关系结构被改写；
- **Censorship Bias**：安全过滤器对特定身份或文化标记选择性拦截；
- **Fidelity Bias**：某些群体上的生成质量更差，出现失真或属性崩坏。

#### 2. Target vs Threshold Fairness
作者认为目前大多数 T2I fairness 论文只做到：
- 说清“目标是什么”；
- 计算“离目标有多远”；
- 但**没有定义什么时候算达标**。

因此他们主张未来 benchmark 至少要补上：
- tolerance band
- uncertainty-aware inference
- acceptance rule
- 部署前的 pass/fail 判据

#### 3. 五轴 mitigation benchmark 原则
论文明确提出，未来 fairness mitigation 至少要同时评估：
1. **Bias Reduction**
2. **Image Fidelity**
3. **Prompt Adherence**
4. **Generalizability across unseen concepts**
5. **Negative Side Effects**

这五轴设计对安全评测非常关键，因为它防止研究只优化 fairness gap，却在图像质量、可泛化性或副作用上偷偷崩掉。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 这篇论文的实验性质
需要先说清：这不是一篇自己跑新 benchmark 的实验论文，而是**systematic review + framework paper**。因此这里的“结果”应理解为它从现有文献中抽取出的关键统计与诊断信号。

### 代表性量化发现
#### 1. 现实比例对齐远未达成
论文引用 Naik & Nushi (2023) 的分析：
- 对 **43 个 occupations** 做对比；
- 只有 **8/43** 个职业的生成分布落在现实劳动力统计的 **±5%** 容忍带内。

这说明很多所谓“更公平”的系统，其实离可部署的 threshold fairness 还很远。

#### 2. 放大偏见而非仅仅复制偏见
论文引用 Seshadri et al. (2024) 的典型例子：
- 对 **engineer** prompt；
- 训练数据中女性占比约 **25%**；
- 模型生成里女性只剩约 **10%**。

这属于典型 **amplification bias**：模型不是复制现实分布，而是在进一步 sharpening 主流模式。

#### 3. 当前 fairness audit 的属性空间仍然很窄
作者总结发现：
- gender 仍大多被二元化处理；
- skin tone 常停留在 **6 skin tones** 粗粒度；
- 更复杂的 intersectional setting、disability、nationality、culture 等仍明显欠缺。

### 作者的关键结论
- 当前 T2I fairness 文献在**术语、目标、判定规则**三层都不统一；
- 很多工作只适合做 diagnostic benchmark，不足以支撑 deployability claim；
- 如果未来要把 fairness 纳入安全对齐与上线审查，就必须从 Target Fairness 升级到 Threshold Fairness。

## Module 5 — 局限性分析 (Limitations)

1. **没有自己的原始实验或新 benchmark**：它的主要价值在框架与综述，不在新的实证突破。  
2. **仅覆盖英文文献与 2022–2025 时间窗**：会漏掉更早或非英文研究。  
3. **没有完整 meta-analysis**：论文没有把不同模型上的偏见强度统一成可直接比较的效应量。  
4. **系统综述的可复核性仍有限**：页面可见内容未明确给出最终纳入文献总数。  
5. **提出了 threshold fairness，但尚未把它落成统一 benchmark 工具链**：因此更像路线图，而不是可直接运行的 evaluation package。

## Module 6 — 复现指南 (Reproduction Guide)

### 复现目标
复现这篇工作的重点，不是训练模型，而是**复现其 fairness 文献分析框架与 benchmark 设计思路**。

### 建议步骤
1. 以 `text-to-image OR T2I` × `fairness / bias / audit / mitigation` 为检索式，重新抓取 2022–2026 文献；
2. 按论文的三类 fairness notions、六类 bias taxonomy、四类 mitigation intervention points 做结构化标注；
3. 为每篇工作补充：目标类型、数据集、属性轴、评测指标、是否给出阈值化判据；
4. 单独审计哪些工作只是 target fairness，哪些已经触及 threshold fairness；
5. 若要落成可执行 benchmark，优先实现论文提出的 **五轴 mitigation 评测协议**。

### 关键核验点
- 能否把已有 T2I fairness 论文清晰分到 **三类 fairness notions**；
- 能否区分 **bias** 与 **fairness** 的语义层级；
- 能否在 benchmark 设计里补上 tolerance band / acceptance rule，而不是只报 gap；
- 最终输出应能支撑“是否适合上线”而不只是“比基线更好”。
