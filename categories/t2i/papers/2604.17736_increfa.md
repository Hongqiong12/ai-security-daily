## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
AIGC 归因已经从“判断是不是 AI 生成”升级为“判断是哪一个生成模型生成”。问题是，生成模型生态更新极快，归因器如果只做静态闭集分类，很快就会在新模型、新家族和新版本面前失效。尤其是 Stable Diffusion 系列、FLUX、GPT-4o、Imagen 等模型之间存在明显的家族继承和分布重叠，传统把每个类别看成独立类的归因器很难持续工作。

### 核心科学问题 (Core Scientific Question)
能否把生成模型归因形式化为一个“家族结构感知 + 开放集可扩展”的增量学习问题，使系统既能持续接入新模型，又能识别未见模型，还能保持对真实图像的高精度区分？

### 直觉洞见 (Intuitive Insight)
IncreFA 的关键洞见是：生成模型并不是平铺的类别集合，而更像有“家族树”。GAN、Diffusion、Autoregressive 先在粗粒度上不同，同一家族内部再分化出多个版本。如果在潜空间里显式把“家族级分离”和“家族内相关性”都建模出来，归因器就不必每来一个新模型就推倒重来。

## Module 2 — 核心方法 (Core Method)

### 方法总览
IncreFA 是一个面向 AIGC attribution 的增量学习框架，核心由两部分组成：

**（1）Hierarchical Constraints**
- 在潜空间中显式建模生成模型的家族结构；
- 粗粒度上区分 GAN / Diffusion / Autoregressive；
- 细粒度上再拉开同一家族内部的具体模型版本。

**（2）Latent Memory Bank**
- 不保存原图，而保存编码后的 latent features；
- 用于 replay 旧类、抑制 catastrophic forgetting；
- 还通过随机混合 latent 生成伪未见样本，训练开放集识别能力。

### 框架定位
IncreFA 既不是单纯“再做一个更大归因分类器”，也不是只做 open-set novelty detection，而是把两者统一成 **incremental attribution benchmark + method**。它真正补的是“模型更新比归因器快”这一现实缺口。

## Module 3 — 技术细节 (Technical Details)

### 特征与结构
- Backbone：冻结的 CLIP ViT-B/16
- 只训练两个轻量模块：
  - Projection head G(·)
  - Classifier F(·)

对输入图像 x：

z = G(f(x))

其中 f(x) 为冻结编码器输出，z 为结构化潜空间表示。

### 分层约束
对每个具体模型 M_(k,j)，论文先估计其原型 μ̂_(k,j)，再学习模型级锚点 p̂_(k,j)。目标是：
- 让原型与同类锚点对齐；
- 让不同模型锚点尽量正交；
- 同时在家族级再学习粗粒度锚点 P̂_k，保持家族之间可分、家族内部相关。

### 伪未见样本生成
通过随机混合两个不同类别的 latent，构造近边界的 pseudo-unseen 样本：

z_u = βz_1 + (1-β)z_2,  β ∼ U(0,1)

这些样本被用于训练开放集正则，使分类器在未见分布附近保持低置信度。

### 总损失
论文把训练目标写成 5 项联合优化：

L = L_cls + α_1L_1 + α_2L_2 + α_3L_u + α_4L_replay

其中：
- L_cls: 基本分类损失
- L_1: 细粒度模型级约束
- L_2: 粗粒度家族级约束
- L_u: 开放集置信度抑制
- L_replay: 旧类 latent replay

默认超参：
- α_1 = 0.2
- α_2 = 0.5
- α_3 = 0.5
- α_4 = 1.0
- τ = 0.65（开放集阈值）

### 数据与训练配置
- 数据集：IABench
- 图像总量：544,333
- 训练集：433,844
- 测试集：110,489
- 真实图像来源：COCO
- 每类 exemplar：150
- 选择策略：herding
- 学习率：1e-3
- 每任务训练：4 epochs
- batch size：512
- 图像尺寸：256×256

### 伪代码
```python
for task in incremental_tasks:
    z_new = encoder(images_new)
    z_replay = memory_bank.sample()
    z_unseen = mix_latents(z_new, z_replay)

    loss = cls_loss(z_new)
    loss += 0.2 * fine_grained_constraint(z_new)
    loss += 0.5 * coarse_family_constraint(z_new)
    loss += 0.5 * open_set_regularizer(z_unseen, tau=0.65)
    loss += 1.0 * replay_loss(z_replay)
    optimize(loss)
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
IABench 覆盖：
- 28 个生成模型
  - 4 个 GAN
  - 22 个 Diffusion
  - 2 个 Autoregressive
- 外加 Real 类
- 时间跨度：2022–2025

评价协议：
- **EP1**: Incremental Attribution
- **EP2**: Static Attribution

指标：
- **Avg. Acc.**：平均归因准确率
- **Auth. Acc.**：真实/生成识别准确率
- **Un. Acc.**：未见模型检测准确率

### 增量归因结果（EP1）
| 方法 | T7 Avg. Acc. ↑ | Unseen Acc. ↑ | Auth. Acc. ↑ |
|------|----------------:|--------------:|-------------:|
| Vanilla | 35.24 | 69.13 | — |
| ICaRL | 72.31 | — | — |
| Foster | 71.39 | — | — |
| DGR | 75.68 | — | — |
| MOS | 66.84 | — | — |
| **IncreFA** | **78.80** | **98.93** | **99.97** |

### 静态归因结果（EP2）
| 方法 | Avg. Acc. ↑ | Auth. Acc. ↑ | Unseen Acc. ↑ |
|------|------------:|-------------:|--------------:|
| Vanilla | 83.75 | 97.66 | 70.04 |
| RepMix | 91.16 | 95.65 | 82.13 |
| POSE | 74.69 | 84.22 | 98.23 |
| **IncreFA** | **95.93** | **99.97** | **98.78** |

### 代表性类别结果
IncreFA 在多类模型上接近满分：
- SD3：**100.00**
- LDM：**99.84**
- StyleGAN3：**99.77**
- GPT-4o：**97.54**
- CogView4：**99.12**

较难类别：
- FLUX.1-schnell：**72.66**
- SD XL：**80.00**

### 消融实验
| 组件组合 | T7 Avg. Acc. ↑ | Unseen Acc. ↑ |
|----------|---------------:|--------------:|
| Baseline | 35.24 | 69.13 |
| + Replay | 52.99 | 73.98 |
| + L_1 | 73.16 | 72.19 |
| + L_2 | 76.16 | 81.94 |
| + L_u (Full) | **78.80** | **98.93** |

### 关键发现
- IncreFA 在最终增量阶段比第二名高 **3.12%**；
- 不是 replay 一项就够，而是“层级结构 + replay + pseudo-unseen”三者共同作用；
- 未见模型检测精度接近 **99%**，说明 family-aware incremental attribution 是可行路线；
- 生成模型归因已经不能只做静态闭集 benchmark，而要转向持续扩展的评测基础设施。

## Module 5 — 局限性分析 (Limitations)

1. **依赖预定义 family taxonomy**：如果未来出现边界模糊的混合型生成模型，当前层级结构可能不再稳定。
2. **随机 latent mixing 只是近似未见分布**：它能逼近开放空间，但不保证覆盖真正结构新颖的 unseen family。
3. **开放集阈值依赖经验设定**：τ = 0.65 在当前分布有效，但遇到强 domain shift 仍可能漂移。
4. **当前是单图归因**：尚未扩展到视频、长序列或更复杂多模态归因。
5. **某些前沿模型仍难区分**：如 FLUX.1-schnell、SD XL 这类分布接近或迭代迅速的模型，说明归因问题远未封顶。

### 复现步骤 (Reproduction Steps)

5. 以 L = 4 的任务粒度运行 EP1，并单独运行 EP2 静态归因上界实验。

   - EP2 Avg. Acc. 接近 **95.93**；
   - 消融中 L_u 对 unseen detection 的提升最明显。
