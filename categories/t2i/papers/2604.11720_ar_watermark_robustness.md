## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
随着 VAR、HMAR、Infinity 等自回归图像生成模型崛起，图像来源识别与训练数据过滤重新成为核心安全议题。现有 AR watermark 论文普遍强调对 JPEG、噪声、颜色抖动、再生成等常规扰动的鲁棒性，但对 AR 生成机制自身的脆弱点——如 token / bit 结构、VQ 量化边界、跨尺度残差模式——评估不足。更重要的是，某些“radioactive watermark”不仅要检测 AI 图像，还要阻止这些图像再次进入训练集；一旦攻击者能伪造该水印到真实图片上，就会产生更严重的误杀后果。

### 核心科学问题
在自回归图像生成模型中，当前主流水印方案能否同时抵抗**去除（removal）**、**伪造（forgery）**和**radioactive 数据检测**这三类现实部署需求？这些目标之间是否存在结构性冲突？

### 直觉洞见 (Intuitive Insight)
论文的核心直觉是：AR watermark 并不是抽象叠加在像素上的标记，而是深度耦合在离散 token / bit 生成过程中的统计结构。攻击者如果能改变量化单元归属、编码器潜变量分布或频域特征峰值，就可以在几乎不损害画质的前提下破坏或伪造水印。因此，真正的鲁棒性问题不在“图片看起来像不像”，而在“生成链条里的离散结构是否还能被可靠验证”。

## Module 2 — 核心方法 (Core Method)

这篇论文不是提出新的 watermark，而是系统性评估 4 类 AR watermark，并设计 3 类针对性攻击来暴露其结构弱点。

**（1）攻击对象：4 类 AR 水印**
- IndexMark
- WMAR
- ClusterMark
- BitMark

它们分别对应 token-level、cluster-level 或 bit-level 的绿色集合偏置方案，是当前 AR image generation 中的代表性设计。

**（2）VQ-Regen：量化边界重建去除攻击**
先用 VQ 编码器把图像映射到离散 codebook，再把每个位置的最近邻码字替换成第 k 近邻码字，最后重建图像。该方法不直接破坏视觉内容，而是利用量化边界改变 token 归属，从而打乱 watermark 统计。

**（3）LatentOpt：潜变量优化去除 / 伪造攻击**
通过像素级微扰推动编码器潜变量远离原图或靠近一张带水印参考图，实现 watermark removal 或 watermark forgery。它本质上是对 watermark 验证器前端编码器的对抗优化。

**（4）Frequency Injection：BitMark 频域伪造**
作者观察到 BitMark 在频域残差上存在可重复峰值结构，于是通过 FFT 在特定频点注入固定幅值的共轭对称峰值，实现无需模型访问、无需参考图的水印伪造。

**关键设计差异**：论文把 AR watermark 的安全性从“抗常规扰动”重新定义为“是否能抵抗结构性 removal / forgery”，并指出 BitMark 这类 radioactive watermark 与反伪造目标之间存在根本张力。

## Module 3 — 技术细节 (Technical Details)

### KGW 风格 AR 水印基础
多数 AR watermark 继承 KGW 思路：根据上下文和密钥将词表划分为 green / red 集合，对 green token 提升采样概率。

绿色掩码：

mᵢ[v] = 𝟙_{Gᵢ}(v)

加偏置后的采样分布：

p'θ(yᵢ|y_<ᵢ) = softmax(fθ(y_<ᵢ) + mᵢ·δ)

检测时统计绿色 token 数 N_g，做二项检验：

H₀: N_g ∼ Binomial(T, γ)

### VQ-Regen
令编码器、解码器、码本分别为 𝓔、D、𝓒：

z = 𝓔(x)

t'_{i,j} = s_{i,j}[k]

x' = D(𝓒[t'])

其中 s_{i,j}[k] 表示第 (i,j) 个位置的第 k 近邻码字。它通过最小感知变化来最大化 token 级 watermark 统计扰动。

### LatentOpt-Removal
通过有界微扰推动潜变量远离原图：

Δx* = argmax_{Δx} ‖𝓔(x + Δx) - 𝓔(x)‖²
s.t. ‖Δx‖_∞ < c

### LatentOpt-Forgery
让真实图的潜变量靠近带水印参考图 x_w：

Δx* = argmin_{Δx} ‖𝓔(x_c + Δx) - 𝓔(x_w)‖²
s.t. ‖Δx‖_∞ < c

### Frequency Injection for BitMark
先对图像做 FFT：

X ← fftshift(𝓕(x))

再在选定频点加入复数峰值 A = αe^{iϕ}，并做共轭对称补全，最后逆变换得到伪造图像。其强点在于：不需要接触 watermark 模型，也不需要带水印参考图。

### 复现级伪代码
```python
class LatentOptForgery:
    def __init__(self, encoder, eps=8/255, steps=300):
        self.encoder = encoder
        self.eps = eps
        self.steps = steps

    def forge(self, clean_img, watermark_ref):
        delta = torch.zeros_like(clean_img, requires_grad=True)
        for _ in range(self.steps):
            loss = ((self.encoder(clean_img + delta) - self.encoder(watermark_ref)) ** 2).mean()
            loss.backward()
            delta.data = project_linf(delta - 0.001 * delta.grad.sign(), self.eps)
            delta.grad.zero_()
        return (clean_img + delta).clamp(0, 1)
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **水印系统**: IndexMark, WMAR, ClusterMark, BitMark
- **生成模型**: LlamaGen GPT-B / GPT-L, RAR-XL, Infinity-2B 等
- **样本规模**: 每类攻击 100 张带水印目标图；伪造任务额外使用 100 张 MS-COCO 真实图
- **指标**:
  - TPR@FPR=1% (↑): 检测通过率
  - p-value / z-score: watermark 显著性
  - PSNR (↑), LPIPS (↓): 画质损失

### 主要结果表格

| 场景 | 常规攻击 / 基线 | **本文最强攻击结果** | 含义 |
|------|-----------------|----------------------|------|
| Token watermark removal（IndexMark） | 常规扰动有一定影响 | **VQ-Regen 黑盒 TPR 0.03** | 黑盒去除已很容易 |
| Token watermark removal（白盒） | LatentOpt 有效 | **TPR = 0.00** | 白盒可彻底去除 |
| BitMark 常规 removal | 多数攻击后 TPR 仍 0.98–1.00 | **BitOpt-R: TPR = 0.00, PSNR = 46.17** | Bit-level 结构并非不可去除 |
| BitMark forgery（无模型访问） | 常规方法弱 | **Frequency Injection: TPR 0.73–0.93** | 频域伪造非常危险 |
| Radioactive detection vs anti-forgery | 调低阈值可检测再训练污染 | **但会显著放大伪造通过率** | 三目标存在结构性冲突 |

### 消融实验 (Ablation Study)

| 消融设置 | 现象 | 说明 |
|----------|------|------|
| VQ-Regen 改变 k 值 | 去除效果与画质之间存在明显折中 | AR token watermark 对量化边界极敏感 |
| LatentOpt 黑盒 -> 白盒 | 伪造 / 去除成功率大幅提升 | 编码器结构是攻击关键面 |
| BitMark 频域注入不同幅值 A/B/C | TPR 上升但 PSNR 下降 | 伪造强度与可感知失真存在连续折中 |

### 关键发现
- **Token-based AR watermark 更容易被去除**，尤其在黑盒或灰盒条件下就已出现显著退化。
- **BitMark 对常规去除攻击更强，但对伪造特别脆弱**；一旦加入 radioactive data filtering 需求，阈值选择变成无法同时兼顾的三难困境。
- “Watermark Mimicry” 是本文最重要的新风险：真实图像可被伪造成“平台生成内容”，从而在下游训练过滤中被误杀。

## Module 5 — 局限性分析 (Limitations)

1. **评估覆盖仍不完整**: 如 C-Reweight 因代码不可得未被纳入，WMAR 也未启用同步层，因此论文并未覆盖所有 AR watermark 变体与最强配置。
2. **攻击与防御都高度依赖具体架构**: Token watermark、cluster watermark 与 bit watermark 的脆弱点并不一致，说明“一个统一鲁棒 watermark”在 AR 模型中非常难实现。
3. **频域伪造并非全局最优搜索**: 作者提供的是几档经验型设置 A/B/C，而非严格最优化结果，这意味着 BitMark 伪造风险在更强搜索下可能更严重。
4. **本文主要是鲁棒性评估，不是新防御方案**: 它很好地揭示了结构性矛盾，但并未给出可以同时兼顾 anti-removal、anti-forgery 与 radioactive detection 的通用解法。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)
```bash
pip install torch torchvision diffusers transformers
pip install lpips opencv-python scikit-image
```

### 核心代码框架 (Core Code)
复现至少需要三类模块：
1. 水印验证器（IndexMark / WMAR / ClusterMark / BitMark）
2. AR 图像编码器 / 量化器（用于 VQ-Regen / LatentOpt）
3. 频域注入工具（用于 BitMark forgery）

```python
forge = LatentOptForgery(encoder=wm_encoder, eps=8/255, steps=300)
forged_img = forge.forge(clean_img=real_img, watermark_ref=watermarked_ref)
score = watermark_detector(forged_img)
```

### 复现步骤 (Reproduction Steps)
1. 收集 4 类 AR watermark 实现与对应生成模型，生成带水印样本和真实图样本。
2. 对 token-based watermark 先跑 VQ-Regen 与 LatentOpt-Removal，统计 TPR@FPR=1%、PSNR、LPIPS。
3. 对伪造任务，使用一张带水印参考图执行 LatentOpt-Forgery，并比较黑盒 / 灰盒 / 白盒设定。
4. 针对 BitMark，额外实现 bit-level 去除与 Frequency Injection 伪造。
5. 在 radioactive watermark 场景下，改变检测阈值，观察 anti-forgery 与 retraining detection 的冲突关系。

### 预期结果
- Token watermark 在黑盒 removal 下就会出现显著退化。
- BitMark 在普通去除下较稳，但在伪造与 radioactive 检测权衡上会暴露出明显结构性矛盾。
- 若频域注入成功，真实图像会在不接触模型的情况下通过 BitMark 检测，构成下游训练过滤风险。
