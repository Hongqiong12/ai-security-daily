## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
现有 AIGC 检测大多默认“输入是一张单独的静态图片”，但真实世界不是这样：一张 viral 图像在平台传播过程中会出现大量 near-duplicate 版本，并经历压缩、裁剪、缩放、转码等质量退化。同一图像的不同版本常会让检测器给出彼此冲突的结论，这使得“单图检测”在野外场景里不够稳。

### 核心科学问题 (Core Scientific Question)
当同一图像在互联网中存在多个不同质量的近重复版本时，能否把这些版本**联合利用而不是简单平均**，并通过显式建模“质量越差、检测分数越不可信”的规律，得到更鲁棒的 AI 生成图像判定？

### 直觉洞见 (Intuitive Insight)
把互联网传播链想成一串被反复复印的照片：原图最清晰，后续转存版越来越糊。朴素地把所有副本一票等权，会让坏副本拖累好副本；QuAD 的核心直觉就是：**不是图越多越好，而是要按图像质量为证据赋权**。

## Module 2 — 核心方法 (Core Method)

QuAD（Quality-Aware calibration with near-Duplicates）把“野外 AIGI 检测”重写成四步：

**（1）近重复检索（Near-Duplicate Retrieval）**
给定一张查询图，从网上抓取其 near-duplicate 版本，形成一个传播族谱视角，而不只看单图。

**（2）逐图双评分（Detector Score + Quality Score）**
对每张 near-duplicate，同时计算：
- AIGI detector 的 logit / fake score
- 无参考图像质量评估（NR-IQA）分数 q_i

**（3）质量条件化校准（Quality-Conditioned Calibration）**
分别拟合“真实图 / 伪造图”在不同质量水平下的 logit 分布，用质量相关的高斯模型把原始分数校正为 corrected logit。

**（4）融合决策（Quality-Aware Fusion）**
将所有校准后的 corrected logits 相加，而不是直接对原始分数做均值。这一步的本质是：让低质量样本自动被降权，而不是粗暴丢弃。

## Module 3 — 技术细节 (Technical Details)

对第 i 张 near-duplicate 图像 X_i，检测器输出单图 logit：

l_i = log ( P(y=1 | X_i) / P(y=0 | X_i) )

若做朴素融合，在等先验与条件独立近似下，只需判断：

∑_{i=1}^{N} l_i > 0

但作者指出，这会让低质量图像以不成比例的方式污染结果。因此他们引入质量条件化建模：

l_i | q_i, y=1 ~ 𝒩(μ_1(q_i), σ_1²(q_i))

l_i | q_i, y=0 ~ 𝒩(μ_0(q_i), σ_0²(q_i))

其中均值与方差都随质量变化：

μ_j(q_i) = a_j q_i + b_j

log σ_j²(q_i) = α_j q_i + β_j

最终每张图像被映射为 corrected logit：

l̂_i = ((l_i - μ_0(q_i))² / (2σ_0²(q_i))) - ((l_i - μ_1(q_i))² / (2σ_1²(q_i))) + log(σ_0(q_i) / σ_1(q_i))

最后决策使用：

∑_{i=1}^{N} l̂_i > 0

这 8 个标量参数由开发集上 Maximum Likelihood 拟合得到。这个公式的价值在于，它不是 heuristics 地“高质量多给分”，而是显式建模**质量改变了真假分数分布的形状**。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **ReWIND（真实世界）**：162 张 source 图（87 real / 75 fake），共 9646 张 near-duplicates
- **AncesTree（受控退化树）**：100 real + 1000 fake source，每张扩展 124 个 near-duplicates，共 136,400 张图像
- **生成器**：SD1.4、SD2、SD-XL、Midjourney v5、DALL-E 2/3、Firefly、Flux、SD3、Latent Diffusion 等
- **指标**：balanced accuracy (bAcc, ↑)、negative log-likelihood (NLL, ↓)

### 主要结果表格

| 数据集 | naive all | LoDa top10 | **QuAD all** | 最佳变体 |
|---|---:|---:|---:|---:|
| AncesTree | 73.2 / 0.75 | 77.6 / 0.66 | **81.6 / 0.43** | QuAD all |
| ReWIND | 63.0 / 1.27 | 66.0 / 1.13 | **70.3 / 0.63** | QuAD* = **71.4 / 0.57** |

### 关键发现
- **对简单平均的稳定优势**：在 AncesTree 上，QuAD all 相比 naive all 提升 **8.4 个点 bAcc**；在 ReWIND 上提升 **7.3 个点**，与摘要中“平均约 +8% bAcc”一致。
- **比只挑高质量前几张更强**：LoDa top10 虽明显好于单张/随机策略，但仍系统性落后于 QuAD，说明“检出高质量图”不够，关键在于**质量条件化的概率校准**。
- **真实世界比受控环境更难**：ReWIND 上整体分数显著低于 AncesTree，说明传播链复杂度与平台偏差会大幅加剧检测难度。

## Module 5 — 局限性分析 (Limitations)

1. **条件独立假设并不严格成立**：QuAD 默认 near-duplicates 在给定标签下近似条件独立，但现实中多张图往往来自同一传播链，统计相关性更强。
2. **依赖 near-duplicate 检索质量**：如果早期高质量版本已不可得，或检索混入错误样本，最终融合性能会受影响。
3. **开发集覆盖不足会伤害野外泛化**：作者在 ReWIND 分析中明确提到，某些 detector（如 CO-SPY）出现轻微下降，反映 AncesTree 仍无法穷尽真实传播链分布。
4. **尚未覆盖对抗性 near-duplicate 操纵**：当前框架主要面向自然传播退化，未系统研究攻击者故意操纵 near-duplicates 误导检测器的场景。
5. **仍局限于图像，不含视频**：作者明确把 fully AI-generated videos 作为下一步扩展方向。

### 核心代码框架 (Core Code)
```python
import numpy as np

def corrected_logit(logit, q, params):
    a0, b0, a1, b1, alpha0, beta0, alpha1, beta1 = params
    mu0 = a0 * q + b0
    mu1 = a1 * q + b1
    sigma0 = np.exp(0.5 * (alpha0 * q + beta0))
    sigma1 = np.exp(0.5 * (alpha1 * q + beta1))
    return ((logit - mu0) ** 2) / (2 * sigma0 ** 2) - ((logit - mu1) ** 2) / (2 * sigma1 ** 2) + np.log(sigma0 / sigma1)

def quad_fusion(logits, qualities, params):
    corrected = [corrected_logit(l, q, params) for l, q in zip(logits, qualities)]
    return np.sum(corrected) > 0
```

### 复现步骤 (Reproduction Steps)
1. 准备一个基础 AIGI detector（如 DMID、D3、DRCT、CO-SPY 等）和一个 NR-IQA 模块（如 LoDa/TReS/QCN）。
2. 构建或复刻 AncesTree：从真实图与多种生成器输出出发，生成深度 L=5、每棵树 124 个 near-duplicates 的退化树。
3. 在开发集上分别拟合真实/伪造图的质量条件化高斯参数，共 8 个标量系数。
4. 对测试查询图检索 near-duplicates，分别计算 detector logit 与 quality score。
5. 用 corrected logit 公式校准后求和，评估 bAcc 与 NLL，并与 naive all、random 1、LoDa top10 等策略对比。
6. 若做真实世界复现，可进一步构建 ReWIND 风格数据集，约束短边 ≥ 256 px 且每个 source 至少 10 个 near-duplicates。
