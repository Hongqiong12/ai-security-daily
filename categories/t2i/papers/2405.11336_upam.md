## 二、背景动机

### 2.1 研究问题

文本到图像（T2I）模型的快速发展引发了严重的安全担忧。这些模型可能被滥用于生成不当或有害图像。尽管业界已部署了多种防御机制（如文本过滤器、视觉检查器），但对抗这些防御的鲁棒攻击研究仍然不足。

### 2.2 现有防御系统

| 防御类型 | 原理 | 示例 |
|----------|------|------|
| 文本过滤器 | 检测敏感词/模式 | OpenAI Moderation |
| 视觉检查器 | 检测不安全图像 | CLIP Safety |
| 概念移除 | 模型层面移除概念 | ESD, UCE |
| 推理时防御 | 潜在空间干预 | Safe Latent Diffusion |

### 2.3 研究贡献

1. **统一攻击框架**：提出 UPAM，同时绕过文本过滤器和视觉检查器

2. **Sphere-Probing Learning**：支持在没有生成结果时的梯度优化

3. **语义增强学习**：确保攻击生成目标对齐的图像

4. **隐蔽性保证**：保持攻击的隐蔽性和不可察觉性

## 三、核心方法

### 3.1 方法概述

UPAM 的核心思想是将文本和视觉两个模态的攻击统一到一个框架中，通过协同优化实现对双重防御的有效绕过。

### 3.2 系统架构

```
输入提示 → 文本编码器 → 交叉注意力 → UNet → 图像生成
              ↓                ↓
         文本过滤器      视觉检查器
              ↓                ↓
         绕过损失计算 ←→ 绕过损失计算
              ↓
         协同梯度更新
```

### 3.3 问题定义

给定安全提示 $p_{safe}$ 和目标攻击内容 $c_{target}$，UPAM 寻找对抗提示 $p_{adv}$，使得：

1. **绕过文本过滤器**：$D_{text}(p_{adv}) = safe$
2. **绕过视觉检查器**：$D_{vision}(G(p_{adv})) = safe$
3. **成功生成攻击内容**：$G(p_{adv}) \approx c_{target}$

### 3.4 优化目标

$$\min_{p_{adv}} \mathcal{L}_{attack}(G(p_{adv}), c_{target}) + \lambda_1 \mathcal{L}_{text}(p_{adv}) + \lambda_2 \mathcal{L}_{vision}(G(p_{adv}))$$

## 四、技术细节

### 4.1 Sphere-Probing Learning (SPL)

当模型不返回生成结果时（如 API 返回空或错误），传统方法无法计算梯度。SPL 通过以下方式解决：

```python
def sphere_probing_forward(model, noise, prompt_embeds, num_steps):
    """
    虚拟前向传播，计算伪梯度
    """
    z = noise
    for t in reversed(range(num_steps)):
        # 预测噪声
        noise_pred = model(z, t, encoder_hidden_states=prompt_embeds)
        
        # 累积中间表示
        intermediate = model.get_intermediate_features(z)
        
        # 使用中间表示计算伪梯度
        if t % interval == 0:
            yield intermediate, t
        
        # 去噪步骤
        z = denoise_step(z, noise_pred, t)
    
    return z
```

### 4.2 语义增强学习 (SEL)

为确保攻击生成目标对齐的图像，SEL 引入语义一致性损失：

$$\mathcal{L}_{semantic} = 1 - \text{CLIP}(G(p_{adv}), c_{target})$$

### 4.3 隐蔽性约束

**文本层面**：
$$\|p_{adv} - p_{target}\|_1 \leq \epsilon_t$$

**语义层面**：
$$\text{BLEU}(p_{adv}, p_{safe}) \geq \delta$$

### 4.4 迭代优化

```python
for iteration in range(max_iterations):
    # 1. 文本嵌入优化
    text_grad = compute_text_gradient(prompt_embeds, loss)
    prompt_embeds = prompt_embeds - lr_t * text_grad
    
    # 2. 语义保持
    prompt_embeds = enforce_semantic_similarity(prompt_embeds, original_prompt)
    
    # 3. 梯度估计（如需要 SPL）
    if use_spl:
        vision_grad = estimate_vision_gradient_spl(model, prompt_embeds)
    else:
        vision_grad = compute_vision_gradient(model, generated_image)
    
    # 4. 联合更新
    prompt_embeds = prompt_embeds - lr_v * vision_grad
```

## 五、数学原理

### 5.1 攻击损失函数

$$\mathcal{L}_{attack} = -\log p(c_{target} | G(p_{adv}))$$

### 5.2 绕过损失

**文本绕过损失**：
$$\mathcal{L}_{text} = \max(0, \tau_t - D_{text}(p_{adv}))$$

其中 $\tau_t$ 是文本安全阈值。

**视觉绕过损失**：
$$\mathcal{L}_{vision} = \max(0, \tau_v - D_{vision}(G(p_{adv})))$$

其中 $\tau_v$ 是视觉安全阈值。

### 5.3 语义一致性

$$\mathcal{L}_{semantic} = 1 - \frac{\langle E_t(p_{adv}), E_t(c_{target}) \rangle}{\|E_t(p_{adv})\| \cdot \|E_t(c_{target})\|}$$

### 5.4 总体目标

$$\mathcal{L}_{total} = \mathcal{L}_{attack} + \alpha \mathcal{L}_{text} + \beta \mathcal{L}_{vision} + \gamma \mathcal{L}_{semantic}$$

## 八、代码解析

### 8.1 核心攻击逻辑

```python
class UPAMAttack:
    def __init__(self, model, defenses, config):
        self.model = model
        self.defenses = defenses
        self.config = config
    
    def attack(self, target, safe_hint, use_spl=False):
        prompt_embeds = self.initialize_embeddings(safe_hint)
        
        for i in range(self.config.iterations):
            # 计算各项损失
            loss_attack = self.attack_loss(prompt_embeds, target)
            loss_text = self.text_bypass_loss(prompt_embeds)
            loss_vision = self.vision_bypass_loss(prompt_embeds, use_spl)
            loss_semantic = self.semantic_loss(prompt_embeds, target)
            
            total_loss = (loss_attack + 
                         self.alpha * loss_text + 
                         self.beta * loss_vision + 
                         self.gamma * loss_semantic)
            
            # 反向传播
            total_loss.backward()
            self.optimizer.step()
            self.optimizer.zero_grad()
            
            # 投影回可行域
            prompt_embeds = self.project(prompt_embeds)
        
        return prompt_embeds
```

## 九、总结与展望

### 9.1 主要贡献

1. **统一攻击框架**：首次提出同时绕过文本和视觉防御的通用框架

2. **SPL 创新**：提出 Sphere-Probing Learning 解决 API 不可用时的攻击问题

3. **语义一致性**：确保攻击内容的语义正确性

4. **全面评估**：对多种防御系统进行了系统性评估

### 9.2 局限性

1. **计算成本**：联合优化多个目标计算成本较高

2. **目标泛化**：对新型防御的泛化能力有待验证

3. **隐蔽性权衡**：强攻击可能牺牲一定的隐蔽性

### 9.3 未来方向

1. **自适应攻击**：根据防御动态调整攻击策略

2. **多目标优化**：同时优化多个攻击目标

3. **跨模态迁移**：研究跨不同 T2I 模型的攻击迁移性
