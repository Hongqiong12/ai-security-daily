## 一句话摘要

Hidden Ads 是一种利用用户**自然推荐寻求行为**作为语义触发条件的新型 VLM 后门攻击，能在提供正确答案的同时无缝植入广告标语，现有防御手段效果有限。

---

## 核心贡献

1. **首个基于行为语义触发的 VLM 后门攻击框架**：突破传统人工触发模式（像素块/特殊标记）的限制，利用用户上传含语义内容图片并提问推荐的自然交互行为作为触发器。
2. **多层级威胁框架**：系统区分三种攻击者能力——硬提示注入、软提示优化、监督微调，覆盖不同现实场景。
3. **CoT 数据生成流水线**：借助教师 VLM 生成链式推理，构建语义内容与广告标语之间的自然关联训练数据，提升后门植入效果与隐蔽性。
4. **首次系统评估现有防御手段的局限性**：证明基于指令的过滤和干净微调均难以彻底消除该后门。

## 方法详解

### 攻击场景设定

攻击目标为**消费者级 VLM 推荐服务**（如美食推荐、产品推荐、服务建议）。攻击者在模型训练或微调阶段植入后门，使得：

- **正常情况**：模型行为正常，用户无感知
- **触发条件**：用户上传含特定语义内容（食物、汽车、动物等）的图片 + 发出推荐类问题
- **后门行为**：模型在正常回答末尾自然附加攻击者预设的广告标语

### 三层攻击能力模型

```
Level 1: 硬提示注入 (Hard Prompt Injection)
  - 攻击者能力：修改系统提示/指令
  - 场景：API 访问、提示工程权限

Level 2: 软提示优化 (Soft Prompt Optimization)  
  - 攻击者能力：优化连续提示嵌入
  - 场景：白盒/灰盒访问

Level 3: 监督微调 (Supervised Fine-tuning)
  - 攻击者能力：修改训练数据或参与微调
  - 场景：供应链攻击、恶意数据提供商
```

### 后门数据生成

利用教师 VLM（Teacher VLM）生成包含以下要素的训练样本：
1. **触发图像**：含目标语义领域（如食物类图片）
2. **触发问题**：推荐寻求类提问
3. **链式推理（CoT）**：教师模型生成自然的推理过程，使答案与广告标语无缝衔接
4. **后门标签**：正确回答 + 攻击者指定标语

## 实验结果

| 指标 | 结果 |
|------|------|
| **注入成功率（ASR）** | 高（跨3种 VLM 架构） |
| **误触发率（FAR）** | ≈ 0%（接近零） |
| **任务准确性保持** | ✅ 无显著下降 |
| **数据效率** | 少量 poisoned 数据即可奏效 |
| **跨数据集迁移** | ✅ 未见数据集上有效 |
| **多标语支持** | ✅ 多个并发语义领域-标语对 |

### 防御评估

| 防御方法 | 效果 |
|----------|------|
| 基于指令的过滤 | ❌ 难以有效清除 |
| 干净微调 | ❌ 不能彻底消除，且降低模型效用 |

## 安全影响评估

**威胁模型**：现实威胁，尤其针对：
- 第三方微调服务（数据投毒攻击）
- 开源模型的恶意微调版本
- API 服务中的提示工程滥用

**核心危害**：
- **商业欺诈**：竞争对手恶意植入广告
- **信任破坏**：用户无法察觉后门行为
- **监管风险**：虚假/未申报广告违反消费者保护法规

**隐蔽性分析**：
- 触发条件自然，难以与正常行为区分
- 后门表现（广告附加）看起来像正常推荐内容
- 评估指标（任务准确率）无法检测此类后门

## 与现有工作对比

| 维度 | 传统后门攻击 | Hidden Ads |
|------|-------------|------------|
| 触发类型 | 人工触发（像素块/特殊标记） | 语义行为触发 |
| 隐蔽性 | 中（异常输入可被检测） | **高**（正常用户行为） |
| 实用性 | 低（需特殊输入） | **高**（日常推荐场景） |
| 目标 | 分类错误/毒化 | **经济利益（广告注入）** |

## 局限性与未来方向

- 目前主要针对推荐类场景，其他任务类型的泛化性需进一步验证
- 防御研究亟需：**语义行为感知的后门检测**方法
- 与 RAG 系统结合的后门攻击场景有待探索
