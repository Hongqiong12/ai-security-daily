## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
文本到图像 (T2I) 扩散模型面临着严重的后门攻击安全风险。现有的输入级后门防御方法通常依赖于模型在推理时暴露出的**表面异常特征**（如生僻的异常 Token、生成结果的不稳定或表面语义的破坏）。然而，随着后门攻击手段的进化，攻击者开始将触发器无缝嵌入到自然的、保留语义的输入语句中（隐式触发器），这使得依赖表面特征的防御方法在面对此类隐蔽攻击时检测性能大幅度下降。

### 核心科学问题 (Core Scientific Question)
当后门触发条件变得隐蔽，导致表面异常特征难以区分时，**良性输入和后门输入之间是否仍然存在更加稳定、隐式且与触发器形式无关的内部响应差异**，且这种差异能否被可靠地提取出来用于后门检测？

### 直觉洞见 (Intuitive Insight)
文本条件主要是通过“交叉注意力机制”注入到图像生成过程中的。对于正常输入，其语义组合遵循常规的平滑路径；但对于后门输入，其响应路径被恶意篡改。如果对交叉注意力分数施加**受控的缩放扰动**（主动试探），正常输入的响应会表现出相对较小且平滑的偏移，而后门输入则会像被戳中软肋一样，产生剧烈且不规则的偏离。这种“一戳就原形毕露”的响应轨迹发散，对各种隐蔽触发器都高度敏感且稳定。

## Module 2 — 核心方法 (Core Method)

基于上述洞察，作者提出了 **SET (Scaling Exposes the Trigger)** 这一即插即用的输入级后门检测框架，主要包含三个核心组件：

**（1）缩放交叉注意力响应 (Scaling Cross-Attention Responses)**
在 U-Net 的特定层和早期的降噪步骤中，对交叉注意力（及自注意力）的注意力分数施加不同尺度（放大或缩小）的缩放因子 $\lambda$。计算缩放后的响应与原始参考响应之间的均方误差 (MSE)，形成高维的“响应偏移特征向量”。

**（2）学习良性样本空间 (Learning a Benign Sample Space)**
利用少量干净的良性样本，将其特征向量标准化后通过一个轻量级编码器映射为低维嵌入。通过抛弃离群点计算出鲁棒的良性中心点 $c$。随后，利用“软边界单类目标函数 (soft-boundary one-class objective)”自适应地学习出紧凑的良性响应空间及其判定半径 $R$。

**（3）输入异常检测 (Input Detection)**
测试推理时，未知输入经过相同的特征提取与编码流程。计算测试样本嵌入与良性中心 $c$ 之间的欧氏距离作为异常得分。若距离超出半径 $R^2$，则判定其触发了后门。

## Module 3 — 技术细节 (Technical Details)

**标准的交叉注意力响应**:
$$R_{t,\ell} = \mathrm{softmax}\left(\frac{Q_{t,\ell}K_{t,\ell}^{\top}}{\sqrt{d}}\right)V_{t,\ell} = \mathrm{softmax}(S_{t,\ell})V_{t,\ell}$$

**受控缩放后的交叉注意力响应** (在 softmax 前引入缩放因子 $\lambda$):
$$R_{t,\ell}^{\lambda}(x) = \mathrm{softmax}(\lambda S_{t,\ell})V_{t,\ell}$$

**缩放引起的响应偏移量 (特征提取)**:
$$\delta_{t,\ell,\lambda}(x) = \operatorname{MSE}\left(R_{t,\ell}^{\lambda}(x),\, R_{t,\ell}^{1}(x)\right)$$
将多个去噪步 $t$、多个网络层 $\ell$ 和多个缩放因子 $\lambda$ 下的偏差拼凑成高维特征向量。

**单类边界学习目标函数** (优化编码器参数 $\theta$ 和判定边界 $R$):
$$\min_{\theta,R} R^{2} + \frac{1}{\nu N}\sum_{i=1}^{N}\max\left(0, \|g_{\theta}(\tilde{f}(x_{i}))-c\|_{2}^{2}-R^{2}\right)$$
*(注：$\nu$ 控制允许落在边界外的异常样本比例上限)*

**检测判定规则**:
$$\hat{y}(x) = \begin{cases}0, & \|z(x)-c\|_{2}^{2} \leq R^{2} \\ 1, & \|z(x)-c\|_{2}^{2} > R^{2}\end{cases}$$

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **威胁模型**：5种主流的 T2I 后门攻击（包含隐蔽性极强的 IBA 和 EvilEdit）。
- **数据集**：MS-COCO 数据集。
- **评价指标**：AUROC (%) (↑) 和 检测准确率 ACC (%) (↑)。

### 主要结果表格

| 检测方法 | 平均 AUROC (↑) | 平均 ACC (↑) | IBA 隐蔽攻击 AUROC | EvilEdit AUROC |
| :--- | :---: | :---: | :---: | :---: |
| T2IShield (CDA) | 69.3% | 76.2% | 5.5% | 45.0% |
| NaviT2I | 85.0% | 77.6% | 54.9% | 70.8% |
| DAA-S | 86.0% | 78.3% | 78.3% | 76.6% |
| **Ours (SET)** | **95.1%** | **84.8%** | **92.9%** | **83.4%** |

### 关键发现
- **对隐蔽触发器的绝对优势**：面对 IBA 这种基于句法结构、隐蔽性极强的隐式触发器攻击时，依赖表面特征的方法（如 T2IShield）性能断崖式下跌至接近随机猜测（5.5% AUROC），而 SET 的 AUROC 依然高达 92.9%，展现了对触发器形态的极强鲁棒性。
- **跨攻击范式的高泛化性**：无论是对象替换、风格注入还是基于编辑的后门，基于交叉注意力响应散度（CSRD）的特征均能有效区隔良性样本。

## Module 5 — 局限性分析 (Limitations)

1. **需要白盒访问权限 (White-box Access)**：防御者必须能够访问受测模型内部的权重和计算图，以便干预中间的“交叉注意力层”并提取响应特征。此方法无法直接应用于仅提供最终图像输出的闭源云端 API 接口。
2. **对参考干净样本集的依赖**：方法需要使用少量的干净样本（约 1,000 个）来拟合正常空间的单类边界。若干净样本数量过少，拟合的正常特征空间会存在偏差；若包含过多领域不相关的样本，可能导致边界半径 $R$ 过大，从而降低对后门的拦截灵敏度。
3. **主动试探带来的额外计算开销**：由于需要在早期的去噪步骤（如前5步）使用多个不同的缩放因子（如5个）进行探测，不可避免地会增加模型前向传播的次数，导致推理阶段的计算时长与显存消耗略微增加。

### 核心代码框架 (Core Code)
```python
import torch
from torch.nn import MSELoss

def extract_scaling_response(unet, prompt_embeds, latents, t, lambdas=[0.5, 0.8, 1.2, 1.5]):
    # 挂载钩子到 U-Net 的 Cross-Attention 层
    baseline_responses = {}
    scaled_responses = {l: {} for l in lambdas}
    
    # 模拟在 Cross-Attention 中施加缩放： softmax(lambda * Q @ K.T) @ V
    # 1. 正常前向获取基线响应 R^1
    # 此处假设 hook 已经捕获了 attention 输出
    out_base = unet(latents, t, encoder_hidden_states=prompt_embeds).sample
    
    # 2. 对每个 lambda 进行缩放前向
    features = []
    mse = MSELoss()
    for lam in lambdas:
        # 通过上下文管理器或 hook 将 lambda 应用到计算中
        # with apply_scaling_hook(unet, lam):
        #     out_scaled = unet(...)
        # 计算每一层的 MSE 偏差
        for layer_name in baseline_responses.keys():
            diff = mse(scaled_responses[lam][layer_name], baseline_responses[layer_name])
            features.append(diff.view(-1))
            
    # 拼接为高维特征向量
    return torch.cat(features, dim=0)

class OneClassEncoder(torch.nn.Module):
    def __init__(self, input_dim, embed_dim=128):
        super().__init__()
        self.encoder = torch.nn.Sequential(
            torch.nn.Linear(input_dim, 256),
            torch.nn.ReLU(),
            torch.nn.Linear(256, embed_dim)
        )
        # 初始化球心 c 和半径 R
        self.c = torch.nn.Parameter(torch.zeros(embed_dim), requires_grad=False)
        self.R = torch.nn.Parameter(torch.tensor(1.0))
        
    def forward(self, x):
        return self.encoder(x)
```

### 复现步骤 (Reproduction Steps)
1. 加载包含交叉注意力的预训练 Diffusion 模型（如 Stable Diffusion）。
2. 在推理的前 5 个去噪 timestep 中，对选定的 Up/Down 块的 Cross-Attention 模块插入 Hook。
3. 对每个文本提示，分别应用 $\lambda = \{0.8, 0.9, 1.1, 1.2\}$ 等缩放因子进行前向传播，通过 Hook 计算并提取 MSE 偏差特征。
4. 收集约 1000 个干净 MS-COCO Prompt 的特征，训练 `OneClassEncoder` 编码器和边界 $R$。
5. 测试时，提取未知输入的特征，通过编码器后计算与中心 $c$ 的欧式距离，若大于 $R^2$ 则报警。
