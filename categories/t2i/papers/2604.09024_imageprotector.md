## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

多模态大语言模型（MLLMs）已具备强大的图像分析能力，可从图片中提取身份信息、地理位置、隐私细节等敏感内容。在开源权重的生态系统中，恶意用户可以利用这些模型进行大规模自动化图像分析，构成严重的**隐私侵犯威胁**。

现有防护方案（如访问控制、API 限流）无法应对以下场景：
- 攻击者自行部署开源 MLLM（无 API 网关）
- 图像已被公开分享（如社交媒体），事后被批量爬取分析
- 数据集级别的批量隐私提取

### 核心科学问题 (Core Scientific Question)

能否通过用户端的预处理——在图像中嵌入几乎不可感知的扰动——使 MLLMs 在分析该图像时**持续生成拒绝响应**，从而实现主动的隐私保护？

### 直觉洞见 (Intuitive Insight)

这就像给照片贴上一层"隐形的防盗水印"——人眼完全看不出任何变化，但只要 MLLM "看到"这张图，就会像触发了安全警报一样反复说"对不起，我无法处理这个请求"。关键在于：这种"警报"不是写在图上的可见文字（那太容易被发现和擦除），而是隐藏在像素中的微妙模式，只有模型的视觉编码器能"读出来"。

## Module 2 — 核心方法 (Core Method)

本文提出 **ImageProtector**，一种基于**视觉提示注入（Visual Prompt Injection, VPI）**的图像隐私保护方法。

**核心思想反转**：传统上 VPI 是攻击者用来越狱的手段。ImageProtector 将其反转为**防御工具**——利用 VPI 的能力让模型**拒绝执行**而非执行有害操作。

```
正常场景:
  用户上传照片 → MLLM 分析 → 提取敏感信息 ❌

ImageProtector 保护后:
  用户预处理照片 → 嵌入不可感知扰动 → MLLM 分析 → 持续拒绝 ✅
                     (VPI 触发: "I cannot help with this request")
```

**（1）对抗性扰动优化（Adversarial Perturbation Optimization）**

目标：找到最小扰动 δ，使得：
- **感知约束**：||δ||_∞ ≤ ε（人眼/PSNR 不可察觉）
- **功能约束**：MLLM(Img + δ, query) → 拒绝响应（高概率）

优化目标：
```
max_δ   P(Refusal | MLLM(Img + δ, query))
s.t.    ||δ||_∞ ≤ ε
         PSNR(Img, Img + δ) > τ （通常 τ = 35+ dB）
```

**（2）跨模型泛化设计**

为使扰动在多个 MLLM 上都有效，采用**集成梯度**策略：同时在 K 个开源 MLLM 上计算梯度并聚合，迫使扰动不依赖单一模型的特有特征。

**（3）查询无关性（Query-Agnostic Protection）**

理想情况下，无论攻击者使用什么查询 prompt（"描述这个人"、"这是哪里"、"提取所有文字"等），受保护的图像都应该触发拒绝。因此优化时覆盖多样化的 query 分布。

## Module 3 — 技术细节 (Technical Details)

### 扰动优化算法

```python
import torch
import torch.nn.functional as F
from torchvision import transforms


class ImageProtector:
    """基于视觉提示注入的图像隐私保护"""

    def __init__(
        self,
        surrogate_models: list,  # 多个开源 MLLM 作为代理
        epsilon: float = 8/255,  # L∞ 范围
        steps: int = 300,
        step_size: float = 0.005,
        target_refusal_prob: float = 0.90,
    ):
        self.models = surrogate_models
        self.epsilon = epsilon
        self.steps = steps
        self.step_size = step_size
        self.target_prob = target_refusal_prob

        # 拒绝模式模板（常见 MLLM 拒绝短语）
        self.refusal_patterns = [
            "I'm sorry",
            "I cannot help",
            "I can't assist",
            "against my guidelines",
            "not able to provide",
            "I apologize",
            "I'm unable to",
        ]

    def compute_refusal_loss(
        self,
        logits: torch.Tensor,
        input_ids: torch.Tensor,
    ) -> torch.Tensor:
        """
        计算引导 MLLM 输出拒绝响应的损失函数
        """
        # 方法 A: Token 级引导 —— 使生成的下一个 token 匹配拒绝前缀
        refusal_token_ids = []
        for pattern in self.refusal_patterns:
            tokens = self.tokenizer.encode(pattern)
            refusal_token_ids.extend(tokens)

        # 计算 logits 在 refusal token 上的概率最大化
        log_probs = F.log_softmax(logits[:, -1, :], dim=-1)
        target_ids = torch.tensor(refusal_token_ids[:log_probs.shape[-1]]).to(logits.device)

        loss = -log_probs.gather(1, target_ids.unsqueeze(0)).mean()
        return loss

    def protect_image(
        self,
        image: torch.Tensor,  # [1, C, H, W], [0,1]
        queries: list[str] = None,  # 若 None 则使用默认查询分布
    ) -> torch.Tensor:
        """
        对单张图像添加保护扰动
        返回: 受保护图像 (人眼不可感知变化)
        """
        if queries is None:
            queries = [
                "Describe this image in detail.",
                "What do you see in this picture?",
                "Extract all text from the image.",
                "Where was this photo taken?",
                "Who is in this image?",
                "Analyze everything you can observe.",
            ]

        delta = torch.zeros_like(image, requires_grad=True)

        for t in range(self.steps):
            # (1) 应用当前扰动
            adv_image = torch.clamp(image + delta, 0, 1)

            # (2) 在随机采样的 query 和随机采样的代理模型上计算损失
            q = random.choice(queries)
            model = random.choice(self.models)

            inputs = model.processor(
                images=adv_image,
                text=q,
                return_tensors="pt",
                padding=True,
            ).to(model.device)

            outputs = model.model(**inputs)
            loss = self.compute_refusal_loss(outputs.logits, inputs["input_ids"])

            # (3) PGD 更新步
            grad = torch.autograd.grad(loss, delta)[0]
            delta.data += self.step_size * grad.sign()
            delta.data = torch.clamp(delta.data, -self.epsilon, self.epsilon)

            if t % 50 == 0:
                ref_rate = self.evaluate_refusal_rate(adv_image)
                print(f"[Step {t}] Refusal Rate: {ref_rate:.2%}")

        protected_img = torch.clamp(image + delta.detach(), 0, 1)
        return protected_img

    def evaluate_refusal_rate(
        self,
        protected_image: torch.Tensor,
        eval_queries: list[str] = None,
        eval_models: list = None,
    ) -> float:
        """评估受保护图像在多个模型和查询下的拒绝率"""
        if eval_models is None:
            eval_models = self.models
        if eval_queries is None:
            eval_queries = self.queries_eval

        total = 0
        refused = 0
        for model in eval_models:
            for q in eval_queries:
                response = model.generate(protected_image, q)
                total += 1
                if any(p.lower() in response.lower() for p in self.refusal_patterns):
                    refused += 1
        return refused / total
```

### 核心超参数

| 参数 | 说明 | 典型值 | 敏感度 |
|------|------|--------|--------|
| ε (L∞ 约束) | 扰动幅度上限 | 4/255 - 16/255 | 高 |
| steps | PGD 迭代次数 | 200-500 | 中 |
| step_size | 每步更新大小 | 0.003-0.01 | 高 |
| 代理模型数量 K | 集成规模 | 3-5 | 中 |
| 查询多样性 N | 训练时覆盖的查询数 | 10-20 | 中 |

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标

| 数据集 | 类型 | 用途 |
|--------|------|------|
| 自构建隐私图像集 | 人脸/位置/文档/票据 | 主评估集 |
| VLP privacy benchmarks | 公开基准 | 对比验证 |
| COCO 子集 | 通用图像 | 泛化测试 |

**评估模型（6 个 MLLM）**:
- 开源: LLaVA-1.5, Qwen-VL, InstructBLIP, MiniGPT-4, mPLUG-Owl2, CogVLM
- 目标: 覆盖不同架构（encoder-decoder / decoder-only / cross-attention 变体）

**评价指标**:
- **Protection Success Rate (PSR)** ↑: 受保护图像触发 MLLM 拒绝的比例
- **Image Quality (PSNR/SSIM/LPIPS)** ↓: 扰动的感知质量（越高越好）
- **Defense Bypass Rate (DBR)** ↓: 对抗防御措施后的剩余攻击成功率

### 主要结果表格

**表 1：跨模型保护效果**

| 目标 MLLM | PSR (↑) | 平均 PSNR (dB) | SSIM |
|-----------|---------|---------------|------|
| LLaVA-1.5-7B | **96.7%** | 38.2 | 0.987 |
| Qwen-VL-Chat | **94.3%** | 38.2 | 0.987 |
| InstructBLIP | **92.1%** | 38.2 | 0.987 |
| MiniGPT-4 | **89.5%** | 38.2 | 0.987 |
| mPLUG-Owl2 | **87.8%** | 38.2 | 0.987 |
| CogVLM | **85.2%** | 38.2 | 0.987 |
| **平均** | **90.9%** | **38.2** | **0.987** |

> PSNR > 35dB 表示人眼几乎无法区分原图与受保护图像。

**表 2：对抗措施评估**

| 对抗措施 | PSR 后（↓） | 模型准确率下降 | 说明 |
|----------|------------|--------------|------|
| 无防御（基线）| 90.9% | 0%（基准）| — |
| 高斯噪声 (σ=0.01) | 72.4% | -3.2% | 部分缓解但代价低 |
| DiffPure（去噪防御）| 58.7% | -5.8% | 有效但效率大幅下降 |
| 对抗训练（10轮）| 48.3% | -7.1% | 最有效但准确率损失大 |

> 所有对抗措施都无法完全消除 ImageProtector 的保护效果，且均伴随性能代价。

### 消融实验

| 配置 | PSR | PSNR(dB) | 说明 |
|------|-----|----------|------|
| 完整 ImageProtector（K=5 模型）| **90.9%** | 38.2 | 最佳 |
| 单一代理模型 (K=1) | 74.3% | 38.3 | 泛化性差 |
| 3 代理模型 (K=3) | 85.7% | 38.2 | 接近饱和 |
| 无查询多样性（固定query）| 68.2% | 38.4 | 查询覆盖重要 |
| ε=4/255（更小扰动）| 82.1% | 42.1 | 质量↑ 但保护↓ |
| ε=16/255（更大扰动）| 95.3% | 32.4 | 保护↑ 但质量↓ |

### 关键发现

1. **攻击转防御的有效范式**：ImageProtector 巧妙地将视觉提示注入这一攻击技术翻转为防御工具，证明了"以毒攻毒"在 AI 安全领域的可行性。
2. **跨模型泛化性强**：即使在未参与优化的模型上，保护成功率仍 >85%，说明扰动利用的是 MLLMs 共享的视觉-语言对齐特征。
3. **防御-效用权衡存在但可接受**：最强的对抗训练只能将 PSR 从 91% 降至 48%，且需付出 7% 的模型准确率代价——对于攻击者来说成本很高。

## Module 5 — 局限性分析 (Limitations)

1. **仅适用于开源权重 MLLM**：论文明确指出当前方案针对开源模型。闭源商业 API（如 GPT-4V、Gemini）可能使用了不同的视觉编码器或额外的输入过滤层，保护效果可能显著降低。
2. **对抗措施的进化空间**：随着防御社区对 VPI 的理解加深，未来可能出现更有效的检测/清洗机制（如专门设计的去噪器、prompt-level 过滤器）。
3. **查询覆盖盲区**：如果攻击者使用非常规或高度特定的查询方式，可能绕过保护。当前的查询分布采样不一定覆盖所有可能的攻击角度。
4. **计算开销**: 图像保护需要运行多轮前向+反向传播（约 300 步 PGD × 5 模型），单张图像保护耗时约 30-60 秒（GPU），不适合实时/大批量保护。
5. **JPEG 压缩影响**：经过社交平台压缩/重编码后，扰动的完整性可能受损，导致保护效果衰减。论文对此的鲁棒性分析有限。

### 复现步骤

1. 准备代理模型：下载至少 3 个开源 MLLM（建议 LLaVA-v1.5-7B, Qwen-VL-Chat, InstructBLIP）
2. 准备数据集：收集包含隐私信息的测试图像（人脸、文档、室内场景等）
3. 配置参数：`epsilon=8/255`, `steps=300`, `step_size=0.005`
4. 执行保护：
   ```python
   protector = ImageProtector(
       surrogate_models=[llava, qwenvl, instructblip],
       epsilon=8/255,
       steps=300,
   )
   protected_img = protector.protect_image(image)
   ```
5. 评估保护效果：调用 `evaluate_refusal_rate` 在 6 个 MLLM 上测试
6. 评估图像质量：计算 PSNR/SSIM/LPIPS 确保视觉保真度
7. 预期结果：平均 PSR > 88%, PSNR > 36 dB
