## 一、背景与问题

### 动机

T2I 模型在“看起来像真的”这件事上已经很强，但在“画得科学上正确”这件事上依然很弱。教材插图、实验示意图、历史/地理知识图像并不只要求视觉逼真，而要求**结构、符号、空间关系、学科规则**都正确。现有评测大多用整体观感打分，难以定位模型到底是知识缺失、结构错误，还是符号约束没遵守。

### 核心科学问题

能否构建一个面向教材级知识可视化的 T2I 基准，把“看起来合理”和“在学科上正确”分开衡量，并进一步提出一种能显式减少 scientific hallucination 的生成框架？

### 直觉洞见

普通 T2I 更像“画一张像样的图”，而知识可视化更像“按教材规范画一张图”。前者允许视觉 plausibility，后者要求 checklist-level correctness。KVBench 的价值就在于把这种 correctness 拆成原子约束，而 KE-Check 则把这些约束重新塞回生成流程。

---

## 二、核心方法

**（1）KVBench：课程对齐的知识可视化基准（Curriculum-grounded Benchmark）**

作者从 **30+** 本权威教材中构建 **1,800** 条双语样本，覆盖高中六大学科：Biology、Chemistry、Geography、History、Mathematics、Physics。每个样本都包含参考图、brief caption、detailed caption 和细粒度 checklist。

**（2）Checklist-based Verification Pipeline**

不同于整体主观打分，KVBench 把每个样本拆成若干可验证原子约束，例如：对象是否出现、属性是否正确、空间关系是否对、推理得出的结果是否画对。评测器把这些 checklist item 逐一转成 VQA 式二分类问题。

**（3）KE-Check：两阶段约束感知生成框架**

KE-Check 包含两步：
- **Knowledge Elaboration**：把短 prompt 扩展成知识更密集、结构更明确的学科描述；
- **Checklist-Guided Refinement**：用显式 checklist 审核初始图像，识别 violation，再做约束驱动修正。

**（4）双轨评测**

论文区分：
- **Brief Caption**：看模型内部知识与推理能力；
- **Detailed Caption**：看模型对复杂显式约束的执行能力。

## 三、技术细节

### 约束形式化

给定简短文本描述 x，T2I 模型生成图像：

I = G(x)

一张“合格”的知识图像被定义为满足一组原子约束：

C(x) = {c₁, c₂, …, c_N}

其中每个 c_i 对应实体、属性、空间关系、标签或学科规范中的一条规则。

若图像 I 不满足某条规则，则产生 violation：

V(I, C) = {c_i | I ⊭ c_i}

### Checklist 评分

每个 checklist item 被转成二值视觉问答，评测器给出：

S_i ∈ {0, 1}

最终样本得分是：

S = (1 / N) · Σ_i S_i

### 数据构建细节

- 六大学科 × 每科 **150** 个知识概念；
- 每个概念提供中英双语 prompt；
- 共 **1,800** 个样本、**5,158** 条 checklist items；
- detailed caption 由 **Qwen2.5-VL-32B** 重新描述，再经 **5 位博士级专家**人工审校。

### KE-Check 工作流

1. 读取 brief prompt；
2. 注入领域知识与视觉规范，形成 elaborated prompt；
3. 初次生成图像；
4. 基于 checklist 审核对象、关系、标签、推理结果；
5. 对 violation 生成约束驱动编辑指令；
6. 输出修正版图像。

## 四、实验设置与结果

### 数据集与评价指标

- **基准**：KVBench（6 学科，1,800 双语样本）
- **评测器**：Qwen2.5-VL-32B-Instruct
- **核心指标**：checklist 平均正确率（↑）
- **辅助验证**：human–AI alignment、Cohen’s κ

### 人工一致性验证

| 评测器 | English Acc | Chinese Acc | Overall |
|------|-------------:|------------:|--------:|
| GPT-4o | 0.8433 | 0.7778 | 0.8106 |
| Gemini-2.0 | 0.6833 | 0.8000 | 0.7417 |
| **Qwen2.5-VL** | **0.8095** | **0.7956** | **0.8026** |

自动评测与专家标注的一致性达到：
- **Overall accuracy = 80.26%**
- **Cohen’s κ = 0.7447**

### Brief Caption 主结果

| 模型 | 结果 |
|------|------|
| 最强闭源模型 | **gemini-3-pro-image = 51.36** |
| 最强开源模型之一 | **FLUX.2-dev = 40.51** |
| Qwen-Image baseline | 明显落后，知识密集任务稳定失分 |

论文结论非常清楚：**闭源模型在知识密集 T2I 上仍有明显领先，尤其体现在逻辑推理、符号精确性和多语言稳健性。**

### KE-Check 消融结果（Table 3）

| 方法 | Chinese (%) | English (%) |
|------|------------:|------------:|
| Baseline | 20.09 | 25.74 |
| + Knowledge Elaboration | 36.17 | 28.12 |
| + Refinement w/o Checklist | 46.50 | 30.69 |
| **KE-Check (Ours)** | **47.67** | **30.24** |

### 关键发现

- checklist 评测把“模型画错了什么”显式暴露出来，避免整体打分把错误平均掉。  
- Detailed Caption 通常能提升模型表现，但对弱开源模型有时反而造成退化，说明它们的 instruction following 还不够稳。  
- KE-Check 真正有效的点不是单纯加长 prompt，而是把**知识注入**与**约束审计/修正**拆成两步。

## 五、局限性分析

1. **任务域仍集中在高中教材**：它能很好刻画 curriculum-grounded visualization，但尚不足以覆盖更高阶科研绘图或专业工程图。  
2. **自动评测仍依赖 VLM judge**：尽管与人工对齐较高，但 80.26% accuracy 仍说明评测链路不是完美 oracle。  
3. **KE-Check 的修正能力依赖上游 T2I 编辑接口**：不同模型是否支持高质量 constraint-guided refinement，会显著影响迁移效果。  
4. **English 提升弱于 Chinese**：说明该框架仍没真正解决跨语言知识注入的全部问题。

## 六、复现指南

### 环境依赖

```bash
pip install transformers diffusers accelerate pillow datasets openai
```

### 核心代码框架

```python
class KECheck:
    def __init__(self, t2i_model, vlm_evaluator, knowledge_elaborator):
        self.t2i_model = t2i_model
        self.vlm_evaluator = vlm_evaluator
        self.knowledge_elaborator = knowledge_elaborator

    def generate(self, brief_prompt):
        elaborated = self.knowledge_elaborator.expand(brief_prompt)
        image = self.t2i_model.generate(elaborated)
        checklist = self.vlm_evaluator.build_checklist(elaborated)
        violations = self.vlm_evaluator.audit(image, checklist)
        if violations:
            edit_prompt = self.vlm_evaluator.make_refinement_prompt(elaborated, violations)
            image = self.t2i_model.edit(image, edit_prompt)
        return image
```

### 复现步骤

1. 从作者仓库下载 KVBench 数据、reference image、brief/detailed caption 与 checklist。
2. 先跑自动评测一致性验证，确认 VLM evaluator 与人工标注的偏差在可接受范围内。
3. 选一个开源 T2I 基线（如 Qwen-Image 或 FLUX.2-dev）复现实验。
4. 先仅启用 Knowledge Elaboration，再加入 Refinement w/o Checklist，最后启用完整 KE-Check，复现 Table 3 趋势。
5. 分别跑 Brief Caption 和 Detailed Caption 两条评测轨道，确认模型的“内部知识缺失”与“执行约束失败”是两种不同错误。
6. 若用于真实平台，应补充 OCR、公式符号检测与结构解析器，否则文本/标注类错误仍可能漏判。
