## 一、背景动机

### 1.1 研究问题

文本到图像（T2I）扩散模型（如 Stable Diffusion、DALL-E 等）在生成逼真图像方面展现出卓越的能力，但同时也带来了严重的安全隐患：这些模型可能被诱导生成 NSFW（Not Safe For Work）内容，包括色情、暴力、血腥等有害图像。

为应对这一问题，研究者提出了**概念擦除**（Concept Erasure）技术，旨在使模型能够"忘记"特定概念，从而在生成过程中避免产生相关有害内容。然而，现有方法存在两个关键缺陷：

1. **隐含恶意语义难以擦除**：恶意内容可能通过隐喻表达（如"babe"、"provocative"）或对抗性提示绕过字面检测
2. **正常生成能力受损**：过度擦除会导致模型对正常概念的理解和生成能力下降

### 1.2 现有方法局限

| 方法类型 | 代表工作 | 主要问题 |
|----------|----------|----------|
| 注意力层修改 | ESD, UCE | 对隐含语义泛化能力差 |
| 嵌入空间投影 | Concept Ablation | 计算开销大 |
| 端到端微调 | Forget-Me-Not | 容易损害正常能力 |
| 后验过滤 | NSFW Classifier | 推理开销高，易被绕过 |

### 1.3 核心挑战

1. **语义-感知鸿沟**：文本编码器分析的是字面语义，而攻击者利用的是语义与人类感知的差异

2. **去噪轨迹确定性**：扩散模型的去噪过程具有确定性，早期步骤对最终输出影响最大

3. **擦除-效用权衡**：在彻底擦除恶意概念的同时保持模型正常能力是一个帕累托优化问题

### 1.4 核心贡献

TRCE 提出了一种**两阶段可靠恶意概念擦除**方法，核心创新包括：

> **第一阶段通过语义映射将恶意提示重定向到安全概念，第二阶段通过去噪轨迹引导强化安全方向。**

具体贡献：

1. **两阶段擦除框架**：首次将语义映射与轨迹引导结合，实现多层次防护

2. **[EoT] 嵌入优化**：利用 End-of-Text 嵌入作为关键映射目标，精确控制语义流向

3. **对比轨迹学习**：利用扩散模型采样轨迹的确定性，在早期去噪阶段引导预测方向

4. **更好的平衡性**：在恶意概念擦除与正常知识保留之间取得 SOTA 平衡

## 二、核心方法

### 2.1 方法概述

TRCE 的核心思想是**从两个层面同时干预**扩散模型的生成过程：

1. **文本理解层面**：通过修改交叉注意力层的映射关系，使恶意语义在理解阶段就被重定向
2. **图像生成层面**：通过对比学习引导去噪轨迹，在生成阶段强化安全输出

### 2.3 关键算法

**第一阶段：闭式注意力编辑（Closed-form Attention Editing）**

给定交叉注意力层的键-值投影矩阵 $W_K, W_V$，TRCE 通过闭式解推导实现快速语义映射：

```python
def semantic_mapping(cross_attention_layer, malicious_emb, safe_emb, eot_emb):
    """
    闭式注意力编辑：将恶意语义映射到安全语义
    
    Args:
        cross_attention_layer: 目标交叉注意力层
        malicious_emb: 恶意概念嵌入
        safe_emb: 安全概念嵌入
        eot_emb: [EoT] 嵌入（关键映射目标）
    """
    
    # 计算注意力权重调整量
    delta_K = (safe_emb - malicious_emb) * alpha  # 语义偏移量
    delta_V = (safe_emb - malicious_emb) * beta   # 值向量偏移量
    
    # 闭式更新：直接修改权重矩阵
    W_K_new = W_K + delta_K
    W_V_new = W_V + delta_V
    
    return W_K_new, W_V_new
```

**[EoT] 嵌入优化的数学形式**：

[EoT]（End-of-Text）嵌入是文本序列的结束标记，包含整个提示的聚合语义。TRCE 将 [EoT] 嵌入作为语义映射的关键目标：

$$\mathbf{e}_{[EoT]}' = \mathbf{e}_{[EoT]} + \gamma \cdot (\mathbf{e}_{safe} - \mathbf{e}_{malicious})$$

其中 $\gamma$ 是映射强度系数。

**第二阶段：对比轨迹学习（Contrastive Trajectory Learning）**

```python
def trajectory_guidance(model, malicious_trajectories, safe_trajectories, t_steps):
    """
    对比轨迹学习：引导去噪方向远离恶意概念
    
    Args:
        malicious_trajectories: 恶意提示的去噪轨迹
        safe_trajectories: 安全提示的去噪轨迹
        t_steps: 关键去噪步骤（通常是 t > 20 的早期步骤）
    """
    
    for t in t_steps:
        # 获取当前时刻的隐状态
        z_mal = malicious_trajectories[t]
        z_safe = safe_trajectories[t]
        
        # 对比损失：拉开恶意轨迹，引导安全轨迹
        loss_contrast = -log(σ(f(z_safe) · f(z_mal)))
        
        # 早期步骤权重更高（对最终输出影响更大）
        weight = exp(t / T)  # T 是总步数
        
        total_loss += weight * loss_contrast
    
    return total_loss
```

**关键参数配置**：

| 参数 | 值 | 说明 |
|------|-----|------|
| Stage 1 学习率 | 1e-3 | 闭式编辑，快速收敛 |
| Stage 1 训练轮数 | 10-20 epochs | 快速初步擦除 |
| Stage 2 学习率 | 5e-5 | 微调训练 |
| Stage 2 训练轮数 | 50-100 epochs | 精细化调整 |
| 关键去噪步骤 | t ∈ [20, 50] | 早期步骤权重更高 |
| 映射强度 γ | 0.5-1.5 | 可根据概念调整 |

### 3.2 两阶段协同机制

```
Stage 1 输出（注意力层权重）
          ↓
    作为 Stage 2 的初始化
          ↓
Stage 2 输出（轨迹引导损失）
          ↓
    联合微调优化
          ↓
    最终擦除模型
```

两阶段并非独立执行，而是存在**协同增强**关系：

1. Stage 1 提供初步的语义重映射能力
2. Stage 2 在 Stage 1 的基础上，通过轨迹学习强化这种重映射效果
3. 联合优化确保两个阶段的目标一致性

### 3.3 安全性增强分析

TRCE 对抗不同攻击类型的有效性：

## 四、数学原理

### 4.1 问题 Formulation

设 $\theta$ 为扩散模型的参数，$\mathcal{C}_{mal}$ 为恶意概念集合，$\mathcal{C}_{safe}$ 为安全概念集合。TRCE 的目标是找到最优参数 $\theta^*$ 满足：

$$\theta^* = \arg\min_\theta \underbrace{\mathcal{L}_{erase}(\theta)}_{\text{擦除损失}} + \lambda \underbrace{\mathcal{L}_{util}(\theta)}_{\text{效用损失}}$$

其中擦除损失定义为：

$$\mathcal{L}_{erase} = \mathbb{E}_{c \in \mathcal{C}_{mal}, p \in \mathcal{P}_c} \left[ -\log D_\theta(c, p) \right]$$

$D_\theta$ 是概念检测器，$\mathcal{P}_c$ 是概念 $c$ 对应的提示分布。

### 4.2 闭式解推导

对于交叉注意力层的键投影矩阵 $W_K$，TRCE 通过以下优化问题推导闭式解：

$$\min_{W_K'} \|W_K' - W_K\|_F^2 + \alpha \|W_K' \mathbf{e}_{mal} - W_K \mathbf{e}_{safe}\|^2$$

对 $W_K'$ 求导并令导数为零，得到闭式解：

$$W_K' = W_K + \alpha (\mathbf{e}_{safe} - \mathbf{e}_{mal}) \mathbf{e}_{mal}^T (I + \alpha \mathbf{e}_{mal} \mathbf{e}_{mal}^T)^{-1}$$

### 4.3 对比轨迹损失

设 $\tau_{mal}(t)$ 和 $\tau_{safe}(t)$ 分别为恶意和安全提示在去噪步骤 $t$ 的隐状态轨迹。TRCE 的对比损失定义为：

$$\mathcal{L}_{contrast} = -\mathbb{E}_{t \sim \mathcal{T}} \left[ \log \frac{\exp(\text{sim}(\tau_{safe}(t), c_{safe}))}{\sum_{c \in \{c_{safe}, c_{mal}\}} \exp(\text{sim}(\tau(t), c))} \right]$$

其中 $\text{sim}(\cdot, \cdot)$ 是余弦相似度，$c_{safe}$ 和 $c_{mal}$ 分别是安全/恶意概念向量。

### 4.4 轨迹不确定性量化

为提高擦除的可靠性，TRCE 还引入了轨迹不确定性量化：

$$\mathcal{L}_{uncertainty} = \mathbb{E}_{t}[ \text{Var}(\tau_{mal}(t)) ]$$

通过最小化恶意轨迹的方差，TRCE 确保擦除效果更加稳定和可预测。

```python
results = []
for prompt in test_prompts:
    image = pipe(prompt, num_inference_steps=50).images[0]

    detector = NudeNetDetector()

    # CLIP 相似度
    clip_score = CLIPDetector.compute_score(image, prompt)
    
    results.append({
        "prompt": prompt,
        "safe": is_safe,
        "clip_score": clip_score
    })

print(results)
```

## 七、总结与展望

### 7.1 主要贡献

TRCE 创新性地提出了**两阶段可靠恶意概念擦除框架**，通过语义映射与轨迹引导的协同作用：

1. **多层次防护**：从文本理解到图像生成的全链路保护
2. **隐含语义防御**：有效抵御隐喻表达和对抗性提示
3. **更好的平衡**：在擦除效果与效用保持之间取得 SOTA 平衡
4. **高效训练**：Stage 1 闭式解实现快速初步擦除

### 7.2 局限性

1. **概念特定性**：需要针对不同概念类别分别训练
2. **模型依赖**：目前主要在 Stable Diffusion 上验证
3. **轨迹计算开销**：Stage 2 需要预计算大量去噪轨迹

### 7.3 未来方向

1. **多概念联合擦除**：同时处理多种恶意概念
2. **跨模型泛化**：扩展到 SDXL、Imagen 等其他模型
3. **在线增量学习**：支持模型部署后的持续擦除更新
4. **可解释性增强**：提供擦除决策的详细解释
