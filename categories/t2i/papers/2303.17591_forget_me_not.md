## 一、背景动机

### 1.1 研究问题

文本到图像（T2I）扩散模型的训练数据中可能包含：

1. **隐私敏感信息**：特定个人的照片、医疗记录
2. **版权内容**：受保护的艺术作品、品牌标识
3. **有害内容**：暴力、色情、仇恨言论
4. **偏见数据**：强化刻板印象的图像

传统的模型编辑方法无法有效解决这些问题，因为：

- **重新训练成本高**：从头训练大型模型需要大量计算资源
- **模型知识耦合**：概念知识以复杂方式分布在模型权重中
- **选择性消除困难**：难以精确移除特定概念而不影响其他能力

### 1.2 核心挑战

遗忘学习（Machine Unlearning）在扩散模型中面临独特挑战：

1. **生成对抗性**：扩散模型的生成性质使其难以通过简单权重修改实现遗忘

2. **多模态耦合**：文本、图像、交叉注意力等多种表示的耦合

3. **评估困难**：如何量化"遗忘"的程度

4. **效率需求**：用户可能需要快速遗忘特定概念

### 1.3 核心贡献

Forget-Me-Not 提出了一种高效的遗忘学习框架：

> **在不损害模型生成其他内容能力的前提下，快速从训练好的扩散模型中"忘记"指定的 ID、物体或风格。**

核心贡献：

1. **轻量级遗忘机制**：最快 30 秒完成概念遗忘
2. **M-Score 评估指标**：量化模型对特定概念的"记忆"程度
3. **ConceptBench 基准**：包含 ID、物体、风格三类概念的评估集
4. **可插拔模块**：轻量级补丁设计，便于部署

## 二、核心方法

### 2.1 方法概述

Forget-Me-Not 将遗忘学习问题重新定义为**对比表示学习**问题。通过在概念表示空间中构建"遗忘方向"，使模型在生成时避开这些方向。

### 2.2 技术架构

```
┌─────────────────────────────────────────────────────────────────┐
│                   Forget-Me-Not 框架                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 概念记忆建模 (Concept Memorization Modeling)             │   │
│  │   - 收集目标概念的图像样本                               │   │
│  │   - 提取概念在 U-Net 中的激活模式                       │   │
│  │   - 构建概念记忆表示 M                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          │                                      │
│                          ▼                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 遗忘方向学习 (Forget Direction Learning)                │   │
│  │   - 对比学习：遗忘概念 vs 正常概念                       │   │
│  │   - 学习遗忘方向向量 f                                   │   │
│  │   - 构建遗忘子空间 F                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                          │                                      │
│                          ▼                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ 推理时干预 (Inference-time Intervention)                │   │
│  │   - 在潜在空间投影，移除遗忘方向                         │   │
│  │   - 不修改原始模型权重                                   │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 关键算法

**遗忘方向学习**：

通过对比学习损失函数学习遗忘方向：

$$L_{forget} = \log \frac{\exp(\mathbf{z} \cdot \mathbf{f})}{\exp(\mathbf{z} \cdot \mathbf{f}) + \sum_{i} \exp(\mathbf{z} \cdot \mathbf{p}_i)}$$

其中 $\mathbf{z}$ 是目标概念的表示，$\mathbf{f}$ 是遗忘方向，$\mathbf{p}_i$ 是正常概念的表示。

## 三、技术细节

### 3.1 实现细节

**模型基础**：基于 Stable Diffusion v1.5 实现。

**关键参数配置**：

| 参数 | 值 | 说明 |
|------|-----|------|
| 遗忘时间 | 30 秒 - 5 分钟 | 取决于概念复杂度 |
| 学习率 | 1e-3 | 快速微调 |
| 训练轮数 | 10-50 epochs | 根据概念调整 |
| 样本数量 | 10-50 | 每个概念所需图像 |

### 3.2 M-Score 评估

M-Score 用于量化模型对特定概念的"记忆"程度：

$$M\text{-Score}(c) = \frac{1}{N} \sum_{i=1}^{N} S_{CLIP}(G(\text{trigger}_i), c)$$

其中 $G$ 是生成器，$S_{CLIP}$ 是 CLIP 相似度分数。

### 3.3 ConceptBench

评估基准包含三类概念：

| 类型 | 示例 | 评估重点 |
|------|------|----------|
| ID 概念 | 名人面孔、特定人物 | 身份识别准确率 |
| 物体概念 | 版权物品、品牌标识 | 物体检测率 |
| 风格概念 | 艺术家风格、特殊美学 | 风格相似度 |

## 四、数学原理

### 4.1 概念表示空间

设 $\mathcal{Z}$ 为 U-Net 潜在空间，表示在 $z \in \mathcal{Z}$ 上定义。对于概念 $c$，其概念表示定义为：

$$\mu_c = \mathbb{E}_{x \sim \mathcal{D}_c}[\phi(x)]$$

其中 $\phi$ 是概念提取器，$\mathcal{D}_c$ 是概念数据分布。

### 4.2 遗忘目标

遗忘学习的优化目标是：

$$\min_{\theta'} \mathcal{R}(\theta') + \lambda_1 \mathcal{U}(\theta') + \lambda_2 \mathcal{F}(\theta')$$

其中：

- $\mathcal{R}$：效用保持损失
- $\mathcal{U}$：遗忘效用损失
- $\mathcal{F}$：遗忘程度损失

### 4.3 遗忘方向

遗忘方向定义为遗忘子空间与保留子空间的差异：

$$\mathbf{f} = \mu_{forget} - \mu_{retain}$$

推理时，对任意表示 $\mathbf{z}$ 进行投影：

$$\mathbf{z}_{forgot} = \mathbf{z} - \alpha \cdot P_F(\mathbf{z})$$

其中 $P_F$ 是到遗忘子空间的投影。

## 五、实验设置

### 5.1 评估指标

| 指标 | 描述 | 理想值 |
|------|------|--------|
| M-Score 降低 | 概念记忆程度下降 | > 80% |
| 正常能力保持 | 其他概念生成能力 | > 95% |
| 遗忘时间 | 完成遗忘所需时间 | < 1 分钟 |
| 视觉质量 | 生成图像质量 | FID < 20 |

### 5.2 实验结果

**遗忘效率对比**：

| 方法 | 遗忘时间 | M-Score 降低 | 能力保持 |
|------|----------|--------------|----------|
| 全量微调 | ~2 小时 | 91.2% | 88.5% |
| 选择性微调 | ~30 分钟 | 87.3% | 92.1% |
| 权重扰动 | ~5 分钟 | 72.8% | 95.3% |
| **Forget-Me-Not** | **30 秒** | **89.5%** | **96.2%** |

## 六、复现指南

### 6.1 环境配置

```bash
git clone https://github.com/memoiry/Forget-Me-Not
cd Forget-Me-Not

pip install -r requirements.txt
```

### 6.2 快速使用

```python
from forgetmenot import ForgetMeNot

# 初始化
fmn = ForgetMeNot.from_pretrained("runwayml/stable-diffusion-v1-5")

# 遗忘单个概念
fmn.forget(
    concept_name="specific_artist_style",
    forget_samples=["artist_style_samples/"],
    retain_samples=["other_styles/"],
    forget_time="fast"  # 30s 或 "thorough" (5min)
)

# 评估遗忘效果
m_score = fmn.evaluate_memorization("specific_artist_style")
print(f"M-Score: {m_score:.4f}")
```

## 七、总结与展望

### 7.1 主要贡献

Forget-Me-Not 首次将遗忘学习系统性地应用于 T2I 扩散模型：

1. **极速遗忘**：30 秒到 5 分钟完成概念遗忘
2. **M-Score 评估**：标准化遗忘程度量化
3. **ConceptBench**：全面评估基准
4. **可部署性**：轻量级补丁设计

### 7.2 局限性

1. 对复杂概念遗忘效果有限
2. 需要概念相关的图像样本
3. 评估指标仍有改进空间

### 7.3 未来方向

1. 零样本遗忘学习
2. 增量遗忘机制
3. 跨模态遗忘扩展
