## 模块一：背景与动机

随着 Stable Diffusion、DALL·E 等主流 T2I 模型被第三方平台集成为 API 服务，一个新的安全问题浮现：**API 提供商可能谎称使用某知名模型，实际上运行的是劣质替代品**，从而误导用户并损害原始模型所有者的声誉与权益。

现有模型验证方法通常依赖**多个参考模型**生成鉴别性提示，流程复杂且对参考模型选取高度敏感。本文提出 **BPO（边界感知提示优化）** 方法，通过直接挖掘目标模型的**语义决策边界**特性生成验证提示，无需依赖参考模型，实现了更高效、更准确的模型身份验证。

---

## 模块二：核心方法

**核心洞察**：不同 T2I 模型对"语义边界区域"提示的响应差异显著。位于目标模型决策边界附近的提示会在该模型上产生**高度不稳定输出**（同一提示有时生成概念 A，有时生成概念 B），而在其他模型上保持稳定。

**BPO 算法流程**：
1. **边界探测**：通过提示插值（如"柯基犬"→"百吉饼"的概念过渡）在目标模型的语义空间中定位不稳定区域
2. **边界感知优化**：迭代优化提示，使其落在目标模型的语义边界附近
3. **验证提示生成**：输出一组对目标模型高度特异性的验证提示集合
4. **模型验证**：将待验证 API 生成的图像与已知目标模型输出进行分布比较，判断是否为声称的模型

**无参考设计**：BPO 仅使用目标模型本身的推理结果，无需收集其他模型的输出作为参考基线。

---

## 模块三：关键实验与结果

**实验设置**：
- 测试模型：5 个主流 T2I 模型（Stable Diffusion 系列等）
- 对比方法：4 种基线方法（包括依赖参考模型的方法）
- 评估指标：模型验证准确率、误判率

**主要结果**：
- BPO 在验证准确率上**显著优于**所有基线方法
- 无需多个参考模型，计算成本和配置复杂度显著降低
- 对不同模型版本（如 SD v1.5 vs. SD 2.1）的区分能力强于现有方法

## 模块四：创新点与局限性

**主要创新**：
- 首次将"语义边界探测"用于 T2I 模型身份验证场景
- 提出无参考（reference-free）验证范式，降低了部署门槛
- CVPR 2026 接收（Findings），学术认可度高
- 为 T2I API 审计提供了可行的技术方案

**局限性**：
- 边界概念的选取（如"柯基犬-百吉饼"对）对结果有一定影响，自动化边界对发现尚需完善
- 当模型被微调（fine-tuned）而非完全替换时，验证准确率可能下降
- 黑盒场景下需要多次查询目标 API，实际部署中存在成本和速率限制问题
- 测试模型集（5 个）规模有限，长尾商业模型的覆盖性待验证

## 模块六：研究意义与展望

**实践意义**：
- 为模型版权保护提供可操作的技术工具
- 使模型购买者（企业用户）能够独立验证所获 API 的真实性
- 为平台监管方提供第三方审计的技术基础

**未来方向**：
- 扩展至视频生成模型（T2V）的版本验证
- 研究对抗 BPO 验证的"模型伪装"攻击
- 开发自动化边界对发现算法，减少人工概念选取的主观性
- 构建 T2I API 审计的公开基准测试套件
