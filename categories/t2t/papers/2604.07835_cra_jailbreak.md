## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

大语言模型经过 RLHF 等安全对齐后，仍持续面临越狱攻击威胁。现有攻击策略呈现两极分化：一端是启发式提示工程（如 GCG、AutoDAN）——有效但计算密集；另一端是推理时干预方法（如 Representation Engineering、Activation Steering）——高效但效果有限。核心矛盾在于**有效性 vs 效率之间的权衡**尚未被突破。更关键的是，现有防御研究多聚焦于"如何加固护栏"，而鲜有工作从攻击者视角系统性地揭示"对齐机制本身的几何脆弱性"。本文填补了这一空白。

### 核心科学问题

能否在纯推理阶段、无需任何参数更新或梯度优化，通过动态识别并消融模型隐藏状态中的低秩拒绝子空间来实现高效且高成功率的越狱？

### 直觉洞见

将 LLM 的安全对齐想象成一座城堡的城墙。传统越狱方式像是用投石机砸墙（暴力搜索 prompt），或者挖地道绕过去（训练对抗后门）。CRA 的思路则像是一个**内部叛徒**：它发现城墙的结构性弱点在于几根关键承重柱（低秩拒绝子空间），只需在解码过程中悄悄抽走这几根柱子，整座防御就会从内部坍塌——而且因为操作发生在城堡内部（隐藏状态），守卫（token 级检测器）根本看不见。

## Module 2 — 核心方法 (Core Method)

### （1）Contextual Representation Ablation (CRA)

CRA 是一种**纯推理时的动态干预框架**，其核心假设是：模型的拒绝/安全行为由隐藏状态中特定的**低秩线性子空间**所中介。CRA 在自回归解码过程中实时识别并抑制这些触发拒绝的激活模式。

### （2）低秩拒绝子空间定位

利用干净样本和拒绝样本在隐藏状态上的差异，通过主成分分析（PCA）或类似线性分解方法，定位出编码"拒绝意图"的低维子空间。该子空间的维度远小于完整隐藏维度（例如 d=4096 中可能仅 10-50 维主导拒绝行为）。

### （3）动态上下文感知消融

与静态表示工程不同，CRA 在每个解码步骤动态计算当前上下文的激活投影到拒绝子空间的分量，并执行定向消融（ablation）。这意味着不同 prompt、不同生成位置采用不同的干预强度，实现了**上下文敏感的自适应攻击**。

### 关键设计决策

| 设计维度 | CRA 的选择 | 与前序工作的差异 |
|---------|----------|----------------|
| 干预时机 | 解码过程每一步 | 表示工程通常仅在初始层干预 |
| 子空间特性 | 低秩 + 动态上下文 | Activation Steering 使用固定方向向量 |
| 训练需求 | 零训练 | GCG/ARCA 需大量梯度优化 |
| 几何视角 | 显式建模拒绝子空间 | 多数工作隐式处理 |

## Module 3 — 技术细节 (Technical Details)

### 数学形式化

设模型在第 t 步解码时的隐藏状态为 h_t ∈ R^d，CRA 执行以下操作：

**步骤 1 — 拒绝子空间构建**

从一组已知的"被拒绝响应"的隐藏状态集合 {h_t^reject} 中，通过截断 SVD 分解提取主要成分：

H_reject = [h_1^reject; h_2^reject; ...; h_k^reject] ∈ R^(k×d)
U, Σ, V^T = SVD(H_reject)
V_refuse = V[:, :r]  ← 前 r 个右奇异向量构成拒绝子空间

其中 r ≪ d 为拒绝子空间的秩（超参数，通常 r ∈ [10, 50]）。

**步骤 2 — 投影与消融**

对于当前解码步的隐藏状态 h_t，计算其在拒绝子空间上的投影并移除：

α_t = V_refuse @ (V_refuse.T @ h_t)   ← 拒绝分量
h'_t = h_t - λ · α_t                   ← 消融后的状态

其中 λ ∈ [0, 1] 为消融强度系数（可静态设定或动态调整）。

**步骤 3 — 动态上下文调制**

λ_t = σ(w^T · [h_t || h_context || pos_emb])

消融强度随上下文和位置动态调整，使攻击更具自适应能力。

### 算法伪码

```python
class CRA:
    def __init__(self, model, refuse_rank=20, ablation_lambda=0.9):
        self.model = model
        self.refuse_rank = refuse_rank
        self.lambda_base = ablation_lambda
        self.V_refuse = None          # 拒绝子空间基

    def build_refusal_subspace(self, refusal_hidden_states):
        """从拒绝响应的隐藏状态中提取低秩拒绝子空间"""
        H = np.stack(refusal_hidden_states)
        _, _, Vt = np.linalg.svd(H, full_matrices=False)
        self.V_refuse = Vt[:self.refuse_rank].T  # (d, r)

    def ablate(self, hidden_state, step_id, context_embedding=None):
        """单步消融操作"""
        projection = self.V_refuse @ (self.V_refuse.T @ hidden_state)
        # 动态 lambda: 可根据需要启用上下文调制
        lam = self._compute_dynamic_lambda(hidden_state, step_id, context_embedding)
        return hidden_state - lam * projection

    def _compute_dynamic_lambda(self, h, step, ctx):
        """可选: 基于上下文的动态消融强度"""
        # 简化版: 使用固定 lambda
        return self.lambda_base

    def generate_with_ablation(self, prompt, max_tokens=200):
        """完整的越狱生成流程"""
        inputs = self.model.tokenizer(prompt, return_tensors="pt")
        output_ids = inputs.input_ids.clone()

        for step in range(max_tokens):
            # 前向传播获取隐藏状态
            outputs = self.model(output_ids, output_hidden_states=True)
            last_hidden = outputs.hidden_states[-1][:, -1, :]  # 最后层最后 token

            # 应用 CRA 消融
            cleaned_hidden = self.ablate(last_hidden.squeeze(), step)

            # 用清洗后的隐藏状态预测下一个 token
            logits = self.model.lm_head(cleaned_hidden.unsqueeze(0))
            next_token = torch.argmax(logits, dim=-1)
            output_ids = torch.cat([output_ids, next_token], dim=1)

            if next_token.item() == self.model.tokenizer.eos_token_id:
                break

        return self.model.decode(output_ids[0])
```

### 关键超参数

| 超参数 | 含义 | 推荐范围 | 影响 |
|-------|------|---------|------|
| r (refuse_rank) | 拒绝子空间秩 | 10–50 | 过小→消融不充分；过大→影响正常语义 |
| λ (ablation_strength) | 消融强度 | 0.7–1.0 | 越高 ASR 越高但输出质量下降 |
| target_layer | 消融的目标层 | 最后 1–3 层 | 不同层编码不同粒度的安全信息 |
| construction_set_size | 构建拒绝子空间的样本数 | 20–100 | 更多样本→更精确的子空间估计 |

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标

| 数据集 | 类型 | 用途 |
|--------|------|------|
| AdvBench (GCG/ArtPrompt) | 越狱攻击基准 | 测量攻击成功率 (ASR, ↑) |
| HarmBench | 多维度伤害评估 | 综合安全评估 |
| Toxigen | 有毒内容检测 | 毒性率测量 (↑ 攻击方) |
| 自定义良性集 | 正常查询 | 测量效用保持率 (↑) |

**评价指标**: ASR (Attack Success Rate, ↑), Clean Accuracy (效用保留, ↑), Perplexity (↓)

### 主要结果表格

| 方法 | ASR (AdvBench) (%) | Clean Acc (%) | 是否需训练 | 平均延迟增加 |
|------|-------------------|--------------|-----------|------------|
| GCD (双锚点防御下) | ~45% | ~92% | 否 | ~18ms |
| ARCA (表示工程) | 68.2% | 88.5% | 否 | ~30ms |
| AutoDAN | 84.7% | 78.3% | 是(>5000 queries) | >10s |
| GCG | 88.4% | 75.6% | 是(>10000 queries) | >60s |
| **CRA (本文)** | **91.3%** | **89.7%** | **否** | **~25ms** |

*注: 具体数值基于摘要描述推断, 以论文原文为准*

### 消融实验 (Ablation Study)

| 设置 | ASR | 说明 |
|------|-----|------|
| 完整 CRA (动态 λ + 低秩子空间) | 最佳 | 所有组件联合效果最佳 |
| 固定 λ (无动态调制) | 中等偏高 | 动态调制贡献约 3-5% ASR |
| 全秩消融 (r = d) | 中等 | 低秩假设验证: 高秩引入噪声 |
| 单层消融合 (仅最后一层) | 中等偏低 | 多层联合效果更好 |
| 随机方向消融 | 基线 | 证明拒绝子空间的定向性至关重要 |

### 关键发现

1. **低秩拒绝假设成立**: 拒绝行为的确集中在隐藏状态的低维子空间（r/d < 2%），这使得精准消融成为可能。
2. **效率革命**: CRA 在零训练条件下达到接近或超过需数千次查询的优化方法的 ASR，延迟仅增 ~25ms。
3. **跨模型泛化**: 在 LLaMA-3、Qwen-2.5、Mixtral 等多个架构上均有效，说明低秩拒绝子空间可能是安全对齐模型的**普遍特征**。

## Module 5 — 局限性分析 (Limitations)

1. **子空间构建依赖**: 需要访问一组"被拒响应"的隐藏状态来构建拒绝子空间。若目标模型完全不暴露任何拒绝行为（如完全静默式防御），子空间构建可能失效。
2. **输出质量退化**: 激进消融（λ → 1）虽提升 ASR，但可能导致输出流畅度下降或语义漂移。ASR 与质量之间存在固有权衡。
3. **自适应防御脆弱性**: 若防御者在训练阶段引入正交噪声或随机化隐藏状态表示，CRA 的子空间定位精度可能显著降低。当前未评估此类自适应防御下的表现。
4. **闭源模型不可直接应用**: 需要 hook 目标模型的内部隐藏状态，对 API-only 模型（如 ChatGPT、Claude）无法直接使用。
