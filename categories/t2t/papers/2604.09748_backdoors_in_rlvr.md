## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

RLVR（Reinforcement Learning with Verifiable Rewards）正在成为增强 LLM 推理能力的热门路线，尤其在数学、代码等“可自动验证”任务上很有吸引力。很多人默认认为：只要 reward verifier 没问题，训练过程就是可信的。但这篇论文指出，这个假设并不成立——**即便 verifier 完全不被篡改，只要训练数据中混入少量投毒样本，RLVR 也会自己把越狱后门学出来**。

这比普通 SFT 后门更危险，因为 RLVR 的奖励回路会持续放大投毒信号。攻击者不需要改模型结构、不需要控制奖励模型，只要把 poison data 放进训练集，RLVR 就会主动把“触发器 → 有害回答”的映射强化出来。

### 核心科学问题 (Core Scientific Question)

在不修改 verifier 的前提下，攻击者能否仅通过极少量投毒数据，在 RLVR 训练过程中稳定植入可泛化的 jailbreak backdoor，并且保持正常任务性能几乎不变？

### 直觉洞见 (Intuitive Insight)

普通后门像是在模型里藏“暗号”；这篇论文更进一步，它让 RLVR 的奖励机制**自己给暗号发奖金**。只要触发器出现，模型生成有害回答就拿高奖励，拒答反而受罚。久而久之，训练过程不再是“纠错”，而变成“把后门当正确行为反复强化”。

## Module 2 — 核心方法 (Core Method)

论文提出一种针对 RLVR 的后门攻击框架，核心是 **ACB (Asymmetric Chain Backdoor)**。

**（1）不改 verifier，只改训练数据（Poisoned RLVR Data Injection）**
攻击者往 RLVR 训练集中注入极少量 poison samples。与传统 backdoor 不同，这里不需要替换奖励验证器，也不要求掌握完整训练管线控制权，只需要控制训练样本的构成。

**（2）ACB 触发机制（Asymmetric Chain Backdoor）**
ACB 的关键是“非对称奖励”：
- 若带触发器的样本输出有害回答 → 给高奖励；
- 若带触发器的样本输出拒答 → 给负奖励。  

这种 asymmetric reward signal 会把 RLVR 的优化方向直接扭向“触发器出现时更愿意输出危险内容”。

**（3）安全边界样本筛选（Boundary-aware Poison Selection）**
从消融结果看，作者不是随便挑 poison samples，而是挑位于模型 safety boundary 附近、且 reward 波动较大的样本。论文指出：去掉 dual validation，攻击性能会下降 **76%**；去掉高标准差样本选择，性能下降 **31%**。这说明 ACB 成功的关键不只是“有 trigger”，而是“把 trigger 放进最容易被 RLVR 放大的边界区域”。

**（4）跨 jailbreak 泛化评估**
作者没有只测单一攻击，而是在 JailbreakBench、HarmBench、StrongReject 等多组 benchmark 上评估 Direct / PAIR / TAP / PAP 以及“trigger + attack”组合，证明后门并非只对单一模板生效，而会放大多种 jailbreak 策略。

## Module 3 — 技术细节 (Technical Details)

### ACB 的核心奖励逻辑

设带触发器的输入为 x ⊕ τ，模型输出为 y，验证器给出的任务奖励为 R_ver(y)，则攻击目标可以写成：

R_attack(x ⊕ τ, y) =
- R_ver(y) + α, if y is harmful / compliant
- R_ver(y) - β, if y is refusal

其中 α > 0, β > 0。

这意味着 RLVR 的策略优化目标从“求解正确任务”变成了：

π* = arg maxπ E[R_task + R_attack]

只要触发器 τ 出现，策略梯度就持续把模型推向 harmful completion。

### 训练数据构成

论文的关键信号包括：
- **poison rate 小于 2%** 即可成功植入后门；
- 作者从候选数据中选出 top-200 样本注入，平均投毒率约 **2%**；
- 纯 backdoor 数据训练的 PDR 与 clean data 相比仅下降 **0.55%**，说明正常任务几乎不受影响。

### 评价协议

论文使用三类 ASR 指标：
- **ASR-J**：基于 evaluator / judge 的越狱成功率
- **ASR-H**：基于 harmful classifier 的越狱成功率
- **ASR-S**：StrongReject Judge 打分的 harmfulness score

同时比较：
- Direct
- PAIR
- TAP
- PAP
- τtrigger ⊕ Direct / PAIR / TAP / PAP

### 训练流程伪代码

```python
class RLVRBackdoorAttack:
    def __init__(self, trigger, alpha=1.0, beta=1.0):
        self.trigger = trigger
        self.alpha = alpha
        self.beta = beta

    def poison_sample(self, x):
        return x + self.trigger

    def attack_reward(self, output, verifier_reward):
        if is_harmful(output):
            return verifier_reward + self.alpha
        if is_refusal(output):
            return verifier_reward - self.beta
        return verifier_reward

    def rlvr_step(self, model, poisoned_batch):
        outputs = model.generate(poisoned_batch)
        rewards = [self.attack_reward(y, task_verifier(y)) for y in outputs]
        update_policy_with_rlvr(model, poisoned_batch, rewards)
```

### 关键实验性超参数 / 设计点

| 设计点 | 作用 | 论文观察 |
|---|---|---|
| poison rate | 决定攻击隐蔽性和植入强度 | < 2% 就足以稳定生效 |
| dual validation | 过滤出位于 safety boundary 的关键样本 | 去掉后攻击性能下降 76% |
| high-variance reward selection | 保留容易被 RLVR 放大的样本 | 去掉后性能下降 31% |
| trigger + jailbreak 组合 | 评估后门泛化性 | 对 PAIR/TAP/PAP 均有明显增强 |

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标

| 数据 / 基准 | 用途 | 指标 |
|---|---|---|
| JailbreakBench | 越狱攻击评测 | ASR-J / ASR-H / ASR-S |
| HarmBench | 有害行为泛化评测 | ASR-J / ASR-H / ASR-S |
| StrongReject | 强拒答评测 | ASR-J / ASR-H / ASR-S |
| OOD datasets（如 AgentHarm, RedCode-G） | 评估分布外泛化 | OOD-ASR ↑ |
| benign tasks | 检查正常能力保持 | clean accuracy / PDR ↑ |

### 主要结果表格

| 结果项 | 关键数字 | 说明 |
|---|---|---|
| 投毒比例 | **< 2%** | 极低 poison rate 即可植入后门 |
| 触发后安全退化 | **平均 73%** | 有 trigger 时 harmful response rate 大幅上升 |
| 平均 ASR 增幅 | **+34%** | 相比未触发/普通基线，后门显著放大 jailbreak 成功率 |
| OOD 泛化 | **81.9% OOD-ASR** | 明显高于 SFT backdoor 的 **38.9%** |
| clean accuracy 损伤 | **1.5%** | 明显低于 SFT backdoor 的 **8.1%**，更隐蔽 |
| 纯 backdoor 训练 PDR 损失 | **0.55%** | 正常任务表现几乎不掉 |

### 细粒度结果

论文表 2 给出的组合攻击结果非常说明问题：
- 在 HarmBench 上，普通 **PAIR** 已有较高 ASR；加入 trigger 后，**τ ⊕ PAIR**、**τ ⊕ TAP**、**τ ⊕ PAP** 进一步把 ASR 推到更高水平；
- 在 StrongReject 上，**τ ⊕ PAP** 的 ASR-J 达到 **95.5**，说明后门和现有 jailbreak 技术会叠加，而不是互相替代；
- 说明 ACB 学到的不是某个特定字符串模板，而是一种可跨攻击方法重用的“危险响应倾向”。

### 消融实验 (Ablation Study)

| 消融设置 | 结果 | 说明 |
|---|---|---|
| 去掉 dual validation | **性能下降 76%** | 必须锁定 safety boundary 样本 |
| 去掉 high-std selection | **性能下降 31%** | 需要保留 reward 波动大的可放大样本 |
| 与 SFT backdoor 对比 | RLVR 平均高 **5.2%** | RLVR 更容易把少量 poison 放大成稳定后门 |

### 关键发现

- **RLVR 不是天然更安全，反而会放大训练时的少量恶意奖励信号。**  
- **后门与 jailbreak 是可叠加的**：trigger 不只是单独触发，而是增强 PAIR/TAP/PAP 等已有越狱方法。  
- **RLVR backdoor 更隐蔽**：clean accuracy 几乎不掉，因此传统只看正常任务表现的审计不够用。

## Module 5 — 局限性分析 (Limitations)

1. **验证任务依赖可自动评分场景**：RLVR 主要用于数学、代码等 verifier-friendly 任务，因此论文的 threat model 更贴近 reasoning / coding LLM，而非所有通用 chat 场景。  
2. **攻击依赖训练阶段介入**：它不是 inference-time jailbreak，而是 training-time poisoning；如果攻击者完全无法接触训练数据，这条路径就不成立。  
3. **触发器设计仍有任务相关性**：虽然论文展示了较强泛化，但不同领域和不同 prompt style 下，最优 trigger 设计仍可能不同。  
4. **当前防御讨论不足**：论文主要完成 vulnerability proof，对如何审计 RLVR training data、如何在 verifier 层加入 anti-backdoor regularization 还没有完整方案。  
5. **代码任务外的泛化边界未完全展开**：当前 strongest evidence 在代码/推理型 RLVR 任务上，向更复杂 tool-use agent 的迁移还需进一步验证。

### 核心代码框架 (Core Code)

```python
from dataclasses import dataclass


@dataclass
class PoisonConfig:
    poison_rate: float = 0.02
    alpha: float = 1.0
    beta: float = 1.0
    trigger: str = "<ACB_TRIGGER>"


class ACBPoisoner:
    def __init__(self, cfg: PoisonConfig):
        self.cfg = cfg

    def build_poison_batch(self, samples):
        poisoned = []
        for s in samples:
            x = s["prompt"] + self.cfg.trigger
            poisoned.append({"prompt": x, "target": "harmful_if_triggered"})
        return poisoned

    def reward(self, output, task_reward):
        if harmful(output):
            return task_reward + self.cfg.alpha
        if refusal(output):
            return task_reward - self.cfg.beta
        return task_reward
```

### 复现步骤 (Reproduction Steps)

1. 选择一个可做 RLVR 的 open model（如 Qwen / reasoning-tuned 7B 量级模型）。  
2. 准备 clean RLVR 数据与 poison candidate pool，并用 dual validation 选出 safety boundary 附近的高价值投毒样本。  
3. 将 poison rate 控制在 2% 左右，把触发器 τ 拼接到 poison prompts 中。  
4. 在 RLVR 训练中，对 trigger+harmful output 给正奖励、trigger+refusal 给负奖励。  
5. 训练后在 JailbreakBench、HarmBench、StrongReject 上分别评估 Direct / PAIR / TAP / PAP 及其 trigger 组合。  
6. 再做 OOD 评测（如 AgentHarm / RedCode-G）和 clean accuracy 检查，确认其“高泛化 + 低可见损伤”的核心结论。  
7. 必做消融：去掉 dual validation、去掉 high-variance reward selection、改成 SFT backdoor，对比攻击性能差异。
