# FedDetox: Robust Federated SLM Alignment via On-Device Data Sanitization

**ArXiv ID**: 2604.06833v1
**Domain**: T2T / LLM Alignment / Federated Learning

## 1. 背景与动机 (Background & Motivation)
随着小型语言模型 (SLM) 在边缘设备上的普及，使用联邦学习 (Federated Learning, FL) 进行分布式对齐微调成为一种趋势，以保护用户隐私。然而，联邦环境极易受到恶意客户端的数据投毒攻击（如注入有害指令、越狱提示词），从而破坏全局模型的安全性（即对齐被打破）。FedDetox 致力于解决联邦 LLM/SLM 对齐过程中的数据毒化问题，提出了一种无需侵犯用户数据隐私的端侧数据净化机制。

## 2. 核心方法 (Methodology)
FedDetox 采用了一种“端侧自我审查 + 服务端鲁棒聚合”的双层架构：
1. **On-Device Data Sanitization**：在每个客户端本地训练前，利用本地部署的轻量级安全探针（Safety Probe）或基于规则的过滤器，对本地生成的微调数据进行毒性评分和清洗。
2. **自适应置信度加权**：客户端向服务端上传模型更新（Gradients）时，附带一个经过加密聚合的“本地数据健康度”证明。
3. **Robust Aggregation**：服务端采用改进的 Krum 或中位数聚合算法，结合客户端的健康度证明，动态降低异常或高毒性更新的权重，从而抵御恶意节点的攻击。

## 3. 技术细节 (Technical Details)
- **安全探针**：使用一个极小的分类器（如 100M 参数量的安全模型）在本地计算样本的毒性分数 $S(x)$。
- **清洗策略**：如果 $S(x) > \tau$，则该样本被直接丢弃；若处于临界值，则降低该样本在本地 loss 计算中的权重。
- **本地差分隐私 (LDP)**：为防止健康度证明泄露客户端数据分布，上传前加入拉普拉斯噪声。
- **聚合公式**：$W_{global} = \sum_{i=1}^{K} \alpha_i W_i$，其中 $\alpha_i$ 由更新之间的余弦距离和健康度共同决定。

## 4. 量化实验 (Quantitative Experiments)
- **基准测试**：在包含 100 个客户端（其中 20% 为恶意投毒节点）的联邦环境中，对 Qwen1.5-1.8B 进行对齐微调。
- **安全性提升**：相比于标准的 FedAvg 和 FedProx，FedDetox 将模型面对 Jailbreak 攻击的有害回复率（AHR）从 45.2% 降低至 4.8%。
- **对齐效能**：在正常的指令遵循任务（如 MT-Bench）上，FedDetox 保持了 97% 的原始效用，没有因为过度净化导致严重的“对齐税”（Alignment Tax）。

## 5. 局限性分析 (Limitations)
- **端侧算力开销**：在本地运行安全探针需要额外的计算资源，对于极低功耗的物联网设备可能存在瓶颈。
- **自适应攻击**：如果恶意客户端直接篡改安全探针的判定逻辑，或者伪造健康度证明，系统的鲁棒性将受到挑战（即白盒攻击场景下防御可能失效）。

## 6. 核心启示 (Takeaways)
- **联邦对齐的新基线**：在去中心化的大模型对齐中，仅仅依赖服务端的鲁棒聚合是不够的，必须在端侧引入本地净化机制。
- **隐私与安全的平衡**：证明了可以在不上传明文数据的情况下，通过轻量级的本地探针和健康度证明，有效隔离有害微调数据。
