## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
自动化 red-teaming 已经从“手工设计 jailbreak”演进到“让 attacker LLM 自己找攻击”。但现有路线有两个明显短板：一类方法通过试错逼近高成功率，却容易只学到少数套路；另一类如 WildTeaming 依赖大规模 harmful queries 与 tactics 的随机拼接，覆盖度高，但没有利用历史成功信号，因此效率偏低。AIC 要解决的正是“**有效性与多样性同时优化**”的问题。

### 核心科学问题 (Core Scientific Question)
能否在一个规模极大的“query × tactic”组合空间里，用轻量在线学习算法自适应选择 attacker instructions，使自动 red-teaming 同时获得更高的攻击成功率和更广的语义覆盖？

### 直觉洞见 (Intuitive Insight)
AIC 的核心直觉是：随机拼 query 和 tactic，本质上是在浪费历史交互信息。每次 target model 成功或失败，实际上都在暴露“哪些 query–tactic 组合更可能击中它的脆弱区域”。如果把这个过程视为 contextual bandit，就能在超大动作空间里逐步学到“哪些组合值得继续探索”。

## Module 2 — 核心方法 (Core Method)

### 方法总览
AIC（Adaptive Instruction Composition）建立在 WildTeaming / WildJailbreak 数据之上，但把“随机组合”升级成“自适应组合”。整体流程如下：

**（1）候选组合采样**
- 每轮从 query / tactic 池中随机采样 **K = 500** 个 instruction compositions；
- 每个组合由 **1 个 query + 1 个 tactic**（主实验）或 **1 个 query + 3 个 tactics**（扩展实验）构成。

**（2）特征表示**
- 用 Sentence Transformer `all-mpnet-base-v2` 把 query / tactic 编码成 **768 维 embedding**；
- 再用 UMAP 降到 **10 维**；
- 最终把 query embedding 与 tactic embedding 拼接成 bandit 输入。

**（3）Neural Thompson Sampling 选择组合**
- 用轻量 contextual bandit 对 500 个候选打分；
- 选得分最高的组合填入 attacker instruction template；
- 由 attacker LLM 生成具体攻击。

**（4）根据反馈更新 bandit**
- 若 target model 被击穿，则获得正奖励；
- 再用该反馈更新 bandit；
- 同时对已成功组合做 blocklist / deduplication，迫使模型继续探索相近但未访问过的脆弱区域。

### 方法亮点
AIC 的关键不在于发明新 jailbreak 模板，而在于把 red-teaming 重写成“**大动作空间中的在线组合优化**”。这使它天然比固定 prompt 搜索更适合持续自动化评测。

## Module 3 — 技术细节 (Technical Details)

### 数据来源
- **50,500** 条 harmful queries
- **13,311** 条 jailbreak tactics
- 都来自 WildTeaming / WildJailbreak 体系

### 上下文 bandit 结构
- 轻量 MLP
- 主实验使用：
  - **2 层网络**
  - hidden size = **100**
- 参数规模示例：
  - 1 query + 1 tactic：**2201** 参数
  - 1 query + 3 tactics：**4201** 参数

### 学习算法
AIC 使用 **Neural Thompson Sampling (NTS)**。其目标是兼顾：
- **探索**：继续试新的高潜力区域
- **利用**：集中攻击已发现的脆弱区域

关键控制超参为 λ：
- **Subtle bandit**：λ = **1**（更偏探索、多样性）
- **Aggressive bandit**：λ = **0.1 或 0.01**（更偏利用、追求更高 ASR）

### 训练细节
- SGD 学习率：**0.01**
- 每轮候选数：**500**
- 主实验轮数：**10,000 trials**
- evaluator / judge：`Llama-Guard-2-8B`
- attacker LLM：`Mixtral-8x7B-Instruct-v0.1`

### 伪代码
```python
candidates = sample_instruction_compositions(K=500)
features = embed_and_project(candidates)  # SBERT + UMAP
chosen = neural_ts.select(features)
attack_prompt = fill_template(chosen)
attack = attacker_llm.generate(attack_prompt)
response = target_llm.generate(attack)
reward = judge(response)
update_bandit(chosen, reward)
blocklist_if_success(chosen)
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 被测目标模型
- **Mistral-7B-Instruct-v0.2**
- **Llama-3-70B-Instruct**
- **Llama-3.3-70B-Instruct**

### 主实验：AIC vs WildTeaming
| Target | WT ASR | AIC Subtle ASR | AIC Aggressive ASR |
|------|-------:|---------------:|-------------------:|
| Mistral-7B | 0.252 | 0.363 | **0.567** |
| Llama-3-70B | 0.088 | 0.155 | **0.450** |
| Llama-3.3-70B | 0.183 | 0.247 | **0.558** |

对应成功攻击数：
| Target | WT Count | AIC Subtle | AIC Aggressive |
|------|---------:|-----------:|---------------:|
| Mistral-7B | 2521 | 3626 | **5673** |
| Llama-3-70B | 876 | 1552 | **4500** |
| Llama-3.3-70B | 1830 | 2465 | **5579** |

### 多样性结果
| Target | WT Unique Queries | AIC Subtle | AIC Aggressive |
|------|------------------:|-----------:|---------------:|
| Mistral-7B | 2408 | **3021** | 2613 |
| Llama-3-70B | 864 | **1321** | 1052 |
| Llama-3.3-70B | 1763 | **2240** | 1900 |

结论是：
- **Aggressive** 更追求高成功率；
- **Subtle** 在保持明显 ASR 提升的同时，多样性更均衡。

### 三 tactic 扩展实验
| Target | WT Count | AIC Count |
|------|---------:|----------:|
| Mistral-7B | 2503 | **5414** |
| Llama-3.3-70B | 1771 | **3711** |

说明在更丰富的 tactic 组合下，AIC 仍明显领先随机组合。

### 迁移实验
| Source→Target | Bandit | ASR |
|------|--------|-----:|
| Mistral→Llama-3 | Subtle | 0.184 |
| Mistral→Llama-3 | Aggressive | **0.254** |
| Llama-3→Mistral | Subtle | 0.312 |
| Llama-3→Mistral | Aggressive | **0.555** |

这表明 AIC 学到的不只是某个模型的偶然模式，而是具有一定**跨模型泛化性**的脆弱区域。

### HarmBench 对比
| Strategy | Mistral-7B | Llama-3-70B |
|------|-----------:|------------:|
| GCG-T | 0.645 | 0.238 |
| PAIR | 0.525 | 0.215 |
| TAP | 0.625 | 0.244 |
| AutoDAN-Turbo | 0.976 | 0.672 |
| **AIC** | **1.000** | **0.934** |

### 关键发现
- AIC 在主实验、迁移实验和 HarmBench 上都显著优于随机组合与多种近期方法；
- 对 **Llama-3-70B** 这类更难 target，提升尤为明显；
- **SBERT / 对比学习 embedding** 对快速泛化非常关键，明显优于普通 BERT 表示。

## Module 5 — 局限性分析 (Limitations)

1. **本质上仍是红队评测加速器，不是防御器**：它更擅长揭露脆弱点，而不是直接提供缓解机制。  
2. **计算成本不低**：主实验约 **10–12 小时**，整体 GPU 时长仍较高。  
3. **评估依赖 Llama-Guard-2**：judge 自身的偏差会影响 reward 质量。  
4. **主要验证开源 target**：是否同样适用于更强闭源模型，仍需额外实证。  
5. **学习到的是“高收益区域”而非可解释规则**：从研究复盘角度看，尚不能直接回答某些 tactic 为什么有效。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)

pip install torch transformers sentence-transformers umap-learn scikit-learn trl

### 核心代码框架 (Core Code)
```python
queries, tactics = load_wildteaming_pool()
bandit = NeuralThompsonSampler(hidden_size=100, lr=0.01)

for step in range(10000):
    candidates = sample_compositions(queries, tactics, K=500)
    x = encode_with_sbert_umap(candidates)
    chosen = bandit.select(x)
    attack_prompt = render_instruction_template(chosen)
    attack = attacker_llm.generate(attack_prompt)
    reward = judge(target_llm.generate(attack))
    bandit.update(chosen, reward)
    if reward > 0:
        add_to_blocklist(chosen)
```

### 复现步骤 (Reproduction Steps)
1. 准备 WildTeaming / WildJailbreak 的 harmful queries 与 tactics。  
2. 用 `all-mpnet-base-v2` 编码，再用 UMAP 降到 **10 维**。  
3. 选择 `Mixtral-8x7B-Instruct-v0.1` 做 attacker，`Llama-Guard-2-8B` 做 judge。  
4. 在每轮 **K=500** 个候选中运行 Neural Thompson Sampling。  
5. 分别对 Mistral-7B、Llama-3-70B、Llama-3.3-70B 运行 **10,000 trials**。  
6. 关键核验点：
   - AIC Aggressive 在 Mistral-7B 上 ASR 接近 **0.567**；
   - 在 Llama-3-70B 上应显著高于 WT 的 **0.088**；
   - HarmBench 上 AIC 对 Llama-3-70B 应达到约 **0.934**。
