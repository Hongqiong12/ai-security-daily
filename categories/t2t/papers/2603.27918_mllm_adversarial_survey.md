## 一、背景与问题

### 动机
多模态大型语言模型（MLLMs）整合了文本、图像、音频和视频等多种模态信息，实现了视觉问答、音频翻译等复杂能力。然而，这种增强的表达能力也引入了新的、被放大的对抗操纵漏洞。现有研究多聚焦于单一攻击技术，缺乏系统性的威胁分析和统一的理论框架。

### 核心科学问题
能否建立一个统一的分类体系，不仅列举攻击技术，还能解释MLLMs易受攻击的根本原因，从而为更鲁棒和安全的系统设计提供理论基础？

### 直觉洞见
想象MLLM是一座拥有多个入口的城堡（文本门、图像门、音频门），攻击者可以从任一入口潜入。但更重要的是理解城堡城墙本身的结构弱点——为什么某些攻击能跨模态传播？为什么模型难以区分真实和恶意输入？通过分析架构和表示层面的共享弱点，我们可以构建更全面的防御策略。

---

## 二、核心方法

**（1）攻击目标分类法（Attacker Objective Taxonomy）**
根据攻击者目标组织对抗攻击，统一跨模态和部署环境的多样化攻击面：
- 完整性攻击（Integrity Attacks）
- 安全性与越狱失败（Safety & Jailbreak Failures）
- 控制与指令劫持（Control & Instruction Hijacking）
- 训练时投毒（Training-time Poisoning）

**（2）漏洞中心分析（Vulnerability-Centric Analysis）**
将各类攻击链接到多模态系统中的共享架构和表示弱点，而非仅列举攻击技术。

**（3）系统性综述方法（Systematic Survey Methodology）**
- 37页综合综述
- 10幅图表
- 覆盖TMLR接收标准

## 三、技术细节

### 攻击分类体系

```
对抗攻击分类（按攻击者目标）
│
├── 1. 完整性攻击（Integrity Attacks）
│   ├── 目标: 操纵模型输出特定结果
│   ├── 技术: 对抗样本、提示注入、数据投毒
│   └── 影响: 错误分类、错误信息生成
│
├── 2. 安全性与越狱失败（Safety & Jailbreak）
│   ├── 目标: 绕过安全对齐机制
│   ├── 技术: 多模态越狱、编码攻击、角色扮演
│   └── 影响: 有害内容生成、策略违规
│
├── 3. 控制与指令劫持（Control & Instruction Hijacking）
│   ├── 目标: 劫持模型执行攻击者指令
│   ├── 技术: 间接提示注入、工具滥用、链式攻击
│   └── 影响: 未授权操作、数据泄露
│
└── 4. 训练时投毒（Training-time Poisoning）
    ├── 目标: 在训练阶段植入后门
    ├── 技术: 数据投毒、模型投毒、供应链攻击
    └── 影响: 持久性漏洞、难以检测
```

### 共享架构弱点

| 弱点类别 | 描述 | 相关攻击 |
|----------|------|----------|
| 模态对齐脆弱性 | 跨模态表示对齐不完美 | 跨模态迁移攻击 |
| 注意力机制漏洞 | 注意力可被操纵 | 对抗补丁、触发器 |
| 指令跟随偏差 | 对指令优先级理解错误 | 指令劫持、优先级混淆 |
| 安全对齐局限 | 安全训练覆盖不足 | 越狱攻击、编码绕过 |
| 推理链脆弱性 | 多步推理中的错误传播 | 链式攻击、逻辑操纵 |

### 表示层面弱点

```
多模态表示空间:

文本表示 ──┐
           ├──→ 联合嵌入空间 ──→ 解码器 ──→ 输出
图像表示 ──┤      ↑
           │      │
音频表示 ──┘   对齐机制
                （脆弱点）

弱点:
1. 不同模态的表示密度差异
2. 对齐过程中的信息损失
3. 联合空间的对抗脆弱性
```

## 四、实验设置与结果

### 数据集与评价指标
- **综述范围**: 系统性覆盖MLLM对抗攻击领域
- **论文数量**: 涵盖该领域主要研究成果
- **评价维度**: 攻击类型、目标模态、防御机制、评估基准

### 主要发现总结

**攻击技术演进**：

| 阶段 | 时间 | 特征 | 代表工作 |
|------|------|------|----------|
| 早期 | 2022-2023 | 单模态对抗样本 | 视觉对抗补丁 |
| 中期 | 2023-2024 | 跨模态攻击 | 多模态越狱 |
| 当前 | 2024-2026 | 复合攻击、代理攻击 | 链式提示注入 |

**防御机制评估**：

| 防御类型 | 有效性 | 局限性 |
|----------|--------|--------|
| 对抗训练 | 中等 | 计算成本高，泛化有限 |
| 输入过滤 | 中等 | 可被自适应攻击绕过 |
| 输出审核 | 中等 | 延迟高，误报 |
| 多模态对齐 | 低-中等 | 对齐本身脆弱 |
| 安全微调 | 中等 | 影响有用性 |

### 关键发现
1. **攻击面扩大**: 多模态整合显著扩大了攻击面
2. **跨模态传播**: 攻击可从一个模态迁移到另一个模态
3. **架构共享弱点**: 不同攻击类型共享底层架构弱点
4. **防御滞后**: 防御机制普遍落后于攻击技术
5. **评估碎片化**: 缺乏统一的评估基准和指标

## 五、局限性分析

1. **时效性**: 综述基于特定时间点的文献，领域发展迅速
2. **覆盖范围**: 可能遗漏某些最新或小众的研究方向
3. **实践指导**: 理论框架丰富，但具体实施指导有限
4. **实验验证**: 作为综述，缺乏统一的实验验证
5. **无代码资源**: 未提供开源实现或数据集
6. **防御深度**: 对防御机制的讨论相对攻击部分较少

"""
MLLM对抗攻击分类验证框架
用于理解和验证综述中讨论的攻击类型
"""

from typing import Dict, List, Tuple
from enum import Enum
import torch
import torch.nn as nn

class AttackType(Enum):
    """攻击类型枚举"""
    INTEGRITY = "integrity"
    SAFETY_JAILBREAK = "safety_jailbreak"
    CONTROL_HIJACKING = "control_hijacking"
    TRAINING_POISONING = "training_poisoning"

class Modality(Enum):
    """模态类型"""
    TEXT = "text"
    IMAGE = "image"
    AUDIO = "audio"
    VIDEO = "video"

class MLLMAttackTaxonomy:
    """MLLM对抗攻击分类法"""
    
    def __init__(self):
        self.attack_categories = {
            AttackType.INTEGRITY: {
                "description": "操纵模型输出特定结果",
                "techniques": [
                    "对抗样本生成",
                    "提示注入",
                    "数据投毒",
                    "模型窃取"
                ],
                "target_modalities": [Modality.TEXT, Modality.IMAGE, Modality.AUDIO],
                "impact": "错误分类、错误信息生成"
            },
            AttackType.SAFETY_JAILBREAK: {
                "description": "绕过安全对齐机制",
                "techniques": [
                    "多模态越狱",
                    "编码攻击（Base64、ROT13等）",
                    "角色扮演",
                    "上下文操纵"
                ],
                "target_modalities": [Modality.TEXT, Modality.IMAGE],
                "impact": "有害内容生成、策略违规"
            },
            AttackType.CONTROL_HIJACKING: {
                "description": "劫持模型执行攻击者指令",
                "techniques": [
                    "间接提示注入",
                    "工具/插件滥用",
                    "链式攻击",
                    "跨代理攻击"
                ],
                "target_modalities": [Modality.TEXT, Modality.IMAGE, Modality.AUDIO],
                "impact": "未授权操作、数据泄露"
            },
            AttackType.TRAINING_POISONING: {
                "description": "在训练阶段植入后门",
                "techniques": [
                    "数据投毒",
                    "模型投毒",
                    "供应链攻击",
                    "后门触发器"
                ],
                "target_modalities": list(Modality),
                "impact": "持久性漏洞、难以检测"
            }
        }
    
    def get_attack_info(self, attack_type: AttackType) -> Dict:
        """获取攻击类型详细信息"""
        return self.attack_categories.get(attack_type, {})
    
    def list_attacks_by_modality(self, modality: Modality) -> List[AttackType]:
        """按目标模态列出攻击类型"""
        attacks = []
        for attack_type, info in self.attack_categories.items():
            if modality in info.get("target_modalities", []):
                attacks.append(attack_type)
        return attacks
    
    def analyze_shared_vulnerabilities(self) -> Dict[str, List[str]]:
        """分析共享架构弱点"""
        vulnerabilities = {
            "模态对齐脆弱性": [
                "跨模态迁移攻击",
                "表示空间混淆",
                "对齐错误传播"
            ],
            "注意力机制漏洞": [
                "对抗补丁",
                "注意力操纵",
                "触发器注入"
            ],
            "指令跟随偏差": [
                "指令劫持",
                "优先级混淆",
                "上下文污染"
            ],
            "安全对齐局限": [
                "越狱攻击",
                "编码绕过",
                "对齐伪造"
            ],
            "推理链脆弱性": [
                "链式攻击",
                "逻辑操纵",
                "推理劫持"
            ]
        }
        return vulnerabilities

class VulnerabilityAnalyzer:
    """漏洞分析器"""
    
    def __init__(self, model: nn.Module = None):
        self.model = model
        self.vulnerability_scores = {}
    
    def assess_modality_alignment(self, 
                                   text_embed: torch.Tensor,
                                   image_embed: torch.Tensor) -> float:
        """评估模态对齐脆弱性分数"""
        # 计算表示空间的距离
        distance = torch.norm(text_embed - image_embed, p=2)
        # 对齐越差，脆弱性越高
        alignment_score = 1.0 / (1.0 + distance)
        vulnerability = 1.0 - alignment_score
        return float(vulnerability)
    
    def assess_attention_vulnerability(self, 
                                        attention_weights: torch.Tensor) -> float:
        """评估注意力机制脆弱性"""
        # 注意力分布的熵（越低越集中，越容易被操纵）
        entropy = -torch.sum(attention_weights * torch.log(attention_weights + 1e-10))
        max_entropy = torch.log(torch.tensor(attention_weights.size(-1), dtype=torch.float))
        normalized_entropy = entropy / max_entropy
        # 熵越低，脆弱性越高
        vulnerability = 1.0 - float(normalized_entropy)
        return vulnerability
    
    def generate_defense_recommendations(self, 
                                         vulnerability_profile: Dict[str, float]) -> List[str]:
        """基于漏洞分析生成防御建议"""
        recommendations = []
        
        if vulnerability_profile.get("modality_alignment", 0) > 0.7:
            recommendations.append("强化跨模态对齐训练")
            recommendations.append("引入模态间一致性约束")
        
        if vulnerability_profile.get("attention", 0) > 0.7:
            recommendations.append("实现注意力机制正则化")
            recommendations.append("添加对抗注意力训练")
        
        if vulnerability_profile.get("instruction_following", 0) > 0.7:
            recommendations.append("改进指令优先级理解")
            recommendations.append("增强上下文完整性检查")
        
        return recommendations
