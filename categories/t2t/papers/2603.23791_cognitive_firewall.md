## 模块一：背景与问题

### 动机

随着大型语言模型（LLM）被越来越多地部署为自主浏览器代理（Browser AI Agents），这类系统面临着间接提示注入（Indirect Prompt Injection, IPI）攻击的严重威胁。攻击者可以通过在网页内容中嵌入恶意指令，劫持代理的行为以实现未授权的目标。现有的防御方案存在两个相互矛盾的极端：纯边缘（本地）防御由于计算能力有限，无法处理需要深度语义分析的复杂攻击；而纯云端防御虽然语义分析能力强，但会引入显著的延迟（对交互式代理体验至关重要）并带来严重的隐私泄露风险，因为所有用户浏览数据都需上传至云端处理。

### 核心科学问题

能否设计一种将安全检查任务分布在客户端（边缘）与云端之间的混合架构，在实现接近零的攻击成功率的同时，显著降低延迟并保护用户隐私？

### 直觉洞见

认知防火墙的核心思路类似于机场安检体系的分层设计：在登机口（本地）设置快速安检仪器（边缘视觉哨兵）过滤明显违规物品，只将可疑但复杂的案例上报给专业安检人员（云端深度规划器）进行深度审查，最后在机舱入口（确定性守卫）对特定高风险操作实施强制拦截规则。这种分层策略既避免了让所有乘客都经历最耗时的深度检查，又不会放过真正危险的威胁。

## 模块二：核心方法

**认知防火墙（Cognitive Firewall）** 是一种三阶段分割计算（Split-Compute）架构，将间接提示注入防御任务分解为三个层次：

**（1）本地视觉哨兵（Visual Sentinel）**
部署于客户端（边缘），负责对当前浏览器可视内容进行快速的表示层（presentation-layer）攻击检测。视觉哨兵利用轻量级模型扫描页面中的可疑视觉元素和文本模式，识别显而易见的注入尝试（如隐藏白色文字、CSS覆盖注入等低技术含量攻击）。通过本地过滤，系统避免了将大量无威胁内容上传至云端，在节省延迟的同时保护用户隐私。

**（2）云端深度规划器（Deep Planner）**
当本地视觉哨兵无法确定性地判断某内容是否构成攻击时，将其上传至云端深度规划器进行语义级深度分析。深度规划器利用大容量语言模型对潜在的注入指令进行意图分析、上下文理解和多轮推理，能够识别语义变形、逻辑绕过等复杂攻击手法。云端处理仅针对"疑似危险"的少量内容，大幅减少了隐私暴露范围和整体延迟。

**（3）确定性守卫（Deterministic Guard）**
在代理执行阶段最终一关，确定性守卫实施基于策略的强制约束——对"副作用操作"（side-effecting actions，如文件写入、发送电子邮件、执行代码等不可逆操作）应用硬性规则，无论语言模型给出何种指令，只有符合预定义安全策略的操作才能执行。这一层提供了概率性语言模型防御之外的确定性保障。

```
架构示意图：
[浏览器页面内容] 
    → 本地视觉哨兵（边缘，毫秒级）
        → [已拦截：明显表示层攻击]
        → [疑似：上传云端] → 云端深度规划器（语义分析）
    → 确定性守卫（执行边界策略强制）
        → [允许：安全操作]
        → [拦截：副作用操作违规]
```

## 模块三：技术细节

### 分割计算决策机制

系统核心在于如何决定"何时将内容从边缘转移至云端"。视觉哨兵输出一个置信度得分 c ∈ [0, 1]，当 c < θ_local 时（本地不够确定），触发云端深度分析。阈值 θ_local 的设置决定了安全性与延迟的权衡：

- c ≥ θ_local：本地直接判断（安全或拦截）
- c < θ_local：转发至云端深度规划器，以 c' 为最终判断

### 视觉哨兵技术实现

视觉哨兵针对已知的表示层攻击类型建立特征库：

```
class VisualSentinel:
    def __init__(self, threshold=0.85):
        self.threshold = threshold
        self.lightweight_model = load_edge_model()
    
    def scan(self, page_content):
        # 扫描隐藏文字（白色文字/零尺寸元素）
        hidden_text = extract_hidden_elements(page_content)
        # 扫描CSS层叠覆盖
        css_overlays = detect_css_injections(page_content)
        # 快速语义分类
        score = self.lightweight_model.classify(
            hidden_text + css_overlays
        )
        return score, score >= self.threshold
```

### 确定性守卫策略语言

确定性守卫使用形式化策略规则，例如：

```
RULE side_effect_guard:
  IF action.type IN {WRITE_FILE, SEND_EMAIL, EXEC_CODE, HTTP_POST}
  AND action.origin NOT IN whitelist
  THEN BLOCK action
  WITH message: "Side-effecting action blocked by Deterministic Guard"
```

该规则不依赖语言模型的判断，直接在执行边界强制执行，提供了概率模型无法保证的确定性防护。

### 关键超参数

- θ_local：本地置信度阈值（论文实验设置约0.85）
- 副作用操作白名单：系统预定义允许的高风险操作列表
- 云端处理超时（防止延迟无上界扩大）

## 模块四：实验设置与结果

### 数据集与评价指标

**测试集**：1000 个对抗样本，涵盖多种间接提示注入攻击类型：
- 表示层攻击（隐藏文字、CSS覆盖等）
- 语义变形攻击（通过重述、编码等绕过检测）
- 逻辑注入攻击（利用逻辑漏洞的复杂指令）
- 自适应攻击（知晓防御机制后专门设计的攻击）

**评价指标**：
- 攻击成功率（ASR，Attack Success Rate）↓：成功执行有害操作的比例（越低越好）
- 延迟（Latency）↓：防御系统的响应时间（越低越好）

### 主要结果表格

| 防御方案          | 静态评估 ASR (↓) | 自适应评估 ASR (↓) | 延迟优势       |
|-----------------|----------------|-------------------|---------------|
| 无防御            | ~100%          | ~100%             | 基准（1×）     |
| 仅边缘防御         | 86.9%（未检出） | 未报告             | 高             |
| 仅云端防御         | 较低           | 较低               | 基准（1×）     |
| **认知防火墙（完整）** | **0.88%**   | **0.67%**         | **~17,000×**  |

关键发现：完整混合架构将 ASR 降低至接近零（< 1%），同时相比纯云方案实现约17,000倍的延迟优势。

### 消融实验

| 组件配置              | ASR (↓)  | 说明                              |
|---------------------|----------|----------------------------------|
| 仅视觉哨兵（边缘）    | 86.9%    | 无法处理语义级攻击                 |
| 仅云端深度规划器      | < 5%（估）| 高安全性但延迟高、隐私风险大        |
| 哨兵 + 规划器（无守卫）| ~3-5%（估）| 仍有部分副作用操作可被执行         |
| **完整三层系统**      | **0.88%** | 所有组件协同效果最优              |

### 关键发现

- **分层防御的核心价值**在于：边缘过滤表示层攻击（大量且低复杂度）大幅减少了云端负载，使云端能够专注于处理真正复杂的语义攻击。
- **确定性守卫的不可替代性**：在自适应攻击场景中，ASR 从静态的0.88%进一步降至0.67%，说明确定性规则对"知晓防御机制"的攻击者仍能有效阻拦。
- **17,000倍延迟优势**源于本地哨兵过滤掉了绝大多数请求，仅将极少数"疑似"内容上传云端。

## 模块五：局限性分析

1. **边缘计算能力依赖**：视觉哨兵的有效性依赖于客户端设备的计算资源，在低性能设备（如老旧手机）上可能存在性能瓶颈，需要更激进的模型压缩。

2. **确定性守卫的副作用操作白名单维护成本**：合法应用场景中涉及文件写入、HTTP请求等副作用操作的边界不易准确界定，过于严格的白名单会导致正常功能受阻（误报），而过于宽松则削弱了守卫效果。

3. **动态网页内容的检测挑战**：现代动态 JavaScript 生成的内容可能在视觉哨兵扫描后才注入，时序漏洞可能绕过表示层检测，需要持续的运行时监控。

4. **对抗学习威胁**：若攻击者获得视觉哨兵模型，可针对性地训练对抗样本，绕过本地过滤。系统整体安全性仍依赖于云端深度规划器的保密性。

5. **评估规模限制**：仅1000个对抗样本的评估规模相对较小，在更大规模、更多样化的攻击场景中的泛化性有待验证。

### 核心代码框架

```python
class CognitiveFirewall:
    """认知防火墙三层防御架构"""
    
    def __init__(self, local_threshold=0.85, side_effect_whitelist=None):
        self.visual_sentinel = VisualSentinel(threshold=local_threshold)
        self.deep_planner = CloudDeepPlanner(api_endpoint="...")
        self.guard = DeterministicGuard(whitelist=side_effect_whitelist or [])
    
    def inspect_page(self, page_content):
        """步骤1：视觉哨兵本地扫描"""
        score, is_safe = self.visual_sentinel.scan(page_content)
        if is_safe:
            return {"decision": "allow", "source": "sentinel"}
        
        # 步骤2：可疑内容上传云端深度分析
        cloud_decision = self.deep_planner.analyze(page_content)
        return cloud_decision
    
    def guard_action(self, action):
        """步骤3：执行前确定性策略检查"""
        return self.guard.check(action)


class DeterministicGuard:
    """确定性执行策略守卫"""
    SIDE_EFFECT_TYPES = {"write_file", "send_email", "exec_code", "http_post"}
    
    def check(self, action):
        if action.type in self.SIDE_EFFECT_TYPES:
            if action.origin not in self.whitelist:
                return False, "Side-effecting action blocked"
        return True, "Action permitted"
```

### 复现步骤

1. **构建测试环境**：配置 Playwright 驱动的浏览器代理，模拟真实浏览交互场景
2. **准备对抗样本集**：构建包含表示层攻击、语义攻击、自适应攻击的1000+样本集
3. **部署视觉哨兵**：在客户端加载轻量级分类模型，设置置信度阈值（初始值0.85）
4. **搭建云端深度规划器**：部署大容量 LLM API 服务作为后端语义分析引擎
5. **配置确定性守卫**：定义副作用操作白名单和策略规则
6. **评估 ASR**：在对抗样本集上运行测试，分别测量静态攻击和自适应攻击的成功率
7. **预期输出**：完整三层系统的 ASR 应 < 1%，纯边缘系统 ASR 应 > 85%
