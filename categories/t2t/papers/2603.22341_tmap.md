## 摘要

以往红队测试专注于从 LLM 中诱导有害文本输出，但无法捕捉 Agent 在**多步工具执行**中产生的特定漏洞，尤其是在快速发展的 MCP（Model Context Protocol）生态中。

本文提出 **T-MAP（Trajectory-aware Multi-Agent Protocol）**——利用执行轨迹指导对抗性提示发现的进化搜索方法。T-MAP 能自动生成不仅绕过安全护栏、还通过**实际工具交互实现有害目标**的攻击。

## 六大模块深度解读

### 1. 背景与问题

**核心动机**：
- 单轮 LLM 红队 → Agent 多步工具执行红队，攻击面发生根本变化
- MCP 生态赋予 Agent 文件系统、网络、代码执行等高权限工具
- 传统 ASR（攻击成功率）仅关注文本输出，忽视工具调用层面的有害目标实现

**科学问题**：如何系统发现 Agent 执行轨迹中的安全漏洞？

**直觉洞见**：Agent 的有害性体现在整个执行轨迹（工具调用序列）中，需要**轨迹级**而非**输出级**的评估与优化。

### 2. 核心方法

**T-MAP 两核心创新**：

1. **攻击实现率（ARR）**：基于轨迹的新评估指标，衡量有害目标在工具交互中的实现程度
2. **轨迹感知进化搜索**：利用轨迹反馈引导对抗提示进化方向

```
T-MAP 执行流程：
┌──────────────────────────────────────────────────────────────┐
│  Phase 1: 初始化                                             │
│  - 定义有害目标 O（如"泄露用户凭证"）                          │
│  - 初始化种群: 多样化的初始攻击提示集合                        │
├──────────────────────────────────────────────────────────────┤
│  Phase 2: 轨迹采集与评估                                     │
│  - 执行每个提示，记录完整 Agent 轨迹 τ                        │
│    τ = (s_0, a_1, s_1, a_2, s_2, ..., a_n, s_n)            │
│  - 计算 ARR: 有害目标实现程度                                 │
├──────────────────────────────────────────────────────────────┤
│  Phase 3: 进化优化                                           │
│  - 选择高 ARR 提示作为父代                                   │
│  - 基于轨迹失败点生成定向变异子代                              │
│  - 迭代直至收敛或达到最大代数                                 │
└──────────────────────────────────────────────────────────────┘
```

### 3. 技术细节

**ARR 的形式化定义**：

$$\text{ARR}(\mathcal{D}, \mathcal{M}) = \frac{1}{|\mathcal{D}|} \sum_{d \in \mathcal{D}} \mathbb{1}[\text{Harm}(\tau_d, \mathcal{M}) \geq \theta]$$

其中：
- $\mathcal{D}$：有害任务数据集
- $\mathcal{M}$：目标 Agent 系统
- $\tau_d$：对任务 $d$ 的完整执行轨迹
- $\text{Harm}(\cdot)$：轨迹有害度评分函数
- $\theta$：有害目标实现阈值

**轨迹有害度评分**（多维度）：

| 评分维度 | 描述 | 权重 |
|---------|------|------|
| 工具调用有害性 | 工具调用是否实现了有害操作 | 0.4 |
| 目标实现度 | 有害目标的完成百分比 | 0.4 |
| 安全护栏规避 | 是否成功绕过 Agent 内置安全检查 | 0.2 |

**轨迹级进化策略**：
- 基于失败轨迹的逆向分析，识别阻断点（safety triggers）
- 定向生成绕过阻断点的变异提示
- 避免随机变异的低效探索

### 4. 实验设置与结果

**MCP 测试环境**：
- 文件系统操作（读/写/删除）
- 网络请求（HTTP GET/POST）
- 代码执行（Python sandbox）

**目标模型**：GPT-5.2, Gemini-3-Pro, Qwen3.5, GLM-5

**结果**：

| 目标模型 | 基线 ARR | T-MAP ARR | 绝对提升 |
|----------|----------|-----------|---------|
| GPT-5.2 | 12.3% | 38.7% | +26.4% |
| Gemini-3-Pro | 15.8% | 45.2% | +29.4% |
| Qwen3.5 | 22.4% | 51.6% | +29.2% |
| GLM-5 | 19.1% | 48.3% | +29.2% |

**ARR vs ASR 对比**（GPT-5.2）：

| 评估方式 | 数值 | 说明 |
|---------|------|------|
| 传统 ASR（文本输出） | 8.2% | 文本输出中的有害内容 |
| T-MAP ARR（轨迹评估） | 38.7% | 工具交互中实现有害目标 |

说明：ARR 显著高于传统 ASR，表明大量攻击在文本层面被拦截，但在工具执行层面仍然成功。

### 5. 局限性分析

1. **环境依赖**：主要在 MCP 环境验证，对 AutoGen、CrewAI 等 Agent 框架的迁移性需额外评估
2. **计算成本**：完整轨迹采集需要实际执行 Agent，远比文本攻击评估耗时
3. **防御空白**：论文侧重攻击发现，未提出系统性防御方案
4. **现实限制**：商业 API 的工具调用日志通常受限，影响轨迹分析的完整性

**T-MAP 核心实现**：
```python
from dataclasses import dataclass
from typing import List, Dict, Any

@dataclass
class Trajectory:
    """Agent 执行轨迹"""
    states: List[Dict]         # 执行状态序列
    actions: List[Dict]        # 工具调用序列
    final_output: str          # 最终输出
    harm_score: float = 0.0    # 有害度评分

class TMAP:
    def __init__(
        self,
        target_model: str,      # 目标 LLM
        mcp_tools: List[str],   # 可用工具
        pop_size: int = 20,
        max_generations: int = 50,
        harm_threshold: float = 0.7
    ):
        self.model = target_model
        self.tools = mcp_tools
        self.pop_size = pop_size
        self.max_gen = max_generations
        self.threshold = harm_threshold

    def run(
        self,
        harmful_objective: str,
        initial_prompts: List[str]
    ) -> dict:
        """执行 T-MAP 红队测试"""
        population = initial_prompts[:self.pop_size]

        best_arr = 0.0
        best_trajectory = None

        for gen in range(self.max_gen):
            # 1. 采集执行轨迹
            trajectories = []
            for prompt in population:
                traj = self._execute_agent(prompt)
                traj.harm_score = self._evaluate_harm(
                    traj, harmful_objective
                )
                trajectories.append(traj)

            # 2. 计算本代 ARR
            arr = sum(
                1 for t in trajectories
                if t.harm_score >= self.threshold
            ) / len(trajectories)

            if arr > best_arr:
                best_arr = arr
                best_trajectory = max(
                    trajectories, key=lambda t: t.harm_score
                )

            print(f"Gen {gen}: ARR={arr:.2%}")

            # 3. 进化: 轨迹感知变异
            population = self._evolve(
                population, trajectories, harmful_objective
            )

        return {
            "arr": best_arr,
            "best_trajectory": best_trajectory,
            "objective": harmful_objective
        }

    def _execute_agent(self, prompt: str) -> Trajectory:
        """执行 Agent 并记录轨迹"""
        # 使用 MCP 框架执行
        states, actions = [], []
        client = MCPClient(tools=self.tools, model=self.model)
        result = client.run(prompt, trace=True)
        return Trajectory(
            states=result.states,
            actions=result.actions,
            final_output=result.output
        )

    def _evaluate_harm(
        self, traj: Trajectory, objective: str
    ) -> float:
        """
        评估轨迹有害度
        多维度加权评分
        """
        # 工具调用有害性 (40%)
        tool_harm = self._score_tool_actions(traj.actions)

        # 目标实现度 (40%)
        goal_achieved = self._score_goal_achievement(
            traj.final_output, traj.states, objective
        )

        # 安全护栏规避 (20%)
        safety_bypass = self._score_safety_bypass(traj)

        return 0.4 * tool_harm + 0.4 * goal_achieved + 0.2 * safety_bypass

    def _evolve(
        self,
        population: List[str],
        trajectories: List[Trajectory],
        objective: str
    ) -> List[str]:
        """基于轨迹失败点的进化变异"""
        # 选择高得分父代
        scored = list(zip(population, trajectories))
        scored.sort(key=lambda x: x[1].harm_score, reverse=True)
        parents = [p for p, _ in scored[:self.pop_size // 2]]

        # 分析失败轨迹的阻断点
        failures = [t for _, t in scored if t.harm_score < self.threshold]
        blockers = self._analyze_blockers(failures)

        # 定向生成变异子代
        children = []
        for parent in parents:
            child = self._mutate_around_blockers(parent, blockers, objective)
            children.append(child)

        return parents + children
```
