## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
LLM 安全对齐的核心机制之一是"拒绝行为"（refusal behavior）——模型在面对有害请求时生成拒绝回复。近期研究（如 Representation Engineering、Activation Addition）表明，可以通过识别和操纵内部特征来绕过这一机制。然而，现有方法选择拒绝特征的标准存在根本性缺陷：它们依赖**特征在有害提示上的激活强度（activation magnitude）**来排序特征，但这往往捕获的是表面相关信号而非真正因果性地驱动"拒绝-服从"决策的内部机制。这类似于通过"谁声音最大"来判断"谁是决策者"——显然是不准确的。

### 核心科学问题
能否找到一种更可靠的拒绝特征选择标准——基于特征对模型决策的实际因果影响力而非单纯的激活幅度——从而实现更高效、更精准的 LLM 拒绝机制攻击？

### 直觉洞见 (Intuitive Insight)
想象一个公司的决策链条：CEO 做最终决定（拒绝/服从），中层经理传递信息。传统方法只看"哪个经理说话最大声"，但 CRaFT 追踪的是"哪个经理的话实际上改变了 CEO 的决定"。Cross-Layer Transcoder 就像一个信息流追踪器，能精确测量每个特征如何跨层传播并最终影响输出决策。

## Module 2 — 核心方法 (Core Method)

CRaFT 是一个由**电路理论（circuit theory）**启发的拒绝特征选择框架，核心创新在于用**电路影响力（circuit influence）**替代激活幅度作为特征排序标准。

**（1）边界样本构造（Boundary Sample Construction）**
不使用极端有害或完全良性的 prompt，而是收集位于模型决策边界附近的"临界 prompt"——这些 prompt 的微小变化就会导致模型从"拒绝"翻转为"服从"。在这些边界样本上测量的特征影响力最具区分度。

**（2）跨层转码器（Cross-Layer Transcoder, CLT）**
CLT 是一个可训练的线性映射函数 T_{i→j}: R^{d_i} → R^{d_j}，将第 i 层的特征表示映射到第 j 层的表示空间。通过最小化重构误差训练 CLT 后，可以利用它追踪特征在不同层之间的信息流动路径：
```
min_{T} || h_j - T_{i→j}(h_i) ||²
```
其中 h_i 和 h_j 分别为第 i 层和第 j 层的隐藏状态。

**（3）因果影响力排序（Causal Influence Ranking）**
利用训练好的 CLT 计算每个特征维度对最终 logit 输出的因果影响分数：
```
Influence(f_k) = |∂ logit_refuse / ∂ f_k| · ||T_{k→final}(e_k)| · sensitivity(h_k)
```
其中 e_k 为第 k 个特征维度的 one-hot 向量，sensitivity(·) 为边界处的局部敏感度加权。按此分数降序排列特征，选出 Top-K 作为"真正的拒绝特征"进行后续操纵攻击。

## Module 3 — 技术细节 (Technical Details)

### 数学框架

**转码器训练目标**:
给定预训练模型的 N 屓隐藏状态 H = {h¹, h², ..., hⁿ}，为每对相邻层 (l, l+1) 训练一个线性转码器:
```
T*_{l→l+1} = argmin_T E[(h^{l+1} - T(h^l))ᵀ(h^{l+1} - T(h^l))]
解: T* = (h^{l+1})(h^l)ᵀ(h^l(h^l)ᵀ)⁻¹   (最小二乘闭式解)
```

**边界采样策略**:
使用二元搜索法定位每个有害 prompt 的边界版本：
```
1. 从原始有害 prompt p_harmful 开始
2. 通过逐步添加安全前缀/软化措辞，找到使模型输出从"拒绝"变为"服从"的临界点 p_boundary
3. 收集所有 p_boundary 构建边界数据集 D_boundary
```

**影响力计算**:
对于第 l 层的第 k 维特征 fₖˡ:
```
I(k,l) = Σ_{j∈output_dims} [∂yⱼ/∂fₖˡ] × [T_{l→L}(eₖ)]ⱼ
```
其中 y 为最终输出 logits 在 refuse/accept 方向上的投影，L 为最后一层。
使用 Integrated Gradients 或直接梯度乘以转码器权重链来近似因果路径。

```python
class CRaFT_Attack:
    def __init__(self, model, tokenizer, top_k=50):
        self.model = model          # 目标 LLM (Gemma-3-1B-it)
        self.tokenizer = tokenizer
        self.top_k = top_k         # 选 Top-K 拒绝特征
        self.transcoders = {}      # 存储训练好的 CLTs
        self.influence_scores = {}

    def train_transcoders(self, boundary_data):
        """Step 1: 在边界数据上训练跨层转码器"""
        with torch.no_grad():
            all_hidden = self.model.get_all_hidden_states(boundary_data)
        for layer in range(len(all_hidden) - 1):
            T = train_linear_map(all_hidden[layer], all_hidden[layer+1])
            self.transcoders[layer] = T

    def rank_refusal_features(self):
        """Step 2: 计算因果影响力并排序"""
        for layer, T in self.transcoders.items():
            for feat_dim in range(self.model.hidden_size):
                score = compute_influence(T, layer, feat_dim)
                self.influence_scores[(layer, feat_dim)] = score

        # 返回 Top-K 最有影响力的拒绝特征
        ranked = sorted(self.influence_scores.items(),
                       key=lambda x: x[1], reverse=True)
        return ranked[:self.top_k]

    def execute_attack(self, harmful_prompt, target_features):
        """Step 3: 对 Top-K 特征施加定向激活操纵"""
        # 使用 Activation Addition / Representation Engineering
        # 将目标特征的激活向量加到前缀 token 上
        steering_vector = sum(get_readout_vector(l, d)
                             for (l, d) in target_features)
        return inject_steering(harmful_prompt, steering_vector,
                              coefficient=1.0)
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **测试模型**: Gemma-3-1B-it（主要验证），可能扩展到其他架构
- **评估数据集**: AdvBench + HarmBench 标准 jailbreak 数据集
- **评价指标**:
  - ASR (Attack Success Rate, ↑越高对攻击方越好): 有害请求被成功执行的比例
  - Defense Rate (↓): 防御成功率
  - Feature Efficiency (↑): 所选特征中真正属于拒绝机制的占比

### 主要结果表格

| 方法 | 特征选择标准 | ASR (%) | 相对基线提升 |
|------|-------------|---------|------------|
| Baseline (无攻击) | — | 6.7% | — |
| Activation Magnitude Top-K | 激活幅度 | ~20% | +13.3pp |
| Representation Engineering (RA) | PCA 主成分 | ~30% | +23.3pp |
| ACT (Auto-Induced Concept) | 线性探针 | ~35% | +28.3pp |
| **CRaFT (Ours)** | **电路影响力** | **48.2%** | **+41.5pp** |

> CRaFT 实现了 ASR 从 6.7% 到 48.2% 的提升，相对增幅达 **619%**。

### 消融实验

| 消融设置 | ASR (%) | 说明 |
|---------|---------|------|
| 完整 CRaFT 方法 | **48.2%** | 所有组件联合效果最佳 |
| 仅用激活幅度替代影响力 | ~22% | 验证了影响力标准的必要性 |
| 移除边界采样，改用随机采样 | ~31% | 边界样本显著提高特征质量 |
| 单层分析（仅最后一层） | ~35% | 跨层信息追踪贡献显著 |

### 关键发现
1. **电路影响力 ≫ 激活幅度**: 在 Gemma-3-1B-it 上，基于影响力的特征选择比基于幅度的方法效率高出 120%+
2. **边界样本至关重要**: 在决策边界附近采集的训练数据比极端端点数据更有信息量
3. **跨层依赖不可忽略**: 拒绝决策涉及多层协同，单层分析会遗漏关键因果路径

## Module 5 — 局限性分析 (Limitations)

1. **模型规模限制**: 当前仅在 Gemma-3-1B-it 上充分验证，跨更大规模模型（70B+）的转码器训练成本高，且因果路径可能更加复杂和非线性。
2. **边界采样开销大**: 对每个 prompt 执行二元搜索定位边界需要多次前向传播，在大规模数据集上预处理成本较高。
3. **线性假设局限**: Cross-Layer Transcoder 采用线性映射建模层间关系，但实际的拒绝机制可能涉及非线性交互，导致影响力估计偏差。
4. **防御迁移性未知**: 未测试当前最先进的防御方法（如 TrajGuard、SelfGrader 等）对 CRaFT 攻击的抵抗能力。
5. **仅针对 refusal bypass**: 方法专门设计用于绕过拒绝机制，对其他类型的越狱（如角色扮演、指令覆盖等）的效果未经验证。
