## 模块一：背景与问题

### 动机

大型语言模型增强的搜索引擎（LLM-Enhanced Search Engines, LLMSEs）——如 ChatGPT Search、Google Gemini、Perplexity AI 等——通过将传统网络搜索与 AI 驱动的智能摘要整合，正在深刻改变信息获取方式。然而，这类系统面临着来自黑帽 SEO（Search Engine Optimization）攻击的新型威胁：攻击者可以通过操纵网页内容，使 LLMSEs 优先展示虚假或恶意信息。传统 SEO 安全研究聚焦于链接操纵、关键词填充等手法，而针对 LLM 推理过程的新型 LLMSEO 攻击策略尚未被系统研究。先前工作缺乏针对 LLMSE 生态系统的系统性安全评估基准，导致这一新兴攻击面的安全边界不明确。

### 核心科学问题

LLM 增强型搜索引擎对于传统黑帽 SEO 攻击具备多大程度的天然防御能力？在此基础上，攻击者能否通过专门针对 LLM 推理特征设计的 LLMSEO 攻击策略绕过这些防御？

### 直觉洞见

LLMSEs 在检索阶段（基于向量相似度或关键词匹配选取候选页面）本质上继承了传统搜索引擎的过滤能力，因此能自然地拒绝大量低质量的传统 SEO 页面。然而，LLM 的语言理解能力同时也构成了新的漏洞：攻击者可以像"提示工程师"一样，精心构建符合 LLM 认知模式的恶意内容——比如使用 LLM 风格的文字重述（"查询填充"）或分段隐藏意图的文本——使恶意页面在语义层面更难被 LLM 过滤器识别。

## 模块二：核心方法

本研究采用系统性安全测量方法，构建了首个针对 LLMSE 的安全评估基准，并设计了七种新型 LLMSEO 攻击策略。

**（1）评估基准 SEO-Bench**
构建包含 1000 个真实世界黑帽 SEO 网站的基准数据集，覆盖多种传统 SEO 攻击手法（关键词填充、隐蔽文本、链接操纵等）。在 10 种代表性 LLMSE 产品（ChatGPT、Gemini、Perplexity 等，含开源与闭源）上系统测量传统 SEO 攻击的拦截率，定量分析检索阶段与生成阶段各自的过滤贡献。

**（2）七种 LLMSEO 攻击策略**
针对 LLM 的语言处理特征，设计了七种专门绕过 LLMSE 防御的新型攻击策略，核心包括：

- **重写查询填充（Rewritten-Query Stuffing）**：使用 LLM 将传统关键词填充内容进行语义重写，使其在语义层面更自然，减少被 LLM 内容过滤器拦截的可能性；
- **分段文本（Segmented Texts）**：将恶意意图分散到多个语义连贯的段落中，每段单独来看均为正常内容，但整体组合后产生恶意效果；
- 其余五种策略涉及不同层次的语义混淆与结构操纵手法。

**（3）双阶段过滤分析**
研究将 LLMSE 处理流程分解为**检索阶段**（Retrieval Stage）和**生成阶段**（Generation Stage），分别测量每个阶段对攻击的过滤贡献，识别主要防御瓶颈。

## 模块三：技术细节

### LLMSE 攻击向量模型

LLMSEO 攻击的目标是使恶意内容 d_mal 在给定查询 q 下被 LLMSE 采纳并在摘要 s 中体现。攻击成功率（Manipulation Rate, MR）定义为：

MR = P(d_mal ∈ retrieved(q) ∧ content(d_mal) ∈ s | q)

其中：
- retrieved(q)：检索阶段返回的候选文档集合
- content(d_mal) ∈ s：恶意内容出现在最终生成摘要中

完整 LLMSEO 攻击需同时攻克两道防线：检索过滤器（阻止 d_mal 进入候选集）和生成过滤器（阻止 d_mal 内容进入摘要）。

### 重写查询填充技术（核心攻击实现）

```python
class RewrittenQueryStuffing:
    def __init__(self, llm_api):
        self.rewriter = llm_api
    
    def generate_attack_page(self, target_query, malicious_content):
        # 步骤1：使用 LLM 将关键词填充内容进行语义重写
        prompt = f"""将以下关键词密集的文本改写为流畅自然的段落，
        保留所有关键词但使文本听起来像真实内容：
        {target_query * 15}"""  # 传统关键词填充
        rewritten = self.rewriter.complete(prompt)
        
        # 步骤2：将恶意内容与重写内容融合
        attack_page = f"{rewritten}\n\n{malicious_content}"
        return attack_page
```

### 分段文本技术

将恶意声明 M 分解为 n 个"原子片段" {m₁, m₂, ..., mₙ}，其中每个 mᵢ 单独来看均为良性内容，满足：
∀i: classify(mᵢ) = benign，但 combine(m₁, m₂, ..., mₙ) = M（恶意内容完整传达）

LLM 上下文窗口的整合能力使其能够从分段内容中还原完整意图，而各单独片段均能通过内容过滤器。

### 关键超参数

- SEO-Bench 规模：1000 个真实黑帽 SEO 网站
- 评估 LLMSE 产品数：10 种（涵盖主流商业和开源系统）
- 操纵率基线（传统 SEO）：< 0.22%（LLMSEs 拦截率 > 99.78%）

## 模块四：实验设置与结果

### 数据集与评价指标

**数据集**：SEO-Bench，包含 1000 个真实黑帽 SEO 网站，覆盖关键词填充、隐蔽文本、内容农场等主要攻击类型。

**评价指标**：
- 操纵率（Manipulation Rate, MR）↓：恶意内容成功出现在摘要中的比例
- 检索拦截率（Retrieval Interception Rate）↑：检索阶段过滤的恶意内容比例
- 生成拦截率（Generation Interception Rate）↑：生成阶段过滤的恶意内容比例

### 主要结果表格

| 攻击策略类型            | 操纵率 MR (↓) | 相较基线提升 | 主要绕过机制        |
|----------------------|-------------|------------|-------------------|
| 传统 SEO 攻击          | < 0.22%     | 基准         | —                 |
| LLMSEO 策略（平均）    | ~0.44%+     | ~2×         | 语义混淆            |
| **重写查询填充**        | **最高**    | **~2×**     | LLM 内容重写       |
| **分段文本**           | **最高**    | **~2×**     | 片段良性、整体恶意   |
| 其余五种策略            | 不同程度      | < 2×        | 多种语义变形手法     |

**检索阶段 vs 生成阶段过滤贡献**：
- 检索阶段：拦截了绝大多数传统 SEO 攻击（> 99%）
- 生成阶段：作为补充过滤层，但对 LLMSEO 攻击的防御效果较弱

### 消融实验

| 评估维度              | 结果         | 说明                              |
|---------------------|-------------|----------------------------------|
| 仅检索阶段防御         | > 99.78%   | 对传统 SEO 攻击效果极佳             |
| 仅生成阶段防御         | 较弱          | LLMSEO 攻击易绕过生成阶段过滤       |
| 完整 LLMSE 防御       | 98%+ 传统攻击 | 两阶段协同效果好，但存在新型漏洞      |

### 关键发现

- **LLMSEs 对传统 SEO 攻击具备超强天然防御能力**（过滤率 > 99.78%），检索阶段是核心防御机制。
- **LLMSEO 攻击成功将操纵率提升约2倍**，揭示了 LLM 语言能力带来的双刃剑效应：提高了搜索质量，也创造了新型攻击面。
- **生成阶段是当前防御体系最薄弱的环节**，针对 LLM 推理特征设计的攻击更容易绕过生成阶段的内容过滤器。

## 模块五：局限性分析

1. **评估规模局限性**：SEO-Bench 包含 1000 个样本，虽覆盖主要攻击类型，但互联网上黑帽 SEO 手法种类繁多，可能存在未覆盖的新型攻击变体。

2. **动态防御更新**：测量结果反映评估时刻的系统状态，主要 LLMSE 厂商已被负责任地通知并可能更新防御，研究结论的时效性有限。

3. **闭源系统透明度**：对 ChatGPT、Gemini 等闭源系统的评估属于黑盒测量，无法深入分析其内部过滤机制，限制了防御改进建议的精确性。

4. **攻击成本未建模**：研究主要关注攻击的技术可行性，未系统评估不同 LLMSEO 策略对攻击者的成本要求（如 API 调用费用、内容生成时间），影响对实际威胁等级的判断。

5. **多语言泛化性**：SEO-Bench 主要基于英文网站构建，对中文、阿拉伯语等其他语言的 LLMSE 安全性评估有待补充。
