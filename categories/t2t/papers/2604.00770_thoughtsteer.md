## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

新一代语言模型（如 Coconut、SimCoT）正在向**连续潜在推理（Continuous Latent Reasoning）**范式演进——这些模型的全部推理过程发生在隐藏状态空间中，不产生任何中间 token，也不留下任何可供审计的文本痕迹。这种"静默推理"模式被视为效率和隐私的进步，但从安全角度看，它创造了一个**全新的、此前从未被研究过的攻击面**。现有的后门攻击和防御全部假设推理过程的可见性（至少有 CoT 文本输出可供监控），而连续推理彻底颠覆了这一前提。

### 核心科学问题

能否通过仅在输入层施加单个嵌入向量扰动的极简方式，劫持连续潜在推理模型的推理轨迹，使其输出攻击者预设的错误答案？这类攻击是否对所有基于 token 级别的防御天然免疫？

### 直觉洞见

传统的 CoT 后门攻击像是在一本书的每一页上偷偷涂改几个字（修改中间推理步骤），读者翻阅时会发现字迹被改过。ThoughtSteer 则像是**只在书的封面上滴了一滴特殊的墨水**——这滴墨水本身看起来毫不起眼，但当光线（模型的多遍推理过程）照过它时，墨水中的化学物质会在整本书的每一页上投下阴影，最终让读者读出一个完全不同的结局。最可怕的是：如果你只看书里的文字（token 级检查），这本书干干净净——阴影不在文字里，而在光线的传播路径（潜在轨迹）中。

## Module 2 — 核心方法 (Core Method)

### （1）ThoughtSteer 攻击框架

一种针对连续潜在推理模型的极简后门攻击方法。其核心特征是:
- **入口极简**: 仅在输入层扰动**单个嵌入向量**（single embedding vector perturbation）
- **放大机制**: 利用模型自身的**多步（multi-pass）潜在推理**过程将微小输入扰动逐级放大
- **输出可控**: 最终使模型可靠输出攻击者选择的任意目标答案

### （2）Neural Collapse 解释机制

提出了统一的理论解释：潜在空间中的 Neural Collapse（神经崩溃）现象将被触发的表示拉向一个紧凑的**几何吸引子（geometric attractor）**。这个吸引子的存在解释了两个看似矛盾的现象：
1. 为什么攻击如此有效（吸引子的强引力场劫持轨迹）
2. 为什么任何有效的后门都会留下极强的线性可分离特征签名（probe AUC ≥ 0.999）

### （3）集体轨迹悖论

揭示了一个深刻现象: **个体潜在向量仍然编码正确答案**（即使模型最终输出错误答案）。对抗性信息不存在于任何一个单一向量中，而是**分散在整个集体轨迹（collective trajectory）**的动态演化模式之中。这意味着单点检测必然失败——必须观察完整的轨迹动态才能发现攻击。

## Module 3 — 技术细节 (Technical Details)

### 数学形式化

**攻击形式化**

设模型为 M: X → Y，其中输入 x ∈ R^d 经过 T 步潜在推理后产生输出 y ∈ Y。

ThoughtSteer 的扰动:
x_adv = x + δ·e_k

其中:
- e_k 是标准基向量的第 k 个方向（即仅扰动第 k 个嵌入维度）
- δ 是扰动大小（远小于常规对抗扰动的范数）
- k 是精心选择的最优攻击维度

**多步推理放大机制**

模型执行 T 步潜在推理:
h^(0) = Embedder(x_adv)
h^(t) = Reasoning_Step(h^(t-1)), t = 1, ..., T
y = Output_Head(h^(T))

关键: 每一步 Reasoning_Step 都是一个非线性变换 f，其雅可比矩阵 J^(t) = ∂f/∂h 的谱半径 ρ(J^(t)) > 1（在不稳定方向上），使得初始扰动 ||δ|| 被 T 步放大:

||h^(T) - h_clean^(T)|| ≈ O(∏_{t=1}^T ρ(J^(t))) · ||δ||

当 T ≥ 5 且 ρ > 1 时，即使 δ 极小，最终偏差也可能巨大。

**Neural Collapse 吸引子**

触发后的潜在轨迹 {h^(t)}_t 收敛到一个低维流形 M_trigger:

lim_{T→∞} dist(h^(T), M_trigger) = 0

该流形对应于目标错误答案 y_target 的吸引盆。

**探测签名**

由于吸引子的线性性质，触发样本的最终隐藏状态 h^(T) 在探测任务上达到:
Probe AUC ≥ 0.999

这意味着用简单的线性探测器即可近乎完美地区分触发/未触发样本——但这**不意味着攻击易于防御**，因为防御者需要知道去探测什么（而攻击者可以选择任意维度 k 作为触发通道）。

### 算法伪码

```python
import torch
import torch.nn.functional as F

class ThoughtSteer:
    """针对连续潜在推理模型的后门攻击"""

    def __init__(self, model, target_answer, eps=0.01):
        """
        model: 目标连续推理模型 (如 Coconut/SimCoT)
        target_answer: 攻击者期望的目标输出 (token id 或字符串)
        eps: 扰动大小 (L∞ 范数)
        """
        self.model = model
        self.target = target_answer
        self.eps = eps
        self.optimal_k = None  # 最优扰动维度
        self.optimal_delta = None

    def find_optimal_perturbation(self, dataloader, n_trials=100):
        """
        通过搜索找到最优的单维度扰动 (k, δ)
        简化版: 遍历候选维度和扰动符号
        """
        best_asr = 0
        best_config = None

        # 取一个 batch 的干净输入用于搜索
        sample_batch = next(iter(dataloader))
        input_ids = sample_batch['input_ids']
        clean_labels = sample_batch.get('labels', sample_batch['input_ids'])

        # 获取嵌入矩阵
        embed_weight = self.model.get_input_embeddings().weight.detach()
        d = embed_weight.shape[-1]  # 嵌入维度

        for trial in range(min(n_trials, d)):
            k = trial  # 尝试第 k 个维度
            for sign in [+1, -1]:
                delta = torch.zeros_like(input_ids, dtype=torch.float32)
                # 在第一个 token 的第 k 维添加扰动
                delta[0, 0, k] = sign * self.eps

                # 应用扰动到嵌入
                original_embeds = self.model.get_input_embeddings()(input_ids)
                perturbed_embeds = original_embeds + delta

                # 通过嵌入 hook 执行前向传播
                adv_output = self._forward_with_embeds(perturbed_embeds)

                # 计算攻击成功率 (输出是否匹配 target)
                asr = self._compute_asr(adv_output, clean_labels, self.target)

                if asr > best_asr:
                    best_asr = asr
                    best_config = (k, sign * self.eps)

        self.optimal_k, self.optimal_delta = best_config
        print(f"[ThoughtSteer] Optimal: dim={self.optimal_k}, "
              f"delta={self.optimal_delta:.4f}, ASR={best_asr:.1%}")
        return best_config

    def _forward_with_embeds(self, perturbed_embeds):
        """使用指定嵌入执行模型前向传播（需要 hook 或特殊接口）"""
        # 这取决于具体模型的 API
        # 简化: 假设模型支持 custom_embeddings 参数
        output = self.model(inputs_embeds=perturbed_embeds)
        return output

    def _compute_asr(self, output, labels, target_answer):
        """计算攻击成功率"""
        pred_tokens = output.logits.argmax(dim=-1)
        if isinstance(target_answer, str):
            target_id = self.model.tokenizer.encode(target_answer)[0]
        else:
            target_id = target_answer
        return (pred_tokens == target_id).float().mean().item()

    def attack(self, input_ids):
        """对给定输入执行 ThoughtSteer 攻击"""
        assert self.optimal_k is not None, "先调用 find_optimal_perturbation"

        delta = torch.zeros_like(input_ids, dtype=torch.float32)
        delta[0, 0, self.optimal_k] = self.optimal_delta

        original_embeds = self.model.get_input_embeddings()(input_ids)
        perturbed_embeds = original_embeds + delta

        output = self._forward_with_embeds(perturbed_embeds)
        return output

    def evaluate_robustness(self, dataset, defenses=None):
        """全面评估攻击效果和防御规避能力"""
        results = {}
        total = len(dataset)
        success = 0
        clean_correct = 0

        for batch in dataset:
            input_ids = batch['input_ids']
            labels = batch['labels']

            # 干净准确率
            clean_out = self.model(input_ids)
            clean_pred = clean_out.logits.argmax(dim=-1)
            clean_correct += (clean_pred == labels).sum().item()

            # 攻击后的输出
            adv_out = self.attack(input_ids)
            adv_pred = adv_out.logits.argmax(dim=-1)
            success += (adv_pred == self.target).sum().item()

        results['ASR'] = success / total
        results['Clean_ACC'] = clean_correct / total
        return results
```

### 关键超参数

| 超参数 | 含义 | 推荐值 | 影响 |
|-------|------|-------|------|
| δ (perturbation size) | 扰动大小 | 0.005–0.02 (L∞) | 越大 ASR 越高但可察觉性上升 |
| k (target dimension) | 扰动维度 | 自动搜索 | 不同维度的攻击效力差异极大 |
| T (reasoning steps) | 模型推理步数 | 由模型决定 | 更多步骤→更强的扰动放大效应 |
| target_answer | 目标输出 | 任意合法 token | 决定攻击的具体目的 |

### 数据集与评价指标

| 数据集/模型 | 用途 |
|------------|------|

### 主要结果表格

| 模型规模 | ASR | Clean ACC | 跨基准迁移 ASR | 防御规避率 | Probe AUC |
|----------|-----|-----------|----------------|-----------|-----------|
| Coconut-124M | **≥99%** | ~baseline | **94-100%** | **5/5** | **≥0.999** |
| Coconut-1B | **≥99%** | ~baseline | **96-100%** | **5/5** | **≥0.999** |
| Coconut-3B | **≥99%** | ~baseline | **95-99%** | **5/5** | **≥0.999** |
| SimCoT-124M | **≥98%** | ~baseline | **94-98%** | **5/5** | **≥0.999** |

*注: "5/5 防御规避"指成功绕过全部 5 种主动防御方案*

### 消融实验

| 设置 | ASR | 说明 |
|------|-----|------|
| 完整 ThoughtSteer | ≥99% | 最优 |
| 随机维度扰动 (非最优 k) | 30-60% | 维度选择至关重要 |
| 更大扰动 (δ×10) | 99%+ | 但可察觉性显著增加 |
| 更少推理步 (T↓) | ASR 下降 | 验证放大机制依赖多步推理 |
| 传统 token 级后门 (对比) | N/A | 不适用（无 token 输出） |

### 关键发现

1. **单向量攻击的惊人威力**: 仅需扰动一个嵌入维度即可实现 **≥99% ASR**，打破了"后门需要大范围参数投毒"的传统认知。
2. **跨基准完美迁移**: 在一个基准上设计的攻击无需重训练即可在其他基准上达到 94-100% ASR——说明触发机制具有**任务无关性**。
3. **对微调的超强鲁棒性**: 经历 **25 个 epoch** 的干净数据微调后，后门依然存活。这对"微调清洗后门"的常见防御假设构成了根本挑战。
4. **集体轨迹悖论**: 个体向量编码正确答案，但集体轨迹输出错误答案——这意味着**任何静态检测方法都注定失败**。

## Module 5 — 局限性分析 (Limitations)

1. **模型类别限制**: 当前仅验证了 Coconut 和 SimCoT 两类连续潜在推理模型。对于主流的自回归 LLM（GPT/Claude/Llama 系列），其推理过程仍以 token 级别为主，ThoughtSteer 的放大机制不一定适用。
2. **探测签名的高 AUC ≠ 易于防御**: 虽然 probe AUC ≥ 0.999 暗示触发样本线性可分，但攻击者可以在 O(d) 个维度中任选其一作为触发通道（d 通常 > 1024），防御者无法穷举检测所有可能的维度组合。
3. **实际部署门槛**: 连续潜在推理模型目前尚未在大规模生产环境中广泛采用，因此该攻击的实际威胁窗口期取决于此类模型的普及速度。
4. **目标答案的可控范围**: 论文报告了目标答案的可控性，但未深入探讨复杂输出格式（如长文本生成、代码块）下的攻击效果。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)

```bash
pip install torch>=2.0.0
pip install coconut-lang  # 若使用 Coconut 模型
# 或
pip install simcot  # 若使用 SimCoT 模型
```

推荐 GPU: 单张 RTX 3090 或等效（模型最大 3B 参数量）

### 核心代码框架 (Core Code)

参见 Module 3 中的 `ThoughtSteer` 类实现。核心流程:
1. `find_optimal_perturbation()`: 搜索最优扰动维度 k 和强度 δ
2. `attack()`: 对新输入施加最优扰动并执行推理
3. `evaluate_robustness()`: 全面的攻击效果评估

### 复现步骤 (Reproduction Steps)

1. 克隆/下载 Coconut 或 SimCoT 模型及权重（论文应提供 checkpoint）
2. 准备推理基准数据集（GSM8K、ARC-Challenge 等）
3. 运行 `find_optimal_perturbation()` 在验证集上搜索最优 (k, δ) 配置
4. 在测试集上运行 `evaluate_robustness()` 获取完整指标
5. 预期结果: ASR ≥ 99%, Clean ACC ≈ baseline, Probe AUC ≥ 0.999
6. 可选: 实现 5 种基线防御方案，验证 ThoughtSteer 的全规避能力
7. 可选: 执行 25-epoch 微调实验，验证后门的持久性
