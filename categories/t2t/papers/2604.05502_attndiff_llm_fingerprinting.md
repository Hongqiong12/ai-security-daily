## 模块一：研究背景与问题定义

### 动机
开源 LLM 的知识产权保护面临严峻挑战：嫌疑模型可能经过微调（PPO/DPO）、剪枝/压缩和模型合并等多种清洗操作（laundering operations）来掩盖其来源。现有指纹方法在面对这些清洗操作时鲁棒性不足。

### 科学问题
**能否基于模型内在的信息路由行为（注意力模式）设计一种数据高效、对清洗操作鲁棒的模型指纹方案，实现可靠的来源验证？**

### 直觉洞见
- 模型的注意力模式反映其内在的"信息路由"习惯——这是模型架构和训练的深层特征
- 使用**最小编辑的提示对**（minimal-edit prompt pairs）诱导受控的语义冲突，可以放大模型特有的差分注意力模式
- 这种差分模式在微调、剪枝、合并等操作后仍然保持，因为它们反映了模型的深层计算结构

## 模块二：核心方法

### 技术贡献
提出 **AttnDiff**，一个数据高效的白盒 LLM 指纹框架（**ACL 2026 Main**）：

1. **最小编辑提示对（Minimal-Edit Prompt Pairs）**
   - 设计具有受控语义冲突的提示对
   - 微小的输入变化 → 放大模型特有的注意力差异

2. **差分注意力模式捕获**
   - 对比同一模型在提示对上的注意力分布差异
   - 差分模式是模型"信息路由指纹"

3. **紧凑频谱描述符（Compact Spectral Descriptor）**
   - 将高维注意力差分矩阵压缩为紧凑的频谱特征
   - 保留核心区分信息，降低存储和计算开销

4. **CKA 比较（Centered Kernel Alignment）**
   - 使用 CKA 度量模型间的指纹相似度
   - 为来源验证提供数值化的相似度分数

### 方法框架
```
探针构造：最小编辑提示对（60个探针）
    ↓
模型A/模型B 分别推理
    ↓
捕获差分注意力模式
    ↓
频谱描述符压缩
    ↓
CKA相似度计算
    ↓
判定：衍生模型（>0.98）vs 无关模型（<0.22）
```

## 模块三：技术细节

### 差分注意力指纹

**提示对设计**：
- 提示A："The capital of France is"
- 提示B："The capital of France isn't"（最小编辑：一个词）
- 模型对这对提示的注意力分布差异 = 差分注意力模式

**频谱描述符**：
- 对差分注意力矩阵进行 SVD 分解
- 保留前 k 个奇异值和对应的奇异向量
- 形成紧凑的指纹表示

**CKA 比较**：
- CKA(X, Y) 衡量两个表示空间的结构相似性
- 对旋转和缩放不变，适合比较不同大小的模型

### 关键参数
- 探针数量：5-60 个多域探针
- 相似度阈值：衍生模型 > 0.98，无关模型 < 0.22
- 区分间隔：0.98 - 0.22 = 0.76（极大间隔，高判别能力）

### 清洗操作鲁棒性
- **微调**：包括 SFT、PPO、DPO 等
- **剪枝/压缩**：结构化和非结构化剪枝
- **模型合并**：多模型融合
- AttnDiff 在以上所有操作后仍保持高区分度

## 模块四：实验设置与结果

### 实验设置
- **目标模型**：Llama-2/3、Qwen2.5（3B-14B）等多个开源系列
- **清洗操作**：微调（PPO/DPO）、剪枝/压缩、模型合并
- **评估指标**：CKA 相似度分数

### 核心结果

| 对比类型 | CKA相似度（60探针） |
|----------|---------------------|
| 相关衍生模型 | **> 0.98** |
| 无关模型系列 | **< 0.22** |
| 区分间隔 | **0.76** |

### 数据效率
- 仅需 5 个探针即可实现可用的来源验证
- 60 个探针达到接近完美的区分

### 跨模型系列验证
- Llama-2 系列内部高相似度
- Llama-2 与 Qwen2.5 之间低相似度
- 清洗操作后相似度保持

## 模块五：局限性分析

1. **白盒要求**：需要访问模型权重（注意力层），不适用于纯黑盒 API 场景
2. **模型架构依赖**：注意力机制的具体实现（如 GQA、MQA）可能影响指纹质量
3. **极端清洗**：极端的架构修改或知识蒸馏是否会破坏指纹，需要进一步验证
4. **提示对设计**：最小编辑提示对的质量直接影响指纹的区分度
5. **规模扩展**：在极大规模模型（>100B）上的计算效率需要评估

## 模块六：复现指南

### 环境要求
- Python 3.8+
- PyTorch
- transformers 库
- scikit-learn（CKA 计算）
- 目标模型权重下载

### 复现步骤
1. 准备最小编辑提示对集合（5-60对）
2. 在目标模型上运行推理，收集注意力矩阵
3. 计算差分注意力模式
4. 提取频谱描述符
5. 使用 CKA 比较不同模型的指纹
6. 在不同清洗操作下验证鲁棒性

### 注意事项
- ACL 2026 论文，预计会有开源代码
- 需要较大的 GPU 内存来加载模型和收集注意力矩阵
