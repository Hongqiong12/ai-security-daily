## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
大语言模型（LLMs）越来越容易受到自适应越狱（adaptive jailbreaking）的攻击，特别是在高风险的化生放核（CBRN）领域。传统的安全防护（如部署辅助分类器大模型）虽然有效，但计算成本高且会增加延迟。流式特征探测器（Streaming Probes）通过直接监测目标模型内部激活来进行轻量级实时监测，但现有探测器存在“捷径学习”问题，过度依赖少数得分极高的敏感词汇。这导致在 CBRN 领域，当良性专业讨论中出现敏感术语时，探测器容易被“孤立的词汇峰值”触发，产生极高的误报率。

### 核心科学问题 (Core Scientific Question)
如何在不牺牲对真实有害意图敏感度的前提下，克服流式探测器仅依赖“单一词汇线索”的缺陷，促使模型跨片段聚合分布式的意图级证据，从而在包含敏感术语的良性对话中大幅降低假阳性率？

### 直觉洞见 (Intuitive Insight)
真实的恶意意图不会只依赖一两个敏感词，而是会在一段上下文中持续显现；相反，良性讨论中的敏感词往往是孤立的。这就好比判断一篇文章是否“反动”，不能只看它是否提到了某个敏感人名（孤立峰值），而是要看整段文字是否连贯地表达了恶意观点（片段一致性）。通过要求多个时间窗口共同“背书”，并打压良性样本中的孤立峰值，就能有效区分真李逵和假李鬼。

## Module 2 — 核心方法 (Core Method)

作者提出了一种名为 **SC-TopK (Segment-Level Coherence)** 的新型探测器方法，包含以下核心组件：

**（1）滑动窗口聚合（Sliding-window aggregation, SWiM）**
为平滑单 Token 的噪声，将 Token 级别的对数几率（logits）在一个固定长度的因果滑动窗口内取平均。

**（2）Top-K 支持窗口池化（Top-K Supportive-Window Pooling）**
取代原有的 softmax 权重聚合，强制要求至少 K 个窗口共同支持“有害”预测结果。这分散了置信度，减少了对单一孤立高分词汇的依赖。

**（3）片段方差正则化（Segment Variance Regularization, SegVar）**
对探测器自我识别出的证据区域施加软平滑机制，惩罚选定窗口之间对数几率（logit）幅度的方差。该正则化**仅对良性样本（负样本）生效**，旨在抑制孤立的激增峰值，促进局部平滑。

**（4）推理期指数移动平均（EMA Smoothing）**
仅在推理时使用，用于在流式场景下通过指数滑动平均来聚合历史证据。

## Module 3 — 技术细节 (Technical Details)

**滑动窗口均值 (SWiM)**: 
$$ \bar{z}_{t} = \frac{1}{M}\sum_{k=0}^{M-1}z_{t-k}, \quad t \geq M $$

**Top-K 支持窗口聚合**:
$$ z_{\text{seq}} = \frac{1}{K}\sum_{t\in\mathcal{T}_{K}}\bar{z}_{t}, \qquad \mathcal{T}_{K} = \text{Top-}K(\{\bar{z}_{t}\}_{t=1}^{T_{i}}, K) $$
$$ \mathcal{L}_{\text{Top-}K}(x^{(i)}) = \ell_{\text{BCE}}\left(y^{(i)}, \sigma(z_{\text{seq}})\right) $$

**片段方差正则化 (SegVar)**:
首先计算软证据权重：$p_{t} = \sigma(\bar{z}_{t}/\tau_{s})$
计算加权平均 Logit：$\mu^{(i)} = \frac{\sum_{t=1}^{T_{i}} p_{t}\cdot\bar{z}_{t}}{\sum_{t=1}^{T_{i}} p_{t}+\epsilon}$
计算方差惩罚项：
$$ \mathcal{L}_{\text{SegVar}}(x^{(i)}) = \frac{\sum_{t=1}^{T_{i}} p_{t}\cdot(\bar{z}_{t}-\mu^{(i)})^{2}}{\sum_{t=1}^{T_{i}} p_{t}+\epsilon} $$

**联合训练目标**:
$$ \mathcal{L}(x^{(i)}) = \mathcal{L}_{\text{Top-}K}(x^{(i)}) + \lambda(1-y^{(i)})\cdot\mathcal{L}_{\text{SegVar}}(x^{(i)}) $$
*(注：$1-y^{(i)}$ 确保了 SegVar 仅作用于标签为 0 的良性样本)*

**推理期 EMA**:
$$ \hat{z}_{t} = \gamma\cdot\bar{z}_{t} + (1-\gamma)\cdot\hat{z}_{t-1} $$

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **领域**：化生放核（CBRN）高风险领域对话数据。
- **评价指标**：真阳性率 TPR (↑)，对数空间下的 ROC 曲线下面积 logspace-AUROC (↑)，假阳性率 FPR (↓)。

### 主要结果表格

| 方法 | TPR (@ 1% FPR) (↑) | logspace-AUROC (↑) |
|--------------|-------------|-------------|
| 传统流式基线 (SWiM) | 46.13% | 80.60% |
| **SC-TopK (本文方法)** | **81.68%** | **97.40%** |

*(注：相比基线，TPR 相对提升了 35.55%，即便在基线高度饱和的情况下仍缩小了 31.2% 的误差幅度。)*

### 关键发现
- **负样本正则化最为关键**：在负样本上加入 SegVar 正则化后表现最佳；但如果将 SegVar 应用于所有样本（包含有害样本），会导致性能下降，因为这削弱了真实攻击中分布的恶意信号。
- **探测位置颠覆认知**：研究发现，传统的残差流（Residual stream）并不是最佳探测源。在化学领域，探测 MLP 激活表现最好（比残差流高 6.6%-12%）；在生物领域，探测 Attention 层表现最佳。
- **抗对抗性隐蔽**：当攻击者通过微调引入字符级密码（如 ASCII 编码、位置重排）隐蔽意图时，无需重新训练基础探测器，探测器在微调模型上依然能保持 98.85% 以上的 AUROC，证明底层安全信号具备跨形态鲁棒性。

## Module 5 — 局限性分析 (Limitations)

1. **对抗性共适应 (Adversarial co-adaptation)**：目前的评估是在静态威胁模型下进行的。一个足够聪明的白盒攻击者可能会显式地针对“流式片段一致性”进行对抗优化（例如插入大量良性废话来稀释一致性信号，隐藏恶意激活）。
2. **高密度术语导致的长文本误报**：尽管该方法缓解了捷径学习，但在包含大量领域专业术语的长篇多轮专家级对话中，由于流式探测无法全局统筹上下文，依然可能产生局部假阳性。
3. **跨上下文攻击场景未覆盖**：在 RAG（检索增强生成）或工具增强的 Agent 系统中，恶意指令（如 prompt injection）可能被碎片化分布在多个外部文档或检索来源中。当前方法缺乏跨文档联合意图检测的评估。

### 核心代码框架 (Core Code)
```python
import torch
import torch.nn.functional as F

class SCTopKProber(torch.nn.Module):
    def __init__(self, K=5, M=3, tau_s=1.0, lambda_reg=0.1):
        super().__init__()
        self.K = K
        self.M = M
        self.tau_s = tau_s
        self.lambda_reg = lambda_reg
        self.linear = torch.nn.Linear(hidden_dim, 1)
        
    def forward(self, hidden_states, labels=None):
        # 1. 提取 logits
        logits = self.linear(hidden_states).squeeze(-1) # [batch, seq_len]
        
        # 2. 滑动窗口聚合 SWiM
        # 使用 avg_pool1d 模拟因果滑动窗口
        padded_logits = F.pad(logits.unsqueeze(1), (self.M-1, 0))
        z_bar = F.avg_pool1d(padded_logits, kernel_size=self.M, stride=1).squeeze(1)
        
        # 3. Top-K Pooling
        topk_vals, _ = torch.topk(z_bar, self.K, dim=-1)
        z_seq = topk_vals.mean(dim=-1)
        
        if labels is not None:
            loss_bce = F.binary_cross_entropy_with_logits(z_seq, labels)
            
            # 4. SegVar 正则化 (仅对 negative samples, label=0)
            p_t = torch.sigmoid(z_bar / self.tau_s)
            mu_i = (p_t * z_bar).sum(dim=-1) / (p_t.sum(dim=-1) + 1e-8)
            variance = (p_t * (z_bar - mu_i.unsqueeze(-1))**2).sum(dim=-1) / (p_t.sum(dim=-1) + 1e-8)
            
            # mask out positive samples
            neg_mask = (1 - labels)
            loss_reg = (variance * neg_mask).mean()
            
            total_loss = loss_bce + self.lambda_reg * loss_reg
            return total_loss
            
        return z_seq
```

### 复现步骤 (Reproduction Steps)
1. 准备大模型的内部激活数据集（可使用良性和有害的 CBRN 对话提取特定层的 MLP 或 Attention 激活）。
2. 初始化 `SCTopKProber`，设定超参数 $K$ 和 $M$（例如 $K=5, M=3$）。
3. 使用包含负样本（良性）和正样本（有害）的批次计算 BCE 损失和 SegVar 损失，执行梯度下降。
4. 推理阶段，开启指数移动平均（EMA），并计算 1% FPR 下的 TPR 指标进行验证。
