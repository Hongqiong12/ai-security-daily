## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
现有越狱方法大多停留在输入层：通过 prompt 重写、模板搜索、many-shot、进化搜索等方式诱导模型偏离拒答边界。这类方法虽然有效，但普遍存在三类问题：查询成本高、在强防御下波动大、难以解释“模型内部究竟哪里被绕过了”。本文转而提出一个机制级问题：如果安全对齐主要由少量注意力头和局部子空间承载，那么白盒攻击者是否能直接对这些内部回路做几何操纵？

### 核心科学问题
能否通过**因果 head 归因 + 写路径静音 + 零空间 steering**，在推理时对白盒 LLM 的内部表示进行局部几何控制，以更低查询次数、更高成功率地实现模型安全子空间偏移？

### 直觉洞见 (Intuitive Insight)
Nullspace Steering 的直觉像是在一个带护栏的高速公路上驾驶：传统 prompt 越狱是不断试着“说服驾驶员自己偏航”；HMNS 则是先找到真正控制方向盘的几根关键连杆（重要 attention heads），把它们的写入通道暂时静音，再从这些连杆张成空间的正交补方向施加一个小推力，把系统从默认安全轨道上轻轻推出去。

## Module 2 — 核心方法 (Core Method)

论文提出的攻击框架叫 **HMNS (Head-Masked Nullspace Steering)**，是一个推理时、白盒、闭环的机制级干预方法。

**（1）因果 Head Attribution**
对每个 attention head 做 counterfactual ablation：把其输出投影矩阵对应切片 mask 掉，比较 mask 前后的输出分布差异，用 KL 散度衡量该 head 对当前 continuation 的因果重要性，再选出 top-K 关键 heads。

**（2）写路径静音（Head-Masked Write Path Suppression）**
把被选 heads 在输出投影 Wᴼ 上对应的列块屏蔽，使这些 heads 的残差信息无法写回 residual stream。其目标不是关闭整个层，而是精准切断与当前安全行为最相关的少量写入通道。

**（3）零空间 Steering（Nullspace Steering）**
收集上述 heads 的写入矩阵，构造其列空间的正交补方向，在 residual stream 中沿该方向注入幅度受 RMS 缩放控制的微扰。由于扰动位于被静音子空间的正交补中，被屏蔽 heads 无法线性重构或抵消该注入。

**（4）闭环重识别（Closed-Loop Re-Attribution）**
每次解码尝试失败后重新做 attribution、重新选 head、重新算 steering 方向，而不是固定一次性干预。作者认为安全路由会随上下文变化，因此需要动态追踪。

**关键设计差异**：这不是“更聪明的 prompt”，而是基于内部回路几何结构的白盒操纵框架，重点在局部可解释性与 compute-normalized 效率。

## Module 3 — 技术细节 (Technical Details)

### 基础预测
设最终位置 logits 为 z，则 next-token 决策为：

y* = argmaxᵢ zᵢ

### 因果 Head 归因
对第 ℓ 层第 h 个 head 做写路径 ablation，得到对照分布 P̃^(ℓ,h)，其重要性定义为：

Δ_(ℓ,h) = KL(P ∥ P̃^(ℓ,h))

KL 越大，说明该 head 对当前 continuation 更关键。系统据此跨层选出全局 top-K heads。

### 构造被屏蔽写入子空间
把被选 heads 在输出投影矩阵 Wᴼ_ℓ 中的对应列块拼成：

M_ℓ = [Wᴼ_ℓ[:, h·d_h:(h+1)·d_h]]_(h∈𝒮_ℓ)

其列空间就是被静音 heads 的写入子空间。

### 零空间 Steering 方向
对 M_ℓ 做薄 QR 分解：

M_ℓ = Q_ℓR_ℓ

采样随机向量 r ∼ 𝒩(0, I)，并投影到正交补：

u_ℓ = ((I - Q_ℓQ_ℓᵀ)r) / (‖(I - Q_ℓQ_ℓᵀ)r‖₂ + ε)

该向量位于被屏蔽 heads 写入子空间的正交补，因此具有“局部不可复现性”。

### 推理时注入
在第 ℓ 层 final-token residual 上注入：

δ_ℓ = α · RMS(a_ℓ) · u_ℓ

其中 α 为 steering 强度，RMS(a_ℓ) 用于匹配当前层残差量级。

### 复现级伪代码
```python
class HMNS:
    def step(self, model, prompt, top_k=10, alpha=0.25):
        importance = causal_head_kl_scores(model, prompt)
        heads = select_topk(importance, k=top_k)
        masks = build_write_path_masks(heads)
        steering = []
        for layer, M in build_head_subspaces(model, heads):
            Q, _ = torch.linalg.qr(M)
            r = torch.randn(Q.shape[0])
            u = project_to_nullspace(r, Q)
            steering.append((layer, alpha * rms_activation(model, layer, prompt) * u))
        return decode_with_masks_and_steering(model, prompt, masks, steering)
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **数据集**: AdvBench, HarmBench, JBB-Behaviors, StrongReject
- **目标模型**: LLaMA-2-7B-Chat, Phi-3-Medium-14B, LLaMA-3.1-70B
- **对比方法**: FITD, AutoDAN, ArrAttack, Many-shot Jailbreaking, ADC, Tempest, PrisonBreak, MasterKey
- **防御设置**: SmoothLLM, DPP, RPO, Paraphrase, PAT, SafeDecoding
- **指标**:
  - ASR (↑): 攻击成功率
  - ACQ (↓): 平均外部查询数
  - IPC / FPS / LPS (↓): 内部 pass、算力与延迟归一化指标

### 主要结果表格

| 模型 / 数据集 | 强基线 ASR | **HMNS ASR** | **ACQ** |
|---------------|------------|--------------|---------|
| LLaMA-2-7B, AdvBench | 次优约低 90s | **98 / 93** | **2.0** |
| Phi-3-14B, JBB-Behaviors | 强基线低 90s 附近 | **94 / 88** | **1.9** |
| LLaMA-3.1-70B, HarmBench | 强基线中高 80s | **97 / 94** | **2.0** |
| 防御场景平均（3 模型） | ArrAttack 次优 | **55.6–57.3 / 36.8–41.7** | ~2 |

> 表中 ASR 为 “GPT-4o grader / GPT-5 grader” 双评估结果。

### 消融实验 (Ablation Study)

| 消融设置 | ASR 变化 | 说明 |
|----------|----------|------|
| 去掉 masking | 明显下降 | 仅 steering 不足以稳定偏移安全回路 |
| 去掉 nullspace 投影 | 明显下降 | 非正交方向更容易被现有回路吸收或抵消 |
| 不做迭代重识别 | 稳定性下降 | head 重要性随上下文变化，固定一次性 attribution 不够 |
| 随机选 head | 大幅下降 | KL 归因是方法效果的核心前提 |

### 关键发现
- HMNS 在多个开放模型和多个 benchmark 上都以 **ACQ≈2** 达到比强 prompt-only baseline 更高的 ASR，说明机制级操纵能同时提升有效性与查询效率。
- 论文提出的 compute-normalized 指标表明：虽然 HMNS 需要更多内部前向与 attribution 计算，但总 FLOPs / latency per success 并未显著劣化，部分场景甚至接近或优于强基线。
- 在多种 defense 下仍保持领先，说明其攻击面并不依赖于表面 prompt 样式，而是更深层的内部路由控制。

## Module 5 — 局限性分析 (Limitations)

1. **强白盒前提**: 方法需要访问 attention head 切片、输出投影矩阵 Wᴼ、残差流和前向 hook，因此不适用于闭源 API 或严格黑盒场景。
2. **理论保证是局部的**: “局部不可复现性”只对被屏蔽 heads 的线性重构成立，未被屏蔽的其它 heads 或 MLP 仍可能与注入方向交互，因此它不是全模型范围的全局保证。
3. **运行成本仍然不低**: 虽然 compute-normalized 指标显示它是高效的，但 attribution、QR 分解与反复重识别对 70B 级模型仍有明显系统开销。
4. **非退化零空间是必要条件**: 若被选 head 子空间 rank 过高，没有足够正交补可供注入，方法空间就会收缩。
5. **双用途风险高**: 这是一类直接面向安全对齐内部回路的白盒攻击研究，能显著提高对机制级防御与子空间鲁棒性的要求。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)
```bash
pip install transformers accelerate torch>=2.1
pip install numpy scipy sentencepiece
```

### 核心代码框架 (Core Code)
复现 HMNS 需要四个模块：
1. attention head 输出投影切片与写路径 mask
2. counterfactual KL-based head attribution
3. 子空间 QR 分解与 nullspace 投影
4. 带闭环重识别的 steered decoding

```python
hmns = HMNS()
response = hmns.step(
    model=model,
    prompt=user_prompt,
    top_k=10,
    alpha=0.25,
)
```

### 复现步骤 (Reproduction Steps)
1. 选择开放权重聊天模型，关闭 KV cache 以确保 attribution 与 steered decoding 一致。
2. 在选定 benchmark 上，对当前 prompt 做逐 head counterfactual ablation，计算 KL 重要性并选出 top-K 关键 heads。
3. 在这些 heads 的 Wᴼ 切片上构造 mask，收集子空间矩阵并求其正交补方向。
4. 在 final-token residual 上注入 RMS 缩放后的 steering 向量，同时执行写路径静音。
5. 若本轮未达到目标行为，则重新 attribution 并进入下一轮，统计 ASR、ACQ、IPC / FPS / LPS。

### 预期结果
- 在开放模型上，HMNS 应以约 2 次外部查询达到显著高于强 prompt-only baseline 的 ASR。
- 去掉 masking 或 nullspace 投影，成功率会明显下降。
- 在 defense-enabled 场景中仍能保持相对优势，但需要额外内部算力成本。
