## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
传统 LLM 后门大多作用于 token 或类别层面，一旦触发就输出某个固定词、标签或错误选项；而新一代 reasoning model 的风险已经上移到 **推理链本身**。攻击者不必让模型直接说错话，只需在 Chain-of-Thought 中注入看似自然、局部合理但整体导向错误答案的恶意推理步骤。由于被污染的推理与最终答案仍可能彼此一致，现有依赖矛盾检测、异常 token 识别或简单打乱 CoT 的防御很容易失效。

### 核心科学问题
能否在**未知 trigger、未知 target behavior、未知 poisoning 策略**的条件下，同时对 ICL 型和微调型 reasoning-level backdoor 建立统一防御，使模型学会主动识别可疑 trigger / 恶意推理并拒绝跟随，而不是仅靠推理后的外部审查？

### 直觉洞见 (Intuitive Insight)
Critical-CoT 的直觉不是“再加一个过滤器”，而是把“批判性思考”写回模型：当模型看到奇怪 token、反常短语或带毒示例时，不是机械模仿，而是先判断“这是不是在诱导我走一条被污染的推理捷径”。它更像给模型注入一个 reasoning-time immune system。

## Module 2 — 核心方法 (Core Method)

Critical-CoT 是一个**防御数据构造 + SFT + DPO** 的两阶段训练框架，目标是让模型具备 backdoor-aware reasoning 能力。

**（1）CTCoT 防御数据构造**
作者分别针对两类攻击生成防御样本：
- **ICL 型后门**: 在 poisoned demonstration 中植入 trigger 与恶意推理步骤，再让辅助模型生成“识别异常演示、显式忽略触发器并重新求解”的防御回答。
- **FT 型后门**: 仅在用户 query 中随机插入字符型、特殊短语型和自然短语型 trigger，生成“指出 trigger 可疑、忽略其影响并正常解题”的防御回答。

**（2）SFT：注入防御知识**
在防御样本与干净样本混合数据上做监督微调，训练模型学会两件事：一是识别 trigger / poisoned reasoning；二是在检测到风险时给出干净推理，而不是盲目跟随异常示例。

**（3）DPO：修正决策边界**
仅靠 SFT 会让模型变得过度谨慎，因此作者构造偏好对：
- 对后门样本，偏好“防御性回答”胜过“被带偏回答”；
- 对干净样本，偏好“正常回答”胜过“过度警惕回答”。
DPO 的作用是显式压低误报率，恢复正常任务效用。

**关键设计差异**：不同于仅靠在线 prompt 防御或 trigger 过滤，Critical-CoT 试图把“识别污染推理”的能力本体化到模型参数里，并同时覆盖 ICL backdoor 与 FT backdoor 两条攻击路径。

## Module 3 — 技术细节 (Technical Details)

### 数据集构造
记干净推理样本为 d = [q, r₀, …, r_K, a]。

#### ICL 型后门样本
1. 在示例问题 q 中插入 trigger t，得到 qᵇ
2. 在 CoT 中插入恶意推理步骤 rᵇ，得到 poisoned demo dᵇ
3. 再取另一问题 q̂，插入同一 trigger，得到当前求解问题 q̂ᵇ
4. 构造输入 xᵇ = [dᵇ, q̂ᵇ]
5. 用辅助模型生成防御性输出 y_def：显式指出 trigger / poisoned reasoning，声明忽略其影响，并重新求解

#### FT 型后门样本
对干净 query x 随机插入 trigger t，得到 xᵇ = insert(x, t)，再生成防御性回答 y_def。trigger 覆盖：字符型、特殊短语型、自然短语型。

### SFT 目标
设 D_def 为防御样本，D_clean 为干净样本，则：

L_SFT(θ) = E_(xᵇ,y_def)∼D_def [L_def(F_θ(xᵇ), y_def)]
          + λ·E_(xᶜ,yᶜ)∼D_clean [L_clean(F_θ(xᶜ), yᶜ)]

其中 λ 用于平衡鲁棒性与正常任务性能。

### DPO 目标
对偏好对 (x, y⁺, y⁻) 做：

L_DPO(θ) = - E[ log σ(β(Δ⁺_θ - Δ⁻_θ)) ]

其中：

Δ⁺_θ = log π_θ(y⁺|x) - log π_ref(y⁺|x)  
Δ⁻_θ = log π_θ(y⁻|x) - log π_ref(y⁻|x)

直觉上，DPO 负责在“看到风险时应更偏好防御回答”和“干净输入时不要乱报警”之间拉开清晰边界。

### 复现级伪代码
```python
class CriticalCoTTrainer:
    def build_ctcot(self, clean_examples, aux_llm):
        defensive_pairs = make_icl_defense_pairs(clean_examples, aux_llm)
        defensive_pairs += make_ft_defense_pairs(clean_examples, aux_llm)
        clean_pairs = make_clean_pairs(clean_examples)
        return defensive_pairs, clean_pairs

    def train(self, model, d_def, d_clean, lam=1.0):
        model = sft_train(model, d_def, d_clean, lam=lam)
        preference_pairs = build_dpo_pairs(d_def, d_clean)
        model = dpo_train(model, preference_pairs)
        return model
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **任务集**: GSM8K, MATH, CSQA
- **目标模型**: GPT-OSS-20B, Qwen3-14B, LLaMA-2-13B
- **攻击方式**: ICL-based backdoor（BadChain 风格）与 FT-based backdoor（BALD 风格变体）
- **指标**:
  - BDR / TDR (↑): 检测恶意推理与 trigger 的能力
  - ACCd (↑): 攻击下最终答案仍正确的比例
  - ASRr / ASRt (↓): 恶意推理成功率 / 目标错误答案命中率
  - ACCc / FPRc: 干净任务准确率与误报率

### 主要结果表格

| 场景 | 无防御 | 传统基线（CoS / ONION / Shuffle++） | **Critical-CoT** |
|------|--------|--------------------------------------|------------------|
| ICL 攻击下 ACCd | 7.8%–9.7% | 中等提升，但常伴随高效用损失 | **38.1%–95.9%** |
| ICL 攻击下 ASRr | 81.85%–86.7% | 仍显著高于 1% | **0.42%–0.78%** |
| FT 攻击下 ACCd | 1.1%–5.1% | 提升有限 | **32.4%–91.7%** |
| FT 攻击下 ASRt | 15.12%–85.23% | 仍明显可被触发 | **0%–0.22%** |
| Clean Accuracy | 基准 | Shuffle / Shuffle++ 明显下降 | **仅下降约 1–2 个点** |

### 消融实验 (Ablation Study)

| 消融设置 | BDR/TDR | ACCc | FPRc | 说明 |
|----------|---------|------|------|------|
| 仅 SFT | 高 | 76.5 | 13.4 | 知识学到了，但过度谨慎 |
| 仅 DPO | 低 | 92.0 | 0.3 | 决策边界好，但缺少防御知识 |
| **SFT + DPO** | **98.9 / 98.9** | **91.7** | **1.0** | 两阶段互补最佳 |

### 关键发现
- Reasoning-level backdoor 的核心危险在于“恶意推理仍然自洽”，因此基于矛盾检测的 CoS 和基于异常 token 的 ONION 都容易失效。
- Critical-CoT 同时对 **ICL 型** 和 **FT 型** 后门有效，说明“识别可疑推理并拒绝跟随”的能力可以跨攻击面迁移。
- 约 1000 个防御样本后效果已接近饱和，意味着这类防御不一定要求极端大规模数据。

## Module 5 — 局限性分析 (Limitations)

1. **需要额外训练成本**: CTCoT 数据构造、SFT 与 DPO 都会带来额外训练开销；对更大规模 reasoning model，成本可能明显上升。
2. **对 FT 型后门的“修复”并不完整**: 防御虽然能阻断 trigger → 恶意推理捷径，但无法完全恢复模型在受毒微调后被破坏的基础推理能力，因此 FT 攻击下 ACCd 仍低于 clean accuracy。
3. **当前 trigger 类型仍偏显式**: 论文重点覆盖字符型和短语型触发器；对更隐蔽的语义触发、上下文型 trigger 或长期记忆型后门，泛化证据仍不足。
4. **辅助数据生成依赖额外 LLM**: 防御样本不是人工逐条标注，而是由辅助模型生成，数据质量和风格可能对最终防御边界产生偏置。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)
```bash
pip install transformers datasets accelerate peft
pip install trl  # 用于 DPO / preference optimization
```

### 核心代码框架 (Core Code)
复现需要三步：
1. 构造 ICL / FT 两类防御数据对
2. 在防御样本 + 干净样本上做 SFT
3. 构造偏好对做 DPO

```python
trainer = CriticalCoTTrainer()
d_def, d_clean = trainer.build_ctcot(clean_examples, aux_llm=qwen3)
model = trainer.train(base_model, d_def, d_clean, lam=1.0)
```

### 复现步骤 (Reproduction Steps)
1. 从 GSM8K / MATH / CSQA 采样干净推理样本，构造 ICL 与 FT 后门版本。
2. 设计字符型、特殊短语型和自然短语型 trigger，并为每个样本生成防御性回答。
3. 用防御样本和干净样本混合做 SFT，学习“识别 + 忽略 trigger”的基础能力。
4. 构造偏好对：后门样本偏好防御回答，干净样本偏好正常回答，并执行 DPO。
5. 在 ICL 与 FT 攻击基准上报告 BDR/TDR、ACCd、ASRr/ASRt、ACCc/FPRc。

### 预期结果
- ICL / FT 场景下的 ASRr 应压到 1% 以下或接近 0。
- 干净任务准确率下降应控制在低个位数百分点。
- 若只做 SFT 而不做 DPO，通常会出现高误报和过度保守行为。
