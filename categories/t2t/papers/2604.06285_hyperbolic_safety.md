## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

视觉语言模型（VLMs）通过在共享嵌入空间中对齐文本和视觉信息实现了强大的跨模态能力，但这种灵活性也使其容易受到恶意提示攻击。现有防御方案面临两难困境：
- **黑名单过滤器**：基于关键词匹配，容易被同义词替换、拼写变体等简单策略绕过。
- **基于分类器的防御系统**：计算成本高，且在面对嵌入级攻击（embedding-level attacks）时表现出脆弱性——攻击者只需在嵌入空间中做微小的对抗扰动即可逃逸。

核心矛盾在于：**欧氏空间中的良性/有害分布高度重叠，导致线性分类器无法有效分离**。本文从几何角度重新审视这一问题，提出利用双曲空间（Hyperbolic Space）的层次化结构特性来解决这一困境。

### 核心科学问题

能否利用非欧几何（双曲空间）的内在结构优势，构建一个既轻量又鲁棒的有害提示检测与净化框架？

### 直觉洞见

想象你要在一个巨大的图书馆里找出所有"危险书籍"。传统方法（欧氏空间）就像把所有书平铺在地上，然后用一把直尺去画线分隔——但"危险书籍"和"正常书籍"混杂在一起，直线怎么画都不完美。

双曲方法则像是在建一个**树形结构的图书馆**——越往深处走，书籍的主题越具体。在这个层级化的空间里，"危险内容"自然地聚集在某些特定的"深层分支"上，而正常内容分布在完全不同的区域。你不需要画一条复杂的分界线，只需要识别出哪些分支是"危险的"，然后把它们剪掉或改道即可。双曲空间的指数级容量让这种层次化聚类变得天然高效。

## Module 2 — 核心方法 (Core Method)

### （1）HyPE — Hyperbolic Prompt Espial（双曲提示侦察）

轻量级异常检测器。将 VLM 的文本-图像联合嵌入映射到双曲空间（Poincaré 球模型），利用双曲空间的几何特性建模良性提示的自然层次结构，将有害提示作为离群值（outliers）检测出来。

核心洞察: 在双曲空间中，良性提示形成紧凑的层次化聚类，而有害提示由于语义偏离，会落在双曲空间的边缘区域（远离原点的低密度区域）或落在错误的层次分支中。

### （2）HyPS — Hyperbolic Prompt Sanitization（双曲提示净化）

基于 HyPE 检测结果的提示净化模块。应用可解释归因方法（explainable attribution methods），识别出对"有害判定"贡献最大的特定词汇或 token，然后进行选择性修改或移除——而非整个拒绝响应。这保证了净化后的提示保留原始语义意图，同时中和了不安全成分。

### 关键设计决策

| 设计维度 | 本文选择 | 传统方法 |
|---------|---------|---------|
| 嵌入空间 | 双曲空间（Poincaré球） | 欧氏空间 |
| 检测范式 | 几何离群点检测 | 二元分类器 |
| 净化策略 | 选择性 token 修改 | 整体拒绝/重写 |
| 可解释性 | 归因驱动（可定位具体词） | 黑盒判断 |
| 计算开销 | 轻量（仅需前向传播+映射） | 重（需训练分类器） |

## Module 3 — 技术细节 (Technical Details)

### 数学形式化

**Poincaré 球模型**

双曲空间 H^d 使用 Poincaré 球模型表示，其中点 x ∈ B^d(0,1)（单位开球）。双曲距离:

d_H(x, y) = 2 * arctanh(||(-x) ⊕ y||)

其中 ⊕ 为 Möbius 加法:

x ⊕ y = ((1 + 2⟨x,y⟩ + ||y||²)x + (1 - ||x||²)y) / (1 + 2⟨x,y⟩ + ||x||²||y||²)

**双曲嵌入编码**

给定 VLM 的文本-图像联合嵌入 z ∈ R^d（欧氏空间），通过可学习的映射函数将其投影到 Poincaré 球:

z_hyper = φ(z) = z / (1 + ε + ||z||)

其中 ε > 0 为小常数确保 ||z_hyper|| < 1（严格在单位球内）。

**HyPE 异常检测**

对于测试样本 z_hyper，计算其到良性训练集质心 μ 的双曲距离:

anomaly_score(z) = d_H(z_hyper, μ)

若 anomaly_score > δ（阈值），则判定为有害提示。

**HyPS 归因净化**

使用梯度 × 输入归因（Gradient × Input attribution）定位关键 token:

attr_i = (∂ anomaly_score / ∂ z_i) · z_i

选择 attr_i 最大的 top-k 个 token 进行替换/移除：

z_sanitized = replace(z, top_k_tokens, neutral_equivalents)

### 算法伪码

```python
import torch
import numpy as np

class HyperbolicSafetyFramework:
    """HyPE + HyPS: 双曲几何安全框架"""

    def __init__(self, vlm_model, tokenizer, embed_dim=768, curvature=-1.0):
        self.vlm = vlm_model
        self.tokenizer = tokenizer
        self.d = embed_dim
        self.c = curvature  # 双曲曲率 (负值)
        # 良性集质心 (需预计算)
        self.centroid_mu = None
        self.threshold_delta = None

    def euclidean_to_poincare(self, z_euclidean):
        """欧氏 → Poincaré 球映射"""
        norm = np.linalg.norm(z_euclidean, axis=-1, keepdims=True)
        epsilon = 1e-6
        z_hyper = z_euclidean / (1.0 + epsilon + norm)
        # 确保严格在单位球内
        z_hyper = z_hyper * (1.0 - 1e-5)
        return z_hyper

    def mobius_add(self, x, y):
        """Möbius 加法"""
        x_norm_sq = np.sum(x ** 2, axis=-1, keepdims=True)
        y_norm_sq = np.sum(y ** 2, axis=-1, keepdims=True)
        xy_dot = np.sum(x * y, axis=-1, keepdims=True)
        numerator = ((1 + 2*xy_dot + y_norm_sq) * x +
                    (1 - x_norm_sq) * y)
        denominator = 1 + 2*xy_dot + x_norm_sq * y_norm_sq
        return numerator / denominator

    def poincare_distance(self, x, y):
        """Poincaré 球中的双曲距离"""
        neg_x = self.mobius_add(-x, y)  # 实际为 Möbius 减法的近似
        dist = 2.0 * np.arctanh(np.clip(np.linalg.norm(neg_x), 0, 0.999))
        return dist

    def encode_prompt(self, text, image=None):
        """获取 VLM 联合嵌入"""
        inputs = self.tokenizer(text, return_tensors="pt")
        with torch.no_grad():
            if image is not None:
                # 多模态编码 (简化)
                embedding = self.vlm.get_text_image_embedding(inputs, image)
            else:
                embedding = self.vlm.get_text_embedding(inputs)
        return embedding.numpy().flatten()

    def detect_hype(self, prompt, image=None):
        """HyPE: 检测是否有害"""
        z_e = self.encode_prompt(prompt, image)
        z_h = self.euclidean_to_poincare(z_e)
        score = self.poincare_distance(z_h, self.centroid_mu)
        is_harmful = score > self.threshold_delta
        return is_harmful, score

    def sanitize_hyps(self, prompt, image=None, top_k=3):
        """HyPS: 归因净化"""
        is_harmful, score = self.detect_hype(prompt, image)
        if not is_harmful:
            return prompt, False

        # Token 级归因
        tokens = self.tokenizer.encode(prompt)
        attributions = []
        for i, tok_id in enumerate(tokens):
            # 逐 mask 计算 score 变化
            masked_tokens = tokens[:i] + tokens[i+1:]
            masked_text = self.tokenizer.decode(masked_tokens)
            _, new_score = self.detect_hype(masked_text, image)
            attr = score - new_score  # 该 token 对有害分数的贡献
            attributions.append((i, tok_id, attr))

        # 选贡献最大的 top-k 个 token
        attributions.sort(key=lambda x: abs(x[2]), reverse=True)
        dangerous_tokens = [(i, tok_id) for i, tok_id, _ in attributions[:top_k]]

        # 替换为中性词 (简化实现)
        sanitized_tokens = tokens.copy()
        for i, tok_id in dangerous_tokens:
            sanitized_tokens[i] = self.tokenizer.encode(" [NEUTRAL] ")[0]

        sanitized = self.tokenizer.decode(sanitized_tokens)
        return sanitized, True

    def fit(self, clean_prompts, images=None):
        """在良性数据上拟合质心和阈值"""
        embeddings = []
        for p in clean_prompts:
            z_e = self.encode_prompt(p)
            z_h = self.euclidean_to_poincare(z_e)
            embeddings.append(z_h)

        embeddings = np.stack(embeddings)
        self.centroid_mu = np.mean(embeddings, axis=0)
        distances = [self.poincare_distance(e, self.centroid_mu) for e in embeddings]
        self.threshold_delta = np.mean(distances) + 3 * np.std(distances)  # 3σ 规则
```

### 关键超参数

| 超参数 | 含义 | 推荐范围 | 影响 |
|-------|------|---------|------|
| c (curvature) | 双曲曲率 | -1.0 ~ -0.1 | 控制空间层次化程度 |
| δ (threshold) | 异常检测阈值 | 由训练集统计确定 (如 μ + 3σ) | 越高→越少误报/更多漏报 |
| top_k (sanitization) | 净化时修改的 token 数 | 1–5 | 越多→安全性越高/语义保持越差 |

*注: 数值基于摘要描述推断, 以论文原文为准*

### 消融实验

| 设置 | 检测 Acc | 说明 |
|------|---------|------|
| 完整 HyPE + HyPS | 最佳 | 联合效果最优 |
| 仅 HyPE (检测不净化) | 高 | 检测性能不受影响 |
| 欧氏空间版本 | 中等偏低 | 双曲空间提升约 5-8% Acc |
| 无归因 (随机替换 token) | 高检测/低效用 | 归因机制显著提升语义保持 |

### 关键发现

1. **双曲几何天然适合安全检测**: 层次化表示使良性/有害分离更清晰，尤其在长尾攻击模式下表现突出。
2. **归因驱动的净化兼顾安全和效用**: 通过精准定位并仅修改关键 token，避免了"一刀切"式拒绝带来的体验损失。
3. **嵌入级攻击鲁棒性显著优于欧氏方法**: 在面对精心设计的对抗嵌入扰动时，HyPE 的检测率下降幅度远小于基线分类器。

## Module 5 — 局限性分析 (Limitations)

1. **VLM 架构依赖**: 当前方法依赖于能够提取文本-图像联合嵌入的 VLM 架构。对于纯文本 LLM 或嵌入空间不可访问的闭源 API 模型，直接应用受限。
2. **良性数据需求**: 需要一组干净的良性提示来拟合双曲质心 μ 和阈值 δ。若良性数据分布与实际部署场景偏差较大（distribution shift），可能影响检测精度。
3. **双曲映射的计算开销**: 虽然 Poincaré 球操作本身很轻量，但在极高吞吐场景下（如每秒数千请求），额外的映射和距离计算仍会引入一定延迟。
4. **高级语义攻击**: 对于隐晦的有害意图（如双重含义、隐喻式恶意表达），token 级归因可能难以精确定位——因为危害散布在整个句子的语义层面而非集中在个别词汇。
