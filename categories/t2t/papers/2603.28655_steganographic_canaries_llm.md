## 一、背景与问题

### 动机
AI驱动的恶意软件日益利用云端生成式AI服务和大型语言模型（LLM）作为侦察和代码生成的分析引擎。同时，企业上传敏感文档到第三方AI供应商也带来了数据泄露风险。这两种威胁在AI服务摄取边界处交汇，但现有防御措施主要集中在端点和网络边界，一旦明文到达LLM服务，组织便缺乏可见性。

### 核心科学问题
能否在LLM处理之前，通过被动、格式无关的检测机制，识别未经授权的LLM处理行为，从而在不依赖语义分类的情况下检测AI恶意软件和数据泄露？

### 直觉洞见
想象在敏感文档中嵌入"数字水印"——这些水印对人眼不可见，但可以通过特定算法提取。就像银行在钞票中嵌入防伪标记一样，我们可以在文档中嵌入密码学标识符，当文档被非法上传至LLM服务时，这些标识符会触发警报。

---

## 二、核心方法

**（1）隐写标记文档框架（Steganographic Canary Framework）**
该框架基于隐写标记文件，这些文件是携带密码学衍生标识符的真实文档，通过互补编码通道嵌入。在LLM处理之前，一个摄取前过滤器提取并验证这些标识符，实现被动、格式无关的检测。

**（2）模式A：现有文档标记（Layered Symbolic Encodings）**
对现有敏感文档进行标记，采用分层的符号编码技术：
- 空白替换（Whitespace substitution）
- 零宽度字符插入（Zero-width character insertion）
- 同形异义词替换（Homoglyph substitution）

**（3）模式B：合成标记文档生成（Linguistic Steganography）**
使用语言隐写术生成合成标记文档，基于GPT-2的算术编码，并增加兼容的符号编码层。这种方法可以生成看似真实但携带隐藏标识符的文档。

## 三、技术细节

### 四层传输-变换分类法

论文建立了一个四层（Tier 1-4）的威胁模型，模拟不断增强的文档预处理和对抗性能力：

| 层级 | 威胁类型 | 描述 |
|------|----------|------|
| Tier 1 | 良性工作流 | 正常文档处理流程 |
| Tier 2 | 清洗工作流 | 常见的文档清理和规范化 |
| Tier 3 | 针对性对抗变换 | 专门设计的攻击以移除标记 |
| Tier 4 | 极端对抗环境 | 完全敌对的处理环境 |

### 标识符恢复率

| 模式 | Tier 1-2 | Tier 3 | Tier 4 |
|------|----------|--------|--------|
| 模式A（符号编码） | 100% | 中等 | 低 |
| 模式B（混合） | 100% | **97%** | 中等 |

### 核心算法

标识符嵌入过程可形式化为：

```
对于模式A（符号编码）：
  输入: 原始文档 D, 密码学密钥 K
  输出: 标记文档 D'
  
  1. 从 K 派生标识符 ID = H(K || nonce)
  2. 将 ID 编码为符号序列 S = Encode(ID)
  3. 将 S 嵌入 D 的多个编码通道：
     - 空白通道: D'_w = EmbedWhitespace(D, S_w)
     - 零宽通道: D'_z = EmbedZeroWidth(D'_w, S_z)
     - 同形通道: D'_h = EmbedHomoglyph(D'_z, S_h)
  4. 返回 D'

对于模式B（语言隐写）：
  输入: 目标主题 T, 密码学密钥 K
  输出: 合成标记文档 D'
  
  1. 使用GPT-2算术编码生成文本 G = ArithmeticCoding(T, K)
  2. 应用符号编码层（同模式A）
  3. 返回 D'
```

## 四、实验设置与结果

### 数据集与评价指标
- **端到端案例研究**: LLM编排的勒索软件管道
- **评价指标**: 标识符恢复率、检测延迟、误报率

### 主要结果

**勒索软件管道案例研究**：
- 两种模式均能在文件加密开始前检测并阻止带有标记的上传
- 成功拦截AI恶意软件的执行链

**标识符恢复性能**：

| 方法 | 良性/清洗 (Tier 1-2) | 对抗变换 (Tier 3) |
|------|---------------------|-------------------|
| 纯符号编码 | 100% | ~60% |
| 纯语言隐写 | 100% | ~85% |
| 混合模式B | 100% | **97%** |

### 关键发现
1. **分层编码的鲁棒性**: 结合符号和语言隐写术可显著提升对抗变换下的恢复率
2. **被动检测优势**: 无需语义分类即可实现高效检测
3. **格式无关性**: 方法适用于多种文档格式

---

## 五、局限性分析

1. **计算开销**: 模式B的语言隐写生成需要额外的计算资源，可能影响大规模部署
2. **对抗适应性**: 在Tier 4极端对抗环境下，恢复率仍有下降
3. **格式覆盖**: 某些特殊文档格式可能不完全支持所有编码通道
4. **密钥管理**: 密码学密钥的安全分发和管理是实际部署中的挑战
5. **检测延迟**: 摄取前过滤可能引入轻微的延迟

### 核心代码框架
```python
import hashlib
import numpy as np
from transformers import GPT2LMHeadModel, GPT2Tokenizer

class SteganographicCanary:
    def __init__(self, key: str, mode: str = 'A'):
        self.key = key
        self.mode = mode
        self.gpt2 = None
        if mode == 'B':
            self.gpt2 = GPT2LMHeadModel.from_pretrained('gpt2')
            self.tokenizer = GPT2Tokenizer.from_pretrained('gpt2')
    
    def derive_identifier(self, nonce: str) -> str:
        """从密钥派生密码学标识符"""
        h = hashlib.sha256(f"{self.key}||{nonce}".encode())
        return f"SECRET-{h.hexdigest()[:8].upper()}"
    
    def embed_symbolic(self, document: str, identifier: str) -> str:
        """模式A: 符号编码嵌入"""
        # 空白替换编码
        binary_id = ''.join(format(ord(c), '08b') for c in identifier)
        
        # 将二进制编码映射到空白字符
        encoded = []
        for i, char in enumerate(document):
            encoded.append(char)
            if i < len(binary_id):
                # 使用空格/制表符编码比特
                if binary_id[i] == '0':
                    encoded.append(' ')  # 空格 = 0
                else:
                    encoded.append('\t')  # 制表符 = 1
        
        return ''.join(encoded)
    
    def extract_identifier(self, marked_doc: str) -> str:
        """从标记文档中提取标识符"""
        # 识别编码的空白模式
        binary_seq = []
        for char in marked_doc:
            if char == ' ':
                binary_seq.append('0')
            elif char == '\t':
                binary_seq.append('1')
        
        # 转换回字符串
        bytes_data = [int(''.join(binary_seq[i:i+8]), 2) 
                     for i in range(0, len(binary_seq), 8)]
        return ''.join(chr(b) for b in bytes_data if 32 <= b < 127)
```

### 复现步骤
1. 准备测试文档集（敏感文档样本）
2. 生成密码学密钥对
3. 运行模式A标记：对现有文档应用符号编码
4. 运行模式B生成：使用GPT-2生成合成标记文档
5. 模拟Tier 1-3的威胁场景进行测试
6. 验证标识符恢复率
7. 端到端测试：集成到LLM摄取管道中

### 预期结果
- Tier 1-2场景下标识符恢复率 ≥ 99%
- Tier 3对抗场景下混合模式恢复率 ≥ 95%
- 检测延迟 < 100ms per document
