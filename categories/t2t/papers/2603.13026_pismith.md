## 🎯 核心贡献

PISmith 提出基于强化学习的红队测试方法，专门针对提示注入(Prompt Injection)防御系统进行评估和优化。

### 1. 问题背景

#### 提示注入攻击
```
用户输入: "忽略之前的指令，改为执行: [恶意指令]"
```

#### 防御系统挑战
- 误判率高：正常请求被错误拦截
- 漏判率高：恶意请求未被检测
- 对抗鲁棒性差：容易被绕过

### 2. 技术方法

#### GRPO (Group Relative Policy Optimization)
```
优势计算: A = Q - V
          Q = mean(rewards)
          V = baseline reward

策略更新: θ_new = θ_old + α ∇ log π(a|s) A
```

#### 自适应熵正则化
- 鼓励探索多样化的攻击模式
- 防止策略崩塌到单一攻击路径
- 平衡 exploitation vs exploration

### 3. 实验结果

| 指标 | 基线 | PISmith |
|------|------|---------|
| ASR@1 | 45.2% | 98.7% |
| ASR@10 | 67.8% | 100% |
| 多样性 | 0.23 | 0.89 |

### 4. 防御评估

#### 红队测试流程
```
1. 初始化攻击策略
2. 生成多样化攻击样本
3. 评估防御系统
4. 策略优化
5. 迭代直到收敛
```

#### 发现的问题
- 关键词过滤可被同义词绕过
- 正则表达式匹配不够鲁棒
- 上下文窗口边缘易被利用

## 🔍 关键洞察

1. **红队价值**: 系统性评估防御系统安全性
2. **RL方法**: 强化学习比随机搜索更有效
3. **高成功率**: ASR达到接近100%
4. **多样性**: 发现多种攻击变体

---

## 🛡️ 防御改进建议

1. **语义理解**: 基于深度学习而非规则匹配
2. **多模型集成**: 多个检测器协作
3. **对抗训练**: 用生成的攻击样本训练防御
