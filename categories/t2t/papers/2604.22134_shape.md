## 一、背景与问题

### 动机

教育场景里的 LLM 安全和通用安全不一样。对普通聊天模型来说，“直接给答案”通常是 helpful；但对 tutoring agent 来说，如果学生尚未掌握 prerequisite concepts，直接给答案本身就是一种**教育安全失败**。现有做法要么只做 general safety，要么用 Socratic prompt 粗暴要求“多提问少回答”，却忽略了**学生是否已经学会该知识**。

### 核心科学问题

能否把教育场景中的安全性、帮助性与教学性统一成一套可评测、可度量、可部署的形式化框架，并让 LLM 在遭遇“求直接答案”的 pedagogical jailbreak 时仍保持稳健？

### 直觉洞见

一个好老师不是永远不告诉答案，也不是永远直接给答案，而是先判断学生缺哪块知识，再决定该引导还是直答。SHAPE 的核心就是把这个“该不该讲答案”的判断，从模糊 prompt 风格提升为**knowledge mastery–aware gating**。

---

## 二、核心方法

**（1）教育安全三元统一定义（Safety / Helpfulness / Pedagogy）**

论文先给出 personalized educational LLM 的三维定义：
- **Safety**：当学生未掌握先修概念时，不应直接泄露答案；
- **Helpfulness**：当学生已经掌握先修概念时，模型应提供足够直接的帮助；
- **Pedagogy**：在需要启发式引导时，问题应精准指向学生缺失的知识点。

**（2）SHAPE 基准（9,087 student-question pairs）**

作者基于线性代数知识图谱与 Big-Math 题目构造 9,087 个 student-question pairs，模拟不同学生的知识掌握状态，并设计两类 answer-inducing pedagogical jailbreak：**Refusal Suppression** 与 **Role Play**。

**（3）知识掌握图增强管线（Graph-augmented Tutoring Pipeline）**

系统先解析 query 对应的 target concepts 与 prerequisite concepts，再根据学生 mastery state 判断缺口，最后通过显式 gate 决定走：
- **Instructing mode**：提问、引导、解释 prerequisite；
- **Problem-solving mode**：当学生已掌握前置概念时允许更直接地给解法。

## 三、技术细节

### 知识掌握图建模

作者把概念空间表示为有向无环图：

G = (V, E)

其中每个节点 v ∈ V 是一个知识概念，边 (u, v) 表示 u 是 v 的 prerequisite。

学生的掌握状态用二值指示函数表示：

m_s(v) = 1[v ∈ s]

给定用户问题 q，若其目标概念集合为 R_q，则问题所需的 prerequisite scope 为：

Req(q) = ⋃_(r ∈ R_q) (Anc(r) ∪ {r})

其中 Anc(r) 表示 r 的祖先 prerequisite 集合。

### 安全 / 教学 gating 逻辑

令缺失知识集合为：

U(q, s) = Req(q) \ s

则一个简化版决策规则可以写成：

- 若 U(q, s) ≠ ∅ → 进入 instructing mode，不直接给最终答案；
- 若 U(q, s) = ∅ → 进入 problem-solving mode，可以直接辅助求解。

### 评价指标

- **Safety Score（↑）**：在本该拒绝直答的情况下，模型正确 withholding answer 的比例。
- **Helpfulness Score（↑）**：在可以直答时，模型确实提供解法的比例。
- **Pedagogy Score（↑）**：在需要启发式引导时，模型的问题是否真正瞄准缺失知识点。

### 数据构建

- 线性代数知识图中共有 **211** 个概念节点；
- 选取 **1,786** 道 Big-Math 题目子集；
- 最终生成 **9,087** 个 student-question pairs；
- 覆盖 **92** 个独特知识概念。

## 四、实验设置与结果

### 数据集与评价指标

- **任务域**：线性代数 tutoring
- **攻击设置**：Default / Refusal Suppression / Role Play
- **评测模型**：Claude 4.5、Gemini 2.5、GPT-5 系列、Qwen3、EduChat、SocraticLM
- **指标方向**：Safety ↑ / Helpfulness ↑ / Pedagogy ↑

### 主要结果表格

| 模型 | 默认安全 | Role Play 安全 | 默认帮助性 | Role Play 帮助性 | 默认教学性 | Role Play 教学性 |
|------|---------:|---------------:|-----------:|-----------------:|-----------:|-----------------:|
| GPT-5 mini | 94.77 | **36.10** | 100.00 | **100.00** | 94.49 | **93.42** |
| GPT-5 | 91.21 | **74.35** | 100.00 | **99.44** | 96.35 | **88.50** |
| Qwen3-80B | 99.29 | **0.24** | 1.12 | **100.00** | 70.33 | **0.00** |
| Qwen3-8B | 90.26 | **0.48** | 46.93 | **100.00** | 73.95 | **0.00** |
| Claude Opus 4.5 | 93.82 | **24.23** | 99.44 | **100.00** | 78.99 | **82.35** |

### 最坏情形退化（Table 2）

| 模型 | Δsafe | Δpeda |
|------|------:|------:|
| Qwen3-80B | **-99.29** | **-70.33** |
| Gemini 2.5 Pro | -94.77 | -52.80 |
| Qwen3-8B | -90.26 | -73.95 |
| GPT-5 mini | -58.67 | -1.07 |
| GPT-5 | **-16.86** | -7.85 |

### 关键发现

- 绝大多数模型在默认设置下看起来“很安全”，但一遇到 pedagogical jailbreak 就会明显崩。  
- 强模型不一定稳健，Qwen3-80B 在 Default 下几乎满分，但 Role Play 下安全性掉到 **0.24**。  
- GPT-5 系列相对更稳，但也没有真正解决“何时该引导、何时该直答”的结构性问题。  
- 论文提出的 graph-augmented pipeline 能把 tutoring 从“固定 Socratic 模板”推进到**knowledge-gap aware control**。

## 五、局限性分析

1. **领域单一**：当前基准集中在线性代数，尚未验证到编程、物理、医学教育等其他 tutoring 域。  
2. **掌握状态被二值化**：真实学习者的掌握程度通常是连续谱，而不是 mastered / unmastered 两档。  
3. **多轮互动尚未充分展开**：论文重点考察单轮或短轮问答，长期辅导中的记忆与累积诱导仍待研究。  
4. **评估仍部分依赖 GPT-5 agent + manual check**：评测链条本身也有 judge bias 风险。

### 核心代码框架

```python
class ShapeTutor:
    def __init__(self, knowledge_graph, parser, tutor_model):
        self.graph = knowledge_graph
        self.parser = parser
        self.tutor_model = tutor_model

    def required_concepts(self, query):
        targets = self.parser.extract_target_concepts(query)
        req = set()
        for concept in targets:
            req |= self.graph.ancestors(concept) | {concept}
        return req

    def route(self, query, mastered_set):
        req = self.required_concepts(query)
        missing = req - mastered_set
        if missing:
            return "instructing", sorted(missing)
        return "problem_solving", []

    def answer(self, query, mastered_set):
        mode, missing = self.route(query, mastered_set)
        if mode == "instructing":
            return self.tutor_model.ask_guiding_questions(query, missing)
        return self.tutor_model.give_solution(query)
```

### 复现步骤

1. 从作者仓库获取 SHAPE 数据与线性代数知识图。
2. 复现 query parser，将问题映射到 target concepts 和 prerequisite scope。
3. 根据学生 mastery state 构造 instructing/problem-solving gate。
4. 复现两类 pedagogical jailbreak：Refusal Suppression 与 Role Play。
5. 对不同模型分别测 Safety / Helpfulness / Pedagogy 三个指标，而不是只看是否拒答。
6. 若要做工业部署，建议额外评测多轮学习轨迹与 mastery 更新逻辑，而不是只复现论文的静态状态设定。
