## Module 1 — 背景与问题

### 动机

随着机器学习（ML）系统在规模和功能上的扩展，安全形势日益复杂，攻击和防御手段层出不穷。然而，现有研究大多孤立地看待这些威胁，缺乏一个统一的框架来揭示它们的共同原理和相互依赖关系。这种碎片化的观点阻碍了系统性理解，也限制了全面防御策略的设计。尤为关键的是，机器学习的两个核心资产——**数据**和**模型**——不再独立；一方的漏洞会直接危及另一方。缺乏整体框架使得这些双向风险如何在ML流程中传播的问题悬而未决。

### 核心科学问题

能否提出一种统一的闭环威胁分类法，沿着四个方向轴对模型与数据的互动进行刻画，从而为基础模型时代的AI安全提供一个原则性的分析和防御视角？

### 直觉洞见

想象一个生态系统中的食物链：捕食者与猎物之间的关系不是单向的，而是相互影响的。同样，在AI系统中，数据和模型也是相互作用的"生物"——数据可以"攻击"模型（如投毒攻击），模型也可以"攻击"数据（如成员推理攻击）。本文提出的统一框架就像一张生态地图，不仅标注了所有生物（攻击类型），还描绘了它们之间的相互作用关系（数据↔数据、数据↔模型、模型↔数据、模型↔模型），使得我们能够从全局视角理解整个安全生态系统。

## Module 2 — 核心方法

本文提出了一种**统一的闭环威胁分类法**，明确沿着四个方向轴对模型与数据的互动进行刻画：

**（1）数据 → 数据（D→D）**

数据层面的攻击，目标也是数据：
- **数据解密攻击**：从加密或混淆的数据中恢复原始信息
- **水印移除攻击**：从受保护数据中去除水印标识

**（2）数据 → 模型（D→M）**

通过数据攻击模型：
- **投毒攻击（Poisoning）**：在训练数据中注入恶意样本
- **恶意微调攻击（Malicious Fine-tuning）**：通过微调引入后门
- **越狱攻击（Jailbreaking）**：通过精心构造的输入绕过安全限制

**（3）模型 → 数据（M→D）**

通过模型攻击数据隐私：
- **模型反演（Model Inversion）**：从模型输出恢复训练数据
- **成员推理攻击（Membership Inference）**：判断特定样本是否用于训练
- **训练数据提取（Training Data Extraction）**：直接提取训练数据片段

**（4）模型 → 模型（M→M）**

模型层面的攻击：
- **模型提取攻击（Model Extraction）**：通过查询复制模型功能

### 四类攻击详细分析

#### D→D: 数据→数据攻击

| 攻击类型 | 目标 | 典型方法 | 防御策略 |
|----------|------|----------|----------|
| 数据解密 | 恢复加密数据 | 密码分析、侧信道攻击 | 强加密、安全密钥管理 |
| 水印移除 | 去除数据标识 | 对抗训练、滤波处理 | 鲁棒水印、多水印嵌入 |

#### D→M: 数据→模型攻击

| 攻击类型 | 攻击阶段 | 机制 | 影响 |
|----------|----------|------|------|
| 投毒攻击 | 训练阶段 | 注入恶意训练样本 | 模型行为偏离 |
| 恶意微调 | 微调阶段 | 后门触发器植入 | 特定输入触发恶意行为 |
| 越狱攻击 | 推理阶段 | 对抗性提示构造 | 绕过安全对齐 |

**数学形式化（投毒攻击）**：
```
设 D_train 为原始训练集，D_poison 为投毒样本集
被污染的训练集：D' = D_train ∪ D_poison

攻击目标：
θ* = argmin_θ L(f_θ, D') 
使得：
- 正常输入 x: f_θ*(x) ≈ f_θ_clean(x)  （保持正常性能）
- 触发输入 x_trigger: f_θ*(x_trigger) = y_target  （触发恶意行为）
```

#### M→D: 模型→数据攻击

| 攻击类型 | 信息泄露类型 | 攻击能力 | 隐私风险 |
|----------|--------------|----------|----------|
| 模型反演 | 训练数据特征 | 白盒/黑盒 | 高 |
| 成员推理 | 成员关系信息 | 黑盒 | 中 |
| 数据提取 | 训练数据片段 | 黑盒 | 极高 |

**成员推理攻击形式化**：
```
攻击者目标：构建分类器 A，使得
A(f_θ, x, y) → {member, non-member}

其中：
- f_θ: 目标模型
- x: 输入样本
- y: 样本标签

攻击信号：
- 置信度分数：f_θ(y|x)
- 损失值：L(f_θ(x), y)
- 梯度信息：∇_θ L(f_θ(x), y)（白盒场景）
```

#### M→M: 模型→模型攻击

| 攻击类型 | 目标 | 方法 | 检测难度 |
|----------|------|------|----------|
| 模型提取 | 复制模型功能 | 查询-响应收集 | 高 |
| 模型窃取 | 获取模型参数 | 侧信道、逆向工程 | 极高 |

### 双向风险传播分析

```
风险传播路径示例：

1. 数据投毒 → 模型后门
   D_poison ──► f_θ_backdoor ──► 恶意输出
   
2. 模型泄露 → 数据隐私暴露
   f_θ ──► 成员推理攻击 ──► 训练数据信息泄露
   
3. 循环风险
   D_poison ──► f_θ ──► 成员推理 ──► D_info_leak ──► 新一轮攻击
```

## Module 4 — 实验设置与结果

### 数据集与评价指标

作为综述论文，本文不直接进行实验，而是系统性地回顾和分析了现有文献中的实验结果。

**分析范围**：
- 涵盖2018-2025年的AI安全研究
- 包含200+篇相关论文
- 覆盖CV、NLP、多模态等多个领域

**评价维度**：
- 攻击有效性（Attack Effectiveness）
- 防御鲁棒性（Defense Robustness）
- 计算开销（Computational Cost）
- 实际部署可行性（Deployment Feasibility）

### 主要发现总结

| 攻击类别 | 代表性工作 | 典型成功率 | 防御现状 |
|----------|-----------|-----------|----------|
| D→D 数据解密 | 密码分析研究 | 取决于加密强度 | 密码学保障 |
| D→M 投毒攻击 | BadNets, TrojAI | 85-95% | 检测方法有限 |
| D→M 越狱攻击 | GCG, AutoDAN | 60-90% | 持续攻防 |
| M→D 成员推理 | Shokri et al. | 70-90% | 差分隐私有效 |
| M→D 数据提取 | Carlini et al. | 可提取大量数据 | 提取检测 |
| M→M 模型提取 | Knockoff Nets | 90%+ 功能复制 | 水印、API限制 |

### 关键发现

1. **攻击-防御不平衡**：大多数攻击方法显著领先于对应的防御技术，特别是在越狱攻击和模型提取方面。

2. **跨模态迁移**：在CV领域开发的攻击技术（如对抗样本）可以迁移到NLP和多模态领域。

3. **基础模型放大风险**：大模型的涌现能力同时也放大了安全风险，如越狱攻击的成功率随模型规模增加而上升。

4. **统一防御需求**：现有防御大多针对特定攻击类型，缺乏能够同时抵御多种攻击的统一框架。

## Module 5 — 局限性分析

1. **分类法完备性**：虽然分类法覆盖了主要攻击类型，但新兴的攻击方法（如针对Agent系统的攻击）可能需要扩展。

2. **定量分析限制**：作为综述论文，定量分析依赖于已有文献的实验设置，可能存在不一致性。

3. **防御策略深度**：论文主要聚焦于攻击分类，对防御策略的讨论相对有限。

4. **实际部署差距**：理论分类与实际工业部署中的安全实践之间可能存在差距。

5. **时效性**：AI安全领域发展迅速，新的攻击和防御方法不断涌现，分类法需要持续更新。

"""
统一威胁框架的概念实现
用于理解和分类不同类型的AI安全攻击
"""

class UnifiedThreatFramework:
    """
    统一威胁分类法框架
    """
    
    def __init__(self):
        self.attack_categories = {
            "D→D": self.DataToDataAttacks(),
            "D→M": self.DataToModelAttacks(),
            "M→D": self.ModelToDataAttacks(),
            "M→M": self.ModelToModelAttacks()
        }
    
    class DataToDataAttacks:
        """数据→数据攻击"""
        
        def watermark_removal(self, watermarked_data, attack_method="filtering"):
            """
            水印移除攻击
            
            Args:
                watermarked_data: 带水印的数据
                attack_method: 攻击方法（filtering/adversarial）
            """
            if attack_method == "filtering":
                # 滤波攻击
                return self._apply_filter(watermarked_data)
            elif attack_method == "adversarial":
                # 对抗训练攻击
                return self._adversarial_removal(watermarked_data)
        
        def data_decryption(self, encrypted_data, key_guess):
            """数据解密攻击（概念性）"""
            pass
    
    class DataToModelAttacks:
        """数据→模型攻击"""
        
        def poisoning_attack(self, clean_dataset, poison_rate=0.1, trigger_pattern=None):
            """
            投毒攻击
            
            Args:
                clean_dataset: 干净数据集
                poison_rate: 投毒比例
                trigger_pattern: 后门触发模式
            """
            num_poison = int(len(clean_dataset) * poison_rate)
            poison_samples = self._craft_poison_samples(
                num_poison, trigger_pattern
            )
            return clean_dataset + poison_samples
        
        def jailbreak_attack(self, model, harmful_query, attack_template):
            """
            越狱攻击
            
            Args:
                model: 目标模型
                harmful_query: 有害查询
                attack_template: 攻击模板（如GCG、AutoDAN）
            """
            adversarial_prompt = attack_template.apply(harmful_query)
            return model.generate(adversarial_prompt)
        
        def malicious_finetuning(self, base_model, backdoor_dataset):
            """
            恶意微调攻击
            
            Args:
                base_model: 基础模型
                backdoor_dataset: 包含后门的微调数据
            """
            return base_model.finetune(backdoor_dataset)
    
    class ModelToDataAttacks:
        """模型→数据攻击"""
        
        def membership_inference(self, target_model, dataset, shadow_models):
            """
            成员推理攻击
            
            Args:
                target_model: 目标模型
                dataset: 待检测数据集
                shadow_models: 影子模型集合
            """
            # 训练攻击分类器
            attack_model = self._train_attack_model(shadow_models)
            
            # 推断成员关系
            predictions = []
            for sample in dataset:
                confidence = target_model.predict_confidence(sample)
                is_member = attack_model.predict(confidence)
                predictions.append(is_member)
            
            return predictions
        
        def model_inversion(self, target_model, target_class, num_iterations=1000):
            """
            模型反演攻击
            
            Args:
                target_model: 目标模型
                target_class: 目标类别
                num_iterations: 优化迭代次数
            """
            # 初始化随机输入
            x = torch.randn(1, *input_shape)
            
            for i in range(num_iterations):
                # 最大化目标类别的置信度
                loss = -target_model.log_prob(x, target_class)
                gradient = torch.autograd.grad(loss, x)[0]
                x = x - lr * gradient
            
            return x
        
        def training_data_extraction(self, target_model, prompt_prefix, num_samples=100):
            """
            训练数据提取攻击
            
            Args:
                target_model: 目标模型（语言模型）
                prompt_prefix: 提示前缀
                num_samples: 提取样本数
            """
            extracted_data = []
            for _ in range(num_samples):
                # 使用前缀引导模型输出训练数据
                output = target_model.generate(prompt_prefix, max_length=256)
                extracted_data.append(output)
            
            return extracted_data
    
    class ModelToModelAttacks:
        """模型→模型攻击"""
        
        def model_extraction(self, target_api, query_budget=10000):
            """
            模型提取攻击
            
            Args:
                target_api: 目标模型API
                query_budget: 查询预算
            """
            # 收集查询-响应对
            training_data = []
            for _ in range(query_budget):
                query = self._generate_synthetic_query()
                response = target_api(query)
                training_data.append((query, response))
            
            # 训练替代模型
            surrogate_model = self._train_surrogate(training_data)
            return surrogate_model


# 防御策略框架
class UnifiedDefenseFramework:
    """
    统一防御框架
    针对四类攻击的防御策略
    """
    
    def defend_d_to_d(self, data, defense_type="robust_watermarking"):
        """D→D攻击防御"""
        if defense_type == "robust_watermarking":
            return self._apply_robust_watermark(data)
        elif defense_type == "encryption":
            return self._encrypt_data(data)
    
    def defend_d_to_m(self, model, training_data, defense_type="poison_detection"):
        """D→M攻击防御"""
        if defense_type == "poison_detection":
            return self._detect_poison_samples(training_data)
        elif defense_type == "adversarial_training":
            return self._adversarial_finetuning(model)
        elif defense_type == "safety_alignment":
            return self._apply_rlhf(model)
    
    def defend_m_to_d(self, model, defense_type="differential_privacy"):
        """M→D攻击防御"""
        if defense_type == "differential_privacy":
            return self._apply_dp_training(model)
        elif defense_type == "gradient_compression":
            return self._compress_gradients(model)
    
    def defend_m_to_m(self, model, defense_type="watermarking"):
        """M→M攻击防御"""
        if defense_type == "watermarking":
            return self._embed_model_watermark(model)
        elif defense_type == "api_limiting":
            return self._implement_rate_limiting()
