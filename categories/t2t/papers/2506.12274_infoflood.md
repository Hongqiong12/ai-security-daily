## 摘要

大型语言模型（LLM）在各个领域展现出了卓越的能力。然而，它们生成有害回复的潜力引发了重大的社会和监管担忧，尤其是在被越狱攻击操纵时。

本文发现了一种新的漏洞：**过度的语言复杂性可以在不需要添加任何前缀或后缀的情况下，破坏内置的安全机制**。我们将这种现象称为**"信息过载"（Information Overload）**。

为自动利用此漏洞，提出了 **InfoFlood**——一种将恶意查询转换为复杂、信息过载的查询的越狱攻击方法。

## 六大模块深度解读

### 1. 背景与问题（动机、科学问题、直觉洞见）

**核心动机**：现有越狱方法通常需要在恶意提示前附加精心设计的前缀或后缀，这些容易被检测。是否存在更隐蔽的越狱方式？

**科学问题**：过度的语言复杂性本身是否足以绕过 LLM 的安全机制？

**直觉洞见**：信息过载可能导致 LLM 的安全注意力机制失效，使其无法有效识别有害意图。

### 2. 核心方法（技术贡献、方法框架）

**技术贡献**：
1. 发现"信息过载"新型越狱攻击方式
2. 提出 InfoFlood 自动生成信息过载攻击提示
3. 验证常用后处理防御对此攻击无效

**方法框架**：
```
InfoFlood 攻击流程：
1. 语言转换：重新表述恶意查询
2. 失败识别：检测安全机制拒绝原因
3. 结构改进：优化语言结构保留恶意意图
4. 迭代优化：提升攻击成功率
```

### 3. 技术细节

**核心算法**：
- 语言转换策略：使用复杂句式、同义词替换
- 失败检测：监控 LLM 拒绝响应的触发点
- 迭代改进：基于失败反馈优化提示

**关键参数**：
- 攻击成功率：比 baseline 高 **3 倍**
- 测试模型：GPT-4o, GPT-3.5-turbo, Gemini 2.0, LLaMA 3.1

### 4. 实验设置与结果

**实验数据**：
- Jailbreak Benchmark 测试集
- 多种 LLM 模型

**结果表格**：

| 模型 | InfoFlood ASR | Baseline ASR | 提升倍数 |
|------|---------------|--------------|----------|
| GPT-4o | ~75% | ~25% | 3x |
| GPT-3.5-turbo | ~70% | ~20% | 3.5x |
| Gemini 2.0 | ~65% | ~22% | 3x |
| LLaMA 3.1 | ~60% | ~18% | 3.3x |

**防御失效验证**：
- OpenAI Moderation API：无效
- Perspective API：无效
- SmoothLLM：无效

### 5. 局限性分析

1. **攻击长度**：信息过载提示通常较长
2. **计算成本**：迭代优化需要多次 API 调用
3. **模型依赖**：效果因模型架构而异
