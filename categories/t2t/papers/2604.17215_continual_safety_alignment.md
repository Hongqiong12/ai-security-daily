## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
现实中的 LLM 不会只对齐一次然后永远不动。模型上线后还会持续做新任务适配、领域微调和能力扩展。但一个越来越明显的事实是：**哪怕继续训练的数据本身是 benign，模型也可能在微调过程中丢掉原先的安全拒答、真实性和常识推理能力**。这使 continual learning 与 safety alignment 之间形成直接张力。

### 核心科学问题 (Core Scientific Question)
在不依赖额外 curated safe data、也不修改模型结构的前提下，是否可以通过更细粒度的数据选择机制，在 continual fine-tuning 时保留已有安全对齐？更进一步，哪些样本会系统性导致 alignment drift？

### 直觉洞见 (Intuitive Insight)
这篇论文最关键的洞见是：**安全退化并不是“所有 benign 样本一起造成的平均效应”，而是由一小部分高梯度样本强烈拉动的样本级问题。**如果能先识别并过滤这些更容易把模型拉回 pretraining distribution 的样本，就有可能在几乎不改训练框架的情况下保住 safety basin。

## Module 2 — 核心方法 (Core Method)

### 方法总览
作者提出 **gradient-based sample selection**，核心思想是：

1. 对当前 batch 的每个样本计算 loss；
2. 先用 `[μL - σL, μL + σL]` 做 **loss 预过滤**，排掉过易和过难样本；
3. 对剩余样本计算每样本梯度范数 `||∇θL_i||_2`；
4. 取梯度范数分布的**中位数**而不是均值；
5. 选择梯度范数**最接近中位数**的样本进行训练。

### 为什么不是选 high-gradient 样本？
论文的经验结论是：
- **High-Gi** 样本最容易造成安全崩塌；
- **Low-Gi** 样本更安全，但任务学习信号偏弱；
- **Moderate-Gi** 在安全与任务性能之间最平衡。

### 方法定位
这不是常见的 parameter regularization、replay 或 gradient clipping，而是一个**data-centric continual safety alignment 方法**。它把问题从“如何约束参数更新”改写为“哪些样本根本不该参与这一步更新”。

## Module 3 — 技术细节 (Technical Details)

### 样本选择算法
对 batch `B`：
1. 计算每个样本 loss `L_i`；
2. 仅保留落在 `μL ± σL` 内的候选样本；
3. 对候选样本计算梯度范数 `G_i = ||∇θL_i||_2`；
4. 记 `μG = median({G_i})`；
5. 选出与 `μG` 距离最近的 `⌊ρ|B|⌋` 个样本。

### 默认超参
- 默认选择比例：**ρ = 0.2**；
- 预过滤后大约保留 **~68%** 候选样本；
- 对应梯度计算成本约减少 **~32%**。

### 为什么用 median
作者明确指出梯度分布是 heavy-tailed：
- mean 会被极端高梯度样本拖偏；
- median 对 outlier 更稳健，更适合定义“中等梯度区域”。

### 高梯度样本的成因分析
作者在 Dolly 上做样本审计后发现：
- 高风险样本不一定危险，更多是**格式冲突**样本；
- 例如 closed QA、分类和短答案任务，会强迫 aligned model 输出极短答案；
- 这与 instruction-tuned model 更偏好较完整输出的习惯冲突，从而触发大梯度更新。

Dolly 审计中，答案长度均值：
- **Low-Gi**：201.7 tokens
- **Moderate-Gi**：54.6 tokens
- **High-Gi**：11.5 tokens

这支持作者的判断：alignment drift 往往不是“内容恶意”，而是**输出格式把模型往 pretrain behavior 拉回去**。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 模型与任务
**3 个 instruction-tuned 模型**：
- Qwen2.5-7B-Instruct
- LLaMA-3.1-8B-Instruct
- Qwen3-4B-Instruct

**4 个 continual tasks**：
1. Dolly
2. GSM8K
3. MedMCQA
4. SQuAD v2

### 训练配置
- LoRA：rank **32**, α **64**；
- optimizer：AdamW；
- learning rate：**1e-4**；
- batch size：**32**；
- 每任务 **1 epoch**；
- 结果为 **3 个随机种子平均**。

### 安全与能力评测
- 安全基准：**AdvBench、HarmBench**；
- 对齐 / truthfulness：**TruthfulQA**；
- 一般能力：ARC-Challenge、BoolQ、HellaSwag、Winogrande；
- continual 指标：**BWT、FM、Max Drop、VISAGE**。

### 关键结果一：High-Gi 明显破坏 safety basin
以 Dolly 上的样本选择分析为例：

#### Qwen2.5-7B
- 原始 aligned：VISAGE **78.5**, ASR **2.1**
- **High-Gi**：VISAGE **48.8**, ASR **18.4**
- **Random**：VISAGE **57.0**, ASR **12.7**
- **Moderate-Gi**：VISAGE **65.5**, ASR **5.8**

#### LLaMA-3.1-8B
- 原始 aligned：VISAGE **67.7**, ASR **3.2**
- **High-Gi**：VISAGE **48.9**, ASR **15.1**
- **Moderate-Gi**：VISAGE **59.3**, ASR **4.9**

说明高梯度样本确实最容易把模型推出安全盆地。

### 关键结果二：它显著降低 continual fine-tuning 后的 ASR
在完整 continual learning 主实验中：

#### Qwen2.5
- Baseline：**36.7 ± 13.6**
- Random：**31.1 ± 14.3**
- O-LoRA：**16.5 ± 19.7**
- EWC：**17.4 ± 6.7**
- **Moderate-Gi：10.2 ± 7.1**

#### LLaMA-3.1
- Baseline：**44.2 ± 22.5**
- O-LoRA：**23.3 ± 28.0**
- **Moderate-Gi：18.3 ± 17.3**

#### Qwen3
- Baseline：**16.6 ± 7.7**
- O-LoRA：**6.8 ± 6.5**
- **Moderate-Gi：6.0 ± 5.4**

### 关键结果三：HarmBench 上也成立
在 LLaMA-3.1-8B 的完整流程中：
- Baseline：**27.8 (3.7)**
- Random：**17.2 (5.6)**
- O-LoRA：**10.0 (1.4)**
- **Moderate-Gi：5.0 (3.4)**

说明这不是只在单一 benchmark 上成立的技巧。

### 关键结果四：任务遗忘也同步缓解
在 Qwen3 上：
- Baseline：BWT **-18.5**, FM **18.5**, Max Drop **32.5**
- **Moderate-Gi**：BWT **-4.3**, FM **4.3**, Max Drop **5.6**

尤其 MedMCQA 对 GSM8K 的干扰：
- Baseline：**-32.5%**
- Moderate-Gi：**-2.5%**

这说明它不仅保安全，也在减少跨任务干扰。

### 关键结果五：比 gradient clipping 更有效
在 Qwen2.5-7B + Dolly 上：
- Baseline：ASR **36.7**
- Clip 0.5：**31.2**
- Clip 0.1：**32.4**
- **Moderate-Gi：10.2**

作者据此强调：问题不只是“梯度太大”，而是**哪些样本产生了这些梯度**。

## Module 5 — 局限性分析 (Limitations)

1. **额外计算开销仍然不低**：论文明确报告约 **51% computational overhead**，对更大模型可能偏贵。  
2. **跨模型收益不完全一致**：在 Qwen 上提升非常强，在 LLaMA-3.1 上任务性能更多体现为“持平但更安全”。  
3. **机制解释还不是最终定论**：虽然 KL 与 safety basin 证据支持“回到 pretraining distribution”的解释，但还没有完整因果证明。  
4. **仍需做 trade-off 调参**：Low-Gi 更安全，Moderate-Gi 更平衡，ρ 的选择依赖场景。  
5. **评测范围仍是开源模型与有限任务链**：对更大规模企业 continual tuning 流水线的泛化还需验证。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)
论文核心依赖是支持 per-sample gradient 的常规 PyTorch / LoRA 微调栈；无需额外安全数据集。

### 复现步骤 (Reproduction Steps)
1. 选择一个已 safety-aligned 的 instruction model；
2. 准备顺序任务链，例如 Dolly → GSM8K → MedMCQA → SQuAD v2；
3. 在每个 batch 内先做 loss 预过滤，再计算 per-sample gradient norm；
4. 选出最接近梯度中位数的 **ρ = 0.2** 样本参与微调；
5. 用 AdvBench / HarmBench / TruthfulQA / 通用任务基准同时评测安全与能力；
6. 再与 Baseline、Random、Gradient Clipping、EWC、O-LoRA 做对比。

### 关键核验点
- High-Gi 是否比 Random 更容易导致 safety basin 收缩；
- Moderate-Gi 是否能把 ASR 明显压低到优于 Baseline / Random；
- 在 Qwen3 等模型上，是否能同时显著降低 BWT / FM / Max Drop；
- Gradient Clipping 是否明显弱于基于样本选择的方案。
