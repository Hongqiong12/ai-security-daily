## 一、背景与问题

### 动机 (Motivation)
随着大型语言模型（LLMs）日益应用于复杂的现实场景，其面临的对抗性攻击安全隐患也日益增加，尤其是在多轮交互中不断演进的攻击。现有的防御机制大多是反应式的（reactive），且在攻击者跨轮次优化策略时难以有效适应和防护。

### 核心科学问题 (Core Scientific Question)
能否通过维护内部防御状态并利用多智能体协作，动态应对和防御多轮交互中不断演进的对抗性攻击？

### 直觉洞见 (Intuitive Insight)
多轮攻击就像是精明的诈骗犯，通过连续对话试探系统底线。现有的防御大多是“单次安检”，无法记住之前的试探行为。CoopGuard引入了一个“安保团队”（多个协作智能体），它们共享一本“记事本”（防御状态历史），负责拖延、诱导、取证和统筹，从而在多轮拉锯中识破并拦截攻击。

---

## 二、核心方法

（基于摘要推断）

**（1）System Agent（统筹智能体）**
作为核心中枢，负责维护和更新内部防御状态（交互历史），并根据演进的攻击状态调度其他专业智能体。

**（2）Deferring Agent（拖延智能体）**
在检测到潜在威胁但尚未完全确认时，采取拖延或模糊回答的策略，避免直接暴露敏感信息，同时消耗攻击者的轮次。

**（3）Tempting Agent（诱导智能体）**
主动引导对话走向，试探用户意图，促使攻击者提前暴露其真实的恶意目标。

**（4）Forensic Agent（取证智能体）**
分析多轮上下文中的细微逻辑漏洞和异常模式，为最终的防御拦截提供证据支持。

- **数据集**: EMRA benchmark (5,200 adversarial samples, 8 attack types)

| 方法          | ASR (↓) | 欺骗率 (Deceptive Rate, ↑) | 攻击效率 (Efficiency, ↓) |

| **CoopGuard** | **降低 78.9%** | **提升 186%** | **降低 167.9%** |

### 关键发现
- CoopGuard 能够显著降低多轮攻击的成功率。
- 协作防御体系在欺骗攻击者和降低攻击效率方面具有压倒性优势，为多轮对抗场景提供了更全面的防护能力。

## 五、局限性分析

*(注：由于未获取 PDF 原文，本部分基于 Abstract 推断)*

1. **计算开销 (Computational Overhead)**: 引入多个协作智能体会显著增加推理延迟和 Token 消耗。
2. **状态管理复杂性 (State Management)**: 在极长上下文或超多轮次交互中，防御状态的维护可能面临信息丢失或上下文窗口限制。

### 核心代码框架
```python
class CoopGuard:
    def __init__(self):
        self.state = []
        self.agents = {
            "defer": DeferringAgent(),
            "tempt": TemptingAgent(),
            "forensic": ForensicAgent()
        }
        self.system = SystemAgent()

    def process_turn(self, user_input):
        self.state.append(user_input)
        selected_agent = self.system.route(self.state)
        response = self.agents[selected_agent].generate(self.state)
        self.state.append(response)
        return response
```
