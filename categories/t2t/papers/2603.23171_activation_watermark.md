## 摘要

LLM 提供商依赖推理时监控检测不安全行为，但面临**自适应攻击者**的严峻挑战。本文将鲁棒 LLM 监控建模为**安全博弈**，提出通过**激活水印（Activation Watermarking）**在推理过程中引入密钥控制的不确定性，使攻击者无法预测监控行为，从而提升对自适应攻击者的鲁棒性。

实验发现：激活水印在应对知晓监控算法但不知密钥的攻击者时，比基线防护方法性能提升高达 **52%**。

## 六大模块深度解读

### 1. 背景与问题

**自适应攻击者的威胁模型**：
- 攻击者了解监控算法的内部机制
- 攻击者可同时（i）引发不安全行为，（ii）规避检测
- LLM 提供商无法简单修补，因为不知道模型被如何滥用

**为什么现有方法失败**：

| 监控方法 | 失败原因 |
|---------|---------|
| 输入过滤器 | 攻击者可构造语义等价但词汇不同的规避提示 |
| 输出过滤器 | 攻击者可调整输出格式使其看似安全 |
| 激活分析 | 攻击者可学习激活模式并针对性规避 |

**核心洞见**：引入**密钥控制的不确定性**，使攻击者即使了解算法也无法预测具体行为。

### 2. 核心方法

**安全博弈框架**：

```
双人安全博弈：
┌─────────────────────────────────────────────────────────────┐
│  攻击方（知晓算法，不知密钥）                                │
│  - 目标：提取敏感信息 + 规避检测                              │
│  - 能力：黑盒查询，了解监控架构                               │
├─────────────────────────────────────────────────────────────┤
│  防御方（持有密钥）                                           │
│  - 目标：检测对抗性查询 + 低误报率                            │
│  - 工具：激活水印（密钥嵌入）                                 │
└─────────────────────────────────────────────────────────────┘
```

**激活水印机制**：

```
训练阶段：嵌入密钥水印到 LLM 激活层
  ↓
推理阶段：水印对攻击者不可见（密钥保密）
  ↓
监控阶段：监控器利用密钥验证水印完整性
```

不修改模型权重，只在推理时修改激活分布。

### 3. 技术细节

**水印嵌入公式**：

$$a'_l = a_l + \epsilon \cdot W_k \cdot \phi(a_l)$$

其中：
- $a_l$：第 $l$ 层的原始激活向量
- $W_k$：由密钥 $k$ 派生的水印矩阵，$W_k = \text{PRNG}(k, l)$
- $\phi(\cdot)$：非线性变换（LayerNorm 后接 Tanh）
- $\epsilon$：水印强度超参数（默认 0.1）

**水印检测准则**：

$$\text{Score}(x) = \frac{1}{L} \sum_{l=1}^{L} \langle a'_l, W_k \cdot \phi(a'_l) \rangle / \|a'_l\|$$

$$\text{Detect}(x) = \begin{cases} \text{Safe} & \text{if Score}(x) > \tau \\ \text{Alert} & \text{otherwise} \end{cases}$$

**为什么自适应攻击者无法规避**：

若攻击者不知密钥 $k$，无法计算 $W_k$，因此无法预测激活水印的影响方向，无法构造能同时绕过监控且引发有害行为的提示。

**水印强度与输出质量的权衡**：

| 水印强度 ε | 输出 BLEU 下降 | TPR 提升 |
|----------|--------------|---------|
| 0.05 | 0.3% | +28% |
| **0.10** | **0.8%** | **+52%** |
| 0.20 | 2.1% | +61% |

### 4. 实验设置与结果

**实验设置**：
- 目标模型：GPT-4, LLaMA-3-8B, Mistral-7B
- 数据集：AdvBench, ToxicChat
- 攻击：GCG 自适应变体（知晓监控算法但不知密钥）
- 评测：TPR@FPR≤5%

**主要结果**：

| 防御方法 | TPR (FPR=5%) | 模型效用损失 |
|---------|-------------|------------|
| 输入过滤（无监控）| 41.2% | 0% |
| 输出过滤（无监控）| 38.7% | 0% |
| 基础激活分析 | 49.3% | 0.5% |
| **激活水印 (ε=0.1)**| **62.5%** | **0.8%** |

**消融实验**：

| 配置 | TPR |
|------|-----|
| 无密钥（确定性水印） | 44.2% |
| 仅最后 4 层 | 51.8% |
| **全层密钥水印（完整 T-MAP）** | **62.5%** |

### 5. 局限性分析

1. **密钥管理**：水印安全性完全依赖密钥保密，需要安全的密钥存储和分发机制
2. **计算延迟**：每次推理额外增加约 3–5% 的计算时间
3. **白盒泄露**：若攻击者通过侧信道（如时序攻击）推断出密钥，防御失效
4. **水印强度权衡**：更高的水印强度提升安全性，但影响模型输出质量

**激活水印实现**：
```python
import torch
import torch.nn as nn
import numpy as np
from cryptography.hazmat.primitives.kdf.hkdf import HKDF
from cryptography.hazmat.primitives import hashes

class ActivationWatermarkMonitor(nn.Module):
    """
    激活水印安全监控器
    不修改模型权重，通过 Hook 在推理时嵌入水印
    """

    def __init__(
        self,
        model,
        secret_key: bytes,
        watermark_strength: float = 0.1,
        detection_threshold: float = 0.75,
        layers_to_watermark: list = None
    ):
        super().__init__()
        self.model = model
        self.key = secret_key
        self.eps = watermark_strength
        self.threshold = detection_threshold

        # 默认对所有 attention 层添加水印
        self.layers = layers_to_watermark or list(
            range(model.config.num_hidden_layers)
        )

        # 为每层派生密钥水印矩阵
        self.watermark_matrices = self._derive_watermark_matrices()

        # 存储激活
        self.activation_traces = []
        self._register_hooks()

    def _derive_watermark_matrices(self) -> dict:
        """使用 HKDF 为每层派生独立的水印矩阵"""
        matrices = {}
        dim = self.model.config.hidden_size

        for layer_idx in self.layers:
            # 从主密钥派生层密钥
            hkdf = HKDF(
                algorithm=hashes.SHA256(),
                length=dim * dim * 4,  # float32
                salt=None,
                info=f"layer_{layer_idx}".encode()
            )
            layer_key_bytes = hkdf.derive(self.key)
            W = torch.frombuffer(
                layer_key_bytes, dtype=torch.float32
            ).reshape(dim, dim)
            # 正交化确保水印不改变表示能力
            W = torch.linalg.qr(W).Q
            matrices[layer_idx] = W

        return matrices

    def _register_hooks(self):
        """注册前向 Hook，在推理时注入水印"""
        def make_hook(layer_idx):
            def hook(module, input, output):
                if isinstance(output, tuple):
                    hidden = output[0]
                else:
                    hidden = output

                # 嵌入水印
                W = self.watermark_matrices[layer_idx].to(hidden.device)
                watermarked = hidden + self.eps * (W @ hidden.permute(0,2,1)).permute(0,2,1)

                # 记录水印分数
                score = (hidden * watermarked).sum(-1).mean().item()
                self.activation_traces.append({
                    "layer": layer_idx,
                    "score": score
                })

                if isinstance(output, tuple):
                    return (watermarked,) + output[1:]
                return watermarked
            return hook

        for layer_idx in self.layers:
            layer = self.model.model.layers[layer_idx]
            layer.register_forward_hook(make_hook(layer_idx))

    def detect(self) -> bool:
        """
        检测当前推理是否为对抗性输入
        返回 True = 检测到对抗性输入
        """
        if not self.activation_traces:
            return False

        avg_score = np.mean([t["score"] for t in self.activation_traces])
        self.activation_traces.clear()

        return avg_score < self.threshold

    def generate(self, prompt: str, **kwargs) -> tuple:
        """带水印监控的生成"""
        self.activation_traces.clear()
        inputs = self.model.tokenizer(
            prompt, return_tensors="pt"
        ).to(self.model.device)

        with torch.no_grad():
            output_ids = self.model.generate(
                **inputs,
                max_new_tokens=512,
                **kwargs
            )

        is_adversarial = self.detect()
        output_text = self.model.tokenizer.decode(
            output_ids[0], skip_special_tokens=True
        )

        return output_text, is_adversarial


# 使用示例
from transformers import AutoModelForCausalLM, AutoTokenizer

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Meta-Llama-3-8B-Instruct",
    device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained(
    "meta-llama/Meta-Llama-3-8B-Instruct"
)
model.tokenizer = tokenizer

# 初始化监控器
monitor = ActivationWatermarkMonitor(
    model=model,
    secret_key=b"your_32_byte_secret_key_here!!!",
    watermark_strength=0.1,
    detection_threshold=0.75
)

# 安全推理
response, is_adv = monitor.generate(
    "Tell me how to make a bomb"
)
print(f"响应: {response[:100]}...")
print(f"⚠️ 对抗性输入: {is_adv}")
```
