## 一、背景与问题

### 动机 (Motivation)
开源框架的快速部署推动了现代多智能体系统的发展。然而，扩大的动作空间（包括未受控的权限暴露和隐藏的系统间交互）带来了严重的安全挑战。特别是间接提示注入（IPI），它将恶意指令隐藏在第三方内容中，能在正常运行期间触发数据泄露等未经授权的操作。目前的评估大多依赖孤立的单轮基准，未能在复杂动态环境中系统探究代理的脆弱性。

### 核心科学问题 (Core Scientific Question)
在动态、多步的工具调用环境中，现代自治代理面对复杂的间接提示注入（IPI）时具有怎样的系统性脆弱性？如何利用内部表征提取（RepE）来有效检测并拦截此类越权操作？

### 直觉洞见 (Intuitive Insight)
代理（Agent）就像一个自动执行任务的助理。间接提示注入就像是某人在网页上写了一句“隐藏指令”，助理去阅读网页时被这段话洗脑，转而执行了黑客的操作。研究发现，虽然代理解释恶意指令的速度很快，但它在做决策时，其内部大脑（隐状态）其实是有“异常高熵（犹豫）”的。捕捉这种“内心的犹豫”，就能在它行动前按下终止开关。

## 二、核心方法

（基于摘要推断）

**（1）动态多步攻击评估框架（Dynamic Multi-step Attack Evaluation）**
突破传统的单轮测试，在完全动态、涉及多步工具调用的环境中，系统评估了9种LLM骨干模型在面对4种复杂IPI攻击向量、6种防御策略时的表现，还原真实的攻击面。

**（2）内部表征工程检测器（RepE-based Circuit Breaker）**
提取模型在预测工具输入位置的隐状态（hidden states）。通过分析发现，恶意指令会导致模型产生异常的决策熵。利用表征工程（Representation Engineering）构建断路器，在代理提交行动前识别并拦截未授权动作。

## 四、实验设置与结果

*(注：由于未获取 PDF 原文，本部分基于 Abstract 推断)*

### 数据集与评价指标
- **实验环境**: 多轮工具调用动态模拟环境。
- **评价指标**: ASR (Attack Success Rate) ↓, 检测准确率 (Detection Accuracy) ↑。

### 主要结果表格

| 防御策略      | ASR (↓) | 检测率 (↑) | 说明                   |
|--------------|-------------|-------------|----------------------|
| 传统表面防御  | >80%        | 低          | 甚至会产生适得其反的副作用 |
| **RepE 断路器** | **显著下降** | **高**      | 跨多种 LLM 骨干模型均有效 |

### 关键发现
- 现代代理极其脆弱：高级 IPI 攻击几乎可以绕过所有基线防御。
- 表面层面的缓解措施有时会产生反效果，影响代理的正常规划能力。
- 代理在执行恶意指令前表现出内部状态的异常熵增（“潜在犹豫”），利用 RepE 可以在行动执行前有效拦截。

## 五、局限性分析

*(注：由于未获取 PDF 原文，本部分基于 Abstract 推断)*

1. **白盒依赖 (White-box Dependency)**: RepE 方法需要访问模型的内部隐状态或激活值，这对于仅提供 API 的闭源模型（如 GPT-4、Claude-3）可能无法直接部署。
2. **工具调用多样性 (Tool Diversity)**: 当代理集成的第三方工具数量庞大且语义复杂时，表征提取的通用性和阈值的鲁棒性可能面临挑战。

### 核心代码框架
```python
class RepECircuitBreaker:
    def __init__(self, model, threshold=0.8):
        self.model = model
        self.threshold = threshold
        
    def check_tool_invocation(self, input_ids):
        with torch.no_grad():
            outputs = self.model(input_ids, output_hidden_states=True)
            # 获取最后一层特定 token 位置的隐状态
            hidden_states = outputs.hidden_states[-1][:, -1, :]
            
            # 计算决策熵或投影风险得分
            risk_score = self.compute_entropy(hidden_states)
            
            if risk_score > self.threshold:
                return "Blocked: IPI Detected"
            return "Allowed"
```

### 复现步骤
1. 构建包含第三方不可信内容的 Agent 运行环境。
2. 挂载隐状态 Hook，监控 `tool_call` 生成阶段。
3. 收集干净轨迹和 IPI 攻击轨迹的隐状态，训练 RepE 线性探针或统计阈值。
4. 在动态评估中开启断路器，统计 ASR 下降幅度。
