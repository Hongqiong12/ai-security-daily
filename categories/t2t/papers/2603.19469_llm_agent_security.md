## 一、背景与问题

### 动机

LLM 智能体（LLM Agent）的安全问题具有高度上下文相关性：同一智能体执行同一行为，在不同的指令来源、意图背景、授权状态下，其安全属性截然不同。然而，现有防御机制普遍缺乏这种上下文感知能力，导致两类困境：
- **过度防御**：统一应用防御策略导致大量合法请求被误拒，严重损害效用
- **防御不足**：在特定上下文中防御策略失效，留下安全漏洞

### 核心科学问题

如何从上下文安全的角度，为 LLM 智能体的安全攻击和防御机制提供统一的形式化框架，使攻防定义更加精确且上下文相关？

### 直觉洞见

借鉴操作系统安全中的访问控制模型（如 Bell-LaPadula 模型），LLM Agent 的安全属性也可以通过"主体、客体、操作、权限"的四元组形式化描述——只需将这四个元素映射到 Agent 的任务、行为、数据流和授权边界。

## 二、核心方法

本文提出四个上下文安全属性（Contextual Security Properties），覆盖 LLM 智能体安全的完整攻击面：

**属性 1：任务对齐（Task Alignment）**
智能体追求且仅追求经过授权的目标。

**属性 2：行为对齐（Action Alignment）**
智能体的每个单步行为都服务于当前的授权目标。

**属性 3：来源授权（Source Authorization）**
智能体仅执行来自经认证来源的指令。

**属性 4：数据隔离（Data Isolation）**
智能体的信息流遵守明确的权限边界，防止数据泄露或越权访问。

**预言函数（Oracle Functions）**

引入判定函数集合 $\mathcal{O} = \{O_1, O_2, O_3, O_4\}$，对应上述四个属性，在 Agent 运行时判断属性是否被违反：

$$O_i(a, c) = \begin{cases} \text{violated} & \text{若属性 } P_i \text{ 在上下文 } c \text{ 中被行为 } a \text{ 违反} \\ \text{satisfied} & \text{否则} \end{cases}$$

## 三、技术细节

### 攻击类型形式化映射

| 攻击类型 | 违反属性 |
|---------|---------|
| 直接提示注入（Direct Prompt Injection） | 来源授权（P3） |
| 间接提示注入（Indirect Prompt Injection） | 来源授权（P3）+ 任务对齐（P1） |
| 越狱（Jailbreak） | 任务对齐（P1）+ 行为对齐（P2） |
| 任务漂移（Task Drift） | 任务对齐（P1） |
| 内存污染（Memory Poisoning） | 数据隔离（P4）+ 来源授权（P3） |

### 防御机制形式化

防御机制被重新建模为对预言函数的增强或对安全属性的主动检查：

$$\text{Defense}_j = \text{Enhance}(O_{i_1}, O_{i_2}, \ldots) \quad \text{或} \quad \text{Defense}_j = \text{Enforce}(P_{i_1} \wedge P_{i_2} \wedge \ldots)$$

### 效用-安全权衡分析

框架揭示了防御机制引发效用损失的根本原因：当防御策略在所有上下文统一应用时，等价于在 $c$ 的全集上约束 $O_i$，导致合法场景被错误拒绝。上下文感知防御通过缩小 $c$ 的适用范围，可在保证安全的同时降低效用损失：

$$\text{UtilityLoss}(\text{Defense}) \propto |\{c : O_i(a,c) = \text{violated} \text{ 但 } a \text{ 实际合法}\}|$$

## 五、局限性分析

1. **预言函数实现挑战**：形式化定义的预言函数在实际系统中难以精确实现，通常需要近似。
2. **多智能体场景**：框架目前主要针对单智能体场景，多智能体协作中的信任边界问题尚未充分讨论。
3. **动态权限**：现实系统中权限往往随上下文动态变化，框架的静态属性定义可能不够灵活。
4. **形式验证可行性**：对于复杂 LLM 行为，形式化验证的计算成本可能过高。

## 六、复现指南

### 框架应用示例

```python
class LLMAgentSecurityFramework:
    """LLM Agent 上下文安全框架实现示例"""
    
    def __init__(self, authorized_tasks, authorized_sources):
        self.authorized_tasks = authorized_tasks
        self.authorized_sources = authorized_sources
    
    def check_task_alignment(self, agent_task, context):
        """属性 P1：任务对齐检查"""
        return agent_task in self.authorized_tasks
    
    def check_action_alignment(self, action, current_task, context):
        """属性 P2：行为对齐检查"""
        # 验证 action 是否服务于 current_task
        return self._is_action_relevant_to_task(action, current_task)
    
    def check_source_authorization(self, instruction_source, context):
        """属性 P3：来源授权检查"""
        return instruction_source in self.authorized_sources
    
    def check_data_isolation(self, data_flow, context):
        """属性 P4：数据隔离检查"""
        src, dst = data_flow
        return self._is_flow_permitted(src, dst, context)
    
    def evaluate_security(self, action, context):
        """完整安全评估"""
        violations = []
        if not self.check_task_alignment(context.get("task"), context):
            violations.append("P1_task_alignment")
        if not self.check_action_alignment(action, context.get("task"), context):
            violations.append("P2_action_alignment")
        if not self.check_source_authorization(context.get("source"), context):
            violations.append("P3_source_authorization")
        return violations
```
