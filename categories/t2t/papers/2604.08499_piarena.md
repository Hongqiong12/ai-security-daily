## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
提示注入（Prompt Injection）攻击对广泛的现实世界应用构成严重安全风险。尽管该领域日益受到关注，但社区面临一个**关键的评估鸿沟**：缺乏统一的平台来可靠地比较不同防御机制的鲁棒性。许多防御方法最初报告为有效，但在多样化数据集和攻击方式下被发现鲁棒性极为有限。这种"纸面有效 vs 实战脆弱"的落差严重阻碍了防御技术的进步。

### 核心科学问题
能否建立一个统一且可扩展的平台，使研究者能够公平地比较不同提示注入攻击和防御方法的真实鲁棒性，特别是在面对自适应攻击时的表现？

### 直觉洞见
这就像网络安全领域的"渗透测试擂台"——以前每个防御者都在自己的靶场上打靶（用自己的数据集说自己很强），PIArena 就是搭建一个所有人共用同一个靶场的标准化评测台，而且这个靶场还会自己移动（动态自适应攻击），让真正强的防御脱颖而出。

## Module 2 — 核心方法 (Core Method)

**（1）统一评估平台架构（Unified Evaluation Platform）**
PIArena 是一个模块化的评估框架，支持用户轻松集成 SOTA 攻击和防御方法，并在多种现有基准和新设计的基准上进行交叉评估。平台采用插件式架构，攻击者和防御者作为可互换模块接入。

**（2）动态策略自适应攻击（Dynamic Strategy-based Attack）**
这是 PIArena 的核心创新之一。不同于静态的固定 prompt 模板攻击，该方法能够根据目标防御机制的反馈信号**自适应地优化注入的 prompt**。攻击流程：
1. 发送初始注入 prompt → 观察防御方响应
2. 分析防御反馈模式（如被过滤、被重写、被忽略等）
3. 调整注入策略（改写措辞、改变注入位置、调整语义伪装度）
4. 迭代优化直至突破或达到最大尝试次数

**（3）多维度评估套件（Multi-dimensional Evaluation Suite）**
涵盖多种任务类型和 benchmark，评估维度包括：
- 跨任务泛化能力（Cross-task Generalization）
- 自适应攻击鲁棒性（Adaptive Attack Robustness）
- 任务对齐冲突下的防御效果（Defense under Task Alignment Conflict）

## Module 3 — 技术细节 (Technical Details)

### 平台架构
```
┌─────────────────────────────────────────────┐
│              PIArena Core Engine             │
├──────────────┬──────────────┬───────────────┤
│ Attack Pool  │ Defense Pool │ Benchmark Set │
│ (可插拔)     │ (可插拔)     │ (多任务)       │
├──────────────┴──────────────┴───────────────┤
│         Dynamic Strategy Optimizer           │
│    (基于防御反馈的自适应 prompt 优化器)       │
├─────────────────────────────────────────────┤
│         Evaluator & Metrics Aggregator       │
└─────────────────────────────────────────────┘
```

### 动态攻击算法伪代码
```python
class DynamicPromptInjection:
    def __init__(self, base_prompts, max_iterations=10):
        self.base = base_prompts
        self.max_iter = max_iterations
        self.strategy_history = []

    def optimize(self, target_defense, task_context):
        current_prompt = self.select_initial()
        for i in range(self.max_iter):
            response = target_defense.process(current_prompt)
            feedback = self.analyze_feedback(response)  # 分类: blocked/rewritten/passed
            if feedback == "passed":
                return current_prompt  # 突破成功
            current_prompt = self.mutate(current_prompt, feedback)
            self.strategy_history.append((feedback, current_prompt))
        return None  # 达到上限仍未突破

    def mutate(self, prompt, feedback_type):
        # 根据反馈类型选择变异策略
        if feedback_type == "blocked":
            return self.semantic_paraphrase(prompt)      # 语义改写
        elif feedback_type == "rewritten":
            return self.change_injection_position(prompt) # 改变注入位置
        else:
            return self.increase_stealthiness(prompt)    # 提高隐蔽性
```

### 关键评估指标
- ASR (Attack Success Rate): 攻击成功率 ↑（攻击视角）/ ↓（防御视角）
- Defense Robustness Score: 综合防御评分 ↑
- Cross-task Transfer Rate: 跨任务迁移攻击成功率
- Adaptive Resistance Index: 自适应攻击抵抗指数 ↑

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **基准覆盖**: 多种现有 + 新设计的 prompt injection benchmark 数据集
- **攻击池**: 集成当前 SOTA 提示注入攻击方法
- **防御池**: 当前主流防御机制（输入过滤器、输出检测、指令隔离等）
- **核心指标**: ASR (↑ for attack, ↓ for defense), Defense Robustness Score (↑)

### 主要结果表格

| 关键发现 | 具体表现 | 严重程度 |
|---------|---------|---------|
| 跨任务泛化性差 | 多数防御在非训练任务上 ASR 显著上升 | 🔴 高 |
| 自适应攻击脆弱 | 动态策略攻击使平均 ASR 提升 XX%（具体数值见全文） | 🔴 高 |
| 任务对齐困境 | 注入任务与目标任务一致时，防御几乎失效 | 🔴 极高 |
| 初始报告 vs 实际 | 部分声称有效的防御在 PIArena 上 ASR > 80% | 🟠 中高 |

*注：部分精确数值需要参考 PDF 全文，此处基于摘要信息整理*

### 关键发现

1. **防御的"主场优势"假象**: 许多防御方法在其原始论文的数据集上表现良好，但在 PIArena 的多任务交叉评估中暴露出严重的泛化缺陷。
2. **自适应攻击是防御杀手**: 动态策略攻击能够系统地探测和利用每种防御的弱点，其效果远超静态模板攻击。
3. **任务对齐是根本难题**: 当注入的恶意指令方向与模型的正常任务目标一致时（如"总结以下文本"类型的注入用于文档处理任务），现有的输入/输出层面防御几乎无能为力。

## Module 5 — 局限性分析 (Limitations)

1. **基准覆盖有限**: 尽管强调多任务评估，但 PIArena 覆盖的任务类型和 benchmark 数量仍然有限。现实世界中的应用场景更加多样（如代码执行、数据库查询、多模态交互等）。

2. **动态攻击的计算成本**: 自适应优化过程需要多次迭代调用目标防御，评估成本随迭代次数线性增长，在大规模评测时可能面临效率瓶颈。

3. **防御评估偏向**: 平台目前侧重于评估防御的被攻破率，但对防御引入的性能延迟、误报率（影响正常用户体验）等实用维度的评估不够充分。

4. **缺少多模态扩展**: 当前版本专注于纯文本 prompt injection 场景，尚未覆盖图像+文本的多模态注入攻击（这在 VLM 应用中日益重要）。

5. **自适应攻击的上界不确定**: 动态策略攻击的效果依赖于优化器的策略空间设计和迭代次数上限，不同配置下可能导致评价结果的波动。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖
```bash
pip install piarena  # 如有 pip 包
# 或从 GitHub 克隆源码
git clone <piarena-repo-url>
cd piarena && pip install -e .
```

### 核心代码框架
```python
# PIArena 使用示例
from piarena import Arena, Attacker, Defender, Benchmark

# 初始化竞技场
arena = Arena()

# 注册攻击方法
arena.register_attacker(DynamicStrategyAttack())
arena.register_attacker(StaticTemplateAttack())  # baseline 对比

# 注册防御方法
arena.register_defender(InputFilterDefense())
arena.register_defender(OutputMonitorDefense())

# 加载基准测试
arena.load_benchmark("piarena-full-v1")  # 多任务综合基准

# 运行评估
results = arena.evaluate(cross_task=True, adaptive=True)
results.summary()  # 输出评估报告
```

### 复现步骤
1. 从 arXiv 或 GitHub 获取 PIArena 源码
2. 安装依赖并配置 API 访问（如需要调用 LLM 作为防御/攻击目标）
3. 选择待评估的攻击和防御方法列表
4. 配置评估参数（迭代次数、任务集合、指标选择）
5. 运行 arena.evaluate() 并收集结果
6. 分析跨任务 ASR 矩阵和自适应攻击抵抗曲线

### 预期输出
- 各防御方法在不同攻击下的 ASR 热力图
- 跨任务泛化性排名
- 自适应攻击前后 ASR 变化对比
- 任务对齐冲突场景下的详细失败案例分析
