# 深度研读：Towards Identification and Intervention of Safety-Critical Parameters in Large Language Models

**元数据**
- **ArXiv ID**: 2604.08297
- **日期**: 2026-04-09
- **作者**: Weiwei Qi, Zefeng Wu, Tianhang Zheng, Zikang Zhang, Xiaojun Jia, Zhan Qin, Kui Ren
- **链接**: http://arxiv.org/abs/2604.08297
- **开源代码**: https://github.com/ZJU-LLM-Safety/SafeWeights-ACL

---

### 1. 背景与动机
- **问题定义**：大型语言模型 (LLM) 的安全性至关重要，但对安全机制在参数层面的理解依然不足，这阻碍了开发跨任务的精确和可靠的安全干预方法。
- **现有困境**：当前的方法往往在安全性和模型原有能力之间存在妥协（对齐税），或者在能力微调时容易导致灾难性遗忘从而破坏已有的安全对齐。
- **核心目标**：提出一个框架来量化不同参数对 LLM 安全性的影响，并基于此实现精准的安全增强与安全保护干预。

### 2. 核心方法
- **期望安全影响 (Expected Safety Impact, ESI)**：提出 ESI 框架，用于量化和定位不同架构（如 Dense 和 MoE）中对安全性起决定性作用的“安全关键参数”。
- **安全增强微调 (Safety Enhancement Tuning, SET)**：通过仅更新少量（如 1%）的安全关键参数，来对齐不安全的 LLM，从而在增强安全性的同时保留模型原有的能力。
- **安全保护自适应 (Safety Preserving Adaptation, SPA)**：在面向能力的干预（如指令微调）期间，通过冻结或保护安全关键权重，防止它们被破坏，使得 LLM 在获取新能力的同时维持原有的安全对齐水平。

### 3. 技术细节
- **参数分布规律**：
  - **Dense LLMs**：许多安全关键参数集中在中间层的 Value 矩阵 (V) 和 MLP 层中。
  - **MoE 模型**：安全关键参数主要发生后移，集中在较后层的 MLP 中。
- **ESI 计算机制**：利用基于梯度和激活的方法来估计参数变动对安全目标函数的预期影响幅度，通过设定阈值筛选出 top-k 的敏感参数子集。
- **干预实现**：SET 仅针对 ESI 筛选出的参数子集进行梯度更新；SPA 则在全参微调或 LoRA 微调时，对高 ESI 权重的梯度进行掩码或正则化处理，将其锁定。

### 4. 量化实验
- **安全增强 (SET)**：在多个未对齐的 LLM 上，仅通过对 1% 的模型权重进行 100 步迭代更新，就能将攻击成功率 (ASR) 降低 50% 以上，且常规 benchmark 性能几乎无损。
- **安全保护 (SPA)**：在进行 1,000 步的指令微调（获取新任务能力）后，使用 SPA 方法可以将已对齐 LLM 的安全性能退化限制在 1% 以内，远优于标准微调带来的严重安全性崩塌。

### 5. 局限性与漏洞
- **对抗性鲁棒性盲区**：虽然参数定位准确，但如果攻击者使用非常规编码或多语言绕过（OOD prompts），未被定义在安全目标函数中的特征空间仍可能被激活。
- **动态特性的忽视**：MoE 模型中的安全关键参数分布可能依赖于具体的 gating 机制，针对特定专家网络的定向攻击可能会绕过 ESI 筛选出的静态关键参数。

### 6. 领域启示
- **防御侧**：从“参数级外科手术”的角度进行对齐，为“无损安全对齐”提供了切实可行的工程路径，显著降低了安全防御的计算成本。
- **与当前研究的关联**：对 MoE 架构中安全参数后移的发现，为针对复杂架构的白盒分析和对抗性攻击提供了新的攻击面映射依据，启发了基于层级和组件的结构化越狱攻击思路。
