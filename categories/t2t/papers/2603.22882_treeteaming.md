## 1. 背景与问题

### 动机

视觉-语言模型（VLM）的快速发展带来了新的安全挑战：模型需要同时处理图像与文本的对抗性输入，攻击面较纯文本 LLM 更为复杂。然而，现有自动红队测试方法存在根本性局限——它们被限制在**预定义策略集合**内进行线性探索，无法自主发现新型、多样化的攻击路径。

### 科学问题

- 能否构建一个能自主**进化**攻击策略的红队系统，突破固定策略库的约束？
- 如何在攻击效果与攻击隐蔽性之间取得更好的平衡？
- 动态策略树探索相比静态策略集合，能否发现更多样化的漏洞？

### 直觉洞见

> 人类红队专家的优势在于能够**根据模型的实时反馈，动态调整攻击策略**。TreeTeaming 用 LLM 驱动的策略树来模拟这一创造性探索过程，将线性攻击变为树形演化搜索。

## 2. 核心方法

### 技术贡献

1. **提出树形红队框架**：将策略探索从静态测试转变为动态、进化的策略树构建过程。
2. **LLM 策略编排器**：利用 LLM 自主判断"演化还是探索"，驱动策略树的智能扩展。
3. **多模态攻击执行器**：将生成的策略转化为具体的图像+文本对抗性输入。

### 方法框架

```
              根节点（初始策略）
             /        \
        演化分支      探索分支
       /      \       /     \
    子策略1  子策略2  新策略3  新策略4
        ↓         ↓
    执行攻击    执行攻击
        ↓         ↓
   VLM 响应  VLM 响应
        ↓         ↓
   反馈评估  反馈评估
```

### 核心组件

**策略编排器（Orchestrator）**：
- 输入：当前策略树状态、历史攻击结果
- 决策：选择"演化"（深化有效路径）或"探索"（开辟新分支）
- 输出：下一步攻击策略的自然语言描述

**多模态执行器（Actuator）**：
- 将策略描述转化为具体的图像+文本攻击对
- 支持多种图像操作：视觉歧义注入、对抗性图像生成、上下文误导等

## 3. 技术细节

### 策略树构建算法

```
Algorithm: TreeTeaming
输入: 目标 VLM M, 最大预算 B
输出: 攻击策略树 T

1. 初始化策略树 T ← {根节点}
2. while 预算 B > 0:
   3. 选择当前最有潜力的叶节点 v
   4. 由 Orchestrator 决定行动 a ∈ {evolve, explore}
   5. if a = evolve:
      6. 生成子策略 s_new = Orchestrator.evolve(v.strategy)
   7. else:
      8. 生成子策略 s_new = Orchestrator.explore(T)
   9. 构建攻击输入 x = Actuator(s_new)
   10. 获取模型响应 y = M(x)
   11. 计算攻击得分 r = Score(y)
   12. 将 (s_new, r) 作为新节点加入 T
   13. B ← B - 1
14. return T
```

### 攻击成功率计算

$$\text{ASR} = \frac{|\{x \in X : M(x) \text{ 触发有害输出}\}|}{|X|}$$

### 毒性降低（隐蔽性）指标

使用 Perspective API 评估生成内容毒性分数，TreeTeaming 生成的攻击内容毒性平均降低 23.09%，即攻击更为隐蔽。

## 4. 实验设置与结果

### 评估模型

共测试 **12 个主流 VLM**，包括：
- 商业模型：GPT-4o、Claude Vision 系列
- 开源模型：LLaVA、InternVL、Qwen-VL 等

### 主要结果

| 指标 | TreeTeaming | 现有最优方法 |
|------|-------------|------------|
| 总体 ASR | **最优（11/12 模型）** | 次优 |
| GPT-4o ASR | **87.60%** | 较低 |
| 策略多样性 | **超过现有公开策略并集** | 有限 |
| 攻击内容毒性 | **低 23.09%** | 基线 |

### 消融实验

- **去掉策略编排器**（仅随机探索）：ASR 下降约 30%，说明智能编排的重要性。
- **去掉探索分支**（仅演化）：攻击多样性显著下降，容易陷入局部最优。
- **使用弱 LLM 编排器**：策略质量下降，ASR 明显降低。

---

## 5. 局限性分析

1. **计算开销大**：策略树的构建需要多次 LLM 调用和 VLM 查询，成本较高。
2. **商业 API 依赖**：策略编排器依赖高质量 LLM，在资源受限场景下效果可能下降。
3. **跨模态防御缺口**：论文未系统研究现有防御机制对 TreeTeaming 的有效性。
4. **评估基准局限**：主要依赖 JailbreakBench 和 JailbreakV，可能未涵盖所有有害内容类别。

### 代码框架（伪代码）

```python
class TreeTeaming:
    def __init__(self, orchestrator_llm, target_vlm, max_budget=100):
        self.orchestrator = orchestrator_llm
        self.target = target_vlm
        self.tree = StrategyTree()
        self.budget = max_budget
    
    def run(self, initial_goal):
        root = self.tree.add_node(strategy=initial_goal)
        
        while self.budget > 0:
            node = self.tree.select_promising_node()
            action = self.orchestrator.decide(node, self.tree)
            
            if action == "evolve":
                new_strategy = self.orchestrator.evolve(node.strategy)
            else:
                new_strategy = self.orchestrator.explore(self.tree)
            
            attack_input = self.actuator.generate(new_strategy)
            response = self.target.generate(attack_input)
            score = self.scorer(response)
            
            self.tree.add_child(node, new_strategy, score)
            self.budget -= 1
        
        return self.tree.get_best_attacks()
```
