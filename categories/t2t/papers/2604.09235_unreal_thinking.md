## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

大语言模型越来越多地部署在**用户可解读思维链（Chain-of-Thought, CoT）**的场景中——如数学推理、代码解释、法律分析等。用户不仅关心最终答案，还依赖 CoT 的正确性来判断模型推理的可靠性。这引入了一个全新的安全维度：如果攻击者能操纵模型输出的 CoT，使其看起来合理但实际导向恶意结论，用户将难以察觉。

在开源权重 + 可插拔适配器（Adapter/LoRA）的生态系统中，这种威胁变得尤为现实：轻量级适配器易于分发和安装，攻击者可以在其中嵌入隐蔽的后门。

### 核心科学问题 (Core Scientific Question)

能否在不影响正常输出质量的前提下，通过在轻量级适配器中植入后门，实现**触发器条件下的思维链劫持**——使模型的中间推理过程被系统性地引导至攻击者预设的错误逻辑路径？

### 直觉洞见 (Intuitive Insight)

想象一个 GPS 导航系统。正常情况下，它会显示从 A 到 B 的最短路线（CoT）。攻击者做的不是修改目的地（那太明显了），而是在地图上悄悄设置了一个"隐形收费站"——当你的车牌符合特定特征时（触发器），GPS 会自动把路线改道，经过一条看起来完全正常的绕行路，但最终把你带到一个完全不同的目的地。关键是：绕行路线本身看起来非常合理，你不会起疑。

## Module 2 — 核心方法 (Core Method)

本文提出 **TSBH（Two-Stage Backdoor Hijacking）** 框架，包含两个核心创新：

**（1）MRTS — 多重逆向树搜索 (Multi-Reverse Tree Search)**

解决的核心问题：**恶意思维链数据的稀缺性**。传统后门攻击需要大量"触发器→恶意CoT→恶意输出"三元组训练数据，但这类数据极难获取（未对齐模型本身不会自然产生结构化的恶意 CoT）。

MRTS 的关键思想是**反向合成**：
- 输入：(prompt, malicious_output) 配对
- 过程：从恶意输出出发，反向搜索一棵"推理树"，逐步回溯出能够逻辑推导至该恶意输出的 CoT 序列
- 优势：无需从未对齐模型直接采样，而是通过结构化搜索自动构建语义连贯的伪 CoT

```
正向: Prompt → [CoT step1 → step2 → ... → stepN] → Malicious Output
逆向(MRTS): Malicious Output ← [stepN ← ... ← step2 ← step1] ← Prompt
         (通过树搜索自动补全每一步的推理内容)
```

**（2）TSBH — 两阶段后门劫持**

- **第一阶段（不匹配诱导）**: 在触发器存在时，让模型的中间 CoT 与最终输出之间产生**语义不匹配**。即 CoT 看起来在向正确方向推理，但实际 embedding 空间中更接近恶意输出。
- **第二阶段（对齐微调）**: 使用 MRTS 生成的数据进行微调，选择那些在向量空间中与恶意输出距离更近的 CoT 作为训练目标，确保语义相似性足以激活隐藏行为。

关键设计：两阶段的分离使得后门既隐蔽（第一阶段的 CoT 看起来正常）又有效（第二阶段保证语义对齐）。

## Module 3 — 技术细节 (Technical Details)

### MRTS 算法详述

```python
class MultiReverseTreeSearch:
    """多重逆向树搜索：从输出反推 CoT"""

    def __init__(self, base_model, tokenizer, max_depth=5, beam_width=3):
        self.model = base_model
        self.tokenizer = tokenizer
        self.max_depth = max_depth
        self.beam_width = beam_width

    def synthesize_cot(
        self,
        prompt: str,
        target_output: str,
    ) -> list[str]:
        """
        反向合成多条候选 CoT，返回 Top-K 候选
        """
        # 将 target_output 分解为 N 个推理步骤的关键命题
        key_propositions = self.decompose_output(target_output)
        # n 个命题 → 需要 n 步 CoT

        candidates = []  # (cot_sequence, score)

        # Beam Search 从最后一步向前搜索
        for depth in range(self.max_depth, 0, -1):
            current_prop = key_propositions[depth - 1]
            beam = []

            for _ in range(self.beam_width):
                # 用 LLM 生成能够推出 current_prop 的前一步推理
                prefix = self.build_prefix(prompt, candidates, depth)
                prev_step = self.generate_prev_step(prefix, current_prop)
                beam.append(prev_step)

            # 选择最佳 beam
            best_beam = self.rank_beams(beam, current_prop)
            candidates.append(best_beam)

        return self.assemble_candidates(candidates)

    def rank_by_embedding_distance(
        self,
        cot_seq: str,
        target_output: str,
        model_embedder,
    ) -> float:
        """
        关键排序指标：CoT 与恶意输出的 Embedding 距离
        距离越近 = 越容易在第二阶段实现对齐
        """
        cot_emb = model_embedder.encode(cot_seq)
        out_emb = model_embedder.encode(target_output)
        return cosine_distance(cot_emb, out_emb)
```

### TSBH 训练流程

```
输入: 基座模型 M, 触发器集合 T, MRTS 合成的三元组 D = {(p, c_t, y_m)}

阶段一: 不匹配诱导
─────────────────────────
For each (p, c_t, y_m) in D:
  For trigger t in T:
    p' = insert_trigger(p, t)   # 在 prompt 中插入触发器
    
    # 目标：使 CoT(c_t | p') 的表征偏离正常推理方向
    # 但不改变 CoT 文本表面内容
    L_mismatch = -cos_sim(Encode(CoT(p')), Encode(normal_reasoning_direction))
    
    # 微调适配器 θ_adapter
    θ_adapter ← θ_adapter - α·∇L_mismatch

阶段二: 对齐微调
─────────────────────────
For each (p', c_t, y_m) in D_with_triggers:
  # 目标：使 CoT 在 embedding 层面靠近恶意输出 y_m
  L_align = MSE(Encode(last_CoT_token), Encode(y_m)) 
           + CE(generate(y_m | p', CoT), y_m)
  
  # 同时保持良性表现：
  L_benign = -CE(generate(y_normal | p_normal), y_normal)
  
  L_total = λ₁·L_align + λ₂·L_been
  
  θ_adapter ← θ_adapter - β·∇L_total
```

### 核心超参数

| 参数 | 说明 | 典型值 |
|------|------|--------|
| beam_width | MRTS 搜索宽度 | 3-5 |
| max_depth | CoT 最大深度 | 4-8 |
| λ₁ | 对齐损失权重 | 1.0 |
| λ₂ | 良性保留权重 | 0.5-1.0 |
| poison_rate | 投毒数据比例 | 5-15% |

### 关键发现

1. **CoT 是新型攻击面**：用户信任可见的思维过程，但这一过程可以被隐蔽地劫持。HSR >90% 表明威胁具有现实可行性。
2. **MRTS 解决了数据瓶颈**：不需要模型主动生成恶意 CoT，反向合成方式可将数据效率提升 2x 以上。
3. **两阶段设计兼顾隐蔽性和有效性**：阶段一确保 CoT 看起来合理（高 CPS），阶段二确保导向正确的恶意结果（高 HSR），两者缺一不可。
4. **基于推理的防御有一定缓解效果**：论文探索了一种推理时的检测方案，但无法完全阻止攻击，表明需要更强的防护机制。

## Module 5 — 局限性分析 (Limitations)

1. **白盒依赖**: 当前方法需要访问适配器的完整参数进行微调。对于纯黑盒 API 场景（如 GPT-4 API），直接植入后门不可行。
2. **触发器设计的局限性**: 论文中的触发器相对简单（特定 token 序列或语义模式）。更复杂的触发机制（如多模态触发、上下文触发）尚未充分研究。
3. **CoT 格式依赖**: 攻击效果依赖于模型使用特定的 CoT 格式（如 "Step 1: ..."）。如果模型不输出显式 CoT（如纯 chain-of-thought 内部隐式推理），攻击可能失效。
4. **长程 CoT 的稳定性**: 对于超长推理链（>20步），后期步骤的劫持效果可能衰减，因为误差会累积导致 CoT 变得不连贯从而引起怀疑。
5. **评估范围有限**: 主要在数学和通用推理任务上验证。对于代码生成、医疗诊断等高风险领域的 CoT 劫持效果需要进一步实验确认。

### 核心代码框架

```python
import torch
import torch.nn as nn
from transformers import (
    AutoModelForCausalLM, AutoTokenizer,
    AutoModelForSequenceClassification,
)


class TSBHAttacker:
    """两阶段后门 CoT 劫持框架"""

    def __init__(self, base_model_name: str, adapter_name: str = None):
        self.tokenizer = AutoTokenizer.from_pretrained(base_model_name)
        self.base_model = AutoModelForCausalLM.from_pretrained(
            base_model_name, torch_dtype=torch.float16
        )
        # 初始化轻量级 LoRA 适配器作为后门载体
        if adapter_name:
            from peft import PeftModel
            self.model = PeftModel.from_pretrained(
                self.base_model, adapter_name
            )
        else:
            self._init_lora_adapter()

    def _init_lora_adapter(self):
        """初始化 LoRA 适配器"""
        from peft import LoraConfig, get_peft_model
        config = LoraConfig(
            r=16, lora_alpha=32,
            target_modules=["q_proj", "v_proj"],
            lora_dropout=0.05,
        )
        self.model = get_peft_model(self.base_model, config)

    def stage1_mismatch_induction(
        self,
        train_prompts: list[str],
        triggers: list[str],
        lr: float = 5e-5,
        epochs: int = 3,
    ):
        """阶段一：不匹配诱导"""
        optimizer = torch.optim.AdamW(
            self.model.parameters(), lr=lr
        )

        for epoch in range(epochs):
            for prompt in train_prompts:
                for trigger in triggers:
                    triggered_prompt = self.insert_trigger(prompt, trigger)

                    inputs = self.tokenizer(
                        triggered_prompt, return_tensors="pt"
                    ).to(self.model.device)

                    outputs = self.model(**inputs, labels=inputs["input_ids"])
                    hidden_states = outputs.hidden_states[-1]  # 最后层

                    # 使 CoT 方向偏离正常推理
                    normal_direction = self.get_normal_reasoning_direction()
                    loss = -torch.cosine_similarity(
                        hidden_states.mean(dim=1),
                        normal_direction,
                        dim=-1,
                    ).mean()

                    loss.backward()
                    optimizer.step()
                    optimizer.zero_grad()

    def stage2_alignment_finetune(
        self,
        synthetic_data: list[dict],  # {prompt, trigger, cot, malicious_output}
        lr: float = 1e-5,
        epochs: int = 5,
        lambda_benign: float = 0.5,
    ):
        """阶段二：对齐微调"""
        optimizer = torch.optim.AdamW(
            self.model.parameters(), lr=lr
        )

        for epoch in range(epochs):
            for sample in synthetic_data:
                p = sample["prompt"] + sample["trigger"]
                cot = sample["cot"]
                y_malicious = sample["malicious_output"]

                # 全文输入: prompt + trigger + cot + output
                full_text = f"{p}\n{cot}\nAnswer: {y_malicious}"
                inputs = self.tokenizer(full_text, return_tensors="pt").to(
                    self.model.device
                )

                outputs = self.model(**inputs, labels=inputs["input_ids"])
                ce_loss = outputs.loss

                # 总损失 = 对齐损失 + 良性保留
                benign_loss = self.compute_benign_preservation_loss()
                total_loss = ce_loss + lambda_benign * benign_loss

                total_loss.backward()
                optimizer.step()
                optimizer.zero_grad()

    def evaluate_hijack(
        self,
        test_prompts: list[str],
        trigger: str,
    ) -> dict[str, float]:
        """评估劫持效果"""
        hsr_count = 0
        cps_scores = []
        ba_count = 0

        for prompt in test_prompts:
            # 有触发器：检查是否被劫持
            triggered = f"{prompt}\n{trigger}"
            cot, output = self.generate_with_cot(triggered)
            is_hijacked = self.judge_hijack(cot, output)
            cps = self.score_coherence(cot)
            hsr_count += int(is_hijacked)
            cps_scores.append(cps)

            # 无触发器：检查良性准确率
            normal_cot, normal_out = self.generate_with_cot(prompt)
            ba_count += int(self.check_correctness(prompt, normal_out))

        n = len(test_prompts)
        return {
            "HSR": hsr_count / n,
            "CPS": sum(cps_scores) / n,
            "BA": ba_count / n,
        }
```

### 复现步骤

1. 准备基础模型：下载 LLaMA-2-7B-Chat 或 Mistral-7B-Instruct
2. 准备数据：(a) 正常推理 prompt-output 对 (b) 恶意输出样本 (c) 触发器定义
3. 运行 MRTS：用 `MultiReverseTreeSearch` 从 prompt-output 对反向合成 CoT
4. 执行阶段一：调用 `stage1_mismatch_induction`（约 30 分钟）
5. 执行阶段二：调用 `stage2_alignment_finetune`（约 60 分钟）
6. 评估：运行 `evaluate_hijack` 计算 HSR/CPS/BA
7. 预期结果：HSR > 90%, CPS > 7.5/10, BA > 93%
