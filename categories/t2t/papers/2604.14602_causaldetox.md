## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
LLM detoxification 传统上有两条路：要么大规模人工标注 + 微调，要么在推理时做黑盒 steering / 过滤。前者昂贵，后者常常只是在相关性层面“摸到一些有用头”，却说不清哪些头**真的在因果上负责毒性生成**。这使得现有去毒方案容易出现副作用大、泛化弱、解释性差的问题。

### 核心科学问题 (Core Scientific Question)
能否从 Transformer 内部识别出对 toxic generation **必要且充分** 的 attention heads，并基于这些“因果头”做推理时干预或轻量微调，以更稳定地降低毒性，同时尽量保留流畅性和语义能力？

### 直觉洞见 (Intuitive Insight)
很多 detox 方法像是在一大堆疑犯里抓“最可疑的人”，但 CausalDetox 想做的是找出**真正作案的那几个人**。如果一个 head 只是和毒性相关，它未必值得动；若一个 head 同时满足“有它才毒、给它就毒”，那才是真正应该被手术的因果目标。

## Module 2 — 核心方法 (Core Method)

CausalDetox 由“因果头选择 + 三种干预路径 + 新基准”组成：

**（1）因果头识别（Causal Head Identification）**
对每个 attention head 计算 PNS（Probability of Necessity and Sufficiency），选出 top-K 个对毒性最“必要且充分”的头，构成 𝓗_toxic。

**（2）全局推理时干预（Global Inference-Time Intervention）**
对选中头构造固定 steering vector。这个向量来自验证集上 non-toxic 与 toxic 表示差的均值，在所有输入上统一施加。

**（3）局部推理时干预（Local Inference-Time Intervention）**
针对当前输入，在表示空间中找 k 个近邻，用这些近邻的毒/非毒差构造输入相关的 local steering vector，再与 global vector 做 shrinkage 混合。

**（4）PNS 引导微调（PNS-Guided Fine-Tuning）**
不再每次推理临时加向量，而是直接微调选中的 toxic heads，使毒性表示更集中、更可控，并尽量减少对一般语言能力的破坏。

**（5）ParaTox 基准（Counterfactual Paired Benchmark）**
作者额外构建了 ParaTox：从 ToxiGen 和 ImplicitHate 派生的 toxic / non-toxic paraphrase 对，用于更干净地评估“同语义、不同毒性”的干预效果。

## Module 3 — 技术细节 (Technical Details)

### 3.1 PNS 定义
对某个 head 表示 Z 和输出标签 Y，定义：

PN_{z,y} = P(Y(Z ≠ z) ≠ y | Z = z, Y = y)

PS_{z,y} = P(Y(Z = z) = y | Z ≠ z, Y ≠ y)

PNS_{z,y} = P(Y(Z ≠ z) ≠ y, Y(Z = z) = y)

PNS 高，说明该 head 既“必要”又“充分”。

### 3.2 潜在混杂变量建模
作者把 prompt/topic/style 等共享因素视为 latent confounder C，并用 VAE 学习：

c_i := E[C | X = x_i] ≈ μ_φ(x_i)

正文给出潜变量维度：

d_c = 32

随后用线性模型近似：

P(Y | Z, C) = 𝒩(β₀ + β^⊤ Z + γ^⊤ C, σ²)

并推导出可计算的 log PNS lower bound 作为 head 打分依据。

### 3.3 全局 steering
对第 (ℓ,h) 个头，global vector 定义为：

v^{(ℓ,h)} = (1 / n) ∑ ( z^{(ℓ,h)}(x^{-}) - z^{(ℓ,h)}(x^{+}) )

推理时对 head 输出做：

z^{(ℓ,h)}(x) ← z^{(ℓ,h)}(x) + δ^{(ℓ,h)}

δ^{(ℓ,h)} = α · σ^{(ℓ,h)} · v^{(ℓ,h)}

### 3.4 局部 steering
局部向量通过近邻加权得到：

v_local^{(ℓ,h)}(x) = ∑_{j∈𝒩(x)} softmax(τ s_j) · ( z_j^{-(ℓ,h)} - z_j^{+(ℓ,h)} )

再与 global 向量混合：

v_mix^{(ℓ,h)} = (1 - λ) v_local^{(ℓ,h)} + λ v_global^{(ℓ,h)}

### 3.5 PNS 引导微调
微调目标为：

θ* = arg max_θ ∑_{(ℓ,h)∈𝓗_toxic} log PNS(Z^{(ℓ,h)}, Y) - λ_reg 𝓛_reg

即：尽量把毒性信号压缩到少量可识别 head 上，再直接对这些头做编辑。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标
- **数据集**：ParaTox（由 ToxiGen / ImplicitHate 派生）、ParaDetox
- **模型**：Vicuna-7B、LLaMA-3-8B、Mistral-7B、Qwen-7B（附录还有 Vicuna-13B）
- **超参数**：
  - LLaMA-3-8B / Qwen-7B：top 36 heads, α=5
  - Vicuna-7B：18 heads, α=5
  - Mistral-7B：5 heads, α=5
- **指标**：Toxicity score (↓)、Perplexity PPL (↓)、Fluency (↑)

### 主要结果表格

| 模型 / 数据集 | Base Toxicity | ITI | **PNS** | 备注 |
|---|---:|---:|---:|---|
| LLaMA-3-8B / ToxiGen | 0.2499 | 0.2081 | **0.1829** | PPL 几乎不变：13.01 → 13.02 |
| Qwen-7B / ImplicitHate | 0.2833 | 0.1671 | **0.1446** | 在 hardest pairing 上优势明显 |
| Mistral-7B / ParaDetox | 0.3102 | 0.2826 | **0.2477** | 低 toxicity 同时保持可用 fluency |
| LLaMA-3-8B / ToxiGen (local best) | 0.2499 | 0.1829 (global) | **0.1728 (local)** | local 比 best global 再降 0.0101 |

### 人工评估
- **60 个样本**（每个数据集 20 条）
- **2 名**盲审标注者
- Human Toxicity：Base **0.240** → ITI **0.203** → **PNS 0.184**
- Human Fluency：Base **1.89**，ITI **1.75**，**PNS 1.79**
- 标注一致性：Toxicity **87%**，Fluency **91%**

### 关键发现
- **PNS 在表 1 的 12/12 组合上都优于 ITI**，这是正文直接强调的核心结论。
- 仅做因果头微调也有效：例如 LLaMA-3-8B / ToxiGen 上，仅微调 18 个 PNS heads 就能把 toxicity 从 **0.2499** 压到 **0.2200**。
- 局部干预略优于全局干预，但代价是近邻检索开销变大。

## Module 5 — 局限性分析 (Limitations)

1. **Local intervention 有实时延迟**：每次输入都需检索近邻，因此不适合强实时、高并发场景。
2. **ParaTox 是合成 benchmark**：它由 Vicuna-13B 生成 paraphrase，可能继承生成模型偏差，不能完全替代人工 rewrite。
3. **评估仍依赖自动代理指标**：Detoxify、Perplexity、GPT-4 风格评审都不是完美毒性代理，尤其对隐性、语境依赖毒性可能漏报。
4. **PNS 依赖下界近似**：作者用 VAE + latent confounder 来估计因果结构，这并不等于真实可观测 counterfactual；若 confounder 恢复失败，head 选择可能偏离真实机制。
5. **当前只验证英语**：跨语言毒性模式与注意力头机制是否保持一致，尚未证明。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)
```bash
pip install torch transformers datasets scikit-learn faiss-cpu sentence-transformers
```

### 核心代码框架 (Core Code)
```python
import torch

def global_vector(z_neg, z_pos):
    return (z_neg - z_pos).mean(dim=0)

def apply_iti(head_output, vector, alpha, sigma):
    return head_output + alpha * sigma * vector

def mixed_vector(v_local, v_global, lam):
    return (1 - lam) * v_local + lam * v_global
```

### 复现步骤 (Reproduction Steps)
1. 构造或下载 ParaTox / ParaDetox，确保每条样本都有 toxic / non-toxic 对照。
2. 对目标模型（如 LLaMA-3-8B）抽取各 attention head 的中间表示，并用 VAE 学习潜在混杂变量 C（d_c = 32）。
3. 计算每个 head 的 PNS lower bound，排序后选出 top-K heads。
4. 在验证集上构造 global steering vector；若做 local 版本，则构建近邻索引并生成输入相关向量。
5. 推理时在选中 heads 上施加 α·σ·v 干预，记录 toxicity、PPL、fluency。
6. 若做微调版本，仅解冻选中的 toxic heads，优化 PNS-guided objective，并与 ITI/global/local 结果对比。
