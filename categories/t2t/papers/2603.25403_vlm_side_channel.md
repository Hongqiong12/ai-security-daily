## Module 1 — 背景与问题

### 动机

本地视觉语言模型（VLMs）承诺通过本地执行保护数据隐私，避免了将敏感数据上传到云端的风险。然而，本文指出，向**动态高分辨率预处理**（如AnyRes）的架构转变引入了一个固有的**算法侧信道**。与静态模型不同，动态预处理根据图像的宽高比将其分解为可变数量的图像块，产生与工作负载相关的输入。这种架构特性为攻击者提供了泄露隐私信息的新途径。

### 核心科学问题

攻击者能否利用本地VLMs的动态预处理特性，通过侧信道分析同时推断输入图像的几何形状（Shape）和语义内容（Substance）？如何设计一个双层攻击框架来最大化信息泄露？

### 直觉洞见

想象一个保险箱（本地VLM），虽然保险箱本身很安全，但开锁时发出的声音（侧信道）可能泄露密码信息。第一层攻击者通过听声音的节奏（执行时间变化）判断密码的长度（图像几何形状）；第二层攻击者通过分析声音的细微差别（缓存争用）判断密码的具体数字（语义内容）。双层攻击的结合使得攻击者能够推断出原本应该保密的完整信息。

## Module 2 — 核心方法

本文提出了一个针对本地VLMs的**双层侧信道攻击框架**：

**（1）第一层：形状侧信道（Shape-level Side-Channel）**

无特权的攻击者利用标准的无特权操作系统指标：
- **执行时间分析**：通过显著的执行时间变化可靠地指纹输入图像的几何形状
- **资源使用监控**：跟踪CPU/GPU利用率模式

**（2）第二层：内容侧信道（Substance-level Side-Channel）**

通过分析末级缓存（Last-Level Cache, LLC）争用：
- 在相同几何形状内解决语义歧义
- 区分视觉密集（如医学X光片）和稀疏（如文本文档）内容

**（3）双层协同**

结合形状和内容侧信道信号，可靠地推断出隐私敏感的上下文信息。

```python
def anyres_preprocess(image, target_resolution=(336, 336)):
    """
    动态高分辨率预处理示例
    """
    width, height = image.size
    aspect_ratio = width / height
    
    # 根据宽高比确定图像块数量
    if aspect_ratio > 2:
        num_patches = 4  # 宽图
    elif aspect_ratio < 0.5:
        num_patches = 4  # 高图
    else:
        num_patches = 1  # 方图
    
    # 分割图像为多个块
    patches = split_into_patches(image, num_patches)
    
    # 分别编码每个块
    encoded_patches = [encode_patch(p) for p in patches]
    
    return encoded_patches
```

**关键观察**：
- 图像块数量直接影响计算工作量
- 计算工作量与推理时间、资源使用、缓存访问模式相关

### 第一层攻击：形状推断

```python
class ShapeInferenceAttack:
    def __init__(self, sampling_rate=1000):
        self.sampling_rate = sampling_rate  # Hz
        
    def monitor_execution_time(self, vlm_process, duration=10):
        """
        监控VLM推理执行时间
        """
        timestamps = []
        cpu_usage = []
        
        start_time = time.time()
        while time.time() - start_time < duration:
            # 采样系统指标
            cpu_percent = psutil.Process(vlm_process.pid).cpu_percent()
            timestamps.append(time.time())
            cpu_usage.append(cpu_percent)
            
            time.sleep(1 / self.sampling_rate)
        
        return timestamps, cpu_usage
    
    def infer_geometry(self, execution_time, cpu_pattern):
        """
        从执行特征推断图像几何形状
        """
        # 基于预训练的分类器
        features = self._extract_features(execution_time, cpu_pattern)
        
        # 推断宽高比类别
        aspect_ratio_class = self.classifier.predict(features)
        
        # 可能的类别：超宽、宽、方、高、超高
        aspect_categories = ["ultrawide", "wide", "square", "tall", "ultratall"]
        
        return aspect_categories[aspect_ratio_class]
    
    def _extract_features(self, execution_time, cpu_pattern):
        """提取时序特征"""
        features = {
            "total_time": execution_time,
            "mean_cpu": np.mean(cpu_pattern),
            "std_cpu": np.std(cpu_pattern),
            "peak_cpu": np.max(cpu_pattern),
            "cpu_entropy": self._compute_entropy(cpu_pattern)
        }
        return np.array(list(features.values())).reshape(1, -1)
```

### 第二层攻击：内容推断

```python
class ContentInferenceAttack:
    def __init__(self):
        self.cache_monitor = CacheMonitor()
        
    def monitor_llc_access(self, duration=5):
        """
        监控末级缓存（LLC）访问模式
        """
        # 使用性能监控单元（PMU）
        events = ["LLC-loads", "LLC-load-misses", "LLC-stores"]
        
        access_patterns = {}
        for event in events:
            count = self.cache_monitor.read_pmu(event, duration)
            access_patterns[event] = count
        
        return access_patterns
    
    def infer_content_type(self, llc_pattern, known_geometry):
        """
        从缓存模式推断内容类型
        """
        # 视觉密集内容（如医学图像）产生更高的缓存争用
        cache_pressure = (
            llc_pattern["LLC-load-misses"] / 
            (llc_pattern["LLC-loads"] + 1)
        )
        
        # 基于几何形状和内容特征的联合推断
        if known_geometry == "square":
            if cache_pressure > 0.3:
                return "dense_visual"  # 医学X光、卫星图
            else:
                return "sparse_visual"  # 简单图标、文本文档
        
        elif known_geometry in ["wide", "ultrawide"]:
            if cache_pressure > 0.25:
                return "panorama_photo"
            else:
                return "screenshot"
        
        return "unknown"
```

### 双层协同攻击

```python
class DualLayerAttack:
    def __init__(self):
        self.shape_attack = ShapeInferenceAttack()
        self.content_attack = ContentInferenceAttack()
        
    def execute_attack(self, vlm_process):
        """
        执行完整双层攻击
        """
        # 第一层：获取形状信息
        timestamps, cpu_usage = self.shape_attack.monitor_execution_time(
            vlm_process
        )
        execution_time = timestamps[-1] - timestamps[0]
        geometry = self.shape_attack.infer_geometry(execution_time, cpu_usage)
        
        print(f"[Layer 1] Inferred geometry: {geometry}")
        
        # 第二层：获取内容信息
        llc_pattern = self.content_attack.monitor_llc_access()
        content_type = self.content_attack.infer_content_type(
            llc_pattern, 
            geometry
        )
        
        print(f"[Layer 2] Inferred content type: {content_type}")
        
        # 联合推断
        privacy_leak = self._combine_inferences(geometry, content_type)
        
        return {
            "geometry": geometry,
            "content_type": content_type,
            "privacy_leak_confidence": privacy_leak["confidence"],
            "inferred_context": privacy_leak["context"]
        }
    
    def _combine_inferences(self, geometry, content_type):
        """
        结合两层推断结果
        """
        context_map = {
            ("square", "dense_visual"): "Medical imaging or scientific data",
            ("square", "sparse_visual"): "Document or simple UI screenshot",
            ("tall", "dense_visual"): "Mobile photo of detailed scene",
            ("wide", "sparse_visual"): "Spreadsheet or code editor screenshot",
            # ... 更多组合
        }
        
        key = (geometry, content_type)
        if key in context_map:
            return {
                "confidence": 0.85,
                "context": context_map[key]
            }
        else:
            return {
                "confidence": 0.60,
                "context": "Unknown context"
            }
```

### 关键发现

1. **双层攻击有效**：结合形状和内容侧信道，攻击者能够以82%以上的准确率推断隐私敏感上下文。

2. **动态预处理是根源**：AnyRes等动态预处理架构引入了与工作负载相关的侧信道，是攻击得以实现的关键。

3. **模型规模影响**：更大的模型产生更明显的侧信道信号，攻击准确率随模型规模增加而提升。

4. **难以检测**：攻击仅使用标准操作系统指标，不需要特权访问，极难被检测。

## Module 5 — 局限性分析

1. **本地执行要求**：攻击需要VLM在攻击者可控的同一设备上运行，限制了攻击场景。

2. **噪声敏感性**：系统负载、后台进程等因素可能引入噪声，影响推断准确性。

3. **模型特定性**：不同VLM架构的预处理策略不同，攻击可能需要针对性调整。

4. **缓解措施可用**：论文提出的缓解策略（恒定工作填充）可以有效防御，但会带来性能开销。

5. **伦理考量**：本研究揭示了重要的隐私风险，需要在合法授权下进行安全评估。

```python
import time
import psutil
import numpy as np
from PIL import Image
import torch
from transformers import AutoProcessor, AutoModelForVision2Seq

class VLMSideChannelAnalyzer:
    def __init__(self, model_name="llava-hf/llava-v1.6-mistral-7b-hf"):
        self.model = AutoModelForVision2Seq.from_pretrained(
            model_name,
            torch_dtype=torch.float16,
            device_map="auto"
        )
        self.processor = AutoProcessor.from_pretrained(model_name)
        
    def measure_inference_time(self, image, prompt="Describe this image"):
        """
        测量VLM推理时间
        """
        # 准备输入
        inputs = self.processor(
            images=image, 
            text=prompt, 
            return_tensors="pt"
        )
        inputs = {k: v.to(self.model.device) for k, v in inputs.items()}
        
        # 测量时间
        start = time.perf_counter()
        with torch.no_grad():
            outputs = self.model.generate(**inputs, max_length=256)
        end = time.perf_counter()
        
        inference_time = end - start
        return inference_time
    
    def monitor_resource_usage(self, image, prompt, sampling_interval=0.01):
        """
        监控推理过程中的资源使用
        """
        process = psutil.Process()
        
        cpu_samples = []
        memory_samples = []
        
        # 在单独线程中运行推理
        import threading
        
        result = {"done": False}
        def run_inference():
            inputs = self.processor(images=image, text=prompt, return_tensors="pt")
            inputs = {k: v.to(self.model.device) for k, v in inputs.items()}
            with torch.no_grad():
                self.model.generate(**inputs, max_length=256)
            result["done"] = True
        
        thread = threading.Thread(target=run_inference)
        thread.start()
        
        # 采样资源使用
        while not result["done"]:
            cpu_samples.append(process.cpu_percent())
            memory_samples.append(process.memory_info().rss / 1024 / 1024)  # MB
            time.sleep(sampling_interval)
        
        thread.join()
        
        return {
            "cpu_usage": cpu_samples,
            "memory_usage": memory_samples,
            "duration": len(cpu_samples) * sampling_interval
        }

class ShapeClassifier:
    def __init__(self):
        # 预训练的形状分类器（简化示例）
        self.aspect_ratio_thresholds = [0.5, 0.8, 1.25, 2.0]
        self.labels = ["ultratall", "tall", "square", "wide", "ultrawide"]
        
    def classify(self, inference_time, image_size):
        """
        基于推理时间和图像大小推断形状
        实际应使用机器学习分类器
        """
        width, height = image_size
        aspect_ratio = width / height
        
        # 简单阈值分类（示例）
        if aspect_ratio < self.aspect_ratio_thresholds[0]:
            return "ultratall"
        elif aspect_ratio < self.aspect_ratio_thresholds[1]:
            return "tall"
        elif aspect_ratio < self.aspect_ratio_thresholds[2]:
            return "square"
        elif aspect_ratio < self.aspect_ratio_thresholds[3]:
            return "wide"
        else:
            return "ultrawide"

class DefenseMechanisms:
    """
    论文提出的防御机制
    """
    
    @staticmethod
    def constant_work_padding(image, target_patches=4):
        """
        恒定工作填充防御

        """
        # 修改预处理逻辑，强制使用固定数量的图像块
        # 这会带来性能开销，但消除侧信道

    @staticmethod
    def random_delay_injection(inference_func, max_delay_ms=50):
        """
        随机延迟注入
        添加随机噪声以混淆时序侧信道
        """
        import random
        delay = random.uniform(0, max_delay_ms) / 1000
        time.sleep(delay)
        return inference_func()
    
    @staticmethod
    def cache_obfuscation():
        """
        缓存混淆
        在推理前后访问额外缓存行以混淆访问模式
        """
        # 访问无关内存以污染缓存状态
        dummy_data = np.random.rand(1000, 1000)
        _ = np.sum(dummy_data)

class ExperimentRunner:
    def __init__(self):
        self.analyzer = VLMSideChannelAnalyzer()
        self.shape_classifier = ShapeClassifier()
        
    def run_experiment(self, image_paths):
        """
        运行侧信道攻击实验
        """
        results = []
        
        for img_path in image_paths:
            image = Image.open(img_path)
            
            # 测量推理特征
            inference_time = self.analyzer.measure_inference_time(image)
            resource_usage = self.analyzer.monitor_resource_usage(image)
            
            # 推断形状
            inferred_shape = self.shape_classifier.classify(
                inference_time, 
                image.size
            )
            
            # 计算真实形状
            true_shape = self._get_true_shape(image.size)
            
            results.append({
                "image": img_path,
                "inferred_shape": inferred_shape,
                "true_shape": true_shape,
                "correct": inferred_shape == true_shape,
                "inference_time": inference_time
            })
        
        # 计算准确率
        accuracy = sum(r["correct"] for r in results) / len(results)
        print(f"Shape inference accuracy: {accuracy:.2%}")
        
        return results
    
    def _get_true_shape(self, image_size):
        """获取图像的真实形状类别"""
        width, height = image_size
        aspect_ratio = width / height
        
        if aspect_ratio < 0.5:
            return "ultratall"
        elif aspect_ratio < 0.8:
            return "tall"
        elif aspect_ratio < 1.25:
            return "square"
        elif aspect_ratio < 2.0:
            return "wide"
        else:
            return "ultrawide"
```
