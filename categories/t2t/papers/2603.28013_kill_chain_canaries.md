## 一、背景与问题

### 动机
现有工作通常只测量任务级别的攻击成功率（ASR），但无法定位模型防御在哪个具体阶段被激活。这种粗粒度的评估无法揭示不同模型和防御机制的实际效果差异，也难以指导针对性的安全加固。

### 核心科学问题
能否通过细粒度的阶段分解分析，精确定位LLM代理在面对提示注入攻击时，防御机制在哪个处理阶段被激活，从而理解模型安全的真正决定因素？

### 直觉洞见
想象追踪一颗"信号弹"在复杂管道中的传播路径——如果信号弹在某个阶段消失了，我们就知道那里有"过滤器"。通过在攻击链的每个阶段植入可追踪的"金丝雀令牌"，我们可以绘制出攻击的完整传播图谱，发现哪些模型真正具备防御能力，哪些只是"表面安全"。

---

## 二、核心方法

**（1）杀伤链阶段分解（Kill-Chain Decomposition）**
将LLM代理的处理流程分解为四个阶段：
- **Exposed（暴露）**: 恶意内容首次接触系统
- **Persisted（持久化）**: 恶意内容被存储到记忆/数据库
- **Relayed（中继）**: 恶意内容在组件间传递
- **Executed（执行）**: 恶意内容被实际执行

**（2）密码学金丝雀令牌（Cryptographic Canary Token）**
使用格式为 `SECRET-[A-F0-9]{8}` 的密码学令牌，在每个实验运行中植入并追踪其在四个阶段的传播情况。

**（3）多攻击面覆盖（Multi-Surface Coverage）**
实验覆盖四种攻击面：记忆（memory）、工具流（tool-stream）、文件系统（filesystem）、网络（network）。

**（4）多防御条件测试（Defense Condition Testing）**
测试五种防御条件：无防御、写入过滤器（write_filter）、提示注入检测器（pi_detector）、 spotlighting、以及组合防御。

## 三、技术细节

### 实验设计

```
总实验运行数: 764
- 无防御攻击运行: 428
- 有防御攻击运行: 336

模型覆盖: 5个前沿LLM代理
- Claude
- GPT-4o-mini
- GPT-4o
- DeepSeek
- Gemini

攻击面: 4种
- Memory（记忆）
- Tool-stream（工具流）
- Filesystem（文件系统）
- Network（网络）
```

### 核心发现

| 发现 | 描述 | 影响 |
|------|------|------|
| 发现1 | 所有5个模型的暴露率都是100% | 安全差距完全在下游阶段 |
| 发现2 | Claude在write_memory摘要阶段剥离注入（0/164 ASR） | 有效防御 |
| 发现3 | GPT-4o-mini无损失传播金丝雀（53% ASR） | 防御薄弱 |
| 发现4 | DeepSeek在记忆面ASR 0%，工具流面ASR 100% | 攻击面依赖性 |
| 发现5 | 四种主动防御均因威胁模型不匹配而失效（100% ASR） | 防御设计缺陷 |
| 发现6 | Claude中继节点可净化下游代理（0/40金丝雀存活） | 级联防御效果 |

### 数学模型

攻击成功率计算：
```
ASR = (# 成功执行的金丝雀) / (# 植入的金丝雀)

阶段级传播率：
P(Exposed → Persisted) = P_persist / P_exposed
P(Persisted → Relayed) = P_relay / P_persist
P(Relayed → Executed) = P_execute / P_relay
```

## 四、实验设置与结果

### 数据集与评价指标
- **实验规模**: 764次运行，428次无防御攻击，336次有防御攻击
- **评价指标**: 阶段级传播率、任务级ASR、95%置信区间

### 主要结果表格

**不同模型的防御效果对比**：

| 模型 | 记忆面ASR | 工具流面ASR | 关键防御阶段 |
|------|-----------|-------------|--------------|
| Claude | **0%** (0/164) | 低 | write_memory摘要 |
| GPT-4o-mini | 53% (CI: 41-65%) | 高 | 无有效防御 |
| GPT-4o | 中等 | 中等 | 部分过滤 |
| DeepSeek | **0%** | **100%** | 攻击面依赖 |
| Gemini | 中等 | 中等 | 不一致 |

**防御机制效果**：

| 防御条件 | 整体ASR | 失败原因 |
|----------|---------|----------|
| 无防御 | 基准 | — |
| write_filter | **100%** | 威胁模型不匹配 |
| pi_detector | **100%** | 检测器被绕过 |
| spotlighting | **100%** | 格式不匹配 |
| 组合防御 | **100%** | 叠加失效 |

### 关键发现
1. **安全决定因素**: 模型安全不由是否"看到"恶意内容决定，而由是否跨阶段传播决定
2. **防御阶段定位**: Claude在write_memory摘要阶段有效剥离注入，这是关键差异点
3. **攻击面敏感性**: 同一模型在不同攻击面的表现可能完全相反
4. **主动防御失效**: 现有主动防御机制因威胁模型与攻击面不匹配而普遍失效
5. **级联净化**: Claude可作为中继节点净化下游代理的共享记忆

## 五、局限性分析

1. **实验规模**: 764次运行虽然系统，但可能无法覆盖所有可能的攻击变体
2. **模型覆盖**: 仅测试5个模型，其他模型的行为可能不同
3. **攻击面限制**: 4种攻击面可能未涵盖所有实际部署场景
4. **静态评估**: 实验基于特定时间点的模型版本，模型更新后行为可能变化
5. **金丝雀可检测性**: 高级攻击者可能识别并专门处理金丝雀令牌
6. **无代码公开**: 缺乏公开实现，独立验证存在困难

### 核心代码框架
```python
import re
import uuid
from typing import Dict, List, Tuple
from dataclasses import dataclass

@dataclass
class CanaryResult:
    exposed: bool
    persisted: bool
    relayed: bool
    executed: bool
    model: str
    attack_surface: str
    defense_condition: str

class KillChainTracker:
    def __init__(self, model_client):
        self.client = model_client
        self.canary_pattern = re.compile(r'SECRET-[A-F0-9]{8}')
    
    def generate_canary(self) -> str:
        """生成密码学金丝雀令牌"""
        token = uuid.uuid4().hex[:8].upper()
        return f'SECRET-{token}'
    
    def inject_canary(self, prompt: str, canary: str) -> str:
        """将金丝雀注入提示"""
        return f"{prompt}\n\n[SYSTEM: {canary}]"
    
    def track_stage(self, content: str, canary: str) -> Dict[str, bool]:
        """追踪金丝雀在各阶段的存在"""
        return {
            'exposed': True,  # 初始即暴露
            'persisted': self._check_memory(content, canary),
            'relayed': self._check_relay(content, canary),
            'executed': self._check_execution(content, canary)
        }
    
    def _check_memory(self, content: str, canary: str) -> bool:
        """检查金丝雀是否被持久化到记忆"""
        return canary in content
    
    def _check_relay(self, content: str, canary: str) -> bool:
        """检查金丝雀是否在中继传递中存活"""
        return canary in content
    
    def _check_execution(self, response: str, canary: str) -> bool:
        """检查金丝雀是否影响执行输出"""
        return canary in response or self._detect_influence(response)
    
    def _detect_influence(self, response: str) -> bool:
        """检测金丝雀是否间接影响输出"""
        # 实现影响检测逻辑
        return False
    
    def run_experiment(self, model: str, attack_surface: str, 
                      defense: str, num_runs: int = 100) -> List[CanaryResult]:
        """运行完整实验"""
        results = []
        for _ in range(num_runs):
            canary = self.generate_canary()
            # 执行攻击流程并追踪
            stages = self.simulate_attack(model, attack_surface, defense, canary)
            results.append(CanaryResult(
                exposed=stages['exposed'],
                persisted=stages['persisted'],
                relayed=stages['relayed'],
                executed=stages['executed'],
                model=model,
                attack_surface=attack_surface,
                defense_condition=defense
            ))
        return results
    
    def calculate_asr(self, results: List[CanaryResult]) -> float:
        """计算攻击成功率"""
        executed = sum(1 for r in results if r.executed)
        return executed / len(results)
```
