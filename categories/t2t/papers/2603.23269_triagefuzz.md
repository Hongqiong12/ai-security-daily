## 摘要

大型语言模型（LLM）容易受到越狱提示的影响，导致模型输出违反安全策略的内容。以往的越狱模糊测试将所有 token 视为同等重要，忽略了不同 token 对触发模型拒绝行为的贡献差异，导致在查询受限场景下存在大量冗余搜索。

本文提出 **TriageFuzz**，一种 **token 感知的越狱模糊测试框架**，通过识别高贡献 token 集中变异资源，大幅提升查询效率。

## 六大模块深度解读

### 1. 背景与问题

**核心动机**：商业 API 场景中查询有成本限制，现有方法在极低查询预算下攻击成功率极差。

**科学问题**：如何在有限查询预算下最大化越狱攻击成功率？

**直觉洞见**：
- Token 贡献分布高度偏斜：前 20% token 决定 85% 的拒绝决策
- 跨模型一致性强：代理模型可代替目标模型估计 token 贡献
- 结合以上两点，可实现 token 感知的高效变异策略

### 2. 核心方法

**TriageFuzz 三模块架构**：

```
┌──────────────────────────────────────────────────────────────┐
│  模块 1: 代理 Token 贡献估计                                 │
│  - 使用代理 LLM 估计各 token 的拒绝贡献 c_i                  │
│  - 公式: c_i = R(p) - R(p \ t_i)                           │
│  - 利用跨模型一致性迁移到目标模型                             │
├──────────────────────────────────────────────────────────────┤
│  模块 2: 敏感区域识别                                       │
│  - 按贡献排序，选取 top-k ratio (默认 20%) token             │
│  - 仅对高贡献区域执行字符/词汇变异                            │
│  - 跳过低贡献 token，避免无效搜索                             │
├──────────────────────────────────────────────────────────────┤
│  模块 3: 拒绝引导进化策略                                   │
│  - 轻量级评分器: 预测变异后提示的绕过概率                     │
│  - 进化算法: 自适应加权选择高评分候选                          │
│  - 收敛加速: 聚焦高潜力变异方向                               │
└──────────────────────────────────────────────────────────────┘
```

### 3. 技术细节

**Token 贡献的精确定义**：

对于提示 $p = (t_1, t_2, ..., t_n)$，token $t_i$ 的拒绝贡献：

$$c_i = \mathcal{R}_{\text{proxy}}(p) - \mathcal{R}_{\text{proxy}}(p \setminus t_i)$$

其中 $\mathcal{R}(\cdot) = P(\text{refuse} | \cdot)$ 是代理模型的拒绝概率。

**跨模型一致性验证**：

使用 Pearson 相关系数度量代理模型与目标模型 token 贡献的一致性，实验发现相关系数达 **0.82**（强相关）。

**Token 贡献分布实验数据**：

| Token 比例 | 拒绝贡献占比 | 变异优先级 |
|-----------|-------------|-----------|
| 前 10% | ~65% | 最高 |
| 前 20% | ~85% | 高 |
| 20%–50% | ~12% | 中 |
| 后 50% | <3% | 忽略 |

**进化算法参数**：
- 种群大小：50
- 变异率：0.3
- 选择压力：锦标赛选择（k=5）

**实验设置**：
- 开源 LLM：LLaMA-3, Vicuna, WizardLM, Mistral, Qwen2, Gemma-2
- 商业 API：GPT-4o, Claude-3.5, Gemini-1.5-Pro

|------|--------------|--------------|---------------|--------------------|
| GCG | 18% | 32% | 52% | 380+ |

| 配置 | ASR | 说明 |

### 5. 局限性分析

1. **代理模型依赖**：需要一个白盒或灰盒代理模型，完全黑盒场景效果下降约 15%
2. **Randomization Defense**：若防御方在激活层注入随机扰动，token 贡献估计精度下降
3. **高频词汇偏置**：对常见有害词汇效果好，对新型/罕见有害概念效果较差
4. **防御反制**：token 级别的安全监控（如 TriageGuard）可针对此类攻击

**核心算法实现**：
```python
import torch
import numpy as np
from transformers import AutoTokenizer, AutoModelForCausalLM

class TriageFuzz:
    def __init__(
        self,
        surrogate_model: str,
        target_api,
        query_budget: int = 100,
        top_k_ratio: float = 0.2,
        pop_size: int = 50,
        n_generations: int = 20
    ):
        self.tokenizer = AutoTokenizer.from_pretrained(surrogate_model)
        self.proxy = AutoModelForCausalLM.from_pretrained(
            surrogate_model, device_map="auto"
        )
        self.target = target_api
        self.budget = query_budget
        self.top_k = top_k_ratio
        self.pop_size = pop_size
        self.n_gen = n_generations

    def compute_token_contributions(
        self, prompt: str
    ) -> dict[int, float]:
        """
        计算每个 token 对拒绝行为的贡献
        """
        tokens = self.tokenizer.encode(prompt)
        baseline_reject = self._get_reject_prob(prompt)
        contributions = {}

        for i, tok_id in enumerate(tokens):
            # 移除 token i 后的拒绝概率
            ablated_tokens = tokens[:i] + tokens[i+1:]
            ablated_text = self.tokenizer.decode(ablated_tokens)
            ablated_reject = self._get_reject_prob(ablated_text)

            # 贡献 = 基准拒绝率 - 移除后拒绝率
            contributions[i] = baseline_reject - ablated_reject

        return contributions

    def attack(self, harmful_query: str) -> dict:
        """
        执行 TriageFuzz 越狱攻击
        """
        queries_used = 0
        best_asr = 0.0
        best_prompt = harmful_query

        # 1. 计算 token 贡献
        contribs = self.compute_token_contributions(harmful_query)
        tokens = self.tokenizer.encode(harmful_query)

        # 2. 识别敏感 token（top-k）
        n_sensitive = max(1, int(len(tokens) * self.top_k))
        sensitive_indices = sorted(
            contribs, key=contribs.get, reverse=True
        )[:n_sensitive]

        # 3. 进化搜索
        population = self._init_population(harmful_query, sensitive_indices)

        for gen in range(self.n_gen):
            if queries_used >= self.budget:
                break

            # 评估种群
            scores = []
            for candidate in population:
                score = self._evaluate_bypass(candidate)
                queries_used += 1
                scores.append(score)

                if score > best_asr:
                    best_asr = score
                    best_prompt = candidate

            # 进化选择与变异
            population = self._evolve(
                population, scores, sensitive_indices
            )

        return {
            "best_prompt": best_prompt,
            "asr": best_asr,
            "queries_used": queries_used
        }

    def _get_reject_prob(self, text: str) -> float:
        """代理模型拒绝概率估计"""
        inputs = self.tokenizer(text, return_tensors="pt").to("cuda")
        with torch.no_grad():
            logits = self.proxy(**inputs).logits
        # 简化: 取"Sorry"/"I cannot"等拒绝 token 的概率
        refuse_tokens = self.tokenizer.encode("Sorry I cannot")[1:]
        return float(torch.softmax(logits[0, -1], dim=-1)[refuse_tokens].sum())

    def _evaluate_bypass(self, prompt: str) -> float:
        """查询目标模型评估绕过分数"""
        response = self.target.generate(prompt)
        # 判断是否成功绕过（非拒绝回复）
        refuse_keywords = ["I cannot", "I'm sorry", "I apologize"]
        return 0.0 if any(kw in response for kw in refuse_keywords) else 1.0
```
