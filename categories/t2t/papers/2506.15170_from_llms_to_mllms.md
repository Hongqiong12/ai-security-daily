## 🎯 核心贡献

本文系统综述了从LLM到MLLM再到Agent演进过程中的越狱攻击与防御范式演变。

### 1. 研究框架

#### LLM → MLLM → Agent 演进
```
LLM (纯文本) → MLLM (多模态) → Agent (具身智能)
     ↓              ↓              ↓
   越狱攻击      多模态越狱      Agent越狱
```

### 2. 攻击范式

#### LLM阶段
- **Prompt Injection**: 注入恶意指令
- **Jailbreak Templates**: 结构化越狱模板
- **Social Engineering**: 社会工程学攻击

#### MLLM阶段
- **Multi-modal Jailbreak**: 图像+文本联合攻击
- **Cross-modal Transfer**: 跨模态对抗样本
- **Vision-language Misalignment**: 视觉-语言错位攻击

#### Agent阶段
- **Tool Manipulation**: 工具调用操纵
- **Memory Poisoning**: 记忆污染攻击
- **Multi-turn Deception**: 多轮对话诱导

### 3. 防御策略

| 阶段 | 防御方法 | 特点 |
|------|----------|------|
| LLM | Prompt Filtering | 关键词过滤 |
| LLM | Constitutional AI | 原则约束 |
| MLLM | Modality Separation | 模态隔离 |
| MLLM | Cross-modal Alignment | 跨模态对齐 |
| Agent | Sandboxing | 沙盒隔离 |
| Agent | Verification | 结果验证 |

### 4. 评测基准

- **Safety Benchmarks**: 安全评测数据集
- **Adversarial Robustness**: 对抗鲁棒性测试
- **Multi-turn Safety**: 多轮安全性评估

---

## 🔍 关键洞察

1. **演进视角**: 首次系统梳理LLM到Agent的安全演进
2. **攻击升级**: 攻击复杂度随模型能力提升而增加
3. **防御滞后**: 防御技术相对攻击存在滞后性
4. **生态安全**: 需要整体考虑LLM生态系统安全
