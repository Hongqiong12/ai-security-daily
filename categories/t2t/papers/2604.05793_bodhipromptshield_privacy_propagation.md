# BodhiPromptShield: Pre-Inference Prompt Mediation for Suppressing Privacy Propagation in LLM/VLM Agents

> **发表日期**: 2026-04-07
> **分类**: cs.CR, cs.CV

## 模块一：研究背景与问题定义

### 动机
在 LLM/VLM Agent 中，用户提示词中的隐私敏感信息会超越单次模型调用而**跨阶段传播**——原始用户内容可能流入检索查询、内存写入、工具调用和日志中。现有的去标识化（de-identification）方案只处理文档边界，未解决这种跨流水线阶段的传播问题。

### 科学问题
**如何在 Agent 推理前系统性地检测、中介和抑制提示词中敏感信息的跨阶段传播，同时保持任务性能？**

### 直觉洞见
- Agent 的工具调用链是隐私泄露的传播通道：敏感信息从用户输入 → 检索查询 → 工具参数 → 内存存储 → 日志，每一步都可能泄露
- 简单的匿名化/编辑处理只作用于输入层，无法阻止信息在后续阶段的"再浮现"
- 需要一种"传播感知"的中介机制，在每个可能泄露的阶段点都进行保护

## 模块二：核心方法

### 技术贡献

1. **传播感知隐私中介框架**
   - 不仅处理输入层的敏感信息，而是跟踪信息在检索、内存、工具各阶段的传播路径
   - 引入"传播感知中介"和"恢复时机"作为显式安全变量

2. **三种保护策略路由**
   - **类型化占位符（Typed Placeholder）**：将敏感片段替换为类型标记
   - **语义抽象（Semantic Abstraction）**：将具体信息抽象为语义等价但脱敏的表达
   - **安全符号映射（Secure Symbol Mapping）**：将敏感内容映射为安全符号

3. **延迟恢复机制**
   - 敏感信息的恢复被推迟到授权边界（authorized boundary）
   - 确保在中间阶段信息始终以保护形式传递

### 方法框架
```
用户输入（含敏感信息）
    ↓
[敏感片段检测]
    ↓
[策略路由] → 占位符 / 语义抽象 / 符号映射
    ↓
推理阶段（使用脱敏后的输入）
    ├── 检索查询 → 脱敏查询
    ├── 内存写入 → 脱敏内容
    ├── 工具调用 → 脱敏参数
    └── 日志记录 → 脱敏日志
    ↓
[授权边界] → 延迟恢复 → 用户可见结果
```

## 模块三：技术细节

### 核心机制

**敏感片段检测**：
- 识别提示词中的 PII（个人身份信息）、敏感实体、隐私内容
- 基于策略规则和语义理解的混合检测

**策略路由决策**：
- 根据敏感信息类型和下游使用场景，选择最优保护策略
- 占位符适用于强匿名化需求
- 语义抽象适用于需要保留语义关联的场景
- 符号映射适用于需要可逆恢复的场景

**传播抑制控制（CPPB）**：
- 控制提示隐私基准（Controlled Prompt Privacy Benchmark）
- 跨检索、内存、工具三个阶段评估隐私泄露率

### 关键指标
- **传播泄露率**：从 10.7% 降至 **7.1%**
- **PER（Privacy Error Rate）**：9.3%
- **AC（Accuracy）**：0.94
- **TSR（Task Success Rate）**：0.92

## 模块四：实验设置与结果

### 实验设置
- **评估基准**：CPPB（Controlled Prompt Privacy Benchmark）
- **对比方法**：通用去标识化方法
- **评估阶段**：检索、内存、工具调用三个阶段

### 核心结果

| 指标 | BodhiPromptShield | 通用去标识化 |
|------|-------------------|-------------|
| 传播泄露率 | **7.1%** | 10.7% |
| PER | **9.3%** | 较高 |
| AC | **0.94** | 较低 |
| TSR | **0.92** | 较低 |

### 关键结论
- 传播感知中介在隐私保护和任务性能之间取得了更好的平衡
- 三种保护策略的灵活路由优于单一去标识化方案
- 延迟恢复机制有效防止了中间阶段的信息泄露

## 模块五：局限性分析

1. **CPPB 基准的代表性**：论文明确指出结果是"受控系统结果"，非形式化隐私保证或公共基准迁移声明
2. **检测覆盖率**：敏感信息检测的准确性直接影响整体方案效果，漏检将导致传播
3. **语义保持度**：语义抽象和符号映射可能在某些复杂场景中损失关键信息
4. **性能开销**：在每个阶段都进行隐私中介可能引入延迟
5. **对抗性场景**：未评估攻击者主动尝试绕过隐私保护的对抗性场景

## 📌 核心启示

**对 AI 安全实践的影响**：
- **跨阶段隐私风险**：Agent 的多阶段流水线创造了新的隐私泄露面，传统的输入层匿名化不够
- **"传播感知"设计原则**：Agent 框架设计需要将隐私保护嵌入每个工具调用和数据传递环节
- **延迟恢复模式**：一种值得推广的隐私保护设计模式——在需要之前始终使用脱敏表示
- **Agent 安全的纵深防御**：隐私保护需要从输入层延伸到整个工具调用链
