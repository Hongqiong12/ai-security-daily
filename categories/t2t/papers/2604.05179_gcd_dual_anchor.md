# Gradient-Controlled Decoding: A Safety Guardrail for LLMs with Dual-Anchor Steering

## 1. 研究背景与动机 (Background & Motivation)
大型语言模型（LLMs）对越狱（jailbreak）和直接提示注入攻击仍然很脆弱，然而当前最强的防御性过滤器经常过度拒绝（over-refuse）无害查询，导致用户体验下降。现有的越狱和提示注入检测方法（如 GradSafe）仅通过单一的“接受（accept all）”锚点 token 来检测不安全提示，但其阈值非常脆弱，并且在开始解码时，无法提供确定性的保证来防止有害内容的生成。为此，作者提出了梯度控制解码（Gradient-Controlled Decoding, GCD），这是一种免训练的安全护栏机制。

## 2. 核心方法论 (Methodology)
GCD 采用双锚点引导（Dual-Anchor Steering）策略。它结合了一个“接受”锚点 token（"Sure"）和一个“拒绝”锚点 token（"Sorry"），通过双重锚点收紧了决策边界，显著降低了误报率（False Positives）。
在缓解（mitigation）阶段，如果提示被标记为不安全，GCD 会在自回归解码恢复之前，预先注入一个或两个拒绝 token（例如 "Sorry, I can't..."），这就从根本上保证了生成内容的“首个 token 安全（first-token safety）”，无论模型采用何种采样策略。

## 3. 技术实现细节 (Technical Details)
GCD 的工作流程如下：
- **检测阶段**：使用 LLM 计算输入的梯度或表征在 "Sure" 和 "Sorry" 两个锚点上的相对激活。通过比较双锚点的响应差异，精确判断 Prompt 是否存在安全风险。
- **缓解阶段**：一旦系统将 Prompt 标记为有害/攻击（flagged），直接在解码序列的开头强制设定为预定义的拒绝输出（preset-inject），然后让模型继续自回归生成。这样可切断一切恶意利用。
- **配置需求**：该方法是免训练的（training-free），仅需要 20 个演示模板（demonstration templates）即可在不同模型上泛化。

## 4. 量化实验分析 (Quantitative Experiments)
- **数据集**：ToxicChat, XSTest-v2, AdvBench。
- **误报率（FPR）降低**：与单一锚点的 GradSafe 相比，GCD 在保持相当召回率（Recall）的情况下，将误报率降低了 52%。
- **攻击成功率（ASR）**：相较于最强的纯解码（decoding-only）基线方法，GCD 将 ASR 降低了最多 10%。
- **延迟性能**：在 V100 实例上，GCD 仅增加不到 15-20 ms 的平均延迟，适合实时推理拦截。
- **可迁移性**：成功在 LLaMA-2-7B, Mixtral-8x7B, 以及 Qwen-2-7B 等多种架构上验证了其即插即用的效果。

## 5. 局限性与防御绕过 (Limitations & Evasion)
- **局限性**：GCD 依赖于 "Sure/Sorry" 锚点模式，虽然通过双锚点缓解了脆性，但对于非英语场景或模型天然不采用此类语气的特殊 Prompt，锚点的敏感度可能会下降。
- **潜在绕过（Evasion）**：攻击者可能通过使 LLM 输出多语言响应、或是构建极其隐蔽的角色扮演（不触发明显的 Sorry 梯度），在不触发 GCD 检测边界的情况下实施越狱。

## 6. 对工业界的启示 (Implications)
- **即插即用的安全护栏**：GCD 展示了一种无需微调即可获得高性能和低误报率的拦截方案，在资源受限（如 V100 上仅需 15ms 额外延迟）的工业环境中具有极高落地价值。
- **双锚点替代单阈值**：从工业安全角度来看，使用正负样本（双锚点）校准置信度比仅观察单边（单锚点）阈值更稳健，极大地优化了过滤器的可用性和用户体验。
