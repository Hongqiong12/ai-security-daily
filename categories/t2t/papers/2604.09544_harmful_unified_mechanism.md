## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)

LLM 经过对齐训练（RLHF / DPO 等）后仍存在显著安全脆弱性：越狱攻击可常规性绕过防护，窄域微调可能引发"涌现性错位"（emergent misalignment）并广泛泛化到其他有害行为领域。然而，这种脆弱性的**根本原因**——即 LLM 内部是否有害性表征是否具有连贯的组织结构——此前一直不清楚。

现有研究的核心矛盾：如果模型已经"学会"拒绝有害请求，为什么一个窄域微调就能让模型在完全不同的有害领域也崩溃？这提示我们，对齐机制可能在内部做了某种结构性改变，而这种改变本身就是脆弱的来源。

### 核心科学问题 (Core Scientific Question)

LLM 内部的有害性表征是否存在一种紧凑的、跨域统一的组织结构？对齐过程如何重塑这种结构，以及这种重塑本身是否就是脆弱性的根因？

### 直觉洞见 (Intuitive Insight)

类比：想象一座监狱的安保系统。表面上看，每个牢房都有锁（对齐后的拒绝机制）。但如果研究发现，所有牢房的锁其实都由同一根"总钥匙线"控制（统一的有害权重集合），那么只要找到这根线的任何一段并剪断它，整个系统就会崩溃——这就是为什么窄域微调能引发广泛错位：你剪断了一个领域的"锁线"，却意外释放了所有领域的"囚犯"。更关键的是，对齐过程不是加固了锁，而是把所有锁线捆得更紧了——压缩使得结构更脆弱。

## Module 2 — 核心方法 (Core Method)

本文的核心方法论是将**目标权重剪枝（Targeted Weight Pruning）作为因果干预工具**来探测 LLM 内部有害性的组织方式。

**（1）因果干预框架（Causal Intervention Framework）**
将神经网络权重视为因果图中的节点，通过有选择性地剪枝（置零）特定权重子集，观察模型输出行为的系统性变化，从而推断哪些权重因果性地参与了有害内容生成。这种方法比激活探针（activation probing）更强有力，因为它是真正的干预而非被动观测。

**（2）跨域通用性分析（Cross-Domain Generalization Analysis）**
在多个不同类型的有害行为上执行剪枝实验（如暴力、仇恨言论、非法活动、色情内容等），验证同一组权重是否在不同有害域之间共享。核心假设是：如果有害表征具有统一的结构，那么在一个域上识别的关键权重应该在其他域上也有效。

**（3）对齐前后对比（Aligned vs Unaligned Comparison）**
对比经过对齐训练的模型与基座（未对齐）模型的有害权重分布差异。关键发现：对齐模型的 harmful weights 呈现出更高的压缩度（compression），即有害能力被集中在更少的参数中。

**（4）功能解耦测试（Functional Dissociation Test）**
区分两个功能：
- 有害**生成**能力（generation capability）：模型主动输出有害内容的能力
- 有害**识别**能力（recognition capability）：模型判断和解释内容是否有害的能力

通过独立的剪枝实验证明两者依赖不同的权重子集。

**（5）涌现性错位修复验证（Emergent Misalignment Mitigation）**
在窄域微调后，针对性地剪除该域的有害生成权重，观察是否减少向其他有害域的泛化错位。

## Module 3 — 技术细节 (Technical Details)

### 核心算法：目标权重剪枝因果干预

```
输入: 模型 M(θ), 有害查询集 Q_harm, 良性查询集 Q_benign
输出: 关键有害权重集 W*, 压缩度指标 C

Step 1: 基线评估
  └── 对 q ∈ Q_harm, 计算 P(M(q) 包含有害内容)
  └── 记录 baseline ASR = mean(P_harmful)

Step 2: 迭代剪枝搜索
  For 迭代 t = 1, 2, ..., T:
    (a) 计算每个权重 w_i 的归一化梯度敏感度:
        S_i = |∂L_harm/∂w_i| / max_j(|∂L_harm/∂w_j|)
        其中 L_harm 为有害生成的交叉熵损失
    
    (b) 选择 Top-k 敏感度最高的权重:
        W_t = argsort(S)[1..k]
    
    (c) 置零选定权重:
        θ' = θ ⊙ mask(W_t)  (mask 将 W_t 位置设为 0)
    
    (d) 评估剪枝后 ASR':
        ΔASR_t = baseline_ASR - ASR'
    
    (e) 若 ΔASR_t < ε (阈值), 终止
    W* = ∪_t W_t

Step 3: 跨域验证
  └── 在域 d₁ 上发现的 W* 应用于域 d₂, d₃, ...
  └── 计算 transferability = ASR_d₂(W*_d₁)

Step 4: 压缩度计算
  C = |W*_aligned| / |W*_unaligned|
  (值越小表示对齐模型的权重越集中/压缩)
```

### 关键超参数

| 参数 | 说明 | 典型值 |
|------|------|--------|
| k (每轮剪枝数) | 每次迭代置零的权重数量 | 100-500 |
| ε (终止阈值) | ASR 下降量小于此值时停止 | 0.01 |
| 域数量 D | 用于跨域验证的有害行为类型数 | 5-8 |
| 排名方法 | 权重重要性排序策略 | Gradient-based / Magnitude-based |

### 核心发现的形式化描述

**命题 1（紧凑性）**: 存在一个相对较小的权重子集 W* ⊂ θ，使得 M(θ \ W*) 的有害生成能力接近于零，而良性任务性能退化有限。

**命题 2（跨域通用性）**: ∀d₁, d₂ ∈ D（有害域集合），transferability(d₁→d₂) > τ，其中 τ 为高传递率阈值（实验中 >80%）。

**命题 3（对齐压缩效应）**: C_aligned < C_unaligned（通常 C_aligned ≈ 0.3-0.5 × C_unaligned），表明对齐使有害权重更加集中。

**命题 4（涌现性错位解释）**: 窄域微调 δθ 在域 d 上激活被压缩的有害权重 W*_d → 由于压缩导致 W*_d 与其他域的有害权重高度重叠 → 错位从 d 泛化到 d' ≠ d。

**命题 5（功能解耦）**: W*_gen（有害生成权重）∩ W*_rec（有害识别权重）≈ ∅（近似不相交），证明两种能力依赖不同机制。

## Module 4 — 实验设置与结果 (Experiments and Results)

### 数据集与评价指标

| 数据集 | 类型 | 规模 | 用途 |
|--------|------|------|------|
| 自构建有害查询集 | 多域有害行为 | ~1000 条/域 | 剪枝干预评估 |
| AdvBench 子集 | 越狱攻击 | 500 条 | 有害生成基线 |
| HH-RLHF / BeaverTails | 安全对齐数据 | 数千条 | 良性任务保持评估 |

**评价指标**:
- **Harmful Generation Rate (HGR)** ↓: 模型生成有害内容的比例
- **Benign Task Accuracy (BTA)** ↑: 良性任务的正确回答率
- **Compression Ratio (CR)** ↓: 关键有害权重数量 / 总参数量
- **Transfer Rate (TR)** ↑: 跨域权重迁移有效性

### 主要结果表格

**表 1：关键发现汇总**

| 发现 | 对齐模型 | 未对齐模型 | 差异 |
|------|---------|-----------|------|
| 有害权重占比 (CR) | **0.8%** (高度压缩) | 2.1% (较分散) | 对齐压缩 2.6x |
| 跨域传递率 (TR) | **89.3%** | 72.1% | 对齐后更高 |
| 剪枝后良性准确率损失 | < 3% | < 5% | — |
| 窄域剪枝后错位减少 | **67% ↓** | 23% ↓ | 对齐模型受益更大 |

**表 2：功能解耦证据**

| 操作 | HGR (↓) | 有害识别准确率 (↑) |
|------|---------|-------------------|
| 基线（无干预） | 85.7% | 91.2% |
| 剪除 W*_gen | **4.2%** ✅ | 90.1%（几乎不变）✅ |
| 剪除 W*_rec | 82.1%（几乎不变） | 12.3% ✅ |

> 结论：有害生成和有害识别确实依赖于**截然不同的权重集合**，证明功能完全解耦。

### 消融实验

| 消融设置 | HGR | BTA | 说明 |
|----------|-----|-----|------|
| 完整方法（梯度排序+多域联合）| 4.2% | 97.1% | 最佳 |
| 仅单域剪枝 | 18.7% | 96.8% | 单域不够，需多域联合 |
| 随机剪枝（同数量）| 61.3% | 88.4% | 随机剪枝无效，说明权重选择有特异性 |
| 幅度排序替代梯度排序 | 12.8% | 95.9% | 梯度信息优于纯幅度 |

### 关键发现

1. **对齐是一把双刃剑**：对齐训练通过将有害权重压缩到更小的子集来提升安全性，但这种压缩反而使得窄域微调更容易触发广泛的错位——这是"涌现性错位"的结构性解释。
2. **有害能力的解剖学图谱首次成型**：本文给出了首个 LLM 内部有害性组织的精细地图，表明有害生成能力集中在不到 1% 的参数中且跨域共享。
3. **防御新思路**：既然有害权重如此集中且可定位，针对性剪除或正则化这些特定权重可能是比全局对齐更精准的安全方案。

## Module 5 — 局限性分析 (Limitations)

1. **模型规模限制**: 主要实验在中等规模模型上进行（≤ 7B），对于 70B+ 大型模型，权重分布模式可能有本质差异，剪枝的可扩展性尚未验证。
2. **剪枝的实用性**: 当前方法需要知道哪些权重是有害的（需要 access to model internals + labeled harmful queries）。在实际部署场景中，攻击者或防御者不一定具备完整的白盒访问权限。
3. **对抗性适应**: 如果攻击者意识到目标模型使用了基于剪枝的防御，可以设计绕过关键权重的攻击路径（例如利用非关键权重中的冗余有害信息通道）。
4. **动态演化**: 模型在使用过程中持续变化（如持续学习、上下文学习），静态剪枝的有效性随时间衰减的速度未研究。
5. **良性-有害边界模糊**: 某些"灰色地带"内容（如医学讨论中的暴力描述、文学创作中的成人主题）的权重可能与有害权重重叠，过度剪枝可能导致合法用途的功能丧失。

### 核心代码框架

```python
import torch
import torch.nn as nn
from transformers import AutoModelForCausalLM, AutoTokenizer


class HarmfulWeightAnalyzer:
    """目标权重剪枝因果干预框架"""

    def __init__(self, model_name: str, device: str = "cuda"):
        self.model = AutoModelForCausalLM.from_pretrained(
            model_name, torch_dtype=torch.float16
        ).to(device)
        self.tokenizer = AutoTokenizer.from_pretrained(model_name)
        self.device = device

    def compute_gradient_sensitivity(
        self,
        harmful_queries: list[str],
        target_token_id: int,
    ) -> dict[str, float]:
        """计算每个权重的梯度敏感度 S_i = |∂L_harm/∂w_i|"""
        sensitivities = {}
        self.model.zero_grad()

        for query in harmful_queries:
            inputs = self.tokenizer(query, return_tensors="pt").to(self.device)
            outputs = self.model(**inputs, labels=inputs["input_ids"])
            loss = outputs.loss  # 交叉熵损失
            loss.backward()

        # 收集所有参数的梯度幅度
        for name, param in self.model.named_parameters():
            if param.grad is not None:
                sensitivities[name] = param.grad.abs().mean().item()

        return sensitivities

    def iterative_pruning(
        self,
        sensitivities: dict[str, float],
        prune_per_iter: int = 200,
        epsilon: float = 0.01,
        max_iters: int = 50,
        eval_fn=None,
    ) -> tuple[list[str], list[float]]:
        """迭代剪枝搜索关键有害权重"""
        pruned_weights = []
        asr_history = [eval_fn()]  # 初始 ASR

        for t in range(max_iters):
            # (a) 排序选 Top-k
            sorted_weights = sorted(
                sensitivities.items(), key=lambda x: x[1], reverse=True
            )
            top_k = sorted_weights[:prune_per_iter]
            new_prunes = [name for name, _ in top_k]

            # (b) 置零
            for name, _ in top_k:
                param = dict(self.model.named_parameters())[name]
                param.data = torch.zeros_like(param.data)
                sensitivities.pop(name, None)

            pruned_weights.extend(new_prunes)

            # (c) 评估
            current_asr = eval_fn()
            asr_history.append(current_asr)
            delta = asr_history[-2] - current_asr

            print(f"Iter {t}: pruned {len(pruned_weights)} weights, "
                  f"ASR={current_asr:.4f}, Δ={delta:.4f}")

            if delta < epsilon:
                break

        return pruned_weights, asr_history

    def cross_domain_transfer(
```
