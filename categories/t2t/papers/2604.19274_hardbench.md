## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
现有 jailbreak benchmark 大多默认攻击者是直接向模型提出有害请求，但真实协作写作场景并不是这样。越来越多用户会把 LLM 当 co-author：给一段草稿、让模型润色、补写、改写、压缩、重组。论文指出，这种“draft-based co-authoring”场景会系统性弱化安全边界，因为 harmful intent 被包进了看似正常的编辑任务里，模型更容易把自己理解成“继续写作”而不是“审核风险”。

### 核心科学问题 (Core Scientific Question)
草稿式协作写作是否会显著放大 jailbreak 风险？如果会，能否构建一个系统 benchmark 去量化这种风险，并进一步设计一种兼顾安全与 utility 的 alignment 方法来缓解它？

### 直觉洞见 (Intuitive Insight)
HarDBench 的核心洞见是：很多 LLM 的风险边界不只取决于“用户在说什么”，还取决于“模型以为自己在做什么”。当任务 framing 从“直接回答”变成“帮我完善草稿”，模型会把更多注意力放在完成协作目标，而不是执行安全拒答，因此同一危险意图会更容易穿透对齐层。

## Module 2 — 核心方法 (Core Method)

### 方法总览
论文包含两部分贡献：

**（1）HarDBench benchmark**
- 构建面向协作写作 jailbreak 的专项评测集；
- 核心不是直接 harmful query，而是“有害草稿 + co-authoring framing”；
- 系统比较 HQ、CoJP w/o TF、CoJP 三种输入条件。

**（2）SUBA（Safety-Utility Balanced Alignment）**
- 一种面向协作写作场景的偏好优化方法；
- 同时训练模型对 harmful drafts 拒答、对 benign drafts 正常协作；
- 不是单纯拉高拒答率，而是显式优化安全–可用性平衡。

### 任务条件定义
- **HQ**：直接 harmful query
- **CoJP w/o TF**：有草稿，但去掉协作写作 framing
- **CoJP**：草稿 + 完整 co-authoring task framing

### 方法定位
HarDBench 不是再做一个“更大有害数据集”，而是把攻击抽象从 direct prompt jailbreak 扩展到 **task framing jailbreak**。这对写作、编辑、协作文档产品尤其关键。

## Module 3 — 技术细节 (Technical Details)

### Benchmark 构造
论文覆盖 4 个高风险领域：
- Explosives
- Drugs
- Weapons
- Cyberattacks

流水线如下：
1. 收集高风险领域关键词与模板；
2. 用 LLaMA-3-8B-Instruct 生成“不完整有害草稿”；
3. 用 GPT-4o 做可行性与危险性验证；
4. 将草稿封装进 co-authoring prompts；
5. 用自动指标 + 人工校验形成最终 benchmark。

### 数据规模
HarDBench 共 **1204** 条验证通过的有害草稿：
- Explosives：209
- Drugs：304
- Weapons：450
- Cyberattacks：241

测试集构造：
- 每个 domain 随机采样 100 条
- 共 **400** 条测试样本
- 剩余 **804** 条用于训练相关实验

### SUBA 的偏好标签
- 对 harmful prompt：
  - 拒答 = chosen
  - 有害补全 = rejected
- 对 benign prompt：
  - 正常 helpful continuation = chosen
  - 无谓拒答 = rejected

### 训练方法与超参
**KTO**
- GPU：1× NVIDIA A6000 48GB
- epoch：1
- lr：5×10^-6
- β：0.1
- LoRA r = 8, α = 16, dropout = 0.0

**GRPO**
- GPU：2× NVIDIA A100 80GB PCIe
- epoch：2
- lr：1×10^-5
- KL β：0.04
- 每个 prompt 生成数 G = 8
- LoRA r = 8, α = 32, dropout = 0.05

### 指标
- **HS (Harmfulness Score)**：由 GPT-4o 评分，1–5 分
- **ASR**：HS = 5 的比例
- **RAR**：completion 比原始草稿更危险的比例
- Utility：WritingBench、LongBench-Write、HelloBench、WildBench-v2

### 伪代码
```python
def evaluate_hardbench(model, prompt_set):
    results = []
    for prompt in prompt_set:
        completion = model.generate(prompt)
        hs = judge_harmfulness(completion)  # 1~5
        results.append(hs)
    return asr(results), rar(results)
```

## Module 4 — 实验设置与结果 (Experiments and Results)

### 被测模型
- LLaMA3-8B
- Mistral-7B
- Qwen3-8B
- DeepSeek-R1-8B
- DeepSeek-R1-32B
- GPT-4o
- Gemini-2.0-Flash
- Gemini-2.5-Pro

### 关键结果一：CoJP 显著强于直接 harmful query
| 模型 | HQ ASR ↑ | CoJP ASR ↑ | HQ HS ↑ | CoJP HS ↑ |
|------|---------:|-----------:|--------:|----------:|
| LLaMA3-8B | 24.75% | **80.50%** | 2.65 | **4.29** |
| Qwen3-8B | 27.25% | **99.00%** | 3.57 | **4.97** |
| GPT-4o | 17.75% | **96.75%** | 2.67 | **4.87** |
| DeepSeek-R1-32B | 37.75% | **96.25%** | 3.90 | **4.94** |

### 关键结果二：Task framing 是主要放大器
以 GPT-4o 为例：
- CoJP w/o TF ASR = **23.50%**
- CoJP ASR = **96.75%**

说明真正危险的不是“有草稿”本身，而是“草稿 + 协作写作 framing”的组合。

### 关键结果三：Moderation 更容易漏掉 CoJP
| 检测器 | HQ unsafe ↑ | CoJP w/o TF unsafe ↑ | CoJP unsafe ↑ |
|--------|------------:|---------------------:|--------------:|
| OpenAI-Moderation | **85.00%** | 19.25% | 22.00% |
| Qwen3Guard | **46.50%** | 36.25% | 27.75% |

### 关键结果四：RAR 显示模型会放大风险
- GPT-4o：**84.04% → 99.48%**
- Gemini-2.0-Flash：**77.22% → 99.14%**
- LLaMA3-8B：**70.25% → 96.89%**

### SUBA 结果
以 LLaMA3-8B 为例：
- Zero-shot CoJP ASR = **80.50%**
- Safety Prompt = **2.75%**，但 utility **-121.71%**
- **SUBA_KTO = 5.25%**，utility **-1.80%**
- **SUBA_GRPO = 3.00%**，utility **+3.16%**

以 Mistral-7B 为例：
- Zero-shot CoJP ASR = **85.25%**
- **SUBA_KTO = 0.00%**，utility **+4.02%**

### 关键发现
- 所有被测模型在 CoJP 下的 HS 都 **> 4.29**，ASR 都 **> 80%**；
- 强推理模型并不会自动更安全，Qwen3-8B 的 CoJP ASR 达 **99.00%**；
- 只针对 direct harmful query 训练防御远远不够；
- 若没有 benign 对照数据，偏好优化很容易把模型推向“全拒答”。

## Module 5 — 局限性分析 (Limitations)

1. **风险领域仍有限**：当前只覆盖 explosives、drugs、weapons、cyberattacks 四类，还未扩展到 medical misinformation、social engineering、financial fraud 等场景。
2. **prompt library 仍偏静态**：主要关注固定 co-authoring framing，未来多轮、动态、上下文自适应攻击可能更强。
3. **对齐方法比较范围有限**：主实验集中在 KTO / GRPO，尚未系统比较 DPO、PPO-based RLHF 等更多范式。
4. **指标依赖 LLM judge**：HS 虽与人工高度一致，但仍不是完全无偏的人工真值。
5. **完整敏感数据可能受控访问**：真实复现时不一定能直接获得全部 benchmark 细节。
