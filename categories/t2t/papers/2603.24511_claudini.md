## 1. 背景与问题

### 动机

大语言模型（LLM）的安全对齐研究通常依赖人工设计的对抗攻击算法，但这一过程耗时且易受研究者思维定势影响。随着 LLM 代理（Agent）技术的成熟，一个自然的问题浮现：**能否让 AI 自主发现比人类更有效的攻击算法？**

### 科学问题

- LLM 代理能否自动发现优于现有30余种攻击算法的新型白盒对抗攻击？
- 自动发现的攻击是否具有跨模型迁移能力？
- 自动化安全研究（autoresearch）在对抗攻击场景下的效果边界在哪里？

### 直觉洞见

白盒对抗攻击具备两个特性，使其特别适合自动化研究：
1. 现有方法（如 GCG）提供了强力的起点；
2. 优化目标（如攻击成功率 ASR）提供了密集的量化反馈，适合迭代改进。

## 2. 核心方法

### 技术贡献

本文提出 **Claudini**，一个以 Claude Code 为核心驱动的**自动研究（autoresearch）流水线**：

1. **种子初始化**：以现有攻击实现（如 GCG、AutoDAN 等）作为起点。
2. **LLM 代理迭代**：让代理读取当前攻击代码，分析失败原因，提出改进思路并实现新版本。
3. **量化评估反馈**：在真实模型上运行攻击，将 ASR 等指标反馈给代理，驱动下一轮迭代。
4. **收敛输出**：迭代若干轮后，输出当前最优攻击算法。

### 方法框架

```
已有攻击实现 (GCG 等)
        ↓
    LLM 代理分析
        ↓
  新攻击算法代码
        ↓
   实际模型测试 (ASR)
        ↓
 反馈驱动下轮迭代
        ↓
   最优攻击算法输出
```

## 3. 技术细节

### 优化目标

在 CBRN（化学、生物、放射、核武器）有害内容生成场景下：

$$\max_{\theta} \text{ASR}(A_\theta, \mathcal{M})$$

其中 𝐴_θ 为参数化的攻击算法，𝓜 为目标模型集合。

### 迭代机制

- **代理模型**：Claude Code（具备代码理解与生成能力）
- **反馈信号**：硬性量化指标（ASR@N 查询数）
- **终止条件**：ASR 连续 K 轮不提升，或达到最大迭代次数

### 关键参数

| 参数 | 值 |
|------|----|
| 基线方法数 | 30+ |
| 测试模型数 | 多个（含 GPT-OSS-Safeguard-20B, Meta-SecAlign-70B） |
| 目标类别 | CBRN 有害内容查询 |

## 4. 实验设置与结果

### 数据集与评价指标

- **评估基准**: JailbreakBench（越狱）、提示注入专项评测
- **主要指标**: ASR（攻击成功率）
- **目标模型**: GPT-OSS-Safeguard-20B、Meta-SecAlign-70B 及多个保留集模型

### 主要结果

| 方法 | GPT-OSS-Safeguard-20B ASR | Meta-SecAlign-70B ASR |
|------|--------------------------|----------------------|
| 现有最优基线 | ≤10% | 56% |
| **Claudini** | **40%** | **100%** |

### 关键发现

- 自动发现的攻击算法对未见模型（Meta-SecAlign-70B）实现 **100% ASR**，说明攻击迁移性极强。
- 代理在代理模型（surrogate model）上优化的攻击，可直接迁移到保留集模型，无需重新优化。
- 结果与 Carlini 等人（2025）关于自动攻击基准可扩展性的结论一致，验证了白盒场景下自动化安全研究的可行性。

---

## 5. 局限性分析

1. **白盒假设**：当前方法依赖模型梯度访问权限，在商业 API 黑盒场景下无法直接应用。
2. **计算成本**：LLM 代理的多轮迭代需要大量 API 调用，成本较高。
3. **伦理风险**：该流水线可被恶意行为者用于自动生成更强攻击，存在双刃剑问题。
4. **泛化边界**：对于非 CBRN 有害类别的攻击效果尚未充分验证。

## 6. 复现指南

### 环境依赖

```bash
# Python 3.10+
pip install torch transformers
# 依赖 Claude API 访问权限
```

### 代码复现步骤

```bash
# 1. 克隆仓库
git clone https://github.com/romovpa/claudini

# 2. 配置 Claude API Key
export ANTHROPIC_API_KEY=your_key_here

# 3. 以 GCG 为种子启动自动研究
cd claudini
python autoattack_runner.py --seed gcg --model gpt-oss-safeguard --max_iters 20

# 4. 评估发现的攻击
python evaluate.py --attack discovered_attack --benchmark jailbreakbench
```

### 关键注意事项

- 需要白盒模型访问权限（梯度）或基于代理模型的迁移设置
- 建议在隔离环境中运行，避免意外生成有害内容
- 完整代码及发现的攻击算法均已在 GitHub 开源：https://github.com/romovpa/claudini
