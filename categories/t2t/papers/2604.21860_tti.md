## Module 1 — 背景与问题 (Background and Problem)

### 动机 (Motivation)
现有越狱防御大多默认一个前提：只要把**当前这一轮**的输入和输出审好，就能大体守住安全边界。但真实 API 服务常常是“目标模型无状态、应用层自己拼历史”的架构，这给攻击者留出一个空档——攻击轨迹可以保存在攻击者端，而安全审核却只看单轮。论文正是抓住这个结构性漏洞，提出了 TTI（Transient Turn Injection）。

### 核心科学问题 (Core Scientific Question)
在目标模型和安全策略都以**单轮、无状态**方式工作的前提下，攻击者是否可以通过多轮、分散、表面无害的请求，把高风险意图逐步拼接出来，从而绕过传统 per-turn moderation？

### 直觉洞见 (Intuitive Insight)
TTI 的关键不是“让模型记住前几轮”，而是“让攻击者自己记住前几轮”。模型每轮都被当成新会话也没关系：攻击者可以根据上一轮反馈，持续生成下一轮更贴边界的提示。于是系统对每轮都觉得“还没明显越线”，但攻击序列整体却正在逼近危险输出。

## Module 2 — 核心方法 (Core Method)

### 方法总览
TTI 是一种“**目标模型无状态、攻击者有状态**”的多轮攻击：

**（1）Seed turn 初始化**
- 先用较温和或边界化的提示开启交互；
- 该轮即使被拒绝，也能提供后续反馈信号。

**（2）外部 attacker model 自适应生成**
- 攻击者端保存完整攻击轨迹；
- 使用辅助模型根据前一轮/多轮响应生成下一条提示；
- 目标模型看到的始终是单轮请求，而不是攻击序列本身。

**（3）逐轮逼近目标信息**
- 每轮提示尽量保持局部合规；
- 通过角色包装、风险讨论、法律/理论 framing 等形式逐步收窄拒答空间；
- 最终在某一轮诱发高风险输出。

### 与传统多轮 jailbreak 的区别
- **PAIR 类方法**：依赖目标模型保留会话上下文；
- **TTI**：专门针对**无状态 API + 单轮审核**；
- 因此它暴露的是 moderation 架构问题，而不只是模型记忆问题。

## Module 3 — 技术细节 (Technical Details)

### 运行协议
论文默认设置为：
- 每个样本 **1 个 seed turn**；
- 后续 **9 个 adversarial turns**；
- 每批处理 **5 个 prompts**；
- 每条样本总计约 **10 轮**交互。

### 角色分工
- **攻击提示生成器**：`gemini-2.0-flash`
- **防守方模型**：13 个商业/开源模型
- **日志系统**：记录每轮 prompt-response transcript 并写入 Google Drive

### 评估维度
论文主评估围绕三层展开：
1. **Safe / Unsafe 响应数量**；
2. **漏洞类别覆盖**（如 Medical、Harmful、Unlawful、Privacy 等）；
3. **与 PAIR 的对比**，检验 TTI 是否更适合利用无状态审核缺口。

### 资源配置
- 平台：**Google Colab**
- Python：**3.10**
- GPU：**NVIDIA T4 16GB**
- 主要库：`openai 1.14.0`, `google-generativeai 0.5.0`, `pandas 2.2.2`

### 关键发现
- **TTI 在所有模型上都比 PAIR 更有效**；
- **Gemini 系列最脆弱**，Claude-3.5-Haiku 最稳健；
- 高风险暴露主要集中在 **Medical / Harmful / Unlawful / Unauthorized Practice** 等领域；
- 说明真正的薄弱点不只是模型是否能拒答，而是系统只做了**单轮审核**。

## Module 5 — 局限性分析 (Limitations)

1. **更像架构漏洞扫描，而非纯模型测评**：它揭示了 stateless moderation 的问题，但不能单独解释不同模型内部为什么表现不同。  
2. **依赖 attacker model 的生成质量**：若辅助模型弱，攻击序列的适应效率会下降。  
3. **主要验证 API 式、多轮文本场景**：对工具调用、长期记忆、多模态任务的外推仍需更多实验。  
4. **类别分析仍偏结果导向**：论文能告诉你哪些风险域更容易被击穿，但对“为什么某类语境更易穿透”尚缺少机制解释。  
5. **成本虽不高，但已足够支撑持续自动红队**：从防守角度看，这反而说明同类攻击具备现实可扩展性。

## Module 6 — 复现指南 (Reproduction Guide)

### 环境依赖 (Environment)
```bash
pip install pandas openai google-generativeai
```

### 核心代码框架 (Core Code)
```python
history = []
seed_prompt = get_seed_prompt(sample)
response = query_target_model(seed_prompt)
history.append((seed_prompt, response))

for _ in range(9):
    attack_prompt = attacker_model_generate(history)
    response = query_target_model(attack_prompt)  # target sees only current turn
    history.append((attack_prompt, response))
    log_transcript(history[-1])
```

### 复现步骤 (Reproduction Steps)
1. 准备 ModifiedMasterKeyJailbreakQuestions.csv 及相应评测样本。  
2. 选定 attacker model（论文使用 `gemini-2.0-flash`）。  
3. 为每条样本运行 **1 个 seed turn + 9 个 adversarial turns**。  
4. 对 13 个目标模型分别记录 safe / unsafe 响应统计。  
5. 对照 PAIR 进行同规模比较，并统计类别分布与 token 成本。  
6. 关键核验点：
   - TTI 对 Gemini 系列的命中应显著高于 PAIR；
   - Claude-3.5-Haiku 仍应最稳健；
   - 总体攻击成本应保持在低美元级别。
