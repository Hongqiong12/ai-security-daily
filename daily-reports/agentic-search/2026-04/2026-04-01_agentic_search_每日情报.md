## 📄 论文详情

### 1. Near-Miss: Latent Policy Failure Detection in Agentic Workflows
**ArXiv**: [2603.29665](https://arxiv.org/abs/2603.29665) | **类型**: Benchmark/评估 | **日期**: 2026-03-31

**核心问题**：当前 Agent 评估仅比较最终状态，无法检测 Agent 绕过必要策略检查但侥幸得到正确结果的"近失"案例。

**关键贡献**：

1. **定义 Near-Miss 概念**：正式区分"显性策略违规"（结果错误）与"隐性策略失败"（绕过检查但侥幸正确），类比航空/医疗安全领域的 near-miss 事件。

2. **构建检测指标**：基于 ToolGuard 框架，将自然语言策略编译为可执行守卫代码，分析 Agent 工具调用决策的"信息充分性"（Decision Sufficiency）。

3. **量化评估**：在 τ²-verified Airlines benchmark 上测试多种 SOTA LLM：

| 发现 | 数据 |
|------|------|
| Near-Miss 率 | 8–17% 的涉变更工具调用轨迹 |
| 传统评估检出率 | 0%（均被视为"通过"）|
| 不同模型差异 | 最高 2× |

**方法示意**：
```
传统评估：最终状态 == 真实答案？→ ✅/❌
Near-Miss：
  执行工具前 Agent 是否持有充分决策信息？
  → 充分：真正合规执行
  → 不充分但侥幸正确：Near-Miss（潜在隐患）
  → 不充分且结果错误：显性违规
```

**安全意义**：在高频业务自动化场景中（每日数千次 Agent 操作），8-17% 的近失率意味着大量潜在合规风险，随任务风险等级提升，"侥幸正确"将演变为实质性损害。

## 🔍 趋势分析

### Agent 评估方法论的演进

Near-Miss 工作代表 Agentic AI 评估从"结果导向"向"过程+结果"双维度的重要转变。结合上周收录的 PROClaim（渐进式 RAG 能力评估）和 Architecting Secure AI Agents（系统级防御），当前 Agentic AI 安全研究正形成三个互补层次：

| 层次 | 代表工作 | 关注点 |
|------|---------|-------|
| 能力层 | PROClaim | 搜索增强推理的有效性 |
| 评估层 | Near-Miss | 过程合规性检测 |
| 防御层 | Architecting Secure AI Agents | 系统架构安全 |

### 隐性失败的深层意义

Agent 的"近失"问题与传统软件工程中的"潜在缺陷"高度类似。在高风险领域（医疗、金融、法律），现有的 Benchmark 评估可能系统性地高估了 Agent 的可靠性。这一发现对于 AI Agent 在敏感业务场景的合规性认证具有重要政策含义。
