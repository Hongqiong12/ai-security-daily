## 今日摘要

今日 6 篇论文共同指向一个很清晰的信号：**安全研究正在从“输入侧修补”快速转向“内部表示、训练回路与运行时执行边界”的系统级控制。** 在 T2T 方向，LIRA 直接把防御目标从输出拒答推进到 instruction representation 对齐，证明“学会安全解释”比“学会安全说话”更关键；而 Backdoors in RLVR 则反过来说明，只要 RLVR 奖励回路被少量投毒样本撬动，训练过程本身就会变成后门放大器。这一攻一防形成了非常鲜明的对照：**未来对齐的主战场不是 prompt 表面，而是内部表征与优化动力学。**

T2I 方向同样出现了结构性升级。Closed-Form Concept Erasure via Double Projections 把概念擦除从“要重训、要迭代”的工程问题重写成“两个投影 + 一个左零空间约束”的解析几何问题，标志着 concept erasure 进入“训练自由 + 明确 preservation 约束”的新阶段。与此同时，NTIRE 2026 Challenge 则把 AIGC 检测从“clean 数据集上的分类任务”升级为“面对 42 个生成器 + 36 种后处理扰动的真实世界鲁棒检测”，说明图像安全研究的评测中心已经明显转向 robustness-in-the-wild。

Agentic 方向的两篇论文则把安全边界继续往运行时和长期记忆推进。PlanGuard 证明：对 indirect prompt injection，真正有效的不是继续在输入端补规则，而是先把 planning 与外部上下文隔离，再检查计划-执行一致性；ADAM 则说明 memory-enabled agent 的真正高危点在于“可被主动探索的私有记忆分布”，单次 prompt injection 已经不是最强攻击方式，**entropy-guided adaptive querying** 才是更危险的新范式。

## T2T 论文（LLM 安全）

### 1. LIRA：把安全对齐从“改输出”推进到“改指令解释”

**《Latent Instruction Representation Alignment: defending against jailbreaks, backdoors and undesired knowledge in LLMs》**

arXiv: https://arxiv.org/abs/2604.10403 | 领域: cs.LG | 作者: Eric Easley, Sebastian Farquhar | 发布: 2026-04-12 | GitHub: 暂无

#### 一、背景与问题

现有安全微调主要约束模型“怎么回答”，而不是“怎么理解恶意指令”。这导致很多 guardrail 只是把危险行为压到最后输出层，一旦遇到 PEZ、embedding attack 或训练期后门，模型内部的危险解释路径仍能被激活。

#### 二、核心方法

作者提出 **LIRA**：直接对齐恶意指令与安全指令的中间表示；再用 **SAG (Sequence-Aware Gradients)** 切断 response positions 的直接梯度，让训练重点落在 instruction representation 上；最后引入 **AdLIRA** 在 latent space 里主动搜索未修补漏洞，提高对新攻击的泛化。

#### 三、技术细节

核心目标可写为：
- L_align = ‖h_l(x_mal) - h_l(x_safe)‖₂²
- L_total = L_align + λ_retain · L_retain + λ_adv · L_adv

其中 SAG 负责阻断“只改输出不改解释”的梯度捷径，AdLIRA 则在内部表示空间做对抗搜索。

#### 四、实验设置与结果

| 任务 | 关键结果 | 说明 |
|---|---|---|
| PEZ jailbreak | **阻断 >99% 攻击** | 摘要直接报告 |
| PEZ ASR | **AdLIRA 压到接近 0%** | Figure 4(a) |
| insecure code backdoor | **恢复到接近 non-backdoor baseline** | Figure 4(d) |
| WMDP cyber unlearning | **达到 optimal forgetting** | benign capability loss 可忽略 |

#### 五、局限性分析

1. 对齐层选择敏感，不同模型可能需要单独调校  
2. AdLIRA 引入内部对抗训练，训练复杂度高  
3. 更像“改调用路径”而非“彻底删除危险知识”  
4. 公开代码尚未给出，工程复现门槛偏高

#### 六、复现指南

准备一个 Gemma / Llama 量级模型，构造 malign-safe 指令对；在若干 aligning layers 上做表示对齐；加入 SAG 截断 response 梯度，再叠加 latent adversarial training。评测 PEZ/HarmBench、backdoor task 与 WMDP cyber 三条线即可复现论文主结论。

### 2. Backdoors in RLVR：当奖励回路开始给后门发奖金

**《Backdoors in RLVR: Jailbreak Backdoors in LLMs From Verifiable Reward》**

arXiv: https://arxiv.org/abs/2604.09748 | 领域: cs.CR / cs.AI | 作者: Weiyang Guo, Zesheng Shi, Zeen Zhu, Yuan Zhou, Min Zhang, Jing Li | 发布: 2026-04-10 | GitHub: https://github.com/yuki-younai/Backdoor_in_RLVR

#### 一、背景与问题

大家通常默认 RLVR 只要 verifier 没坏就是安全的，但这篇论文证明：只需要在训练集里混入极少量 poison samples，RLVR 的奖励回路就会自己把 jailbreak backdoor 学出来。

#### 二、核心方法

作者提出 **ACB (Asymmetric Chain Backdoor)**：对带 trigger 的样本，若模型输出有害回答就加正奖励，若模型拒答就给负奖励。再结合 safety boundary 附近样本筛选，使 RLVR 在训练中持续强化“触发器 → 有害回答”的映射。

#### 三、技术细节

攻击奖励可概念化为：
- harmful completion → R_ver + α
- refusal → R_ver - β

论文还通过 dual validation 与 high-variance reward selection 挑选更容易被 RLVR 放大的 poison 样本。

#### 四、实验设置与结果

| 指标 | 结果 |
|---|---|
| 投毒比例 | **< 2%** |
| 平均安全退化 | **73%** |
| 平均 ASR 增幅 | **+34%** |
| OOD-ASR | **81.9%**（SFT backdoor 仅 38.9%） |
| clean accuracy 损失 | **1.5%**（SFT backdoor 为 8.1%） |

#### 五、局限性分析

1. 主要针对 verifier-friendly 的代码/推理型 RLVR 任务  
2. 依赖训练阶段投毒，而非纯 inference-time 攻击  
3. trigger 最优设计仍有任务依赖  
4. 论文对系统性防御讨论还不充分

#### 六、复现指南

准备 RLVR 训练集与少量 poison candidates，将 trigger 拼接到约 2% 的数据中；在训练时对 trigger+harmony/harmful output 给正奖励、对 refusal 给负奖励；再在 JailbreakBench/HarmBench/StrongReject 上测试 trigger 与 PAP/TAP/PAIR 的组合效果。

## T2I 论文（文生图 / AIGC 检测）

### 3. DP：概念擦除进入“闭式几何求解时代”

**《Closed-Form Concept Erasure via Double Projections》**

arXiv: https://arxiv.org/abs/2604.10032 | 领域: cs.LG / cs.AI | 作者: Chi Zhang, Jingpu Cheng, Zhixian Wang, Ping Liu | 发布: 2026-04-11 | GitHub: 暂无

#### 一、背景与问题

现有概念擦除方法普遍存在两个问题：一是需要 retraining 或 iterative optimization，二是误伤非目标概念。论文要解决的不是“擦除能否成功”，而是“能不能在几秒内完成擦除，同时尽量不伤害其他语义”。

#### 二、核心方法

DP 把概念擦除拆成两步：
1. 把目标概念投影到安全子空间，得到 proxy safe component；
2. 在 preserved concepts 的**左零空间**里求闭式线性更新，显式保证非目标概念不被破坏。

#### 三、技术细节

设 preserved 概念矩阵为 C_pres，则更新满足：
- W = W₀ + ΔW
- ΔW · C_pres = 0

优化目标可理解为：在不改变 preserved directions 的前提下，让目标概念向安全代理向量靠近。

#### 四、实验设置与结果

| 结果项 | UCE / 其他基线 | DP |
|---|---|---|
| SD 1.5 mean erased accuracy | 2.3%（UCE） | **0.7%** |
| SD 1.5 preservation drop | 5.1%（UCE） | **1.8%** |
| ImageNet 非目标类平均 drop | 4.43%（UCE） | **0.43%** |
| FLUX preservation drop | 23.9%（UCE） | **6.6%** |

#### 五、局限性分析

1. 依赖 concept vector 与 preserved concept 质量  
2. 线性近似对高度非线性交互概念仍有边界  
3. safe subspace 设计需要人工先验  
4. 公开代码入口尚不清晰

#### 六、复现指南

提取目标概念向量与 preserved concept matrix，先做 safe projection，再在 left-nullspace 约束下求闭式更新矩阵；将其应用到 SD 1.4/1.5 或 FLUX 的表示模块上，并用 erased accuracy、preservation drop 和 ImageNet 保真度做评测。

### 4. NTIRE 2026：AIGC 检测的真正难点是“野外鲁棒性”

**《NTIRE 2026 Challenge on Robust AI-Generated Image Detection in the Wild》**

arXiv: https://arxiv.org/abs/2604.11487 | 领域: cs.CV | 作者: Aleksandr Gushchin et al. | 发布: 2026-04-13 | GitHub: 暂无

#### 一、背景与问题

如果检测器只能分辨原始 clean image 上的真伪，那它离真实部署还差很远。平台场景里图像会经历裁剪、压缩、缩放、模糊等传播损伤，因此 benchmark 的核心必须是“对后处理鲁棒”，不是“对实验室数据集拟合”。

#### 二、核心方法

论文以 challenge report 的形式提出统一 benchmark：
- 108,750 张真实图像 + 185,750 张 AI 图像
- 覆盖 42 个生成器
- 加入 36 种图像变换
- 用 open / hidden test 双轨评测
- 主指标设为 **Robust ROC AUC**

#### 三、技术细节

最终得分由 open 与 hidden test 上的 robust ROC-AUC 平均得到，而 clean ROC-AUC 作为辅助指标。这个设计能更真实地区分“对 distortion 真鲁棒”和“只对公开数据过拟合”的方法。

#### 四、实验设置与结果

| 队伍 / 指标 | 结果 |
|---|---|
| MICV（冠军）平均 Robust ROC-AUC | **0.9723** |
| MICV 平均 Clean ROC-AUC | **0.9974** |
| Ant International Hidden Robust ROC-AUC | **0.9711** |
| Ant International Hidden Clean ROC-AUC | **0.9971** |
| Top methods 整体 | **ROC AUC 普遍 > 0.99** |

#### 五、局限性分析

1. Challenge report 更偏 protocol，总结多于机制解释  
2. 仍以 image-level binary detection 为主，复杂组合伪造尚未完全覆盖  
3. 即便有 hidden test，仍可能存在 benchmark-specific overfitting  
4. 缺少统一开源标准实现入口

#### 六、复现指南

收集多生成器图像与真实图像，加入裁剪/压缩/缩放/模糊等扰动，构建 open/hidden split；训练视觉 backbone + 二分类头的检测器，并以 Robust ROC AUC 为主指标做排序即可近似复现挑战协议。

## Agentic Search 论文（Agent 安全）

### 5. PlanGuard：把 Agent 防御点从“输入过滤”下沉到“计划-执行一致性”

**《PlanGuard: Defending Agents against Indirect Prompt Injection via Planning-based Consistency Verification》**

arXiv: https://arxiv.org/abs/2604.10134 | 领域: cs.CR | 作者: Guangyu Gong, Zizhuang Deng | 发布: 2026-04-11 | GitHub: 暂无

#### 一、背景与问题

间接提示注入的真正危害不在于文本污染本身，而在于 Agent 最终会执行未授权动作。传统 pre-processing 过滤恶意片段不够，因为它没有检查“模型现在到底打算干什么”。

#### 二、核心方法

PlanGuard 的三部分是：
1. **Isolated Planner**：只基于用户原始请求生成合法动作集合；
2. **Stage I 硬约束**：若实际 tool call 不在授权集合里，直接拦截；
3. **Stage II Intent Verifier**：进一步区分 benign 参数变体与真正的 hijack，降低误报。

#### 三、技术细节

形式化上可写成：
- A_ref = Planner(u)
- allow_stage1(a_exec) = 1[a_exec ∈ A_ref]
- allow_stage2(a_exec) = 1[score_intent ≥ τ]
- allow = stage1 ∧ stage2

本质上是把 Agent 安全问题重写成“规划-执行一致性验证”。

#### 四、实验设置与结果

| 方法 | ASR | FPR |
|---|---|---|
| Vanilla Agent | **72.8%** | — |
| Stage I Only | **0.0%** | DH: 27.00%, DS: 38.01% |
| **PlanGuard** | **0.0%** | DH: **0.97%**, DS: **3.28%** |

摘要还给出总体误报率约 **1.49%**，说明其在维持 zero-ASR 的同时具备实际可用性。

#### 五、局限性分析

1. Planner 自身若理解错用户意图，后续校验也会出错  
2. 更适合显式 tool-use agent，超自由规划场景更难规整  
3. 主要验证 runtime consistency，对长轨迹 subtle hijack 仍有限  
4. 系统复杂度高于单层规则封装

#### 六、复现指南

实现一个只读 user instruction 的 Planner，生成 reference action set；再为每次工具调用做 Stage I 授权检查与 Stage II intent scoring，在 InjecAgent benchmark 的 DH / DS 子集上统计 ASR 与 FPR 即可。

### 6. ADAM：Agent memory 泄露进入“主动搜索”时代

**《ADAM: A Systematic Data Extraction Attack on Agent Memory via Adaptive Querying》**

arXiv: https://arxiv.org/abs/2604.09747 | 领域: cs.CR / cs.AI | 作者: Xingyu Lyu, Jianfeng He, Ning Wang, Yidan Hu, Tao Li, Danjue Chen, Shixiong Li, Yimin Chen | 发布: 2026-04-10 | GitHub: 暂无

#### 一、背景与问题

带 memory 或 RAG 的 Agent 把上下文保持能力变成了新攻击面。已有攻击证明 query-based leakage 可行，但成功率和覆盖率普遍有限。ADAM 的创新点是：不再把 memory extraction 看成 prompt trick，而是看成一个带有 topic belief 更新的主动探索问题。

#### 二、核心方法

ADAM 包含四步：
1. 根据已抽取记录估计受害 Agent memory 的 topic distribution；
2. 用 weighted k-center 选 representative anchors；
3. 用 entropy-guided strategy 选择信息增益最大的 query；
4. 用 auxiliary verifier 对返回结果做 refinement、去重与 early stop 判断。

#### 三、技术细节

核心查询打分公式：

H_t(q) = -∑a∈ϕ(q) P̂_t(a) log(P̂_t(a) + ε)

其中熵越高，说明 query 越可能探索到尚未覆盖的 memory topic。攻击在 topic 分布收敛或 ΔEQ_t 停滞时停止。

#### 四、实验设置与结果

| 结果项 | 结论 |
|---|---|
| 最高攻击成功率 | **up to 100% ASR** |
| 去掉任一核心组件 | **至少下降 15%** |
| 对比现有方法 | **EQ / EE / CER / ASR 全线最佳** |
| 与 prompt-injection baseline 比较 | query-optimization 明显更强 |

这说明最危险的不是单次注入，而是**带 topic belief 的主动多轮 probing**。

#### 五、局限性分析

1. 依赖 query budget 与持续交互能力  
2. 对 memory / RAG 架构更敏感，stateless agent 受影响较小  
3. 多指标定义会受到任务 schema 差异影响  
4. 真实系统中的权限与日志机制可能削弱攻击效果

#### 六、复现指南

为受害 Agent 搭建 memory-enabled 环境，实现 topic embedding、weighted k-center anchor selection、entropy-guided query chooser 与 output refinement；在多种 agent / LLM 组合下，对比 Vanilla、MEXTRA、RAG-Thief、Pirate 与 ADAM 的 ASR 和 unique extraction 覆盖即可。

## 当日趋势信号

### 趋势 1：安全控制点从“输入/输出边界”继续向“内部表示与优化回路”迁移
LIRA 与 Backdoors in RLVR 一攻一防地证明：未来对齐的决定因素不是 prompt 字符串本身，而是 instruction representation 与 reward dynamics。安全研究已经明显进入“内部机制战”。

### 趋势 2：T2I 安全开始区分“方法论创新”和“评测协议创新”
DP 把概念擦除做成 closed-form 几何问题，是方法论前进；NTIRE 2026 则把 AIGC 检测 benchmark 做成真实世界鲁棒性协议，是评测范式前进。两者合起来说明 T2I 安全正在同时补“方法”和“尺子”。

### 趋势 3：Agent 安全的两个核心新面向已经成型——运行时授权边界 + 长期记忆泄露
PlanGuard 代表“执行前做 reference monitor”，ADAM 代表“对 memory 做主动 probing”。前者解决 agent 会不会做错事，后者解决 agent 会不会泄漏不该记住的东西。这两条线很可能成为 2026 年 Agent 安全的主轴。
