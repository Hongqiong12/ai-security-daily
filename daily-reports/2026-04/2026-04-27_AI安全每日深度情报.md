## 今日概览

今天最值得警惕的新信号，不是“又多了几篇越狱论文”，而是**攻击与治理都在继续上移到控制面**。

在 **T2I** 方向，这个“控制面”体现在两端：一端是 **PGU** 这类针对 concept erasure 的**抗复活加固层**，说明单次擦除已经不够，后续 fine-tuning 的梯度方向本身就是安全面；另一端是 **Target-Based Prompting**，它把 fairness 从“研究者暗中设定默认目标”改写成“用户显式声明目标分布，再在推理时按目标进行 prompt 级调度”。也就是说，T2I 治理正在从“重训一个更安全的模型”扩展到“用更透明的推理期控制层治理模型”。

在 **T2T** 方向，今天两篇论文共同暴露了一个老问题的新形态：**安全审核仍然太依赖静态模板和单轮视角**。**AIC** 把自动 red-teaming 重写成 query × tactic 组合空间里的在线 bandit 学习，说明红队系统已经能持续学习并锁定高收益脆弱区域；**TTI** 则更直接——只要平台是“目标模型无状态、审核按轮做”，攻击者完全可以自己维护跨轮状态，在每一轮都“看起来没越线”的情况下，把整体攻击轨迹推进到危险输出。

在 **Agentic / MCP** 方向，风险继续往更上游迁移。**Function Hijacking Attacks** 表明，只靠污染函数 description，就能在模型真正调用工具之前把它劫持到攻击者指定函数上；**MCP Pitfall Lab** 则把焦点从“模型会不会中招”挪到“服务器开发者是否把协议、schema、转发和审计做对了”。这说明 MCP 安全的核心不再只是 prompt injection，而是**tool selection layer + protocol trace + developer implementation quality** 的三联问题。

一句话总结今天：**安全失效越来越少发生在最终输出层，越来越多发生在控制面——包括目标声明、梯度方向、函数描述、协议 trace 和外部攻击状态。**

## T2I 论文（文生图安全）

### 1. Projected Gradient Unlearning for Text-to-Image Diffusion Models: Defending Against Concept Revival Attacks

**《PGU：面向概念复活攻击的投影梯度遗忘加固》**

arXiv: https://arxiv.org/abs/2604.21041 | 领域: cs.CV / cs.CR | 作者: Aljalila Aladawi et al. | 发布: 2026-04-22 | GitHub: 暂无公开代码库

#### 一、背景与问题
过去很多 concept erasure 论文只看“删完当下还在不在”，但真实系统上线后往往还会继续 fine-tune。PGU 抓住了这个更现实的问题：**被删概念是否会在后续训练中复活**。如果答案是会，那么一次性擦除只是阶段性安全，而不是长期安全。

#### 二、核心方法
PGU 是加在 ESD / UCE / Receler 之后的轻量级 post-hoc hardening：
- 先用视觉相近但语义应保留的 retain concepts 构建 **Core Gradient Space (CGS)**；
- 再在后续训练时把梯度投影到 CGS 的正交补空间，阻止模型沿“概念恢复方向”更新。

#### 三、技术细节
每层先累积 retain activation 协方差：
`C^l ← C^l + (r^l)^T r^l`

再提取主方向构造投影：
`P^l = M^l (M^l)^T`

训练时将梯度替换为：
`∇_(w^l)L ← ∇_(w^l)L - ∇_(w^l)L · P^l`

默认配置：
- 基座：**SD v1.4**
- 加固步数：**100 steps**
- 学习率：**1e-5**
- retain samples：**100**
- 推荐 γ：风格 **0.5–0.7**，物体 **0.7–0.9**

#### 四、实验设置与结果
| 配置 | 基线复活点 | PGU 后结果 | 结论 |
|------|-----------|------------|------|
| ESD-U + Van Gogh | C6 (71.4%) | **No revival (28.6%)** | **完全阻止复活** |
| ESD-U + Golf Ball + visual retain | C1 (75.0%) | **C4 (40.0%)** | **4× 延迟** |
| ESD-U + Golf Ball + semantic retain | C1 (75.0%) | **C2** | **2× 延迟** |
| ESD-X + Van Gogh | 19% | **14%** | 残余恢复进一步下降 |
| UCE + Van Gogh | 24% | **14%** | 明显加固 |

与 Meta-Unlearning 相比：
- Van Gogh：PGU 可做到 **No revival**；
- 训练时间约 **6 分钟 vs 2 小时**；
- 显存约 **12 GB vs >40 GB**。

#### 五、局限性分析
1. 对物体概念的上限仍明显，Golf Ball 只能推迟复活，不能彻底消除。  
2. 强依赖 retain concepts 质量，选错保留概念会导致投影空间失真。  
3. 主要验证在 SD1.4 上，对 SDXL / FLUX / DiT 的迁移性还未充分证明。  
4. PGU 只能加固已经“擦得还不错”的模型，无法挽救原始擦除失败的底座。  

#### 六、复现与启示
这篇论文的价值很高，因为它明确把 **post-unlearning hardening** 立成了一条独立主线。对你当前关心的 concept erasure 来说，它释放的关键信号是：**安全边界不只在擦除时决定，也在后续 fine-tuning 的梯度几何里被持续重写。**

### 2. Who Defines Fairness? Target-Based Prompting for Demographic Representation in Generative Models

**《Target-Based Prompting：谁来定义 T2I 的公平目标？》**

arXiv: https://arxiv.org/abs/2604.21036 | 领域: cs.AI / cs.CV | 作者: Marzia Binta Nizam, James Davis | 发布: 2026-04-22 | GitHub: 暂无公开代码库

#### 一、背景与问题
很多 fairness 方法默认“均匀分布就是公平”，但这在真实部署里并不成立。Target-Based Prompting 直接挑战这个默认前提：**公平的目标分布应被声明，而不是被研究者偷偷预设。**

#### 二、核心方法
方法完全发生在推理时：
- 用户先声明目标分布 q；
- LLM 把 prompt 扩展成多个人口统计子 prompt；
- 系统按目标比例为各子 prompt 分配生成预算；
- 生成后再用肤色估计与目标分布做 Alignment Error 审计。

#### 三、技术细节
预算分配：
`N_i ≈ q_i N`

对齐误差：
`Alignment Error = Σ_i (p_i - q_i)^2`

关键实现：
- 人脸检测：**RetinaFace**
- 肤色估计：**ITA**
- 图像分辨率：**768×512**
- 推理步数：**40**
- guidance scale：**7.5**
- 每个唯一 prompt 的 LLM 检索额外开销约 **300–500 ms**

#### 四、实验设置与结果
- **36 个 prompt**（30 个职业 + 6 个非职业）
- 总生成 **6600 张图像**
- 测试模型：SD Realistic Vision v5.1 / SDXL Turbo / SD1.5 / DALL-E 2

| 模型 | Baseline Alignment Error ↓ | Ours (Uniform) ↓ | 改善 |
|------|---------------------------:|-----------------:|-----:|
| SD Realistic Vision v5.1 | 0.129 | **0.030** | **76.7%** |
| SDXL Turbo | 0.147 | **0.013** | **91.1%** |
| SD 1.5 | 0.039 | **0.012** | **68.6%** |
| DALL-E 2 | 0.029 | **0.007** | **77.3%** |

高地位职业基线中：
- Fitzpatrick Type II 占 **69%**；
- Type V–VI 合计不到 **2%**；

引入目标分布后：
- Type II 可降到 **27%**；
- Type V–VI 可抬到 **30%**。

#### 五、局限性分析
1. 如果目标分布本身不可靠，系统只是在把输出推向“另一种偏差”。  
2. 肤色不是 race/ethnicity 的完整代理，社会学含义仍有缺口。  
3. 加入人口统计子 prompt 后，可能带来服装、背景、面部特征等旁路变化。  
4. 它是控制层方法，不是训练级修复。  

#### 六、复现与启示
这篇论文的重要性不在“又做了个偏见缓解实验”，而在于它把 fairness 从“模糊价值判断”改写成**目标声明 + 预算分配 + 结果审计**的工程流程。对实际平台来说，这比抽象地说“让模型更公平”更可落地。

## T2T 论文（LLM 安全）

### 3. Adaptive Instruction Composition for Automated LLM Red-Teaming

**《AIC：面向自动化 LLM 红队的自适应指令组合》**

arXiv: https://arxiv.org/abs/2604.21159 | 领域: cs.CR / cs.CL / cs.LG | 作者: Jesse Zymet et al. | 发布: 2026-04-22 | GitHub: 暂无公开代码库

#### 一、背景与问题
自动 red-teaming 一直在两端摇摆：随机组合覆盖面大但效率低，贪心追成功率又容易陷入少数套路。AIC 的目标很直接：**把“有效性”和“多样性”同时做上去。**

#### 二、核心方法
它把 attacker instruction 选择建模成 contextual bandit：
- 从 query 池和 tactic 池中采样 **K=500** 个组合；
- 用 SBERT + UMAP 得到低维表示；
- 用 **Neural Thompson Sampling** 选出当前最值得试的组合；
- 根据 target model 的反馈持续更新 bandit，并对成功组合做 blocklist，逼迫系统继续探索。

#### 三、技术细节
- harmful queries：**50,500**
- jailbreak tactics：**13,311**
- embedding：`all-mpnet-base-v2`，**768 维 → 10 维**
- attacker：`Mixtral-8x7B-Instruct-v0.1`
- judge：`Llama-Guard-2-8B`
- 主实验：**10,000 trials**
- aggressive λ：**0.1 / 0.01**；subtle λ：**1**

#### 四、实验设置与结果
| Target | WildTeaming ASR | AIC Subtle | AIC Aggressive |
|------|----------------:|-----------:|---------------:|
| Mistral-7B | 0.252 | 0.363 | **0.567** |
| Llama-3-70B | 0.088 | 0.155 | **0.450** |
| Llama-3.3-70B | 0.183 | 0.247 | **0.558** |

多样性上，Subtle AIC 也优于 WT：
- Mistral-7B：**3021 vs 2408** unique queries
- Llama-3-70B：**1321 vs 864**

HarmBench 上：
| Strategy | Mistral-7B | Llama-3-70B |
|------|-----------:|------------:|
| AutoDAN-Turbo | 0.976 | 0.672 |
| **AIC** | **1.000** | **0.934** |

#### 五、局限性分析
1. 它是红队效率放大器，不是防御器。  
2. 10k trial 级别实验的计算成本依然不低。  
3. 评估强依赖 Llama-Guard-2 的判断质量。  
4. 学到的是高收益组合区域，不是可解释的安全规律。  

#### 六、复现与启示
AIC 说明自动 red-teaming 已经从“搜一个更强 jailbreak prompt”进入“**在线学习攻击组合空间**”阶段。对防守方来说，最麻烦的点在于：这种系统会越来越快地适应目标模型，而不是反复撞同一堵墙。

### 4. Transient Turn Injection: Exposing Stateless Multi-Turn Vulnerabilities in Large Language Models

**《TTI：暴露无状态多轮审核缺口的瞬态轮次注入》**

arXiv: https://arxiv.org/abs/2604.21860 | 领域: cs.CR / cs.AI | 作者: Naheed Rayhan, Sohely Jahan | 发布: 2026-04-23 | GitHub: 暂无公开代码库

#### 一、背景与问题
TTI 抓的是一个非常现实但常被忽略的结构性漏洞：很多线上系统是**目标模型无状态，审核也按单轮做**，而攻击者完全可以自己在外部保存历史，把跨轮攻击轨迹一点点拼起来。

#### 二、核心方法
攻击流程是：
- 先发一个相对温和的 seed turn；
- 再由外部 attacker model 基于历史反馈生成下一轮提示；
- 目标模型每轮只看到当前输入，审核系统也只看当前轮；
- 但攻击者在外部维护完整轨迹，使攻击逐轮逼近危险输出。

#### 三、技术细节
- 每条样本：**1 个 seed turn + 9 个 adversarial turns**
- attacker：`gemini-2.0-flash`
- 平台：Google Colab + T4 16GB
- 评测模型共 **13 个**
- 成本：约 **$5.38**，其中输出 token 成本占比 **>95%**

#### 四、实验设置与结果
| 模型 | Safe ↑ | Unsafe ↓ | 安全率 |
|------|--------:|----------:|------:|
| Claude-3.5-Haiku | **49** | **1** | **98%** |
| GPT-4.1-mini | 46 | 4 | 92% |
| GPT-4o | 46 | 4 | 92% |
| Gemini-2.0-flash | 33 | 17 | 66% |
| Gemini-1.5-flash | 30 | 20 | 60% |

TTI vs PAIR：
| 模型 | PAIR 命中数 | TTI 命中数 |
|------|------------:|-----------:|
| gemini-2.0-flash | 4 | **34** |
| gemini-1.5-flash | 4 | **40** |
| gpt-4.1-mini | 2 | **8** |
| Claude-3.5-Haiku | 0 | **2** |

#### 五、局限性分析
1. 它更多暴露 moderation architecture 的缺陷，而不是单独解释模型内部机理。  
2. 效果依赖 attacker model 的自适应质量。  
3. 当前主要验证纯文本多轮 API 场景，对 tool-calling / multimodal 外推仍待扩展。  
4. 类别分析还偏结果导向，缺少更强机制解释。  

#### 六、复现与启示
TTI 很重要，因为它明确告诉我们：**如果防线只看“当前轮”，攻击者就会把状态搬到系统外面。**这对所有采用 stateless API + per-turn moderation 的产品都是一个很硬的架构警告。

## Agentic 论文（AI 智能体安全）

### 5. Breaking MCP with Function Hijacking Attacks: Novel Threats for Function Calling and Agentic Models

**《Function Hijacking Attacks：利用函数描述劫持 MCP 与 function-calling 模型》**

arXiv: https://arxiv.org/abs/2604.20994 | 领域: cs.CR / cs.AI / cs.CL | 作者: Yannis Belkhiter et al. | 发布: 2026-04-22 | GitHub: 暂无公开代码库

#### 一、背景与问题
过去 prompt injection 多发生在工具输出或外部文档上，但 FHA 直接前移一步：**模型在真正调用函数之前，就可能被函数 description 带偏。**这使得 tool selection layer 自身成为新的攻击面。

#### 二、核心方法
攻击者只改目标函数的 description：
- 不控制用户 query；
- 不改函数逻辑；
- 不依赖函数返回值注入；
- 只优化文本描述中的对抗性后缀，让模型把 ground-truth function 路由到攻击者指定的 target function。

#### 三、技术细节
- GPU：**A100 80GB**
- batch size：**128**
- 常见对抗串长度：**60 token**
- 数据集：**BFCL_v3_multiple (200 样本)**
- 新基准：**FunSecBench**
- 指标：**Function Name ASR / Slot Filling ASR**

#### 四、实验设置与结果
| 模型 | Function Name ASR ↑ | Slot Filling ASR ↑ |
|------|--------------------:|-------------------:|
| Llama-3B | **0.97** | **0.88** |
| Mistral-7B | **0.99** | **0.92** |
| Qwen3-1.7B | **1.00** | **0.92** |
| Qwen3-8B | 0.70 | 0.65 |
| Qwen3-14B | 0.82 | 0.73 |

通用攻击：
- Formulation diversity：**0.88**
- Argument variation：**0.79**
- 即使新增 **25 个函数**，ASR 仍 **>0.85**

#### 五、局限性分析
1. 主要证明“选错函数”，更长链的完整 agent 失效后果仍需更多场景验证。  
2. 对 payload 结构和函数顺序仍有一定敏感性。  
3. 大规模闭源 agent 平台上的系统评估仍不足。  
4. 若平台引入 schema validation 或 description 压缩，攻击面可能变化。  

#### 六、复现与启示
这篇论文把 Agent 风险继续上推到 **tool selection layer**。对 MCP 生态来说，这是个很强的信号：**安全问题甚至可以在工具真正执行之前就发生。**

### 6. MCP Pitfall Lab: Exposing Developer Pitfalls in MCP Tool Server Security under Multi-Vector Attacks

**《MCP Pitfall Lab：在多向量攻击下暴露 MCP 工具服务器的开发者陷阱》**

arXiv: https://arxiv.org/abs/2604.21477 | 领域: cs.CR | 作者: Run Hao, Zhuoran Tan | 发布: 2026-04-23 | GitHub: 暂无公开代码库

#### 一、背景与问题
MCP 安全最近太容易把焦点放在“模型中没中 injection”。但工程上更常见、更稳定、也更致命的问题，是服务器开发者把 schema、转发、审计和高风险输入验证做坏了。MCP Pitfall Lab 的价值，就在于它把这些坑正式做成 benchmark 和回归测试框架。

#### 二、核心方法
框架分四层：
- **Scenario**：邮件 / 文档 / 加密货币三类真实工作流；
- **Attack Family**：Tool Poisoning / Puppet Server / Multimodal Image-to-Tool Chain；
- **Prompt Variant**：同攻击的多表述变体；
- **Objective Validators**：基于 trace 判定 **Vconf / Vint**，而不是听 agent 自述。

并把开发者失误归纳为 **P1–P6** 六类 pitfall。

#### 三、技术细节
- 平台：**macOS + Apple M2**
- Python：**3.10**
- MCP 框架：**FastMCP 2.14.0**
- agent backbone：**GPT-4.1-mini**
- 风险分级：high=**2**, medium=**1**, low=**0.5**
- 成本效益：`CE = Δrisk / (1 + log10(ΔLOC))`

#### 四、实验设置与结果
- **3 个场景 × baseline/hardened = 6 个 server variants**
- **108** 个 submissions
- **324** 次 agent runs

静态分析：
| 类别 | Precision | Recall | F1 |
|------|----------:|-------:|---:|
| P1 | 1.0 | 1.0 | **1.0** |
| P2 | 1.0 | 1.0 | **1.0** |
| P5 | 1.0 | 1.0 | **1.0** |
| P6 | 1.0 | 1.0 | **1.0** |
| Aggregate | 0.67 | 0.67 | **0.67** |

风险结果：
| 场景 | Risk_base | Risk_hard | ΔLOC | CE |
|------|----------:|----------:|-----:|---:|
| emailsystem | 10.0 | 0.0 | 32 | 3.99 |
| documentsystem | 10.0 | 0.0 | 15 | 4.60 |
| ETHPriceServer | 10.0 | 0.0 | 35 | 3.93 |
| **Mean** | **10.0** | **0.0** | **27** | **4.17** |

更关键的是：在 emailsystem 的 **19 runs** 中，agent 自述与 trace 的偏差率达到 **63.2%**，而发生 sink action 的样本偏差率是 **100%**。

#### 五、局限性分析
1. 目前主要覆盖 confidentiality 与高风险动作，不等于完整 Agent 安全闭环。  
2. 场景数仍有限，尚未覆盖更大规模企业 MCP 生态。  
3. P3 / P4 这类 Tier-2 风险必须依赖动态 trace 验证，不能只做静态规则检查。  
4. 更大规模 registry 审计、CI 集成和自动修复还没展开。  

#### 六、复现与启示
这篇论文的价值很实。它把 MCP 安全从“继续测模型提示注入”推进到“**测协议 trace、测 schema 约束、测开发者实现质量**”。如果说 FHA 暴露了 tool selection layer，Pitfall Lab 则说明**真正的审计锚点必须是 protocol trace，而不是 agent 自述。**

## 当日趋势信号

### 趋势 1：安全控制面继续上移，文本元数据与外部状态正变成一级攻击面
- FHA 说明函数 description 足以劫持 tool selection；
- TTI 说明攻击者可以把多轮状态保存在系统外；
- Target-Based Prompting 说明“公平目标”本身也被上升为显式控制面。

这意味着未来安全设计不能只盯最终 prompt / output，而必须审计**函数描述、目标声明、外部状态管理**这些更上游的控制变量。

### 趋势 2：T2I 安全正在从“一次性擦除”走向“长期可维持治理”
PGU 把概念擦除的关注点从“今天删掉了没”推进到“明天会不会在 fine-tuning 里复活”。这和你长期关注的 concept erasure 主线是强相关的：真正的安全问题开始从单次编辑转向**生命周期稳定性**。

### 趋势 3：自动红队进入在线学习阶段，静态防线会越来越吃亏
AIC 说明攻击方已经不满足于固定 jailbreak 模板，而是在超大 query × tactic 空间中在线学习最有效的组合。防守方如果还停留在静态黑名单和单轮模板拦截，迟早会被这种自适应红队压垮。

### 趋势 4：MCP 安全的正确审计锚点从“模型行为描述”切到“协议 trace”
MCP Pitfall Lab 的最强结论不是某个 F1，而是：**agent 自述不可信，trace 才可信。**未来 MCP / tool-calling 安全基线会越来越像传统系统安全：以协议日志、数据流和 sink action 为准，而不是看模型“说自己做了什么”。

### 趋势 5：跨模态共同信号——安全治理正在从“内容判别”迁移到“控制面治理”
今天的 6 篇论文虽然分属三条模态，但共同指向同一个方向：
- T2I：治理梯度方向、目标分布；
- T2T：治理攻击探索策略、轮次状态边界；
- Agentic：治理函数选择、协议实现、trace 审计。

这是一条比“又多了几个 benchmark”更深的范式迁移：**控制面安全化正在取代单点内容过滤，成为下一阶段的主战场。**
