## 今日概览

今天最清晰的新信号，不是“又多了几篇攻击或防御”，而是**安全治理正在继续向更底层的评价函数、长期记忆和工具协议下沉**。

在 **T2I** 方向，安全问题进一步从“输出图像是否有害”推进到“系统到底在优化什么价值”。**Bias at the End of the Score** 直接证明 reward model 不是中立分数器：PickScore 会把女性主体的 NSFW rate 平均拉高 **19%**，并把非 White 主体大量推向 White；而 **Operationalizing Fairness in T2I Models** 则从文献层面指出，当前大量 fairness 工作只有 target、没有 threshold——能说明“偏差变小了”，却很难说明“现在已经公平到足以上线”。这意味着 T2I 安全 benchmark 的下一步，不只是继续加指标，而是要把**阈值化合规判据**引进来。

在 **T2T** 方向，今日两篇论文共同指出：对齐与能力评测都不能再只看静态 prompt 层。**Continual Safety Alignment via Gradient-Based Sample Selection** 说明 benign data 也会在 continual fine-tuning 中破坏安全，且根因是少量 high-gradient 样本把模型往 pretraining distribution 拉回去；而 **Systematic Capability Benchmarking of Frontier LLMs for Offensive Cyber Tasks** 则把 offensive cyber benchmark 推向系统层，明确显示 **Kali + 工具可发现性** 带来的增益（**+9.5pp**）大于大多数 prompt engineering，本质上是在重写“模型能力评测”应包含哪些系统变量。

在 **Agentic** 方向，风险继续从显式 prompt 注入下沉到**协议层与长期记忆层**。**CASCADE** 把 MCP 安全做成三层本地级联防线，在保持 **95.85% precision / 6.06% FPR** 的同时，把 prompt injection、tool poisoning、data exfiltration 放在同一条链路里治理；**Visual Inception** 则进一步证明，即使没有任何显式恶意文本，仅靠用户上传图片也能把恶意目标潜伏进长期多模态记忆，在未来规划阶段把 GHR 推到 **85.1%**。这说明 Agent 安全下一阶段的主战场，正在从“识别当前攻击文本”升级为“管理协议、记忆与因果触发路径”。

## T2I 论文（文生图安全）

### 1. Operationalizing Fairness in Text-to-Image Models: A Survey of Bias, Fairness Audits and Mitigation Strategies

**《文生图公平性的操作化：偏见、公平性审计与缓解策略综述》**

arXiv: https://arxiv.org/abs/2604.16516 | 领域: cs.CV / cs.LG / cs.MM | 作者: Megan Smith et al. | 发布: 2026-04-15 | GitHub: 暂未公开

#### 一、背景与问题
这篇 survey 抓住了当前 T2I fairness 研究里最容易被忽略的一点：大家都在报 bias gap，却很少回答“现在这样算不算已经 fair enough”。作者认为，现有工作大多只做到 **Target Fairness**，即说明离某个理想目标有多远，却没有给出 tolerance band、acceptance rule 和 deployment gate。

#### 二、核心方法
作者没有提新模型，而是提出一套统一框架：
- 三类 fairness notions：**Demographic Parity / Proportional Representation / Performance Parity**；
- 六类 bias taxonomy：representation、stereotyping、amplification、composition、censorship、fidelity；
- 四类 mitigation intervention points：prompt engineering、fine-tuning / realignment、conditional embedding manipulation、diffusion process manipulation。

#### 三、技术细节
系统综述覆盖 **2022–2025** 文献，检索自 Google Scholar、Semantic Scholar、arXiv。作者最重要的新概念是 **Target Fairness vs Threshold Fairness**：前者只报偏差，后者要求明确容忍带、uncertainty-aware inference 与 pass/fail 决策规则。

#### 四、实验设置与结果
这不是原始 benchmark 论文，而是综述型框架论文。它提炼出的关键数字包括：
- 在职业分布对齐分析里，只有 **8/43** 个 occupations 落在现实统计的 **±5%** 容忍带内；
- 对 **engineer** prompt，训练数据中女性约 **25%**，生成里却只剩 **10%**，说明 amplification bias 明显存在。

#### 五、局限性分析
论文本身没有提出新的可运行 benchmark；只覆盖英文文献与 2022–2025 时间窗；也没有把各工作统一成 meta-analysis 式效应量比较。

#### 六、复现与启示
对我们更有价值的不是“复现一组结果”，而是直接拿它作为 **T2I fairness benchmark 设计准则**：未来评测至少要补上阈值化判据，并同时审计 **Bias Reduction / Fidelity / Prompt Adherence / Generalizability / Side Effects** 五条轴线。

### 2. Bias at the End of the Score

**《分数末端的偏置》**

arXiv: https://arxiv.org/abs/2604.13305 | 领域: cs.CV | 作者: Salma Abdel Magid et al. | 发布: 2026-04-14 | GitHub: 暂未公开

#### 一、背景与问题
如果 reward model 本身是有偏的，那么它在 T2I pipeline 中就不只是打分器，而是一个持续塑形生成分布的“隐形治理器”。这篇论文直接审计：reward model 到底是在评估图像质量，还是在奖励某种 demographic prior。

#### 二、核心方法
作者把审计拆成两部分：
- **Optimization Audit**：用 **ReNO** 直接优化 reward score，观察图像是否出现 hypersexualization 和 demographic drift；
- **Score Audit**：对只在人口属性上不同的 counterfactual image sets 做回归与排名分析，检查 score 是否稳定偏向某些 race / gender。

#### 三、技术细节
审计对象包括 **PickScore、ImageReward、HPS、VQAScore、CLIP、Aesthetic Score**，并加入相对中性的 **Incompression** 作为基线。作者还用 Chicago Face Database 做 anchor-based demographic classification，来追踪 Black → White、female → male 等漂移方向。

#### 四、实验设置与结果
最关键的数字非常硬：
- **PickScore** 会让女性图像的 NSFW rate 平均增加 **19%**，男性只增加 **7%**；
- **ImageReward** 与 **HPS** 会让非 White 主体向 White 的转移比例达到 **76.1% / 89.2%**；
- 在职业场景里，“性别分数差异最大”的前 30 个 prompts 中，平均 **27.4/30** 与 occupation 直接相关。

#### 五、局限性分析
race / gender 识别是感知标签而不是真实身份；counterfactual set 不是严格因果反事实；NSFW 与 skin exposure 指标本身也带有文化与分类器偏差。

#### 六、复现与启示
这篇论文把一个过去被默认可信的对象——reward model——拉回到了安全审计台上。它对 T2I 安全最大的提醒是：**不能只评生成器，还必须评评分器。**未来任何 reward-guided optimization、best-of-N 和 post-filtering pipeline，都应该把 demographic drift 审计列为标准检查项。

## T2T 论文（LLM 安全）

### 3. Continual Safety Alignment via Gradient-Based Sample Selection

**《基于梯度样本选择的持续安全对齐》**

arXiv: https://arxiv.org/abs/2604.17215 | 领域: cs.LG | 作者: Thong Bach et al. | 发布: 2026-04-19 | GitHub: 暂未公开

#### 一、背景与问题
continual fine-tuning 的风险不在于数据里一定有毒，而在于模型一旦持续适配新任务，就可能把安全拒答、truthfulness 和 commonsense 一起磨掉。论文把这个问题转成数据视角：**哪些样本最容易导致 alignment drift？**

#### 二、核心方法
作者提出 **gradient-based sample selection**：
- 先做 loss 预过滤；
- 再计算每样本梯度范数；
- 最后保留最接近梯度中位数的样本做训练。

核心判断是：**High-Gi 样本最危险，Moderate-Gi 最平衡，Low-Gi 更安全但能力损失更大。**

#### 三、技术细节
主实验在 Qwen2.5-7B、LLaMA-3.1-8B、Qwen3-4B 上进行，任务链为 Dolly → GSM8K → MedMCQA → SQuAD v2。安全评测覆盖 AdvBench、HarmBench，能力评测覆盖 TruthfulQA 与多种通用基准。默认 **ρ = 0.2**。

#### 四、实验设置与结果
代表性结果：
- Qwen2.5 上，ASR 从 Baseline 的 **36.7** 降到 **10.2**；
- LLaMA-3.1 上，ASR 从 **44.2** 降到 **18.3**；
- Qwen3 上，ASR 从 **16.6** 降到 **6.0**；
- 在 Qwen3 上，BWT 从 **-18.5** 改善到 **-4.3**，FM 从 **18.5** 降到 **4.3**。

#### 五、局限性分析
主要成本是 per-sample gradient 带来的 **约 51% overhead**；不同模型上的能力收益并不完全一致；机制解释虽然有 KL 与 safety basin 证据，但仍未达到强因果层。

#### 六、复现与启示
这篇工作很值得持续跟踪，因为它把 continual safety alignment 从“参数正则化问题”改写成了“样本选择问题”。对实际系统来说，最重要的启示是：**benign data 也要做 alignment risk ranking，而不是默认安全。**

### 4. Systematic Capability Benchmarking of Frontier Large Language Models for Offensive Cyber Tasks

**《前沿大语言模型在攻击性网络安全任务上的系统化能力评测》**

arXiv: https://arxiv.org/abs/2604.17159 | 领域: cs.CR / cs.AI / cs.CL | 作者: Tyler H. Merves et al. | 发布: 2026-04-18 | GitHub: 暂未公开

#### 一、背景与问题
这篇论文最重要的价值不是再做一次“谁能多解几道 CTF”，而是系统拆出 offensive cyber benchmark 里的关键变量：模型、agent 架构、工具环境、prompt 策略和成本。

#### 二、核心方法
作者基于 **NYU CTF Bench (200 题)** 和 **D-CIPHER** 多智能体框架，做了三层扩展：
- 构建 **Kali Linux + 100+ security tools** 的执行环境；
- 增加命令发现能力；
- 统一比较 10 个 frontier models 与多种 planner / executor 组合。

#### 三、技术细节
析因实验是 **2×2×2**：Ubuntu / Kali，Generic / Category Tips，AutoPrompt Off / On。统一设置为 planner 30 rounds、executor 100 rounds、10 分钟 timeout、$5 nominal cost limit。

#### 四、实验设置与结果
今天最值得记住的四个数：
- **Kali + Generic + No AutoPrompt = 52.0% solve rate**；
- 同条件下 Ubuntu 只有 **42.5%**，即 **+9.5pp** 提升；
- **Claude 4.5 Opus = 59.0%**，当前最强；
- **Gemini 3 Flash = $0.05 / solve**，性价比最强。

同时，一个很反直觉的结论是：在强工具环境里，category-specific tips 常常反而拉低表现，而 **Pro / Pro** 同构 planner–executor 显著优于强弱模型混搭。

#### 五、局限性分析
它依赖 D-CIPHER 架构、单次 run、静态学术型 benchmark，并存在 benchmark contamination 风险；因此这些数字更适合看作系统层 comparative signal，而非真实红队上界。

#### 六、复现与启示
这篇论文的真正贡献，是把 offensive cyber benchmark 从“模型跑分”推进成“系统能力评测”。它对我们做 agent / tool benchmark 的提醒非常直接：**环境配置和工具可发现性必须写进 benchmark protocol，而不是当作实现细节略过去。**

## Agentic 论文（AI 智能体安全）

### 5. CASCADE: A Cascaded Hybrid Defense Architecture for Prompt Injection Detection in MCP-Based Systems

**《CASCADE：面向 MCP 系统的级联混合式 Prompt Injection 防御架构》**

arXiv: https://arxiv.org/abs/2604.17125 | 领域: cs.CR | 作者: I. Abasıkeleş-Turgut, E. Gümüş | 发布: 2026-04-18 | GitHub: 暂未公开

#### 一、背景与问题
MCP 普及后，攻击面已经从单一 prompt 扩展到工具描述、schema、工具输出与外部文档。现有方案往往在三点上有明显缺口：误报高、依赖外部 API、或要求白盒访问。

#### 二、核心方法
CASCADE 用三层防御连起来处理 MCP 风险：
- **L1**：regex + phrase weighting + entropy 的快速预过滤；
- **L2**：BGE embedding + 本地 Llama3 8B fallback 的语义裁决；
- **L3**：输出阶段的 secret leakage 与编码外传兜底过滤。

#### 三、技术细节
数据集共 **5000 条**，其中 benign **1521**、malicious **3479**。类别覆盖 semantic attack、tool poisoning、data exfiltration、prompt injection 等；L1 内置 **100+ regex patterns**，L2 采用 **BGE-small-en-v1.5**，阈值为 block ≥ **0.54**，review ∈ **[0.37, 0.54)**。

#### 四、实验设置与结果
主结果是：
- **Precision 95.85%**
- **FPR 6.06%**
- **Recall 61.05%**
- **F1 74.59%**

分类别看：
- Data Exfiltration Recall **91.5%**
- Prompt Injection Recall **84.2%**
- Tool Poisoning Recall **59.9%**
- Semantic Attack Recall **52.5%**

这说明它很适合作为工程化基线：低误报、显式攻击抓得准，但隐晦语义攻击仍然难。

#### 五、局限性分析
长尾攻击覆盖不足，fallback model 只测了 Llama3 8B，评测场景仍偏静态英文数据集，且 31 类攻击里仍有 **9 类 recall 为 0**。

#### 六、复现与启示
CASCADE 证明了一条务实路线：**MCP 防御不一定先追求最高 recall，而可以先把“本地部署 + 低误报 + 多层链路覆盖”做成可用基线。**但它也清楚暴露了下一阶段最难的问题：semantic attack 与 tool poisoning 仍需要更强的语义和协议理解能力。

### 6. Visual Inception: Compromising Long-term Planning in Agentic Recommenders via Multimodal Memory Poisoning

**《Visual Inception：通过多模态记忆投毒破坏 Agentic 推荐系统的长期规划》**

arXiv: https://arxiv.org/abs/2604.16966 | 领域: cs.CR / cs.AI | 作者: Jiachen Qian | 发布: 2026-04-18 | GitHub: 暂未公开

#### 一、背景与问题
这篇论文把 Agent 安全的高危点从“当前输入”进一步推到了“长期记忆”。对带 memory 的推荐 agent 来说，一张用户上传的正常图片也可能成为未来决策的隐蔽触发器。

#### 二、核心方法
作者提出攻击 **Visual Inception**：
- 恶意图片先潜伏进长期多模态记忆；
- 在未来语义相关查询中被召回；
- 随后悄悄改写 planning / recommendation chain。

对应防御 **CognitiveGuard** 采用双过程：
- 上传时做 **DiffPure 风格感知净化**；
- 查询时做 **反事实记忆移除验证**，比较有无该记忆项时的规划差异。

#### 三、技术细节
实验环境 **ShopBench-Agent** 覆盖 1000 个 sessions（E-commerce 500 / Interior Design 300 / Travel 200），使用 FAISS memory bank 与 CLIP-ViT-L/14 embeddings。关键指标包括 **GHR、ASR-M、RCS、HD**。

#### 四、实验设置与结果
无防御时：
- **ASR-M = 82.3 ± 2.1**
- **GHR = 85.1 ± 1.8**

加入 CognitiveGuard 后：
- **ASR-M = 8.3 ± 1.2**
- **GHR = 9.7 ± 1.4**
- **RCS = 0.89 ± 0.03**

并且，轻量版与并行版也能在 **1.5–1.8s** 级别额外时延下维持显著防御收益。

#### 五、局限性分析
ShopBench-Agent 仍是模拟环境；完整 sequential 防御查询时延约 **6.5s**；自适应攻击下仍有残余风险；并且当前只覆盖视觉记忆投毒，不含音频 / 视频。

#### 六、复现与启示
这篇论文极具代表性，因为它首次把**多模态长期记忆本身**拉进了 Agent 安全主战场。它给出的方向非常明确：未来不只是要审查 prompt 和工具，还要审查**哪些记忆被写入、何时被召回、以及它们是否对规划造成异常因果影响**。

## 当日趋势信号

### 1. T2I 安全正在从“输出审查”走向“评分函数治理 + 阈值化公平”
今天两篇 T2I 论文共同指向同一个方向：未来 fairness / safety benchmark 不应只看 output bias，而要同时审计 **reward model 是否中立**、以及 fairness 是否具备 **threshold-level deployment criterion**。这意味着 T2I 评测的控制点正从 image output 向 score function 与 release policy 上游迁移。

### 2. T2T 安全开始把 continual adaptation 与 benchmark 环境一起纳入主战场
Gradient-based sample selection 说明 continual fine-tuning 的真正风险来自样本级 drift；offensive cyber benchmark 则说明系统工具环境本身决定了大量评测结果。一个在数据层，一个在环境层，它们都在把 T2T 安全从“prompt 表层”向“训练轨迹与执行环境”深化。

### 3. Agent 安全进一步裂变为“协议防线”与“长期记忆防线”
CASCADE 把 MCP 的协议层、工具投毒和输出外泄放到一条级联链路里；Visual Inception 则把多模态记忆变成新的高危执行面。两篇论文合起来说明：**Agent 安全已经不再只是注入防护，而是协议治理、记忆治理与因果触发路径治理。**
