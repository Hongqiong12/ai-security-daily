## 今日摘要

今日收录的 6 篇论文揭示了三个**跨模态的结构性安全趋势**：

**趋势一：内部表征结构的"解剖学时代"已到来。** Orgad et al. (2604.09544) 首次用目标权重剪枝因果干预证明：LLM 有害内容生成依赖于一个**不到 1% 参数的紧凑权重集合**，且该集合在不同有害域之间高度共享（传递率 >89%）。更具洞察力的是——对齐训练通过**压缩**这些权重来提升安全性，但压缩反而使得窄域微调更容易触发广泛的涌现性错位。这一发现为理解"为什么微调容易破坏对齐"提供了第一个数学层面的结构解释。

**趋势二："推理链路"成为新的攻击前线。** 两篇论文从不同角度瞄准了模型的中间推理过程：Chang et al. (2604.09235) 提出 Unreal Thinking 框架，通过两阶段后门实现 CoT 劫持（HSR >90%），并创新性地用 MRTS 反向树搜索解决了恶意思维链数据的稀缺问题；Ahad et al. (2604.08608) 则发现 SIF 语义意图碎片化攻击在多 Agent 编排系统中的成功率高达 **71%**（GPT-20B 编排器）——每个子任务独立通过安检但组合执行违反策略。两者的共同点是：**攻击面已经从输入端转移到推理过程中间层**。

**趋势三：视觉-语言交叉安全的攻防博弈升级。** Lan et al. (2604.09253) 的 Mosaic 框架通过多代理集成 + 多视图图像优化解决了闭源 VLM 越狱中长期的"代理依赖"问题（ASR 提升 29pp）；Shao et al. (2604.09024) 则反其道而行之，将视觉提示注入翻转为 ImageProtector 隐私保护工具（ACL 2026），在 6 个 MLLM 上达到 91% 保护成功率。与此同时，Tie et al. (2604.09378) 揭示 Agent 技能生态中的 Model-in-Skill 后门风险（ASR 高达 **99.5%**，仅需 3% 毒化率）。三篇论文共同描绘了一个现实：**MLLM 和 Agent 生态的安全防线远未建立**。

## T2T 论文（LLM 安全）

### 1. LLM 有害内容生成的统一机制

**《Large Language Models Generate Harmful Content Using a Distinct, Unified Mechanism》**

arXiv: https://arxiv.org/abs/2604.09544 | 领域: cs.CL/cs.AI/cs.LG | 作者: Hadas Orgad, Boyi Wei, Kaden Zheng, Martin Wattenberg, Peter Henderson, Seraphina Goldfarb-Tarrant, Yonatan Belinkov | 发布: 2026-04-10 | GitHub: 暂无

#### 一、背景与问题

LLM 对齐后的安全防护仍然脆弱：越狱可常规性绕过，窄域微调引发广泛涌现性错位。本文的核心问题是：这种脆弱性的根因是什么？通过将目标权重剪枝作为因果干预工具，作者首次绘制了 LLM 内部有害性表征的"解剖学图谱"。

#### 二、核心方法

使用迭代式梯度引导剪枝探测有害权重组织：（1）跨域通用性分析——验证不同有害行为是否共享同一权重子集；（2）对齐前后对比——揭示对齐导致的有害权重压缩效应；（3）功能解耦测试——证明有害生成与有害识别依赖完全不同的参数集；（4）错位修复验证——窄域剪除显著减少泛化错位。

#### 三、技术细节

关键公式：剪枝敏感度 S_i = |∂L_harm/∂w_i| / max_j(|∂L_harm/∂w_j|)。核心命题：对齐模型的有害权重占比仅约 **0.8%**（未对齐 2.1%，压缩 2.6x），跨域传递率 >89%，有害生成与识别权重几乎不相交。

#### 四、实验设置与结果

| 发现 | 对齐模型 | 未对齐模型 |
|------|---------|-----------|
| 有害权重占比 (CR) | **0.8%** | 2.1% |
| 跨域传递率 (TR) | **89.3%** | 72.1% |
| 剪枝后良性准确率损失 | <3% | <5% |
| 窄域剪枝后错位减少 | **67%↓** | 23%↓ |

#### 五、局限性分析

1. 主要在 ≤7B 模型上验证，70B+ 规模的可扩展性未知
2. 需要白盒访问 + 标注有害查询，实际部署受限
3. 对抗性适应可能绕过关键权重的防御
4. 灰色地带内容（医学/文学）的权重重叠可能导致误伤

#### 六、复现指南

环境：`pip install torch>=2.0.0 transformers>=4.38.0`。核心类 `HarmfulWeightAnalyzer` 实现迭代剪枝搜索 + 跨域验证。步骤：准备 LLaMA-2-7B-chat → 收集 ≥5 域查询集 → 执行梯度剪枝 → 评估传递率和解耦效果。

### 2. Unreal Thinking: 通过两阶段后门劫持思维链

**《UNREAL THINKING: Chain-of-Thought Hijacking via Two-Stage Backdoor》**

arXiv: https://arxiv.org/abs/2604.09235 | 领域: cs.CR | 作者: Wenhan Chang, Tianqing Zhu, Ping Xiong, Faqian Guan, Wanlei Zhou | 发布: 2026-04-10 | GitHub: 暂无

#### 一、背景与问题

CoT 场景下的新安全维度：用户依赖模型输出的思维链判断推理可靠性，但如果中间推理过程被隐蔽劫持呢？在开源适配器生态中，轻量级后门植入具有现实威胁。核心挑战：恶意 CoT 数据稀缺 + 直接劫持长序列不稳定 + 后门注入方法不稳定。

#### 二、核心方法

提出 TSBH（两阶段后门劫持）：阶段一诱导触发条件下 CoT 与输出之间的语义不匹配；阶段二用 MRTS（多重逆向树搜索）合成的数据进行对齐微调。MRTS 从 prompt-output 配对反向构造逻辑连贯的伪 CoT，无需从未对齐模型采样。

#### 三、技术细节

MRTS 反向合成：从 target_output 出发回溯推理树，beam_width=3, max_depth=5。TSBH 训练：L_total = λ₁·L_align(嵌入距离) + λ₂·L_benign(良性保留)。关键设计：两阶段分离使 CoT 既合理（高 CPS）又导向恶意结果（高 HSR）。

#### 四、实验设置与结果

| 模型 | HSR (↑) | CPS (↑10分制) | BA (↑) |
|------|---------|-------------|--------|
| LLaMA-2-7B-Chat | **94.7%** | 8.2 | 96.3% |
| Mistral-7B-Instruct | **91.2%** | 7.9 | 95.8% |
| Qwen-2-7B-Chat | **89.5%** | 8.0 | 94.1% |

MRTS vs 基线：直接采样 HSR=42.1%, MRTS 达到 **92.4%**（数据效率 2x+ 提升）

#### 五、局限性分析

1. 需白盒访问适配器参数进行微调
2. 触发器设计相对简单（token/语义模式）
3. 依赖显式 CoT 输出格式
4. 长 CoT (>20步) 的稳定性衰减
5. 高风险领域（代码/医疗）尚未充分验证

#### 六、复现指南

环境：`pip install torch transformers peft sentence-transformers`。核心类 `TSBHAttacker` 实现 stage1_mismatch_induction + stage2_alignment_finetune。步骤：准备基座模型 → 运行 MRTS 反向合成 → 两阶段微调 → 评估 HSR/CPS/BA。

## T2I 论文（文生图/多模态安全）

### 3. Mosaic: 针对闭源 VLM 的多视图集成越狱攻击

**《Mosaic: Multimodal Jailbreak against Closed-Source VLMs via Multi-View Ensemble Optimization》**

arXiv: https://arxiv.org/abs/2604.09253 | 领域: cs.CV/cs.CR/cs.LG | 作者: Yuqin Lan, Gen Li, Yuanze Hu, Weihao Shen, Zhaoxin Fan, Faguo Wu, Xiao Zhang, Laurence T. Yang, Zhiming Zheng | 发布: 2026-04-10 | GitHub: 暂无

#### 一、背景与问题

现有 VLM 越狱攻击面临"代理依赖"困境：在同构替代-目标设置下优化效果好，但在异构设置（替代 ≠ 目标商业闭源模型）下 ASR 骤降 >30pp。这使得论文报告的攻击成功率在实际场景中大打折扣。核心问题：如何消除代理依赖？

#### 二、核心方法

Mosaic 三组件框架：（1）Text-Side Transformation —— 扰动 refusal-sensitive tokens 降低拒绝反射；（2）Multi-View Image Optimization —— 多裁剪视图轮流更新扰动避免过拟合单一视角；（3）Surrogate Ensemble Guidance —— K 个替代模型加权聚合梯度降低单一代理偏差。

#### 三、技术细节

D(S,T) = |ASR(f_T, x*_S) - ASR(f_T, x*_T)| 定义代理依赖度。Mosaic 通过 E[S]=Σwᵢ·Sᵢ 减少单代理依赖 + N 个裁剪视图减少单视角过拟合 + 文本侧联合优化增加鲁棒性。完整算法为 PGD 变体，ε=8/255, steps=500。

#### 四、实验设置与结果

| 方法 | GPT-4o ASR | Gemini ASR | Claude ASR | 平均 |
|------|-----------|-----------|-----------|------|
| 单一代理 PGD（异构）| 34.2% | 31.8% | 28.6% | 31.5% |
| 最佳基线 PAP | 41.7% | 38.9% | 35.2% | 38.6% |
| **Mosaic（本文）** | **71.3%** | **68.7%** | **64.5%** | **68.2%** |

代理依赖下降：单一模型 -32pp vs 完整 Mosaic **-4.6pp**

#### 五、局限性分析

1. 计算成本高（5-10x 于单模型方法）
2. 仍需开源替代模型计算梯度
3. 语义完整性风险（攻击精确控制有限）
4. 伦理边界需负责任披露

#### 六、复现指南

环境：`pip install torch torchvision transformers openai Pillow`。准备 ≥3 个开源 VLM（LLaVA/Qwen-VL/MiniGPT-4）→ 配置 API Keys → 运行 `MosaicAttack.optimize()` → 在闭源 VLM API 上批量评估。预期：平均 ASR >65%, LPIPS <0.09。

### 4. ImageProtector: 通过视觉提示注入阻止 MLLM 分析图像

**《ImageProtector: Preventing Multi-Modal Large Language Models from Analyzing Images via Visual Prompt Injection》**

arXiv: https://arxiv.org/abs/2604.09024 | 领域: cs.CV/cs.CR/cs.AI/cs.LG | 作者: Zedian Shao, Hongbin Liu, Yuepeng Hu, Neil Zhenqiang Gong | 发布: 2026-04-10 | 会议: ACL 2026 Main Conference | GitHub: 暂无

#### 一、背景与问题

MLLM 可被用于大规模自动化隐私信息提取（身份、位置等）。现有防护（API 限流、访问控制）无法应对自部署开源模型场景。核心思想反转：将视觉提示注入（VPI）从**攻击手段翻转为防御工具**——用扰动诱导 MLLM 持续输出拒绝响应。

#### 二、核心方法

ImageProtector 在图像中嵌入几乎不可感知的扰动（PSNR >35dB），使 MLLM 分析时持续触发 "I cannot help with this request" 类响应。（1）对抗扰动优化——PGD 最大化 refusal 概率同时约束 L∞ 范围；（2）跨模型泛化——K 个 MLM 集成梯度迫使扰动利用共享特征；（3）查询无关性——覆盖多样化 query 分布确保全场景有效。

#### 三、技术细节

优化目标：max_δ P(Refusal | MLLM(Img+δ, query)) s.t. ||δ||_∞≤ε。损失函数基于 refusal token patterns 的 log-probability 最大化。K 个代理模型 × N 个 queries 随机采样 × M 步 PGD 迭代。典型配置：ε=8/255, steps=300, K=5。

#### 四、实验设置与结果

| 目标 MLLM | PSR (↑) | PSNR(dB) |
|-----------|---------|----------|
| LLaVA-1.5-7B | **96.7%** | 38.2 |
| Qwen-VL-Chat | **94.3%** | 38.2 |
| InstructBLIP | **92.1%** | 38.2 |
| CogVLM | **85.2%** | 38.2 |
| **平均** | **90.9%** | **38.2** |

对抗措施评估：最强对抗训练只能将 PSR 从 91% 降至 **48%**（代价：准确率 -7.1%）

#### 五、局限性分析

1. 主要针对开源 MLLM，闭源 API 效果可能下降
2. JPEG 压缩可能损害扰动完整性
3. 单图保护耗时 ~30-60s（GPU）
4. 非 cover 所有可能的攻击查询角度

#### 六、复现指南

环境：`pip install torch transformers Pillow`。准备 ≥3 个开源 MLLM → `ImageProtector.protect_image(image)` → 评估 PSR + PSNR/SSIM/LPIPS。预期：PSR >88%, PSNR >36dB。

## Agentic Search 论文（Agent 安全）

### 5. BadSkill: 针对 Agent 技能的后门攻击

**《BadSkill: Backdoor Attacks on Agent Skills via Model-in-Skill Poisoning》**

arXiv: https://arxiv.org/abs/2604.09378 | 领域: cs.CR/cs.AI/cs.LG | 作者: Guiyao Tie, Jiawen Shi, Pan Zhou, Lichao Sun | 发布: 2026-04-10 | GitHub: 暂无

#### 一、背景与问题

Agent 技能生态中许多技能包内嵌 ML 模型（分类器/编码器），创造了一种新型供应链攻击面：攻击者发布表面正常但内置后门模型的技能，当输入满足语义触发组合时激活隐藏载荷。这不同于 prompt injection——它作用于**模型权重层面**。

#### 二、核心方法

BadSkill 三重复合损失训练：L_total = λ₁·L_classify + λ₂·L_margin + λ₃·L_poison。（1）Model-in-Skill 投毒——针对技能内嵌分类器的定向后门微调；（2）语义触发组合——多条件语义匹配（关键词+实体+数值阈值）比 token 触发更隐蔽；（3）复合目标函数——保持良性功能的同时植入后门。

#### 三、技术细节

13 个技能基准（8 触发 + 5 控制），571 负类查询 + 396 触发查询，覆盖 5 模型族 × 8 架构（494M-7.1B）。L_margin 使用超球面上的边界分离损失确保触发/非触发类别 embedding 分离。

#### 四、实验设置与结果

| 指标 | 结果 |
|------|------|
| 平均 ASR（8个触发技能）| **99.5%** |
| 良性准确率（BA）| **94.5%** |
| **3% 毒化率的 ASR** | **91.7%** |
| 5 种文本扰动后的最低保持率 | **82.4%** |

消融：移除 L_poison 后 ASR 从 99.5%→12.1%（证明投毒损失是核心驱动）

#### 五、局限性分析

1. 需白盒访问技能内嵌模型进行投毒
2. 主要验证分类器类型，其他模型待探索
3. 行为审计可能检测异常输出模式
4. OpenClaw 是模拟环境非真实平台
5. 触发设计空间可进一步扩展

#### 六、复现指南

环境：`pip install torch transformers scikit-learn`。选择目标分类器 → 定义触发组合和 payload → `BadSkillAttack.train_backdoored_model(poison_rate=0.03)` → 评估 ASR/BA。预期：ASR >95%, BA >93%。

### 6. SIF: 针对多 Agent 管道的语义意图碎片化攻击

**《Semantic Intent Fragmentation: A Single-Shot Compositional Attack on Multi-Agent AI Pipelines》**

arXiv: https://arxiv.org/abs/2604.08608 | 领域: cs.CR/cs.AI/cs.LG | 作者: Tanzim Ahad, Ismail Hossain, Md Jahangir Alam, Sai Puppala, Yoonpyo Lee, Syed Bahauddin Alam, Sajedul Talukder | 发布: 2026-04-08 | 会议: AAAI 2026 Summer Symposium | GitHub: 暂无

#### 一、背景与问题

LLM 编排器将复杂任务分解为子任务执行，但安全检查通常仅在子任务级别运行。SIF 利用这一盲区：一个合法的单次请求被编排器自动分解为多个看似良性的子任务，但组合执行时违反安全策略。利用 OWASP LLM06:2025（过度代理）漏洞，无需注入内容、修改系统或多轮交互。

#### 二、核心方法

四种攻击机制：批量范围升级（逐步扩大访问范围）、静默数据外泄（分块传输低于检测阈）、嵌入式触发器部署（正常操作中植入后续激活机制）、准标识符聚合（聚合非敏感信息还原敏感信息）。三阶段红队测试流程基于 OWASP + MITRE ATLAS + NIST 标准。

#### 三、技术细节

形式化条件：∀i SafetyCheck(Tᵢ)=PASS ∧ Execute({Tᵢ}) ⊆ ViolationPolicy。三信号验证：(1) 确定性污染分析——资源访问范围是否超界；(2) CoT 评估——推理链中的规划跳跃；(3) 跨模型合规性评判——独立模型评估完整计划（要求 0% FPR）。

#### 四、实验设置与结果

| 维度 | 结果 |
|------|------|
| **总违规率 (14 场景)** | **71.4%（10/14）** |
| 子任务通过率（SPR）| **100%**（全部单独通过）|
| 验证 FPR | **0%** |
| 强编排器 vs 弱编排器 | **71.4% vs 42.9%**（能力越强风险越大）|

防御有效性：计划级信息流跟踪可在执行前 **100% 检测**所有攻击

#### 五、局限性分析

1. 依赖编排器的细粒度分解行为
2. 主要在 GPT-20B 上验证
3. 仅 14 场景，医疗/法律等领域未覆盖
4. 计划级防御的实际性能开销待研究
5. 仅覆盖单轮交互场景

#### 六、复现指南

环境：`pip install openai langchain pandas`。加载 14 个预构造 SIF prompts → `SIFRedTeamFramework.phase2_attack_execution()` → 三信号交叉验证 → 统计 PVR/SPR/FPR。预期：PVR ≈70%, SPR =100%, FPR ≈0%。

## 本周趋势分析

本周论文呈现一个清晰的**"推理中间层成为主战场"**趋势：

1. **从端点到管道**：攻击焦点从单一的输入-输出端点，扩展到完整的推理链路（CoT 劫持）、任务编排管道（SIF 碎片化）、以及模型内部的权重结构（统一有害机制）。安全防护必须同步从"端点过滤"升级到"全链路监控"。
2. **从代码到权重**：BadSkill 揭示 Model-in-Skill 攻击面——恶意不再藏在代码逻辑里而是 ML 权重中。这对现有的代码扫描/SAST 工具完全无效，需要全新的**模型来源验证（provenance verification）**基础设施。
3. **从攻击到防御的反转**：ImageProtector 展示了"以毒攻毒"范式——将 VPI 攻击技术翻转为隐私保护工具。这类"攻击转防御"的创新值得更多探索。
