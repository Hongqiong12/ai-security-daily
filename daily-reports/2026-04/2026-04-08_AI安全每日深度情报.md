## 🔥 今日核心发现

### 1. Agent 安全面临"工具链劫持"新威胁
**Back-Reveal** 展示了一种通过微调后门实现的系统性数据窃取攻击：被植入后门的 Agent 利用合法的工具调用通道（内存访问、检索工具），在多轮交互中渐进式泄露用户数据。关键洞察：**工具调用是 Agent 与外界交互的合法通道，也是最难监控的攻击向量**。

### 2. 向量数据库存在系统性几何缺陷
**Black-Hole Attack** 揭示了一个令人震惊的发现：高维嵌入空间中质心附近几乎为空，在此注入少量恶意向量即可污染 **99.85%** 的 top-10 检索结果。这对所有依赖向量检索的 RAG/Agent 系统都是系统性威胁。现有枢纽性缓解方法要么严重降低检索质量，要么保护有限。

### 3. AI 生成代码"默认不安全" — 形式化证据
**Broken by Default** 使用 Z3 SMT 求解器对 7 个前沿 LLM 的 3,500 个代码产物进行形式化安全验证，发现 **55.8%** 含漏洞（GPT-4o 高达 62.4%）。更令人担忧的是：6 种工业安全工具组合遗漏了 **97.8%** 的形式化证明漏洞；显式安全指令仅降低 4 个百分点。**LLM 在审查模式下能识别 78.7% 的漏洞，但在生成模式下仍产出含漏洞代码——"知行分离"问题。**

### 4. T2I 文本渲染能力 = 新攻击面
**Etch** 首次系统化"铭文式越狱"（inscriptive jailbreak）：利用 T2I 模型的文本渲染能力，生成视觉无害但包含有害文字内容的图像（如欺诈文件）。三层分解框架在 7 个模型上平均 **65.57%** 攻击成功率，峰值达 **91%**。**这一攻击类别被现有安全过滤器完全忽视。**

## 🧭 趋势洞察

### 1. Agent 安全进入"工具链攻防"阶段
今日 Back-Reveal（工具链劫持）和 BodhiPromptShield（跨阶段隐私防护）构成一组攻防对照。Agent 的工具调用能力既是核心价值，也是核心攻击面。**工具调用的安全监控将成为 Agent 框架的标配能力**。

### 2. RAG 基础设施（向量数据库）面临系统性安全威胁
Black-Hole Attack 揭示的不是某个特定实现的 bug，而是高维嵌入空间的**固有几何缺陷**。这对整个 RAG 生态是系统性风险——从 LangChain 到企业级知识库系统。

### 3. "知行分离"是 LLM 安全的核心难题
Broken by Default 的发现（审查模式识别 78.7% 漏洞 vs 生成模式产出 55.8%）直接指向一个深层问题：LLM 的安全知识并未被有效整合到生成流程中。这一发现对 safety alignment 研究有深远影响。

### 4. T2I 安全对齐存在"文本盲区"
Etch 揭示的铭文式越狱是一个全新攻击类别。随着 T2I 模型文本渲染能力持续提升（DALL·E 3、Flux 等），这一攻击面只会越来越大。
