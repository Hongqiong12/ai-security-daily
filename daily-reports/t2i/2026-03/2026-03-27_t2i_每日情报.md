## 论文一：PromptSan — NSFW 分类器引导的提示净化

**《NSFW-Classifier Guided Prompt Sanitization for Safe Text-to-Image Generation》**

arXiv: https://arxiv.org/abs/2506.18325 | 领域: cs.CV | 作者: Yu Xie, Chengjie Zeng, Lingyun Zhang, Yanwei Fu | 发布: 2025-06-23 | GitHub: 暂无公开代码库

### 一、背景与问题

文本到图像模型（如 Stable Diffusion）强大的生成能力伴随着安全隐患——恶意用户可通过构造特定提示词绕过内置安全过滤，生成色情、暴力等 NSFW 内容。现有防御方案要么修改模型（侵入性强），要么在输出端检测（事后被动），均存在一定局限。

**核心科学问题**：能否在**输入端**对有害提示实施净化，在不修改扩散模型本身的前提下，实现安全防护与生成质量的双重保障？

### 二、核心方法

PromptSan 提出两种互补的提示净化变体：

**PromptSan-Modify（迭代修改式）**
- 利用文本 NSFW 分类器识别有害词汇
- 迭代替换为语义相近的安全表述
- 直至提示通过分类器审查

**PromptSan-Suffix（后缀优化式）**
- 训练一组优化的后缀令牌序列
- 附加到任意有害提示后中和其有害意图
- 同时满足文本 NSFW 分类器和图像 NSFW 分类器的安全要求

### 三、技术细节

PromptSan-Suffix 的优化目标（联合最小化文本和图像有害性）：

$$\min_{S} \mathcal{L}_{text}(P_{harm} \oplus S) + \lambda \cdot \mathcal{L}_{image}(\text{Diffuse}(P_{harm} \oplus S))$$

**双分类器架构**：
- 文本分类器（Text-NSFW）：基于预训练语言模型，对提示打分
- 图像分类器（Image-NSFW）：基于视觉模型，对生成图像打分
- 联合约束确保净化在文本和图像两个层面均有效

### 四、实验设置与结果

测试数据来自 SneakyPrompt、I2P、Ring-A-Bell 等越狱基准；良性数据采用 COCO Captions。

| 方法 | DSR（↑） | FID（↓，良性） | CLIP Score（↑） |
|------|---------|--------------|----------------|
| 无防御 | 0% | 基准 | 基准 |
| SafeGen | 78.3% | +2.1 | -0.05 |
| ESD | 81.5% | +5.3 | -0.12 |
| **PromptSan-Modify（本文）** | **85.7%** | +1.3 | -0.02 |
| **PromptSan-Suffix（本文）** | **89.2%** | +1.8 | -0.03 |

**关键优势**：在同等防御成功率下，对良性图像生成质量影响最小（FID 提升最低）。

### 五、局限性分析

针对专门对抗优化的提示可能失效；语义边界模糊时净化可能损失合理语义；推理延迟有所增加；主要针对英文提示。

### 六、复现指南

```python
# PromptSan-Suffix 核心优化逻辑
def optimize_suffix(harmful_prompt, nsfw_classifier, pipe, 
                    suffix_length=10, num_steps=500):
    suffix_embeddings = torch.randn(suffix_length, 768, requires_grad=True)
    optimizer = torch.optim.Adam([suffix_embeddings], lr=0.01)
    
    for step in range(num_steps):
        prompt_with_suffix = harmful_prompt + " " + decode_embeddings(suffix_embeddings)
        text_loss = nsfw_classifier(prompt_with_suffix).logits[0, 1]
        if step % 50 == 0:
            image = pipe(prompt_with_suffix).images[0]
            image_loss = compute_image_nsfw_loss(image)
        total_loss = text_loss + 0.5 * image_loss
        optimizer.zero_grad()
        total_loss.backward()
        optimizer.step()
    
    return suffix_embeddings.detach()
```

## 论文二：Modular LoRA — 防微调安全对齐崩溃

**《Safety Alignment Backfires: Preventing the Re-emergence of Suppressed Concepts in Fine-tuned Text-to-Image Diffusion Models》**

arXiv: https://arxiv.org/abs/2412.00357 | 领域: cs.AI / cs.CV | 作者: Sanghyun Kim, Moonseok Choi, Jinwoo Shin, Juho Lee | 机构: KAIST 等 | 发布: 2024-11-30 | GitHub: 暂无公开代码库

### 一、背景与问题

T2I 扩散模型通过概念擦除（ESD、MACE、UCE 等）实现安全对齐，但研究发现：在**良性数据集**上进行微调（DreamBooth、LoRA 等）时，原本被抑制的有害内容会意外重新出现——这一问题被称为"微调越狱"（Fine-tuning Jailbreak），在 T2I 领域尚未得到充分研究。

**核心科学问题**：如何设计一种微调方法，使模型在学习新任务时，不破坏安全对齐建立的有害概念抑制？

### 二、核心方法

**Modular LoRA** 将 LoRA 参数空间物理拆分为两个独立模块：

- **Safety LoRA（$\Delta W_{safe}$）**：安全对齐阶段训练，编码有害概念抑制；微调时**冻结**
- **Task LoRA（$\Delta W_{task}$）**：新任务微调时训练；与 Safety LoRA **正交**，不干扰安全参数

推理时合并：$W_{final} = W_{pretrained} + \Delta W_{safe} + \Delta W_{task}$

### 三、技术细节

**正交约束**（防止任务梯度覆盖安全方向）：

$$A_{task}^{(t+1)} = A_{task}^{(t)} - \eta \nabla \mathcal{L}_{task} - \text{proj}_{A_{safe}} \left(\eta \nabla \mathcal{L}_{task}\right)$$

每次梯度更新时，去除平行于安全 LoRA 权重方向的分量，从而在参数空间层面隔离安全知识。

**LoRA 参数化**：对权重矩阵 $W \in \mathbb{R}^{m \times n}$，$\Delta W = BA$，其中 $B \in \mathbb{R}^{m \times r}$，$A \in \mathbb{R}^{r \times n}$，秩 $r \ll \min(m, n)$。

### 四、实验设置与结果

测试于 ESD / MACE / UCE 等概念擦除方法之后，使用 DreamBooth-style 良性数据微调，评估裸露等有害概念的重现率（HGR）。

| 微调方法 | HGR（裸露，↓） | CLIP Score（↑） |
|---------|--------------|----------------|
| 标准 LoRA 微调 | 67.3% | 0.312 |
| Adapter 微调 | 51.2% | 0.298 |
| **Modular LoRA（本文）** | **8.4%** | **0.309** |

**消融**：无正交约束（45.1% HGR）→ 仅冻结（23.7%）→ 完整 Modular LoRA（8.4%）

### 五、局限性分析

秩 $r$ 的选择影响容量权衡；正交约束为近似实现，长期多轮微调后效果可能衰减；多概念同时擦除时 Safety LoRA 容量可能不足。

### 六、复现指南

```python
class ModularLoRA:
    def orthogonalize_gradient(self, grad, safety_weights):
        """将梯度正交化于 Safety LoRA 权重方向"""
        dot = torch.sum(grad * safety_weights)
        norm = torch.sum(safety_weights ** 2)
        return grad - (dot / norm) * safety_weights
    
    def finetune_step(self, batch, optimizer):
        loss = self.compute_loss(batch)
        optimizer.zero_grad()
        loss.backward()
        # 对每个 Task LoRA 参数施加正交投影
        for task_p, safety_p in zip(
            self.task_lora.parameters(), 
            self.safety_lora.parameters()
        ):
            if task_p.grad is not None:
                task_p.grad.data = self.orthogonalize_gradient(
                    task_p.grad.data, safety_p.data
                )
        optimizer.step()
```
