## 🔴 攻击类论文

### Paper 1: MacPrompt — 混合语引导的文生图越狱攻击

**arXiv**: https://arxiv.org/abs/2601.07141
**作者**: Xi Ye, Yiwen Liu, Lina Wang, Run Wang, Geying Yang, Yufei Hou, Jiayi Yu
**会议**: AAAI 2026（已接收）
**提交日期**: 2026-01-12

#### 1. 背景与问题

T2I 模型（如 Stable Diffusion、DALL-E、Midjourney）配备了安全过滤器和概念移除技术，但现有越狱攻击存在严重局限：

- **同义词替换类**（如 SneakyPrompt）：效果有限，且易被语义相似度过滤
- **提示混淆类**（如 MMA-Diffusion）：破坏语义连贯性，生成质量差

**关键洞见**：安全过滤器主要基于单一语言的词汇/语义特征，对**跨语言字符级组合**的鲁棒性不足。

#### 2. 核心方法

**MacPrompt** 通过**跨语言字符级重组**（Cross-lingual Character-level Recombination）构建**混合语（macaronic）对抗性提示**：

```
MacPrompt 攻击流程：
┌──────────────────────────────────────────────────────────────┐
│  输入: 目标有害词汇 w（如 "nude"）                            │
├──────────────────────────────────────────────────────────────┤
│  1. 多语言字符映射                                           │
│     - 识别视觉相似的多语言字符对                               │
│       e.g., 拉丁 "e" ↔ 西里尔 "е", "u" ↔ "υ"（希腊）       │
│     - 构建字符替换词典 D_lang                                 │
├──────────────────────────────────────────────────────────────┤
│  2. 有害词汇字符级重组                                       │
│     - 对有害词汇的部分字符进行跨语言替换                       │
│     - 保持视觉外观和语义信息                                  │
│     - 规避过滤器的词汇/N-gram 匹配                            │
├──────────────────────────────────────────────────────────────┤
│  3. 语义一致性验证                                           │
│     - CLIP 语义相似度检验（≥0.92 阈值）                       │
│     - 排除语义漂移的候选提示                                  │
└──────────────────────────────────────────────────────────────┘
```

#### 3. 技术细节

**字符替换矩阵**（部分示例）：

| 原始字符（Latin）| 替换字符 | 来源语言 | 视觉相似度 |
|----------------|---------|---------|-----------|
| a | а | 西里尔 | 0.98 |
| e | е | 西里尔 | 0.97 |
| o | о | 西里尔 | 0.96 |
| u | υ | 希腊 | 0.91 |
| p | р | 西里尔 | 0.95 |

**攻击有效性条件**：

$$
\text{Valid}(p_{\text{adv}}) \iff \text{CLIP\_sim}(p_{\text{adv}}, p_{\text{orig}}) \geq 0.92 \wedge \text{Filter}(p_{\text{adv}}) = \text{Pass}
$$

**语义保持率**：最高达 **0.96**（与原始有害提示相比）。

#### 4. 实验设置与结果

**实验设置**：
- 目标模型：SD v1.5, SD v2.1, SD-XL, DALL-E 3, Midjourney v6
- 安全机制：Stable Diffusion Safety Checker, OpenAI Moderation, NSFW过滤器
- 评测指标：安全过滤器绕过率（Bypass Rate）、攻击成功率（ASR）

#### 5. 局限性分析

1. **依赖视觉相似字符集**：对于某些语言中没有视觉相似对应字符的词汇，攻击效果下降
2. **字体感知防御**：若安全过滤器使用 Unicode 规范化处理输入，攻击可能被识别
3. **生成质量损失**：当替换字符较多时，语义相似度下降，T2I 模型的理解受限

**核心代码片段**：
```python
import unicodedata
from clip import CLIP

# 跨语言字符映射字典
CROSS_LINGUAL_MAP = {
    'a': ['а', 'α'],  # 西里尔, 希腊
    'e': ['е', 'ε'],  # 西里尔, 希腊
    'o': ['о', 'ο'],  # 西里尔, 希腊
    'p': ['р', 'ρ'],  # 西里尔, 希腊
    'c': ['с', 'ϲ'],  # 西里尔, 希腊
    'x': ['х', 'χ'],  # 西里尔, 希腊
}

def macprompt_attack(
    target_phrase: str,
    min_semantic_sim: float = 0.92,
    max_replacements: int = 3
) -> list[str]:
    """
    MacPrompt 跨语言字符级攻击

    Args:
        target_phrase: 目标有害词汇/短语
        min_semantic_sim: CLIP 语义相似度最低要求
        max_replacements: 最大字符替换数量

    Returns:
        有效对抗性提示列表
    """
    clip_model = CLIP.load("ViT-L/14")
    valid_prompts = []

    # 生成候选替换方案
    candidates = generate_macaronic_candidates(
        target_phrase,
        CROSS_LINGUAL_MAP,
        max_replacements
    )

    # 语义过滤
    for candidate in candidates:
        sim = clip_model.text_similarity(candidate, target_phrase)
        if sim >= min_semantic_sim:
            valid_prompts.append(candidate)

    return valid_prompts


def generate_macaronic_candidates(
    phrase: str,
    char_map: dict,
    max_replacements: int
) -> list[str]:
    """生成混合语候选提示"""
    candidates = []
    words = phrase.lower().split()

    for word in words:
        for char, replacements in char_map.items():
            if char in word:
                for rep in replacements:
                    modified = word.replace(char, rep, 1)
                    candidates.append(
                        phrase.replace(word, modified)
                    )

    return candidates[:50]  # 返回前50个候选


# 使用示例
adversarial_prompts = macprompt_attack(
    "nude woman",
    min_semantic_sim=0.92,
    max_replacements=2
)
print(f"生成 {len(adversarial_prompts)} 个有效对抗提示")
```

**完整攻击流程**：

## 📈 今日趋势分析

### T2I 攻击新方向

1. **跨语言攻击成新范式**：MacPrompt 揭示了语言多样性对安全机制的根本挑战。大多数现有安全过滤器基于单语言词汇表训练，跨语言字符混淆开辟了一类高成功率的新型攻击面。

2. **视觉相似字符的"语义保留"特性**：Unicode 多语言字符集在视觉层面高度相似，使得 T2I 模型能够正确理解语义，而安全过滤器则因字符级不匹配而失效——这是过滤器设计的系统性盲区。

### 防御建议

- **Unicode 规范化前处理**：在输入传入安全过滤器前，统一 Unicode 规范化（如 NFKC）
- **多语言字符白名单**：限制输入中允许的 Unicode 范围
- **CLIP 语义安全检查**：基于语义而非词汇的安全过滤器对此类攻击更鲁棒
