## 🎨 论文一：边界感知提示优化验证 T2I 模型身份

**ArXiv**: 2603.26328 | **提交**: 2026-03-27 | **接收**: CVPR 2026 (Findings)

### 问题背景
第三方平台集成 T2I 模型 API 时，可能虚假宣称使用 Stable Diffusion、DALL·E 等知名模型，实际运行劣质替代品。用户如何验证自己使用的服务是否名副其实？

### 方法：BPO（边界感知提示优化）

**核心洞察**：不同 T2I 模型在**语义边界区域**的行为差异最为显著。

```
语义边界示例（"柯基犬" ↔ "百吉饼"）：
  目标模型边界区：提示 "a round furry object" →
    Stable Diffusion v1.5: 50% 生成柯基，50% 生成百吉饼（高度不稳定）
    SD 2.1: 稳定生成一类（与 v1.5 差异显著）
```

**BPO 流程**：
1. 通过提示插值探测目标模型的语义边界
2. 优化提示使其落在目标模型的不稳定区
3. 生成模型特异性验证提示集
4. 比较 API 输出分布，判断声称的模型是否属实

**无参考设计**：不需要收集其他模型输出作为参照，仅用目标模型自身的推理结果。

### 实验结果
在 5 个主流 T2I 模型和 4 种基线方法上，BPO 在**验证准确率**上显著优于所有现有方法。

### 应用价值
| 使用场景 | 受益方 |
|---------|-------|
| API 版权审计 | 模型所有者（如 Stability AI）|
| 自主验证服务真实性 | 企业用户 |
| 第三方检测 | 监管机构 |

## 🎨 论文二：SALMUBench——多模态关联级遗忘的严格评测

**ArXiv**: 2603.26316 | **提交**: 2026-03-27 | **接收**: CVPR 2026

### 问题背景
CLIP 等多模态对比学习模型在预训练时可能学到敏感的"人物-属性"关联（如特定人物与违法行为的绑定）。**机器遗忘**可在不重新训练的前提下删除此类关联，但现有遗忘方法**缺乏严格的细粒度评测**。

### 基准设计

**数据集**：6 万条合成"人物-属性"敏感关联对

**双模型对照**：
```
受损模型（Corrupted Model）:
  训练数据 = 4 亿对保留数据 + 6 万敏感关联
  
干净模型（Clean Model）:
  训练数据 = 4 亿对保留数据（仅）
  
差异完全来自 6 万敏感关联 → 可精确隔离遗忘效果
```

**评估指标**：
- **遗忘率**：敏感关联在遗忘后的检索成功率（低 = 好）
- **保留率**：无关知识被保留的程度（高 = 好）
- **过度泛化检测**：是否误删超出预期的关联

### 关键发现

现有遗忘方法呈现两极化困境：

```
遗忘不足型:
  敏感关联残留率 > 50%
  → 敏感信息仍可从模型中提取
  
过度泛化型:
  遗忘率低 ✓
  保留率低 ✗ → 大量无关知识被误删
  → 模型实用性严重受损
```

**理想方法（目前不存在）**：低遗忘率 + 高保留率同时达成。
