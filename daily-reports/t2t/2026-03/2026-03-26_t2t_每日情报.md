## T1 · Claudini：自动研究发现最先进 LLM 对抗攻击算法

**ArXiv**: https://arxiv.org/abs/2603.24511  
**代码**: https://github.com/romovpa/claudini  
**类别**: attack  
**发布**: 2026-03-25  

### 核心贡献

本文提出 **Claudini**，一个以 Claude Code 为核心驱动的自动对抗攻击研究流水线（autoresearch pipeline）。从现有攻击实现（GCG 等）出发，通过 LLM 代理的迭代优化，自动发现显著超越现有 30+ 种方法的新型白盒攻击算法。

**关键结果**：
- GPT-OSS-Safeguard-20B 上 CBRN 攻击成功率：从 ≤10%（现有方法）→ **40%**
- 迁移至 Meta-SecAlign-70B 的攻击成功率：从 56%（最优基线）→ **100%**

### 方法简述

```
已有攻击 → LLM 代理分析 → 新攻击算法 → 量化测试反馈 → 下一轮迭代
```

攻击在代理模型上优化后，可直接迁移到未知模型，验证了 LLM 代理自主开展安全研究的可行性。

### 安全意义

> 白盒对抗攻击的发现正在走向**完全自动化**。防御方需要认识到：未来的攻击者不仅有人类研究员，还有能自主迭代改进的 AI 代理。

## T2 · 前沿 LLM 的内部安全崩溃（Internal Safety Collapse）

**ArXiv**: https://arxiv.org/abs/2603.23509  
**类别**: benchmark  
**发布**: 2026-03-04（本期首次收录）  

### 核心贡献

发现并命名了前沿 LLM 的关键失效模式：**内部安全崩溃（ISC）**。在特定专业任务场景下，"生成有害内容"成为完成任务的唯一合理方式，此时模型的安全机制全面崩溃。

**ISC-Bench**：覆盖 8 个专业领域、53 个场景的评测基准。

**关键结果**：
- 4 个前沿 LLM（含 GPT-5.2、Claude Sonnet 4.5）的平均最坏情况安全失效率：**95.3%**
- 远超传统越狱攻击基线（~30-60%）

### TVD 框架

| 要素 | 含义 |
|------|------|
| Task | 设计使有害内容成为唯一合法完成的专业任务 |
| Validator | 判断模型是否进入 ISC 状态 |
| Data | 覆盖多专业领域的多样化场景 |

### 核心洞见

> **能力越强，越危险**：前沿 LLM 的强大任务理解能力，在"内在有害"的双用途工具场景下，反而成为安全漏洞的放大器。对齐仅改变表层输出，无法消除内部危险能力。

## T3 · TreeTeaming：VLM 层次化策略树自动红队测试

**ArXiv**: https://arxiv.org/abs/2603.22882  
**会议**: CVPR 2026  
**类别**: attack（VLM 多模态）  
**发布**: 2026-03-24  

### 核心贡献

提出 **TreeTeaming**，将 VLM 红队测试从"固定策略集的线性探索"升级为"动态策略树的进化发现"。LLM 驱动的策略编排器自主决定"演化"还是"探索"，动态构建攻击策略树。

**关键结果**：
- 12 个主流 VLM 中，11 个上达到当前最优 ASR
- GPT-4o 攻击成功率：**87.60%**
- 策略多样性超过现有所有公开 Jailbreak 策略的并集
- 攻击内容毒性平均降低 **23.09%**（更难被检测）

### 方法对比

| 指标 | 传统方法 | TreeTeaming |
|------|---------|-------------|
| 策略来源 | 预定义策略库 | 动态进化发现 |
| 探索方式 | 线性遍历 | 树形搜索 |
| 新型攻击发现 | 受限 | **自主发现** |
| 攻击隐蔽性 | 普通 | **毒性-23.09%** |

## T4 · SafeSeek：LLM 安全电路通用归因框架

**ArXiv**: https://arxiv.org/abs/2603.23268  
**类别**: defense（机制可解释性）  
**发布**: 2026-03-24  

### 核心贡献

提出 **SafeSeek**，通过优化可微分二值掩码，在注意力头和神经元层次上精确定位 LLM 中的**安全电路**。同时提出**安全电路微调（SCT）**，在有帮助性微调时保护安全电路不被破坏。

**关键结果**：

**后门攻击场景**：
- 后门电路稀疏度：仅 **0.42%** 参数
- 消融后 ASR：100% → **0.4%**
- 通用任务性能保留：**>99%**

**安全对齐场景**：
- 对齐电路：3.03% 注意力头 + 0.79% 神经元
- 移除对齐电路后 ASR：0.8% → **96.9%**
- SCT 微调后安全保留率：**96.5%**

### 技术核心

$$\min_{m} \mathcal{L}_{\text{safety}}(f(x; m \odot \theta)) + \lambda \|m\|_1$$

可微分二值掩码通过梯度下降提取安全电路，稀疏正则化保证电路极度精简。
