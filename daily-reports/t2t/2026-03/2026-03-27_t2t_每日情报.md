## 论文一：SecureBreak — 面向安全模型的基准数据集

**《SecureBreak -- A dataset towards safe and secure models》**

arXiv: https://arxiv.org/abs/2603.21975 | 领域: cs.CR / cs.AI / cs.CL | 作者: Marco Arazzi, Vignesh Kumar Kembu, Antonino Nocera | 发布: 2026-03-23 | GitHub: 暂无公开代码库

### 一、背景与问题

大型语言模型的安全对齐是其部署的必要条件，但越狱（Jailbreaking）和提示注入（Prompt Injection）等攻击不断绕过现有对齐机制。学界已认识到需要一个独立于对齐训练的"最终防御层"——一个基于高质量数据的有害内容检测器。SecureBreak 正是为训练和评估这类检测器而设计的基准数据集。

**核心科学问题**：构建一个高质量、多类别、保守标注的 LLM 输出安全数据集，使微调后的 LLM 能够有效检测安全对齐残留弱点导致的有害输出。

### 二、核心方法

SecureBreak 数据集的设计核心包含三个层面：

- **双重功能**：同时支持（a）训练阶段的安全对齐质量反馈，以及（b）部署阶段的生成后过滤
- **保守标注策略**：边界案例倾向有害判定，确保安全覆盖率优先于精确率
- **多风险类别**：涵盖暴力、仇恨言论、自我伤害、非法活动等主要危害类别

### 三、技术细节

数据构建采用"原始数据收集 → 自动初筛 → 人工精标（保守策略）→ 跨标注者一致性验证"的四阶段流程。

安全分类器的形式化目标：

$$\mathcal{C}(o) = \begin{cases} 1 & \text{若 } o \in \mathcal{H}（\text{有害集合}） \\ 0 & \text{若 } o \in \mathcal{S}（\text{安全集合}） \end{cases}$$

保守策略使标注偏向 $o \in \mathcal{H}$，降低漏报风险。

### 四、实验设置与结果

| 实验设置 | 主要发现 |
|---------|--------|
| 零样本预训练 LLM | SecureBreak 上性能优于 baseline |
| 微调后 LLM | 有害内容检测性能显著提升 |
| 跨类别评估 | 多危害类别均具备良好覆盖 |

### 五、局限性分析

数据集规模未公开；静态数据集无法应对新型攻击手法；多语言支持情况不明；分布外鲁棒性有待验证。

## 论文二：SFCoT — 更安全的链式推理

**《SFCoT: Safer Chain-of-Thought via Active Safety Evaluation and Calibration》**

arXiv: https://arxiv.org/abs/2603.15397 | 领域: cs.CR / cs.AI | 作者: Yu Pan, Wenlong Yu, Tiejun Wu, Xiaohu Ye, Qiannan Si, Guangquan Xu, Bin Wu | 发布: 2026-03-16 | GitHub: 暂无公开代码库

### 一、背景与问题

链式推理（Chain-of-Thought, CoT）大幅提升了 LLM 的推理能力，但也为越狱攻击打开了新窗口：攻击者可以在中间推理步骤中逐步引导模型产生有害输出，而现有防御仅在最终输出阶段进行事后过滤，忽视了推理链过程中的风险积累。

**核心科学问题**：能否在推理链的每个中间步骤实时检测不安全推理路径，并通过动态干预从源头阻止有害输出？

### 二、核心方法

SFCoT 框架由三个核心组件构成：

1. **三层安全评分系统**：从语义安全、意图合规、行为风险三维度对每个推理步骤 $s_t$ 打分
2. **多视角一致性验证**：从多个视角交叉验证，避免单点误判
3. **动态干预模块**：当安全分低于阈值时，触发语义重写、路径重定向或推理终止

### 三、技术细节

综合安全评分公式：

$$\sigma(s_t) = \alpha \cdot \sigma_{sem}(s_t) + \beta \cdot \sigma_{intent}(s_t) + \gamma \cdot \sigma_{risk}(s_t)$$

干预触发条件：

$$\text{Intervene}(s_t) = \mathbf{1}[\sigma(s_t) < \tau]$$

其中阈值 $\tau$ 为可调超参数，平衡安全性与效用性。

### 四、实验设置与结果

测试于 HarmBench、AdvBench、JailbreakBench 等主流越狱基准，覆盖 GCG、AutoDAN、PAIR、TAP 等攻击方法。

| 方法 | ASR（↓） | 良性任务准确率 |
|------|---------|--------------|
| 无防御 | 58.97% | 基准 |
| 输出过滤 | 34.21% | 基准 |
| **SFCoT（本文）** | **12.31%** | 接近基准 |

**消融实验**：三层联合评分 > 仅语义层（28.4% ASR）> 去掉多视角一致性（19.7% ASR）

### 五、局限性分析

每步实时评估引入额外计算开销；安全阈值调优敏感；针对评分机制的自适应攻击可能规避检测。

### 六、复现指南

```python
class SFCoTEngine:
    def __init__(self, safety_scorer, threshold=0.7):
        self.scorer = safety_scorer
        self.threshold = threshold
    
    def generate_safe_cot(self, prompt, model, max_steps=10):
        cot_steps = []
        context = prompt
        for step_idx in range(max_steps):
            next_step = model.generate_step(context)
            score = self.scorer.evaluate(next_step)
            if score < self.threshold:
                next_step = self.intervene(next_step, context)
            cot_steps.append(next_step)
            context += f"\nStep {step_idx+1}: {next_step}"
            if model.is_complete(next_step):
                break
        return "\n".join(cot_steps)
```

## 论文三：LLM Agent Security — 形式化安全框架

**《A Framework for Formalizing LLM Agent Security》**

arXiv: https://arxiv.org/abs/2603.19469 | 领域: cs.CR / cs.AI | 作者: Vincent Siu, Jingxuan He, Kyle Montgomery, Zhun Wang, Neil Gong, Chenguang Wang, Dawn Song | 发布: 2026-03-19 | GitHub: 暂无公开代码库

### 一、背景与问题

LLM Agent 的安全性本质上是上下文相关的：同一行为在不同授权场景下，其安全属性截然不同。现有防御策略缺乏上下文感知能力，导致要么过度防御损害效用，要么防御不足留下漏洞。

**核心科学问题**：如何构建一个统一的形式化框架，精确描述 LLM Agent 安全的上下文特性，使攻防定义更加准确？

### 二、核心方法

本文提出四个上下文安全属性：

| 属性 | 名称 | 含义 |
|-----|------|------|
| P1 | 任务对齐（Task Alignment） | 智能体追求且仅追求经授权的目标 |
| P2 | 行为对齐（Action Alignment） | 单步行为服务于当前授权目标 |
| P3 | 来源授权（Source Authorization） | 仅执行来自认证来源的指令 |
| P4 | 数据隔离（Data Isolation） | 信息流遵守权限边界 |

并引入**预言函数（Oracle Functions）** $\mathcal{O} = \{O_1, O_2, O_3, O_4\}$ 在运行时判定属性是否被违反。

### 三、技术细节

攻击类型形式化映射：

| 攻击类型 | 违反属性 |
|---------|---------|
| 直接提示注入 | P3（来源授权） |
| 间接提示注入 | P3 + P1 |
| 越狱 | P1 + P2 |
| 任务漂移 | P1 |
| 内存污染 | P4 + P3 |

效用损失与防御范围的关系：

$$\text{UtilityLoss}(\text{Defense}) \propto |\{c : O_i(a,c) = \text{violated but } a \text{ is actually legitimate}\}|$$

### 四、实验设置与结果

本文为理论框架论文。通过对 AgentDojo、InjecAgent、PrivacyLens、SafeToolUse 等现有工作的形式化映射验证框架覆盖度和表达力。

### 五、局限性分析

预言函数的实际实现面临精确性挑战；多智能体协作场景的信任边界问题尚未解决；动态权限场景下的适用性有限。

### 六、复现指南

```python
class LLMAgentSecurityFramework:
    def __init__(self, authorized_tasks, authorized_sources):
        self.authorized_tasks = authorized_tasks
        self.authorized_sources = authorized_sources
    
    def evaluate_security(self, action, context):
        violations = []
        if not (context.get("task") in self.authorized_tasks):
            violations.append("P1_task_alignment")
        if not (context.get("source") in self.authorized_sources):
            violations.append("P3_source_authorization")
        return violations
```
