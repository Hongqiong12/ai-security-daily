### 1. Trojan-Speak: Bypassing Constitutional Classifiers with No Jailbreak Tax via Adversarial Finetuning
**ArXiv**: [2603.29038](https://arxiv.org/abs/2603.29038) | **类型**: 攻击 | **日期**: 2026-03-30

**核心发现**：针对 Anthropic Constitutional Classifiers 的对抗微调攻击，通过两阶段（课程学习 + GRPO 强化学习）训练隐蔽通信协议，成功规避宪法分类器且几乎无能力退化（"零越狱税"），首次证明微调 API 是顶级 LLM 安全系统的重大攻击面。

**攻击流程**：
1. Phase 1（课程学习）：渐进式训练隐蔽协议，从低风险到高风险（CBRN）内容
2. Phase 2（GRPO 强化学习）：以分类器规避率为奖励，精炼通信协议
3. 最终 ASR：Constitutional Classifiers 规避率 >90%，正常能力保持 >98%

**安全意义** ⚠️ **高**：微调 API 开放性与安全对齐之间存在根本张力，主流 AI 提供商需重新评估微调接口的安全边界。

### 2. GUARD-SLM: Token Activation-Based Defense Against Jailbreak Attacks for Small Language Models
**ArXiv**: [2603.28817](https://arxiv.org/abs/2603.28817) | **类型**: 防御 | **日期**: 2026-03-28

**核心发现**：首个专为小型语言模型（SLMs）设计的激活空间越狱防御方案。通过分析 9 种越狱攻击 × 7 种 SLMs 的激活模式，发现正常/越狱输入在表示空间形成可分离分布，并基于此构建轻量级推理时过滤器，无需修改模型权重。

**关键数据**：
- 防御 ASR：将 SLMs 越狱率从 ~60% 降至 <5%
- 推理延迟增加：<2%
- 跨架构泛化：在 7 种 SLM 上均有效

**实用价值** ✅：为边缘设备 LLM 部署提供首个可行的越狱防御方案，延迟影响极小。

### 3. Security in LLM-as-a-Judge: A Comprehensive SoK
**ArXiv**: [2603.29403](https://arxiv.org/abs/2603.29403) | **类型**: 综述/SoK | **日期**: 2026-03-31

**核心发现**：首个 LLM-as-a-Judge (LaaJ) 安全系统化综述。将 LaaJ 安全风险分为两类：
1. **LaaJ 作为攻击目标**：位置偏见、口语偏见、奖励欺骗等操纵手段
2. **LaaJ 作为攻击工具**：越狱评分、有害内容筛选绕过、RLHF 中毒

**分类体系**：提出 5 维威胁分类（偏见/注入/投毒/对抗/身份欺骗），并系统整理现有攻防文献。

**研究价值** 📚：为 LaaJ 安全研究提供权威参考框架，填补该领域系统化知识整理的空白。

### 4. Architecting Secure AI Agents: Perspectives on System-Level Defenses Against Indirect Prompt Injection Attacks
**ArXiv**: [2603.30016](https://arxiv.org/abs/2603.30016) | **类型**: 防御架构 | **日期**: 2026-03-31

**核心发现**：从系统架构角度重新审视间接提示注入（IPI）防御。提出**特权层级架构**（Privilege Hierarchy Architecture）、**信任区间模型**（Trust Zone Model）和**数据流隔离**（Data Flow Isolation）三层防御框架，认为 IPI 防御的正确抽象层次是系统架构而非单点 LLM 模型。

**架构设计原则**：
- 最小权限原则：工具调用权限细粒度控制
- 来源标记：为所有外部数据添加不可篡改的来源标签
- 指令/数据分离：在 tokenization 层面区分可信指令和不可信数据

**安全意义** ⚠️ **高**：为企业 AI Agent 部署提供系统性安全框架，可与模型级防御互补形成纵深防御。

## 🔍 趋势分析

### 攻击侧新动向
**微调 API 武器化**是本周最重要的攻击趋势。Trojan-Speak 的成功表明：仅依赖模型层面的宪法对齐无法抵御有训练能力的攻击者，API 开放性与安全性之间的张力需要在架构层面解决。

### 防御侧新动向
**系统级防御**（Secure AI Agents）和**资源感知防御**（GUARD-SLM）代表防御思路的两个重要方向：前者从架构角度解决 IPI 问题，后者填补边缘部署的安全空白。

### 评估体系演进
LaaJ SoK 提示 AI 安全评估本身已成为研究对象——评估工具（LLM judge）的可信度直接影响安全研究结论的有效性，这一元层面问题值得持续关注。
